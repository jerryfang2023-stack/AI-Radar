# MVP 验证计划｜采购应付 Agent 运营层

- hypothesis：采购和应付团队愿意为“异常识别 + 催办草稿 + 人工确认 + 过程记录”的轻量 Agent 服务包付费。
- target_users：制造业采购负责人、财务共享中心负责人、分销贸易企业老板、ERP/数字化负责人。
- test_method：8 个客户访谈 + 1 个半自动流程样张 + 1 周样本数据模拟。
- success_signal：至少 3 个客户确认痛点强；至少 1 个客户愿意提供样本数据或付费试点。
- failure_signal：客户不愿提供数据、流程太不规范、负责人认为只是低价值效率优化。
- feedback_items：
  - 哪类异常最痛？
  - 是否允许 AI 读取邮件和附件？
  - 哪些动作必须人工确认？
  - 客户更愿意按月付费、按量付费还是项目服务费？
- decision：pending
