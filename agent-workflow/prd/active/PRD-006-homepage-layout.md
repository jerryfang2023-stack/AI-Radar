# PRD-006：首页布局与排版优化

更新时间：2026-05-02  
owner：`pm`  
协作 agent：`ui-ue`、`copy`、`data`、`dev`、`qa`

## 1. 背景

当前首页承载内容过多，信息密度偏高，容易让观澜 AI 看起来像资讯流或后台数据页。首页应回到商业化入口的核心任务：建立品牌信任、说明判断价值、引导用户进入 Daily Brief 或机会库。

## 2. 用户任务

用户进入首页，5 秒内需要明白：

1. 观澜 AI 是做什么的。
2. 为什么它值得登录或试读。
3. 今天有什么代表性判断值得看。
4. 完整内容在哪里进入。

## 3. 产品目标

将首页优化为“高价值判断入口”，而不是内容大杂烩。

## 4. 非目标

- 不在首页展示全部 Signals、Trends、Opportunities。
- 不展示后台同步、字段、文档沉淀等内部说明。
- 不做花哨营销页。
- 不让首页承担完整栏目页功能。

## 5. 推荐信息结构

### 首屏

- Logo 与品牌名：观澜AI｜WaveSight AI。
- 定位语：观AI之澜，识商业之势。
- 主标题：围绕“从 AI 热点中看见商业之势”表达。
- 副标题：说明观澜 AI 提供信号、趋势、机会观察和判断依据。
- CTA：查看今日判断 / 申请试读或登录。
- 右侧视觉：雷达、信号、趋势、机会的克制商业可视化。

### 第二屏：今日判断入口

- 最新 Daily Brief 摘要。
- 只展示 1 个今日判断、2-3 条关键 Signals、1-2 个趋势变化。
- 引导进入 Daily Brief 详情。

### 第三屏：机会观察

- 展示 3 张代表性 Opportunity 卡。
- 每张卡包含机会方向、优先级、证据等级、趋势状态。
- 不展示公司名作为卡片标题。

### 第四屏：方法感与信任感

- 用简洁表达说明观澜 AI 如何从 Signals 到 Trends，再到 Opportunities。
- 不展开内部流程，不说 Markdown、同步、字段、脚本。

### 底部

- 注册/试读/会员入口。
- 企业版或私享内参申请入口。
- 非投资建议说明。

## 6. 排版要求

- 首页每屏只完成一个主要任务。
- 大标题减少数量，避免每个模块都同等强调。
- 卡片数量控制，首页以精选为主。
- 桌面端使用清晰栅格和大留白。
- 移动端按“品牌价值 -> 今日判断 -> 机会观察 -> 进入栏目”的顺序阅读。
- 文本不要被卡片、按钮或图片遮挡。

## 7. 交给 UI/UE Agent 的任务

- 输出首页桌面端和移动端页面结构。
- 明确首屏、今日判断、机会观察、信任说明和底部 CTA 的视觉层级。
- 更新 `04-Site/DESIGN.md`。

## 8. 交给 Copy Agent 的任务

- 优化首页主标题、副标题、CTA、模块标题。
- 文案必须克制、有吸引力、有商业判断感。
- 不直接罗列“老板/行业大咖/资源方/投资者”等人群标签。

## 9. 验收标准

- 首页 5 秒内能看懂观澜 AI 的价值。
- 首页不再像资讯流、数据面板或后台页。
- 首页只展示精选内容，其余进入栏目页。
- 首页无后台话术和后台控件。
- 桌面端和移动端都不拥挤。
- CTA 明确但不制造焦虑。
