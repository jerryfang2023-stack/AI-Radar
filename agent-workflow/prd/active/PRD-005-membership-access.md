# PRD-005：账号登录、订阅会员与 Admin 权限边界

更新时间：2026-05-02  
owner：`pm`  
协作 agent：`commercial-site`、`ui-ue`、`copy`、`dev`、`qa`

## 1. 背景

观澜 AI 需要从本地内容网站升级为可商业化运营的 AI 商业情报产品。新的访问模型不再采用“申请访问 / 等待审批”的普通内容访问流程，而是采用更清晰的账号体系：

```text
未登录用户 -> 注册 / 登录 -> 获得默认阅读有效期 -> 订阅延长阅读权限 -> 会员持续访问
```

Admin 仍然独立存在，只负责后台运营、内容管理、用户与订阅权限管理，不与普通前台混用。Admin 后续功能、设计、页面和文案升级已拆入 `PRD-007-admin-console.md`。

## 2. 本轮核心调整

1. 不保留“申请试读”入口。
2. 去掉普通内容的“申请访问”流程。
3. 普通用户只有两种基础状态：未登录、已登录。
4. 用户注册后自动获得默认阅读有效期。
5. 默认阅读有效期当前为 7 天，后续可在 Admin 中调整。
6. 登录用户可以通过订阅获得更多阅读时间权限。
7. 需要新增注册页、登录页、订阅套餐页、购买与支付页。
8. 首页增加注册 / 登录入口。
9. 去掉旧首屏和页面尾部的“申请访问”入口或按钮。

## 3. 用户状态

| 状态 | 说明 | 权限 |
|---|---|---|
| 未登录用户 | 尚未注册或未登录 | 可访问首页、样例内容、注册页、登录页、订阅说明 |
| 已登录有效用户 | 注册或订阅后仍在阅读有效期内 | 在有效期内访问受限内容 |
| 已登录到期用户 | 阅读权限已到期 | 可访问账户页、订阅页、公开内容；受限内容显示续订提示 |
| 订阅会员 | 已购买月 / 季 / 年套餐且仍在有效期内 | 访问完整 Daily Brief、Signals、Opportunities、Trends |
| 管理员 | 后台运营身份 | 全部内容、用户、订阅、同步、编辑、质量检查能力 |

不再使用“试读用户”作为独立入口类别。注册后的默认阅读有效期是已登录用户的一段权限有效期，而不是一个单独申请流程。

## 4. 用户任务

未登录用户需要：

1. 在首页快速理解观澜 AI 的价值。
2. 看到明确的注册 / 登录入口。
3. 注册账号后直接获得默认阅读有效期。
4. 在访问受限内容时看到登录或注册提示。

已登录用户需要：

1. 看到自己的阅读权限状态。
2. 知道当前阅读权限或订阅何时到期。
3. 到期后能进入订阅页续费。
4. 订阅后获得对应周期的完整内容访问权限。

管理员需要：

1. 查看和管理用户状态。
2. 调整单个用户的阅读或订阅有效期。
3. 以周 / 月 / 季 / 年为单位延长或缩短阅读权限。
4. 管理订阅套餐展示。
5. 查看支付状态或手动标记支付结果。
6. 保持内容同步、编辑、合并、下架和质量检查能力。

## 5. 页面规划

### 5.1 首页

需要调整：

- 右上角显示 `注册`、`登录`。
- 已登录后显示账户入口或会员状态。
- 删除首屏旧的“申请访问 / 申请查看详情”按钮。
- 删除页面尾部旧的“申请访问”入口。
- CTA 改为：
  - `注册`
  - `登录`
  - `查看订阅方案`

### 5.2 注册页面

路径建议：

```text
register.html
```

页面目标：

- 让用户创建账号。
- 明确注册后获得默认阅读有效期。
- 文案克制，不承诺结果。

核心字段：

- 手机号或邮箱
- 验证码或密码
- 用户名 / 公司名称，可选
- 同意服务条款与隐私政策

注册成功后：

- 创建登录状态。
- 写入阅读权限开始时间。
- 写入阅读权限到期时间。
- 跳转到账户页或 Daily Brief。

### 5.3 登录页面

路径建议：

```text
login.html
```

页面目标：

- 已注册用户登录。
- 未注册用户可跳转注册。
- 管理员不从普通登录入口暴露后台能力。

核心字段：

- 手机号 / 邮箱
- 密码或验证码

登录成功后：

- 根据权限状态跳转：
  - 权限有效：进入 Daily Brief 或上一次访问页。
  - 权限到期：进入订阅页或账户页。

### 5.4 订阅套餐页面

路径建议：

```text
pricing.html
```

套餐：

| 套餐 | 权限周期 | 说明 |
|---|---|---|
| 月度会员 | 1 个月 | 适合短期观察和试用 |
| 季度会员 | 1 个季度 | 适合持续跟踪 AI 商业变化 |
| 年度会员 | 1 年 | 适合长期订阅 Daily Brief、Signals、Opportunities、Trends |

页面内容：

- 三档套餐卡。
- 每档包含访问权限、周期、适合场景、价格占位。
- 明确“观澜 AI 提供商业信号、趋势和机会观察，不构成投资、经营或合作建议”。

### 5.5 购买与支付页面

路径建议：

```text
checkout.html
```

第一阶段可以先做支付流程占位，不接真实支付。

页面目标：

- 确认购买套餐。
- 展示订单摘要。
- 展示支付方式占位。
- 支付成功后延长阅读权限。

核心模块：

- 套餐名称
- 订阅周期
- 应付金额
- 支付方式
- 支付状态

### 5.6 账户 / 权限状态页面

路径建议：

```text
account.html
```

页面目标：

- 显示当前登录用户身份。
- 显示阅读权限或订阅到期时间。
- 到期前提示续订。
- 到期后引导进入订阅页。

## 6. 权限规则

### 6.1 未登录

可以访问：

- 首页
- 注册页
- 登录页
- 订阅页
- 样例内容

点击完整内容：

- 弹出或进入登录 / 注册提示。
- 不出现申请访问表单。

### 6.2 已登录且权限有效

可以访问：

- Daily Brief
- Signals
- Opportunities
- Trends

页面显示：

- 可显示轻量会员状态，例如“有效期至 YYYY-MM-DD”。
- 不显示后台控件。

### 6.3 已登录但权限到期

可以访问：

- 首页
- 账户页
- 订阅页
- 样例内容

访问完整内容时：

- 显示续订提示。
- 引导进入订阅套餐页。

### 6.4 管理员

管理员能力只出现在 Admin：

- 内容同步。
- Daily Brief 发布。
- Signals 编辑与合并。
- Opportunities 编辑与合并。
- Trends 状态维护。
- Tags 管理。
- 用户权限管理。
- 试读 / 订阅时长调整。
- 支付状态手动标记。
- 系统日志与质量检查。

普通前台不得出现：

- 页面编辑
- 恢复原始数据
- 访问状态
- 同步
- JSON 编辑
- 后台日志

## 7. Admin 配置项

Admin 需要支持：

| 配置项 | 说明 |
|---|---|
| 默认注册有效期 | 默认 7 天 |
| 单个用户有效期 | 可手动调整 |
| 延长单位 | 周 / 月 / 季 / 年 |
| 用户状态 | 未登录不可见，已登录 / 到期 / 会员 / 管理员 |
| 套餐状态 | 月 / 季 / 年套餐是否显示 |
| 支付状态 | 待支付 / 已支付 / 失败 / 手动确认 |

## 8. 非目标

- 本轮不做真实支付网关接入。
- 本轮不做复杂企业 CRM。
- 本轮不做企业版销售流程。
- 本轮不保留普通内容的申请访问表单。
- 本轮不做公开“申请试读”入口。
- 本轮不把 Admin 能力暴露在普通前台。

## 9. 文案边界

推荐表达：

- `注册`
- `登录查看完整内容`
- `订阅后继续阅读`
- `当前阅读权限有效至 YYYY-MM-DD`
- `观澜 AI 提供商业信号、趋势与机会观察，不替代最终经营、投资或合作判断。`

避免表达：

- `申请访问`
- `申请试读`
- `立即抢占机会`
- `必投机会`
- `保证回报`
- `内部审批通过后查看`

## 10. 验收标准

### 前台

- 首页有注册 / 登录入口。
- 首屏和尾部不再出现申请访问按钮。
- 未登录用户访问受限内容时，只出现注册 / 登录提示。
- 注册后获得默认阅读有效期。
- 已登录用户能看到阅读权限状态。
- 到期用户看到订阅提示。
- 月 / 季 / 年订阅套餐可见。
- 购买和支付页面有完整流程占位。

### Admin

- Admin 可以查看用户列表。
- Admin 可以调整用户阅读权限到期时间。
- Admin 可以按周 / 月 / 季 / 年延长权限。
- Admin 可以管理套餐展示状态。
- Admin 可以手动标记支付结果。
- Admin 仍保留同步、编辑、合并、下架和质量检查能力。

### 权限边界

- 普通页面无编辑、同步、恢复、JSON、访问状态等后台控件。
- 已登录用户不再进入申请访问流程。
- Admin 页面集中承接所有后台能力。

## 11. 建议开发顺序

1. 清理旧申请访问入口和按钮。
2. 建立本地账号状态模型。
3. 新增注册页。
4. 新增登录页。
5. 新增账户页。
6. 新增订阅套餐页。
7. 新增购买 / 支付占位页。
8. 更新首页右上角状态。
9. 更新受限内容权限判断。
10. Admin 增加用户权限与订阅管理。
