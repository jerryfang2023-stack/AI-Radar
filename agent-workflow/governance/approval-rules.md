# 审批规则

## Agent 可直接执行

- Markdown 命名检查。
- frontmatter 补齐建议和低风险补齐。
- 重复项检测。
- 标签初步归类。
- 网站数据同步。
- 健康检查。
- 低风险文案润色初稿。
- 普通页面的错字和断行修复。

## Agent 产出草案，人工确认

- 首页核心文案。
- 栏目增删合并。
- Scoring / Priorities 展示规则变更。
- Opportunity 是否成立。
- 机会优先级最终排序。
- 标签体系大类调整。
- 视觉风格明显调整。
- PRD 定稿。

## 必须人工审批

- 项目定位修改。
- 对外品牌主张定稿。
- 访问申请审批。
- 删除或合并核心历史内容。
- 云端上线。
- 权限策略变更。
- 涉及真实客户、合作、投资判断的公开表达。

## 状态流转

```text
pending -> in_progress -> verify -> passed
                     \-> blocked
```

规则：

- `verify` 表示 agent 做完了，但还需要检查。
- `passed` 只能在验收标准满足后标记。
- 需要人工确认的任务，agent 不能自行标为 `passed`。
