# Window Dispatch Hub｜调度窗口规则

本文件只定义当前调度方式。

## 调度模式

当前窗口可以作为调度中枢，只做：

- 生成派发单；
- 输出新窗口指令；
- 接收 closeout；
- 验收并回填看板；
- 清理无效文件。

大型开发、页面、内容生产和脚本改造默认交给独立执行窗口。

## 新窗口最小读取

默认只读：

1. `AGENTS.md`
2. `context/context-index.md`
3. 当前任务派发单
4. 任务直接相关的 1-3 个 current 文件或 Skill
5. 高风险流程必须加入 `context/06-execution-harness.md`

如果上下文不足，先说明缺什么，不要自行全库扫描。

## 快捷口令

- `派发：<任务>`：新建派发单和新窗口指令。
- `执行：<Task ID>`：读取派发单并输出新窗口指令。
- `收口：<closeout 路径>`：读取 closeout、验收并回填。
- `看板` / `状态`：查看当前任务。

## 派发单必须包含

- Task ID。
- 阶段边界。
- 最小读取文件。
- 允许范围。
- 禁止范围。
- 输出路径。
- 验证要求。
- 高风险流程对应的 harness 和 Stop 条件。

## 收口文件必须包含

- 实际改动范围。
- 修改文件清单。
- 运行验证。
- 未完成项和风险。
- 是否需要调度窗口继续派发。
- 高风险流程必须说明使用的 harness、固定读取、质量门和下游放行结论。

## 当前原则

- 不把过程任务、过程 closeout、截图记录或临时规划写入新提示词。
- Agent 按流程节点设置。
- Skill 按能力模块沉淀。
- 不新增栏目常驻 Agent。
