# Plan-first Policy

适用范围：新增或调整一级栏目、权限、数据模型、自动化、部署、商业化路径，以及跨 Product / Intelligence / Experience / Build 四个流程节点的任务。

## 必须先计划

- 改变导航、会员态、权限或商业化路径。
- 改变内容 schema、自动化链路、来源等级或数据写入位置。
- 改变 GitHub Pages、生产同步或发布方式。
- 任务需要多个流程节点连续交付。

## 最小计划内容

- 目标和不做范围。
- 牵头 Agent 与需要调用的 Skill。
- 只读文件清单。
- 改动范围。
- 风险与回滚方式。
- 验收方式。
- 回填位置。

## 禁止

- 不把过程记录写进计划。
- 不把已停止链路当作当前基线。
- 不在计划中引用已删除文档。
- 不用计划替代用户确认。

## 回填

计划通过后，只更新当前需要的文件：

- `context/09-v3-3-current-action-index.md`（仅在当前动作索引确需更新时）
- 必要 closeout
