# Source Intelligence Model

更新时间：2026-05-03  
owner：`data-agent` / `workflow`  
状态：active

## 1. 定位

Source Intelligence 是观澜AI的输入质量治理模型。

它负责管理：

- 每日 Signals 的监测来源。
- The Point 的 Builder / Podcast / Blog 来源。
- 关键词池。
- 噪音来源。
- 来源升权、降权和淘汰。

目标是让观澜AI不是被热点牵着走，而是持续拥有更高质量、更早、更可验证的 AI 商业判断输入。

## 2. 来源分层

| 层级 | 类型 | 处理规则 |
|---|---|---|
| S | 一手证据 | 优先采信，可进入高分 Signal 候选 |
| A | 高质量商业媒体 | 可作为 Signal 证据，但尽量回找一手来源 |
| B | 垂直行业媒体 / 数据库 | 适合补行业场景和需求侧信号 |
| C | 社媒 / 社区 / 聚合 | 只作为线索，不直接作为高证据 |
| D | 噪音或低可信来源 | 降权或移除 |

## 3. Signals 来源池

### S 级优先来源

- 公司官网、产品更新、博客、开发者文档。
- 客户案例、合作公告、上线公告。
- 监管文件、政策文件、招投标公告。
- 招聘页、价格页、API 文档、云市场、插件市场。
- GitHub、App Store、Chrome Web Store、云市场 listing。

### A 级商业媒体

- 海外：WSJ、FT、Bloomberg、The Information、TechCrunch、Axios、CNBC。
- 中文：36氪、晚点、甲子光年、钛媒体、财新、机器之心、量子位。

### B 级行业与数据来源

- 医疗、工业、金融、营销、企业服务行业媒体。
- 融资数据库、并购公告、工商变更。
- G2、Product Hunt、招投标平台、政府采购。

## 4. The Point 来源池

第一阶段继续使用 `follow-builders` 默认来源：

- 25 位 X / Twitter AI 建造者。
- 6 个 AI 播客。
- 2 个官方 Blog 来源。

准入原则：

- 建造者优先于评论者。
- 原创观点优先于搬运。
- 能影响产品、模型、基础设施、商业化、组织或风险判断。
- 有稳定原文来源。

降权原则：

- 连续多次只有转发或情绪表达。
- 原文不可追溯。
- 与观澜AI商业判断任务无关。
- 内容高频重复但无新增观点。

## 5. 关键词池

关键词分为 5 类：

| 类别 | 作用 | 示例 |
|---|---|---|
| 赛道词 | 定位方向 | AI Agent、AI Coding、AI营销、企业知识库 |
| 证据词 | 判断商业证据 | 融资、客户、收入、部署、复购、招投标 |
| 场景词 | 判断落地场景 | 客服、销售、法务、财务、医疗、工业 |
| 风险词 | 捕捉反证 | 监管、诉讼、安全、隐私、裁员、暂停 |
| 行动词 | 捕捉商业动作 | 合作、采购、试点、集成、上线、并购 |

## 6. 质量指标

### 来源指标

- 有效 Signal 产出率。
- 一手证据比例。
- 平均 Signal Score。
- 首发或早发现能力。
- 重复转载率。
- 噪音率。
- 关联 Opportunity / Trend 转化率。

### 关键词指标

- 命中数量。
- 有效 Signal 数量。
- 平均 Signal Score。
- 噪音率。
- 重复率。
- 进入 Daily Brief 的数量。
- 转化为 Opportunity / Trend 的数量。

### Builder 指标

- 高分 Point 数量。
- 原创性。
- 商业相关性。
- 观点后续被 Signal / Trend / Opportunity 吸收的比例。
- 观点分歧或反证价值。

## 7. 升权、降权和淘汰

### 升权

满足任一条件：

- 连续 3 次产出高分 Signal。
- 率先提供客户采用、收入、招投标或监管证据。
- 多次进入 Daily Brief。
- 支撑新 Opportunity 或 Trend。
- Builder 观点形成长期共识或重要反证。

### 降权

满足任一条件：

- 连续 7 天只有转载或低分内容。
- 噪音率高于有效 Signal 产出率。
- 标题党、缺来源、不可验证。
- 与 AI 商业机会判断无关。

### 淘汰

满足任一条件：

- 长期不可访问。
- 高重复且无增量。
- 来源可信度低或版权风险高。
- 多次污染自动化结果。

## 8. 每周报告

每周输出：

```text
agent-workflow/reports/source-intelligence-weekly-YYYY-WW.md
```

结构：

```markdown
# Source Intelligence Weekly

## 1. 本周结论
## 2. 高产来源
## 3. 低效来源
## 4. 新增候选来源
## 5. 候选降权来源
## 6. 高产关键词
## 7. 低效关键词
## 8. Builder 池变化
## 9. 需要更新自动化的事项
```

## 9. 自动化应用

### 对 `ai-the-point`

- 读取本文件与 `the-point-model.md`。
- 使用 Builder 池准入和降权原则。
- 不因单个来源失败生成空文件。
- 将来源质量问题写入 `daily-run-log.md` 或运行报告。

### 对 `ai-2`

- 读取本文件与 `signal-system.md`。
- 优先使用 S / A 来源。
- 对 C 级社媒线索必须回找一手证据。
- 记录高产词、低效词、噪音来源。

### 对 `ai-3`

- 不直接使用来源池，但同步报告应保留来源质量软提醒。
- 若内容状态为 `needs_review`，不得入站。

## 10. 与 Tags 的关系

来源和关键词不直接等于 Tags。

只有当关键词能稳定连接多个 Signal、Point、Trend 或 Opportunity，才可进入 `tag-taxonomy.md` 的正式标签字典。
