# 观澜AI｜WaveSight AI 设计规范

## 设计定位

观澜AI是面向老板、行业操盘者、资源方和投资观察者的 AI 商业情报产品。页面要像高端商业内参和专业研究产品，而不是普通 SaaS 官网、资讯流、后台系统或工具站。

## 设计硬规则：VI / SVG / Token First

以后所有设计相关工作必须先读取：

```text
docs/brand/wavesight-ai-vi/USAGE.md
docs/brand/wavesight-ai-vi/visual-identity-guidelines.md
docs/brand/wavesight-ai-vi/typography-guidelines.md
docs/brand/wavesight-ai-vi/brand-tokens.css
docs/brand/wavesight-ai-vi/motion-tokens.css
docs/brand/wavesight-ai-vi/executable-svg/README.md
docs/brand/wavesight-ai-vi/executable-svg/manifest.json
```

强制要求：

- 不根据截图重画元素。
- 所有 Logo、符号、信息卡片和动效分镜都优先引用 SVG 资产。
- 页面颜色使用 `brand-tokens.css`。
- V2 公开页面全局背景使用 `#FFFDF8` / `--gl-bg-page`，与导航栏融合；卡片和内容容器使用 `rgba(255,253,248,0.60-0.72)` 或 `--gl-paper` 形成纸面层次。`#F7F4EF` / `--gl-bg` 保留为品牌纸感底色、物料底色或局部背景，不再作为 V2 网站公开页面唯一全局背景。
- 动效使用 `motion-tokens.css`。
- 如果需要新组件，必须基于现有 SVG 风格扩展，保持深海蓝、云白、雾灰和少量香槟金。

品牌核心表达：

```text
观 AI 之澜，识商业之势
看懂变化，判断影响，找到行动点
```

后续页面和视觉资产必须以用户已给定的 Logo、澜线、地平线、商业内参符号、信息元素和动效规范为准。不得随意更改、重画或替换以下资产：

- Logo 主结构：一条远方地平线 + 三道流动澜线 + `观澜 AI` + `WAVESIGHT AI`。
- Logo 默认使用最新高还原 SVG 资产；正式横版 Logo 网页默认宽度不低于 `160px`，报告封面建议不低于 `220px`；小于 `120px` 宽时优先使用 symbol。
- Logo 母题：澜线代表变化和趋势，地平线代表视野和判断边界。
- 图形语言：澜线、地平线、细分割线。
- 商业内参符号系统：Signal、Point、Trend、Opportunity、Quote、Divider、Timeline Node、Data Snapshot、Rating Chip、Source Note。
- 商业内参信息元素：期号编号、日期、会员内参标签、阅读时长、核心判断框、引述框、数据卡、趋势时间线、机会矩阵、来源脚注。
- 动效原则：轻、慢、稳，像水面微动，不做炫技。

如果后续任务确需调整上述资产，必须先由 PM / UI / Verification 评审，并说明原因、范围、风险和是否需要用户确认。未经确认，不得改变澜线数量、去掉金色地平线、添加信号点、雷达圈、眼睛、机器人、蓝紫科技渐变或新的品牌母题。

核心气质：

- 克制
- 高级
- 清晰
- 有留白
- 有判断感
- 有商业可信度

## 参考方向

每次 UI/UE Agent 改页面前，必须从以下方向里选 1-2 个作为当次参考：

- Apple：留白、克制、主标题有冲击力。
- Linear：信息密度适中、模块干净、边界清晰。
- Stripe：商业产品感、图表和说明的专业平衡。
- Bloomberg / FT / Economist 内参式阅读：标题、导语、正文、侧栏有清楚层级。
- `awesome-design-md` 中的真实 `DESIGN.md`：用于补充具体页面结构和组件规则。

不允许只说“高级”“苹果风”，必须说明本次参考的是哪一种页面逻辑。

## 色彩

主色：

- 暖白 `#FFFDF8`：V2 公开页面全局背景、导航融合底色。
- 云白 / 米白 `#F7F4EF`：品牌纸感底色、物料底色或局部背景。
- 纸白 `#FFFFFF`：卡片、报告页、浮层。
- 墨海蓝 `#071827` / `#0B1D2A`：标题、导航、重点判断、深色背景。
- 深澜蓝 `#0D355C`：主要品牌色、链接、趋势线。
- 澜线蓝 `#1F5D8C`：辅助图形和数据线。
- 雾蓝灰 `#6F7F8F` / 远山灰 `#A7ADB4`：说明文字、山形背景、弱信息。
- 香槟金 `#C8A766`：地平线、分隔线、少量强调。

禁止：

- 大面积紫色渐变。
- 霓虹科技蓝。
- 高饱和互联网蓝。
- 大面积纯黑压迫。
- 花哨装饰渐变。

## 字体与字号

原则：

- 字体系统必须遵守 `docs/brand/wavesight-ai-vi/typography-guidelines.md`。
- 中文标题使用 `Noto Serif SC / Source Han Serif SC / Songti SC` 一类衬线字体，形成东方智识和商业判断感。
- 中文正文与界面使用 `Noto Sans SC / Source Han Sans SC / PingFang SC / Microsoft YaHei` 一类黑体，保证长时间阅读清晰。
- 英文品牌、标签和栏目英文使用 `Inter / IBM Plex Sans / Avenir Next`，品牌名统一写作 `WAVESIGHT AI`。
- 数字、日期、期号、数据卡和图表数值使用 `IBM Plex Mono / JetBrains Mono / Roboto Mono`，少量封面大数字可用 Georgia / Times New Roman。
- 大标题只用于真正的页面主判断。
- 卡片内标题必须克制，不得像首页 hero。
- 正文行高要舒适，中文建议 `1.6-1.85`。
- 字体不能靠无限放大制造高级感。

桌面建议层级：

- Hero 主标题：`56px / 72px / 600-700 / 中文衬线`
- 页面标题：`44px / 58px / 600 / 中文衬线`
- 内参文章标题：`40px / 56px / 600 / 中文衬线`
- 模块标题：`28px / 40px / 600 / 中文衬线或黑体`
- 卡片标题：`20px / 30px / 500-600 / 中文黑体`
- 正文：`16px / 28px / 400 / 中文黑体`
- 摘要说明：`14px / 24px / 400 / 中文黑体`
- 标签 / 日期：`12px / 18px / 400-500 / 英文或中文黑体`
- 英文小标签：`11px / 16px / 500-600 / Inter`

移动建议层级：

- Hero 主标题：`34px / 46px / 中文衬线`
- 页面标题：`30px / 42px / 中文衬线`
- 内参文章标题：`28px / 40px / 中文衬线`
- 模块标题：`22px / 32px / 中文黑体或衬线`
- 卡片标题：`18px / 28px / 中文黑体`
- 正文：`15px / 26px / 中文黑体`
- 摘要说明：`13px / 22px / 中文黑体`
- 标签 / 日期：`12px / 18px / 英文或中文黑体`

禁止：

- 同一屏出现多个超大黑体标题。
- 小卡片标题超过 `28px`。
- 文本容器过窄导致中文频繁断行。
- 文本行宽过宽导致阅读疲劳。
- 英文品牌名大小写不统一。
- 大段正文使用衬线宋体。
- 过度书法化、国潮、圆体、科技窄体或花体英文。

## 布局原则

页面要先确定“阅读路径”，再设计卡片。

优先结构：

1. 页面判断 / 导语
2. 主阅读区
3. 侧栏或补充证据
4. 关联内容

避免：

- 卡片套卡片。
- 三栏内容完全等权但信息量不同。
- 太多边框、阴影、徽章同时出现。
- 每个元素都像按钮。
- 公开页面出现后台编辑、恢复数据、JSON 等痕迹。

## 栏目标题规范

一级栏目页标题必须保持统一的视觉锚点，避免每个栏目看起来像不同产品。

硬规则：

- Daily Brief、Signals、Opportunities、Trends 等一级栏目标题区必须使用同一套位置、字号、行高、留白和内容宽度节奏。
- 标题必须是一句话价值表达，优先一行显示；不在标题下追加解释型副标题、功能说明或内部使用提示。
- 标题背景默认与页面背景一致，不额外添加突兀色块、装饰块或孤立渐变。
- 新增或修改任一栏目标题后，必须对照至少两个既有栏目截图，检查标题左边距、顶部位置、字号、行高和首屏节奏是否一致。
- 标题不能为了“有设计感”而牺牲清晰度；如果文案过长，应先让 Copy Agent 压缩文案，而不是放大容器。

## 成品感与精致度

观澜AI不是临时数据看板，公开前台不能粗糙、简陋或像组件堆叠。

硬规则：

- 首屏必须有明确主次：主标题、核心信息、主要动作不能互相抢权。
- 页面不能用大面积占位、空洞指标或内部解释制造“功能区”；每一块都必须服务用户阅读和判断。
- 信息密度要像商业内参：克制、可扫读、有层级；不能像新闻瀑布流，也不能像后台字段表。
- 卡片、徽章、边框、阴影和分割线只作为层级辅助，不作为装饰堆砌。
- 详情页应像研究报告或商业笔记，优先用正文节奏、分隔线、侧栏摘要和少量链接组织信息，不把每个字段都包成大卡片。
- 桌面端和移动端都要检查：是否主次清楚、是否有过大留白、是否文字过长、是否横向溢出、是否按钮或徽章压过正文。
- 如果用户反馈“粗糙、简陋、主次不明、占位过大、文案像内部话术”，默认视为结构问题，先调整信息架构和阅读路径，再处理视觉细节。

## Design Director 风格美观质检

页面设计不能只通过功能验收，还必须通过风格美观质检。Design Director 必须把以下标准同时作为创作要求和交付后的质检标准。

### 五项质检

| 维度 | 重点 | 分值 |
|---|---|---:|
| Style Purity / 风格一致性与纯净度 | 视觉方言是否纯粹，是否有异类元素、廉价装饰或素材风格混杂 | 20 |
| Proportion & Rhythm / 比例与韵律感 | 负空间、构图、排版建筑感、标题与模块节奏是否成立 | 20 |
| Color Sophistication / 色彩深度与平衡 | 色调是否和谐，灰阶是否通透，光影与材质是否统一 | 20 |
| Craftsmanship / 细节的艺术处理 | 圆角、图标、字体渲染、对齐、hover / active / empty / loading / error 状态是否精致 | 20 |
| Emotional Resonance / 情感共鸣 | 第一秒气场是否符合目标用户，是否有克制但可记忆的精致瞬间 | 20 |

### 眯眼测试

每次页面设计完成后，必须执行眯眼测试：

- 忽略文字内容，只看色块、明暗分布、留白、轮廓和视觉重心。
- 如果眯眼后页面骨架仍清晰、平衡、有节奏，视觉结构通过。
- 如果眯眼后只剩散乱卡片、随机色块、过多按钮、无主次装饰或空洞大留白，必须重做结构。

### 分数规则

- `90-100`：优秀，可作为母版或参考样张。
- `80-89`：通过，可进入 Dev 或 QA，但记录可优化项。
- `70-79`：勉强通过，只能在无阻塞项时继续，必须列出后续优化点。
- `<70`：不通过，必须重做，不得交给 Dev 或不得验收通过。

页面类型通过线：

- 首页 / 全站母版 / 核心首屏 / 海报 / 视觉资产：低于 `85` 必须重做。
- 一级栏目页 / 详情页 / 会员页：低于 `80` 必须重做。
- Admin 工作台和后台模块：低于 `75` 必须重做。
- 所有页面任一单项低于 `14` 必须重做；基础运营页面如需降低标准，必须由 PM 明确说明，且不得低于 `10`。

任何单项低于 `10`，或出现以下阻塞项，即使总分达到 `70` 也必须重做：

- 风格混杂，像多个模板拼接。
- 海报、插图或截图样张不符合观澜AI商业情报调性。
- 比例失衡，标题、卡片、按钮、徽章争抢注意力。
- 配色脏、廉价、高饱和或偏离本规范色彩系统。
- 细节粗糙，圆角、图标、字重、边框、间距、hover / active 状态不统一。
- 页面第一秒缺少可信、有判断、有边界、值得读的情绪。

### 证据化质检要求

风格美观质检必须基于证据，不接受只写主观分数。

页面已经实现或已有可访问页面时，质检必须包含：

- 设计基准引用：本次对照 `DESIGN.md` 哪一节、哪一种参考方向、哪一个页面母版。
- 桌面端截图路径。
- 移动端截图路径。
- 五项评分及逐项扣分原因。
- Squint Test 结论和视觉骨架问题。
- 必须重做清单。
- 可延后优化清单。
- Dev 实现偏差清单。
- QA 复核建议。

如果没有截图、没有扣分原因、没有重做清单或没有 Dev 偏差清单，风格美观质检无效，不得作为 accepted 依据。

| 同屏超过 3 种卡片风格、圆角或阴影系统 | -10 |
| 按钮、徽章、数字和摘要争抢注意力 | -8 到 -12 |
| 首屏没有明确主任务或视觉重心 | -12 到 -18 |
| 海报或首屏视觉像通用 AI 氛围图 | 直接阻塞 |
| 前台像后台组件堆叠 | 直接阻塞 |
| 移动端标题断行难看、按钮拥挤或横向溢出 | -10 到直接阻塞 |
| 配色脏、廉价、高饱和或偏离本规范色彩系统 | -8 到 -15 |
| 圆角、图标线宽、边框、hover / active 状态不统一 | -6 到 -12 |

## Daily Brief 规范

Daily Brief 是“每日商业内参”，不是数据列表。

列表页：

- 每天一张主卡。
- 卡片可以点击进入详情页。
- 卡片只展示三类内容：
  - 关键 Signals
  - 机会卡
  - 趋势观察
- 短期不展示机会优先级，因为评分需要长期积累才有说服力。
- 页面左右留白不能过大，桌面端主内容宽度建议接近 `1520-1680px`。

详情页：

- 用“主 Signal 阅读区 + 右侧机会/趋势侧栏”。
- 首条 Signal 要明显更强，像今日主线。
- 右侧侧栏只做摘要，不抢主阅读区。
- 文案必须克制，强调“证据关系”和“观察”，不替用户下最终判断。

禁止：

- 三栏平铺且每栏等权。
- 一屏内堆太多小盒子。
- 详情页标题只有日期，没有判断含义。
- 侧栏文字过长。

## 首页 Decision Brief 规范

首页 Decision Brief 是给目标客户的第一层判断入口。

第一栏必须优先展示 Signals，而不是短期评分。

推荐三栏：

- 今日关键 Signals
- 正在升温的趋势
- 值得继续观察的机会

每条内容必须有：

- 标题
- 一句解释
- 必要时有 Signal Score / 来源层级 / 赛道

禁止：

- 首页显示过多后台指标。
- 用“优先级评分”作为首页第一判断依据。
- 文案像内部工作流说明。

## Opportunities 规范

Opportunities 是机会库，不是公司库。

机会卡名称不能以公司为核心命名。公司只能作为证据、案例或来源出现。

机会卡应展示：

- 机会名称
- 一句话机会定义
- 适用行业 / 场景
- 证据等级
- 相关 Signals
- 相关 Trends

Priority Engine 可以出现在 Opportunities 中，但必须作为排序和复盘辅助，不作为前台独立主栏目。

## Signals 规范

Signal 标题必须是“事件 + 商业含义”，不能只是公司名。

Signal 卡片优先展示：

- 事件标题
- 内容简介
- Signal Score
- 来源层级
- 相关趋势 / 机会

Signals 栏目页补充规则：

- Signals 栏目页是商业信号筛选和进入详情的页面，不是完整报告页，也不是内部雷达说明页。
- 单条 Signal 的“指向机会”数量使用主 Opportunity 口径；前台不显示由弱标签、赛道或反向关系放大的机会数量。
- 列表行要让标题和事实摘要成为主信息，赛道、信源状态、机会数、趋势数和 CTA 只做辅助。
- 详情页承担完整来源、事实、商业含义、机会拆解和关联内容，栏目页不重复堆叠详情正文。

## 文案原则

文案要有观点，但不替客户下最终判断。

推荐语气：

- “值得继续观察”
- “证据正在增强”
- “需要保留反证”
- “可能形成连续证据”
- “当前更像早期信号”

避免：

- “一定会”
- “必须投”
- “立刻行动”
- “确定机会”
- “保证收益”

## UI Agent 工作流程

每次修改页面前，UI/UE Agent 必须输出：

1. 本次页面目标。
2. 使用的设计参考，例如 `观澜 VI + Apple/Linear/Stripe + design-taste-frontend / awesome-design-md`；不得把 `frontend-design` 写入本轮执行技能或设计基准。
3. 页面阅读路径。
4. 要删除或弱化的信息。
5. 桌面端和移动端验收点。

每次修改页面后，必须检查：

- 是否使用了本 `DESIGN.md`。
- 是否过度卡片化。
- 是否有过大留白或过度拥挤。
- 文字是否断行难看。
- 普通用户前台是否出现后台痕迹。
- 是否需要截图验收。

## DESIGN v2 草案：商业情报桌面

更新时间：2026-05-04  
适用范围：首页、Daily Brief、Signals、The Point、Opportunities、Trends、会员页、Admin 和后续视觉资产任务。  
状态：历史草案。2026-05-10 后如与 `docs/brand/wavesight-ai-vi/`、`brand-tokens.css`、`WSD-20260510-vi-home-daily-closeout.md` 冲突，以最新 VI 与收口文件为准。

### Art Direction

观澜AI的界面应像一套“商业情报桌面”，而不是 AI 科技官网、资讯流、工具站或后台数据大屏。

核心方向：

- 产品气质：高端商业内参、研究桌面、判断样张、克制会员产品。
- 视觉关键词：克制、研究感、纸面秩序、深墨绿重点、连续证据。
- 目标用户感受：可信、有判断、有边界、值得读。
- 记忆点：用户第一眼看到“AI 变化被压缩成商业判断”的样张，而不是抽象能力图。

非目标视觉：

- 不做 AI 紫蓝渐变、霓虹科技大屏、机器人头像、抽象雷达流程图、模板 SaaS 卡片墙。
- 不做投融资排行榜、公司榜、热度榜或泛数据仪表盘。
- 不用大面积装饰图证明“科技感”；视觉资产必须服务判断阅读。

### 色彩系统

| 用途 | 色值 | 使用规则 |
|---|---|---|
| 页面背景 | `#FFFDF8` | V2 公开页面全局背景，与导航栏融合 |
| 品牌纸感底色 | `#F7F4EF` | 物料底色或局部背景，不再作为公开网站唯一全局背景 |
| 纸面 / 卡片 | `rgba(255,253,248,0.60-0.72)` / `#FFFFFF` | 内容容器，少量浅阴影，不做玻璃拟态 |
| 主文字 | `#17231F` | 标题、主判断、正文重点 |
| 次级文字 | `#5E6B66` | meta、来源、说明 |
| 深墨绿 | `#13251F` | 重点面板、主按钮、Admin 侧栏 |
| 铜棕 | `#A45F36` | eyebrow、日期、少量强调 |
| 低饱和青绿 | `#0F6F62` | 链接、状态、趋势 |
| 边框 | `#D8D0C4` | 细线、表格、列表分隔 |

执行规则：

- 首页和栏目页不再使用大面积雾化渐变作为高级感来源。
- 深墨绿每屏只承担一个重点区域，不整页铺满。
- 铜棕只做小面积识别，不做大色块。
- 不新增紫蓝、霓虹蓝、彩色光束或高饱和节点色。

### 字体层级

| 层级 | 桌面建议 | 使用场景 |
|---|---:|---|
| 首页 H1 | `64-72px / 1.08` | 仅首页主标题 |
| 栏目页 H1 | `52-60px / 1.10` | Daily、Signals、The Point、Opportunities、Trends |
| 详情页 H1 | `38-46px / 1.18` | Signal、Trend、Opportunity、Point 详情 |
| Section H2 | `26-32px / 1.22` | 模块标题 |
| 卡片标题 | `18-22px / 1.35` | 列表与内容卡 |
| 正文 | `15.5-17px / 1.7-1.85` | 详情正文、报告文本 |
| Meta / Eyebrow | `11-13px / 1.2` | 日期、来源、状态 |

执行规则：

- 不用视口宽度无限放大文字；页面必须有稳定层级。
- 栏目页标题是一句话价值表达，不追加解释型副标题。
- 卡片标题不得拥有 hero 级尺寸。
- 详情页正文优先像报告，不像字段卡片堆叠。

### 页面母版

#### 首页母版

结构：

```text
顶部导航
首屏左侧：品牌锚点 / H1 / 导语 / 两个 CTA / 三个依据标签
首屏右侧：Intelligence Desk 判断样张
首屏下沿：露出今日主判断或最新 Brief
第二屏：今日主判断 + Signals / The Point / Opportunities / Trends 四条轻量入口
后续：方法感叙事与会员动作
```

规则：

- 删除旧抽象雷达图和四张 poster 利益卡。
- 首屏只完成“说明价值 + 进入今日判断”一个主任务。
- 右侧样张必须像商业判断样张，不像后台面板或静态海报。
- CTA 不制造焦虑，优先 `查看今日简报` 和 `申请完整情报层`。

#### 栏目页母版

结构：

```text
统一标题区
轻量 command bar
主列表 / 主卡组
侧栏摘要或筛选结果，只在必要时出现
```

规则：

- 标题区高度、左边距、字号、行高和模块起点必须统一。
- 指标卡、筛选控件和徽章都降级，不能抢列表标题。
- Signals 是商业信号检索台，不是后台仪表盘。
- Opportunities 是机会库，不是榜单；`机会排行` 应改为 `优先观察` 或 `验证序列`。
- Trends 应按阶段和反证组织，不做趋势词卡片墙。
- The Point 应围绕共识、分歧与边界，不做人物墙或日期归档页。

#### 详情页母版

结构：

```text
顶部：类型 / 日期 / 来源状态
主栏：H1 + 主判断 + 正文报告节奏
侧栏：摘要、来源、关联 Signal / Trend / Opportunity
底部：相关内容
```

规则：

- 正文主栏宽度建议 `760-860px`。
- 侧栏宽度建议 `320-380px`。
- 引用只出现一次，不重复截断。
- 不做卡片套卡片；字段要转成报告阅读结构。

#### 会员页母版

结构：

```text
权限状态
可读内容层级
注册 / 登录 / 订阅动作
克制说明
```

规则：

- 登录、注册、邀请申请、Account、Pricing、Checkout 要像情报订阅产品，不像通用 SaaS 套餐页。
- 不使用焦虑 CTA，不承诺即时开通、收益或确定机会。
- 本地演示能力必须在 Admin 或上线规划中说明，不写给普通用户。

#### Admin 母版

结构：

```text
左侧窄导航
顶部状态条
高密度工作区
列表 / 筛选 / 审批 / 编辑区
```

规则：

- Admin 可以高密度、低装饰，但不能像前台栏目页。
- Admin 模块框起点、模块标题起点和工具栏高度必须一致。
- JSON、同步、恢复、字段、发布检查等内部语言只允许在 Admin 出现。
- 邀请码、访问统计、订单等本地演示能力上线前必须进入生产边界复查。

#### 移动端母版

规则：

- 本阶段移动端只做底线验收：不横向溢出、不遮挡、不破坏阅读。
- 首屏不强行占满一整屏，避免空白过大。
- 复杂右侧样张收缩为一张可扫读卡。
- command bar 可折叠，不把筛选项堆成后台表单。

### 组件克制规则

- 卡片只用于独立内容项、工具面板、模态框；页面区块不要全部卡片化。
- 徽章最多用于状态和少量分类；同一行超过 3 个徽章时必须折叠或降级为 meta。
- 列表和表格优先使用细分隔线，不使用重阴影制造层级。
- 主要按钮每屏最多一个；次动作可用文本链接。
- 图标服务快速识别，不替代关键文字。
- 首页视觉资产不嵌入关键中文，关键文案由 HTML/CSS 承载。

### 首页 Intelligence Desk 视觉资产规则

P0-11 只有在 P0-10 accepted 后执行。资产必须满足：

1. 内容是“商业判断样张”，不是抽象 AI 氛围。
2. 桌面端画面包含 Brief 摘要、2 条 Signal、1 条 Opportunity、1 个 Trend 状态。
3. 关系线必须克制，表达从 Signal 到 Opportunity / Trend 的连接。
4. 不出现 `Scoring`、`JSON`、`同步`、`后台`、`字段`、`编辑`。
5. 不出现机器人头像、霓虹科技背景、纯雷达网格、股市 K 线堆叠或通用 SaaS 插画。
6. 如用生成图，只做纸面结构和氛围，不嵌中文小字。
7. 如用 HTML/CSS 样张，优先使用真实站内数据并截断为 1-2 行。

### 审美阻塞项

以下情况不得进入 Dev 或不得验收通过：

- 页面粗糙、简陋、像模板页或临时拼装页。
- 首页继续使用旧雷达图、四张 poster 卡或泛营销利益表达。
- 栏目标题位置、字号、行高、模块起点不一致。
- 页面像后台组件堆叠，主次不明。
- 徽章、数字、按钮、摘要争抢注意力。
- 详情页把字段直接摊开，没有报告阅读路径。
- 文案过长破坏布局，但 UI 未退回 Copy 压缩。
- 普通前台出现 Admin、JSON、同步、编辑、恢复、字段、后台等痕迹。

### 后续 Dev 执行边界

后续页面开发不得继续简单追加 `final / override / lock / polish` 样式层来修视觉问题。Dev 应先按母版整理：

1. tokens：颜色、字体、间距、阴影、边框。
2. layouts：首页、栏目页、详情页、会员页、Admin。
3. components：卡片、列表、command bar、侧栏、按钮、徽章。
4. page-specific：只保留必要页面差异。

如需要临时覆盖，必须在 closeout 说明原因、影响页面和后续清理路径。
