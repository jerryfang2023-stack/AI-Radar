# Column Architecture｜栏目架构与边界

更新时间：2026-05-24
状态：current / V2.2.1

栏目按“用户要完成的判断任务”设计，而不是按“我们有什么数据”设计。

## V2.2.1 前台导航

```text
今日观察 / 商业信号 / 趋势追踪 / 商业内参
```

Admin 为后台入口，不进入普通用户导航。Tags 不作为一级导航，只用于搜索、筛选、关键词表和关系网络。

不作为一级栏目：

- 前沿观点：作为商业信号相邻内容流和后台判断资产。
- 机会判断：作为趋势追踪和商业内参中的文章段落。
- Scoring / Priority Engine：后台能力。
- Method：内部产品资产，不进入当前前台导航。

## 栏目任务

| 栏目 | 用户任务 | 核心内容 | 不应承担 |
|---|---|---|---|
| 今日观察 | 用一篇 newsletter 式主稿看懂一天最重要的 AI 商业变化 | 当日主文章、关联商业信号、观点参照、日期 / 标签筛选 | 新闻列表、内部流程看板、Card 扩写 |
| 商业信号 | 浏览和精读可追溯的事实信号 | 产品 / 服务、融资、案例三类正式 `signal_card`，成熟变化短专题，观点参照 | 大而全数据库、无证据短卡、公司名列表 |
| 趋势追踪 | 判断一组变化是否正在形成连续方向 | 趋势候选、趋势快报、深度趋势报告、关联信号和观点 | 卡片聚合页、公司案例库、执行方案 |
| 商业内参 | 阅读跨周期、跨栏目、可归档的组合判断 | 周 / 月 / 专题内参，趋势和信号组合，风险边界与复盘线索 | 泛泛观点合集、单篇趋势报告换壳 |

## 今日观察

今日观察不是普通日报、热点摘要、趋势预告或 Card 扩写。它每天只选一个最有商业冲突和叙事价值的对象，写成一篇有商业钩子、有角色变化、有公开场面、有可带走判断的商业观察。

固定页面构成：

1. 日期 / 标签 / 搜索筛选栏。
2. 当天主文章入口和摘要。
3. 当天关联商业信号卡。
4. 当天关联观点卡。
5. 相关文章详情页。
6. 关联内容索引。

写作边界：

- 不把企业内部购买动机、真实 ROI、预算审批、客户内部验收、销售转化、组织阻力或长期满意度写成事实，除非公开材料直接证明。
- 不展示 Raw、Pool、gate、字段、同步、入库等内部生产语言。
- 不恢复“今日判断”作为首页或栏目主口径。

## 商业信号

商业信号是“少数值得精读的商业事实库”。当前正式前台信号只保留三类：

```text
product_service / funding / case
```

生成规则：

- eligible fact `core_pool` + `raw_qc_decision=allow` 默认全部生成正式前台 `signal_card`；`important_viewpoint_or_article` 走观点卡 / 观点索引。
- 不再用单日数量上限、人工精选名额或 selected change cards 截流。
- 如果某条不适合前台，应回到 Raw-to-Pool 修正 `core_pool` 资格，不在卡片生成阶段静默丢弃。

商业信号页固定能力：

1. 日期查看。
2. 关键词和正式 tags 检索。
3. 融资 / 产品服务 / 案例分组。
4. 详情页展示事实、商业变量、来源、边界和相关内容。
5. 关联观点卡和趋势候选。

## 前沿观点

前沿观点作为商业信号相邻内容流，不作为独立一级栏目。观点只证明“谁在何时何处说了什么”，不替代公司公告、客户证据、财报、监管文件等事实来源。

固定链路：

```text
opinion_intake -> duplicate merge -> four-tier rating -> cardcopy -> site sync
```

四档：

| 档位 | 前台位置 |
|---|---|
| `feature` | 今日观察主推观点区或重点观点入口 |
| `sidebar` | 商业信号页观点模块 / 侧栏 |
| `archive` | 不进前台 |
| `discard` | 不进前台 |

观点详情页按“人 + title + 观点 + 时间线”组织。列表可截短翻译；观点详情页和人物详情页必须读取完整翻译字段。

## 趋势追踪

趋势追踪不是卡片聚合页，也不是公司案例库。它判断一组 AI 商业变化是否正在变成连续方向、谁会受影响、谁可能付钱、证据是否足够、风险在哪里。

趋势资产：

- `trend_candidate`：候选状态记录，负责观察方向、证据增强、分化、降温、修正。
- `trend_report`：正式趋势文章，分为趋势快报和深度报告。

当前规则：

- 单条新闻、单张卡或单个观点不能触发正式趋势。
- 正式趋势必须有多源事实、场景或案例支撑，并写清风险边界、信息缺口和后续观察变量。
- 不再把“反证”作为硬准入条件。
- 不再要求固定 `7 / 30 / 90` 时间窗口；如果使用时间窗口，必须来自页面或报告的具体内容设计，而不是硬门槛。
- 不写投资建议，不写“马上行动”。

趋势追踪栏目页固定能力：

1. 日期、关键词和 tags 筛选。
2. 正在观察的方向。
3. 真实趋势候选。
4. 趋势报告入口。
5. 关联商业信号、观点和内参。
6. 详情页展示支撑信号、观点参照、风险边界和后续变量。

## 商业内参

商业内参是高阶组合判断报告，不是单篇趋势深度报告，也不是普通专题文章。

它回答：

- 一个方向是否正在形成结构性变化。
- 哪些机会只是短期热度，哪些可能进入长期判断。
- 哪些公司、客户、场景、资本、技术、监管和观点变量正在一起变化。
- 老板应如何理解这个方向的中期机会和风险。

商业内参可引用商业信号、成熟变化短专题、趋势候选、趋势报告和前沿观点，但不能绕过事实证据、来源边界和质量门。

## 合并与降级规则

1. 同一数据的不同排序方式，优先做成筛选或视图，不新增栏目。
2. 用户任务不同，才保留独立栏目。
3. 只服务后台生产的能力，不进入前台导航。
4. 对外解释不清的能力，先降级为后台能力。
5. 无法关联到商业信号、趋势追踪、商业内参或明确人工判断来源的内容，不进入正式前台展示。
