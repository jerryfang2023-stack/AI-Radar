# 观澜数据中心 V4 前台信息架构

状态：已确认  
版本：`SITE-V4.0-data-center-frontstage-ia-v1.2`  
日期：2026-07-16  
上游文档：`SITE-V4.0-data-center-frontstage-prd-v1.2`

## 1. 架构目标

本信息架构用于确定观澜数据中心 V4 的栏目、页面、模板、页面模块和跳转关系。

页面以事件为核心，以最少必要信息支持内部浏览、搜索、筛选和关联查询。数据层不设置数据概览、来源评价、质量覆盖、推荐、评分或教育性模块。

## 2. 全站结构

```text
观澜
├─ 数据中心
│  ├─ 商业事件（默认）
│  ├─ FDE 实施
│  ├─ AI 硬件
│  ├─ 社群情报
│  ├─ 一线观点
│  └─ 实体索引（公司 / 机构 / 模型 / 产品 / 服务）
└─ 应用中心
   └─ 行业报告
      ├─ 机会地图
      ├─ 周报
      └─ 月报
```

网站不设置独立首页。根入口进入当日商业事件列表。

全站顶栏只展示正式横版 Logo，不展示数据中心/应用中心切换或全局搜索。左侧栏分为“数据中心”和“应用中心”两个分组，承载全部栏目导航。

Tag 结果页属于功能页，不进入左侧导航。

## 3. 页面模板

### 3.1 数据列表模板

由六个数据栏目共用：

- 商业事件；
- FDE 实施；
- AI 硬件；
- 社群情报；
- 一线观点；
- 实体索引。

统一模块顺序：

1. 仅含正式 Logo 的全站顶栏；
2. 数据中心/应用中心分组侧边栏；
3. 栏目标题；
4. 栏目搜索与筛选工具栏；
5. 数据列表；
6. 需要时显示数字分页。

不设置：

- 首屏宣传型大标题；
- 数据概览；
- 统计卡片；
- 来源与证据模块；
- 质量与覆盖模块；
- 推荐内容。

### 3.2 数据详情模板

由事件、FDE 实施、AI 硬件、社群情报和一线观点共用基础结构：

- 简洁路径；
- 主内容；
- 关联侧栏；
- 下方延伸数据。

不同数据类型只替换必要字段和关联模块。

### 3.3 实体详情模板

由公司和产品共用：

- 实体档案；
- 关联实体；
- 事件时间线；
- 其他关联数据。

### 3.4 结果模板

由栏目搜索结果和 Tag 结果页共用：

- 当前关键词或 Tag；
- 数据类型切换；
- 分组结果或单类型完整结果；
- 数字分页。

### 3.5 应用模板

行业报告中的机会地图、周报和月报沿用现有主要样式、结构和核心交互，仅统一：

- 全站导航；
- 数据记录链接；
- 页面末尾的“相关数据”模块。

行业报告不设置关联路径模块。

## 4. 数据列表页

### 4.1 商业事件列表

事件默认页采用仅展示当天记录的单列事件流。

页面模块：

1. 仅含正式 Logo 的全站顶栏；
2. 数据中心/应用中心分组侧边栏；
3. 栏目名“商业事件”、栏目说明与辅助数据日期；
4. 合并为一行的关键词搜索、事件类型、Tag 与“更多筛选”；
5. 当日事件流。

事件工具栏保持紧凑，包含：

```text
搜索事件标题、公司、产品或关键词
搜索
全部事件类型
Tag
更多筛选
清除条件
```

事件类型使用下拉筛选，不再占用独立横向标签行，不显示类型数量。

事件列表示例：

```text
[模型、产品与服务] 中文事件标题  [Tag] [Tag]
[商业合作]         中文事件标题  [Tag] [Tag]
[部署与案例]       中文事件标题  [Tag] [Tag]
```

每条事件只显示：

- 一个事件类型；
- 中文标题；
- 最多两个 Tag。

整行可点击进入事件详情。不显示摘要、来源、统计、原始英文标题或逐条日期。

事件默认页只显示当天记录，不显示历史日期分组、独立时间行或数字分页。当前数据日期以小字显示在栏目标题旁，作为辅助信息。历史事件通过事件栏目搜索和筛选进入结果列表。

### 4.2 实体索引

公司与产品合并为一个按名称排列的索引栏目，不按日期分组。栏目位于“一线观点”之后。

索引顶部提供紧凑切换：

```text
全部  公司与机构  产品与模型
```

公司索引项显示公司名称、可用别名和最多两个 Tag；产品索引项显示产品或模型名称、所属公司、产品类型和最多两个 Tag。点击后分别进入原有公司详情或产品详情。

旧 `view=companies` 与 `view=products` 地址自动映射到实体索引的对应类型筛选。

### 4.3 日期型数据列表

以下栏目按日期倒序并按天分组：

- FDE 实施；
- AI 硬件；
- 社群情报；
- 一线观点。

各栏目沿用事件页的单列记录结构：

- FDE 实施：中文事件标题、实施阶段、最多两个 Tag；
- AI 硬件：中文事件标题、硬件类型、最多两个 Tag；
- 社群情报：中文标题、最多两个 Tag；
- 一线观点：中文观点标题、人物、最多两个 Tag。

除事件默认页外，其他完整列表均为每页二十条并采用数字分页。

## 5. 日期与分页

### 5.1 日期筛选

事件默认页仅展示当天数据，日期以栏目标题旁的小字显示，不单独成行。

其他日期型栏目默认展示最新数据，不限定只看当天。

日期筛选提供：

- 今天；
- 近 7 天；
- 近 30 天；
- 自定义。

其他日期型栏目选择单日时只展示该日期分组。当前日期条件始终可见并可以清除。事件历史查询的日期条件收纳在“更多筛选”中。

首期不设置：

- 日历视图；
- 复杂时间轴；
- 上一天或下一天按钮。

### 5.2 数字分页

分页形式：

```text
上一页  1  2  3  4  …  20  下一页
```

规则：

- 每页二十条；
- 翻页保留关键词和筛选条件；
- 返回列表时恢复页码和滚动位置；
- 不提供每页条数切换；
- 不使用无限滚动。

## 6. 详情页

### 6.1 事件详情

采用“主内容 + 关联侧栏 + 下方延伸数据”的双栏结构。

主栏：

1. 中文标题；
2. 原始标题；
3. 日期、精确事件类型、最多五个 Tag；
4. 主体、动作、对象和已披露数据；
5. 原始来源。

侧栏：

- 相关公司；
- 相关产品；
- 相关 FDE 实施；
- 相关 AI 硬件。

下方：

- 相关社群情报；
- 相关一线观点；
- 相关事件。

无数据的关联模块不展示。“未披露”只在所属事实字段中显示，不形成单独大模块。原始来源位于主栏末尾，不设置独立证据链区域。

### 6.2 公司详情

采用“实体档案 + 事件时间线”结构。

顶部：

- 公司名称；
- 实体类型；
- 最多五个 Tag。

侧栏：

- 基本身份；
- 相关产品。

主栏：

- 按天分组的相关事件时间线。

下方：

- FDE 实施；
- AI 硬件；
- 社群情报；
- 一线观点。

每个关联模块默认显示最近三条并提供“查看全部”。无数据模块不展示。

### 6.3 产品详情

采用“实体档案 + 事件时间线”结构。

顶部：

- 产品名称；
- 产品类型；
- 最多五个 Tag。

侧栏：

- 所属公司；
- 产品类型信息。

主栏：

- 按天分组的相关事件时间线。

下方：

- FDE 实施；
- 社群情报；
- 一线观点。

每个关联模块默认显示最近三条并提供“查看全部”。无数据模块不展示。

### 6.4 FDE 实施详情

采用双栏结构。

主栏：

- 中文标题；
- 日期、实施阶段、最多五个 Tag；
- 客户需求；
- 使用场景；
- 交付组成；
- 已披露结果；
- 原始来源。

侧栏：

- 相关公司；
- 相关产品；
- 原始事件。

下方：

- 相关事件；
- 同实体 FDE 记录。

### 6.5 AI 硬件详情

采用双栏结构。

主栏：

- 中文标题；
- 日期、硬件事件类型、最多五个 Tag；
- 部件与算力层级；
- 供需主体；
- 已披露制程、容量、地点、金额和出货日期；
- 原始来源。

侧栏：

- 相关公司；
- 相关产品；
- 原始事件。

下方：

- 相关事件；
- 同实体硬件记录。

### 6.6 社群情报详情

采用“内容正文 + 关联侧栏”结构。

主栏：

- 中文标题；
- 日期、最多五个 Tag；
- 原始内容；
- 原始入口。

侧栏：

- 作者；
- 社群；
- 相关公司；
- 相关产品。

下方：

- 相关事件。

不生成额外摘要或编辑解读。

### 6.7 一线观点详情

采用“内容正文 + 关联侧栏”结构。

主栏：

- 中文观点标题；
- 日期、最多五个 Tag；
- 中文翻译全文；
- 原文；
- 原始入口。

侧栏：

- 人物；
- 角色；
- 机构；
- 相关公司；
- 相关产品。

下方：

- 相关事件。

不展示热度、评分、立场判断或编辑点评。

## 7. 搜索与 Tag 结果页

### 7.1 数据类型切换

```text
全部
事件
实体索引
FDE 实施
AI 硬件
社群情报
一线观点
```

### 7.2 “全部”状态

结果按数据类型分区，每类先显示最多五条，并提供“查看全部”。

### 7.3 单类型状态

切换到单一类型后显示完整列表和数字分页：

- 事件、FDE、硬件、社群、观点按日期分组；
- 实体索引按名称排列。

不显示搜索摘要、结果评分、热度或推荐。

### 7.4 栏目搜索

- 在当前栏目工具栏输入关键词并按回车进入栏目结果页；
- 支持中文、英文和专有名词；
- 搜索结果页保留原关键词；
- 不提供 AI 问答、联想答案或复杂即时下拉；
- 从详情返回时恢复数据类型、页码和滚动位置。

### 7.5 Tag 结果

点击任一 Tag 进入统一 Tag 结果页：

- 顶部显示当前 Tag；
- 默认展示“全部”；
- 支持数据类型切换；
- 支持继续输入关键词；
- 使用与栏目搜索一致的结果模板。

## 8. 应用中心回链

机会地图、周报和月报保留现有主体结构。

新增统一“相关数据”模块：

- 正文中的公司、产品和事件名称可以直接跳转；
- 页面末尾按事件、公司、产品、FDE 实施和 AI 硬件分组；
- 每条只显示中文标题或实体名称、日期和类型；
- 默认展示必要记录，其余通过“查看全部”进入对应数据列表；
- 社群情报和一线观点仅在被应用内容实际引用时出现；
- 不显示证据评分、覆盖率或引用数量。

## 9. 跳转与返回

详情页顶部显示简洁路径，例如：

```text
数据中心 / 商业事件 / 当前事件
```

规则：

- 列表页不显示多余路径导航；
- 列表进入详情后，返回时恢复筛选、页码和滚动位置；
- 从实体页进入关联事件后，返回时恢复实体页位置；
- 站内关联记录在当前页面打开；
- 原始来源和原始入口作为外部链接在新窗口打开。

“查看全部”不创建新的关联专页：

- 公司页的全部事件进入事件列表并带入公司筛选；
- 产品及其他实体关联进入对应栏目列表并带入条件；
- 应用中心的全部相关数据复用对应栏目列表；
- 页面顶部显示当前生效条件并允许一键清除。

## 10. 空状态

- 无当日事件：`当日暂无事件`
- 筛选无结果：`未找到符合当前条件的数据`
- 搜索无结果：`未找到“关键词”相关数据`
- 无数据的关联模块直接隐藏；
- 不推荐其他内容；
- 不自动扩大搜索范围；
- 提供“清除筛选”或“返回全部”。

默认事件页示意图已经确认，实施时必须体现：

- 观澜 VI 配色与正式横版 Logo；
- 顶栏只保留 Logo；

- 栏目标题与辅助数据日期同处一行；
