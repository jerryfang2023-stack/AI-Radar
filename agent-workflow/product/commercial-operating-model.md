# Commercial Operating Model

更新时间：2026-05-03  
owner：`strategy` / `pm` / `commercial-site`  
状态：draft

## 1. 定位

本文件定义观澜AI第一阶段商业化运营模型。

观澜AI不是免费资讯站，也不是投资建议服务。它的商业化核心是：

> 为商业决策者提供持续更新、证据可追踪、边界清晰的 AI 机会判断内参。

## 2. 商业化阶段

### 第一阶段：会员制情报访问

核心交付：

- 今日要点
- 关键信号
- 机会解码
- 商业内参
- 标签、搜索和关系网络

路径：

```text
访客 -> 注册 -> 7 天试读 -> 月/季/年订阅
```

### 第二阶段：行业定制简报

面向高意向用户，围绕特定行业、赛道、客户类型或资源需求输出定制判断。

### 第三阶段：机会连接与项目化服务

围绕机会方向提供资源连接、项目筛选、验证路径和合作撮合。

## 3. 免费与付费边界

| 用户状态 | 可见内容 | 目标 |
|---|---|---|
| 访客 | 首页、少量样例、注册入口 | 理解价值 |
| 注册试读 | 部分今日要点、样例关键信号、少量机会解码 | 体验判断质量 |
| 试读到期 | 公开内容和账户页 | 引导订阅 |
| 付费会员 | 完整今日要点、关键信号、机会解码、商业内参、搜索和关系网络 | 持续决策辅助 |
| 企业/私享 | 行业简报、闭门报告、资源连接 | 高价值服务 |

## 4. 会员权益

第一阶段建议会员权益：

- 每日今日要点完整阅读。
- 关键信号访问，含 Front Signal、Structured Signal 和 Builders 观点。
- 机会解码长文访问。
- 商业内参访问。
- 趋势温度、标签、搜索和关系网络访问。
- 每周精选机会摘要。
- 未来 newsletter / signals letter。

不承诺：

- 投资回报。
- 确定性商业成功。
- 替用户下最终决策。
- 独家内幕消息。

## 5. 样例内容策略

样例内容应展示“判断能力”，不是展示全部数据库。

可公开：

- 1 篇样例今日要点。
- 3-5 条样例关键信号。
- 1-2 篇样例机会解码。
- 1 个趋势温度样例。
- 1 个 Builders 观点样例。

样例内容要求：

- 有观点。
- 有证据。
- 有风险边界。
- 不含后台流程语言。
- 不让用户误解为投资建议。

## 6. Newsletter 转化

newsletter 第一阶段定位为“转化入口”，不是完整付费交付替代品。

建议结构：

1. 今日一句判断。
2. 2-3 条关键 Signal。
3. 1 条 Builders 观点。
4. 1 个机会观察。
5. 进入完整内参的 CTA。

CTA 应克制：

```text
查看完整判断依据
进入今日内参
继续观察这个方向
```

避免：

```text
马上投资
错过就没了
稳赚机会
```

## 7. 企业版 / 私享内参

保留申请入口，但不与普通会员路径混淆。

适用场景：

- 特定行业机会扫描。
- 某类客户或渠道资源判断。
- 赛道趋势复盘。
- 闭门简报。
- 机会连接和项目筛选。

企业版交付必须有边界：

- 提供判断依据、风险边界和后续观察问题。
- 不替客户做投资或经营决策。
- 不承诺融资、成交或收益。

## 8. 到期与续订路径

试读到期用户：

- 可继续访问公开内容。
- 访问受限内容时提示订阅。
- 显示月 / 季 / 年方案。

付费到期用户：

- 保留账户与历史基础信息。
- 提示续订恢复完整访问。
- 不使用焦虑式倒计时或夸张收益承诺。

## 9. 商业化指标

第一阶段建议跟踪：

- 首页注册转化率。
- 注册到试读激活率。
- 今日要点阅读完成率。
- 试读期内访问频次。
- 机会解码详情点击率。
- Builders 观点详情点击率。
- 试读到订阅转化率。
- 到期续订率。
- 企业版咨询线索数。

## 10. 与产品模块关系

| 模块 | 商业化作用 |
|---|---|
| 首页 | 解释为什么值得注册 |
| 今日要点 | 每日打开理由，第一付费交付 |
| 关键信号 | 证明判断有证据，沉淀可精读的信号报告 |
| Builders 观点 | 增强观点解释层和行业鲜活度 |
| 机会解码 | 长期订阅资产 |
| 商业内参 | 高阶组合判断能力 |
| Tags / Search | 提升复访和探索效率 |
| Admin | 后台运营，不对普通用户展示 |

## 11. 下一步任务

1. Copy Agent 统一会员页、订阅页、购买页文案。
2. UI/UE Agent 复核会员路径的页面节奏。
3. Dev Agent 后续准备真实支付和云端权限方案。
4. QA Agent 做未登录、试读有效、试读到期、付费会员、Admin 多身份验收。
5. PM Agent 判断 newsletter 何时进入 P1。
