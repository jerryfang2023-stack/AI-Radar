# 观澜AI｜WaveSight AI 文案规范

## 文案定位

观澜AI的文案不是普通营销页文案，也不是技术产品说明，而是“面向商业决策者的 AI 商业内参表达”。

它要让目标用户感到：

- 有判断力
- 有证据边界
- 有商业敏感度
- 值得继续阅读
- 值得申请访问

但不能让用户感到：

- 被煽动
- 被下指令
- 被推销
- 被技术术语压迫
- 像在看内部运营说明

## 核心表达原则

### 0. 言之有物，简洁克制，精炼有力

每句话都必须提供信息、判断或边界。

推荐：

- “市场形成共识前，先看证据在哪里累积。”
- “少追概念，多看谁正在变成生意。”
- “当前更像早期信号，仍需观察客户采用和商业化证据。”

避免：

- “我们致力于打造领先平台。”
- “赋能企业拥抱智能新时代。”
- “帮助用户实现高质量发展。”
- “全面提升商业认知水平。”

检查标准：

- 删除后不影响理解的句子，删掉。
- 不能回答“所以呢”的句子，重写。
- 只有形容词、没有证据或判断的句子，重写。
- 一句话能说清，不写两句。
- 能用具体词，不用大词。

### 0.1 不用内部话术，不以说服为目标

观澜AI公开前台的文案只做三件事：说明发生了什么、它可能意味着什么、边界在哪里。不要写成产品团队对页面功能的解释，也不要写成销售话术。

推荐：

- “从 AI 变化中，筛出值得继续观察的商业信号。”
- “这条信号目前由单信源支撑，后续需要继续观察客户采用。”
- “它更像一个早期变化，而不是已经完成验证的机会。”

避免：

- “这页只做筛选和判断入口。”
- “本模块用于展示关系网络。”
- “帮助用户快速识别高价值机会。”
- “强证据说明该机会已经成立。”
- “建议立即验证并进入行动清单。”

检查标准：

- 凡是像在解释页面功能的句子，改成客户能得到的信息或判断边界。
- 凡是像在劝用户相信某个结论的句子，改成事实、来源、观察或不确定性。
- 凡是内部团队才会说的词，删除或转译。

### 1. 有观点，但不替客户下最终判断

推荐：

- “这类信号值得继续观察。”
- “来源和事实正在从概念走向商业化。”
- “当前更像早期变化，仍需要反证。”
- “它的价值不在单点新闻，而在后续是否形成连续信号。”

避免：

- “必须立刻行动。”
- “一定是机会。”
- “确定值得投资。”
- “马上入局。”
- “保证回报。”

### 2. 对外讲利益，不讲内部流程

推荐：

- “更快看清哪些 AI 变化值得继续跟踪。”
- “把碎片信息压缩成可讨论的商业判断。”
- “从新闻、融资、客户采用和产品变化中寻找连续证据。”

避免：

- “新增 Markdown 后自动沉淀。”
- “同步脚本会更新网站。”
- “拆解数据字段。”
- “自动生成 JSON。”

### 3. 说人话，不堆概念

推荐：

- “谁在增长，谁被客户采用。”
- “哪些变化只是热闹，哪些正在形成商业证据。”
- “先看信号，再看机会，再看趋势是否延续。”

避免：

- “AI 赋能生态闭环。”
- “全链路智能化矩阵。”
- “商业认知升级引擎。”
- “一站式 AI 决策中台。”

### 4. 克制但有吸引力

文案可以有力度，但不鸡血。

推荐语感：

- “从热闹里筛信号，从信号里看机会。”
- “别等行业洗牌后，才开始看 AI。”
- “市场形成共识前，先看证据在哪里累积。”
- “少追概念，多看谁正在变成生意。”

避免：

- “颠覆未来。”
- “错过再等十年。”
- “财富密码。”
- “风口红利。”

## 品牌固定表达

品牌名：

- 观澜AI｜WaveSight AI

定位语：

- 观AI之澜，识商业之势。

品牌解释：

- 面向商业决策者的 AI 信号、趋势与机会观察系统。
- 从 AI 热点中提取信号，从信号中形成判断，从判断中发现可继续验证的机会。

## 首页文案规范

首页目标：

- 让用户在 5 秒内知道观澜AI做什么。
- 让老板、行业大咖、资源方、投资观察者觉得“这里有判断价值”。
- 不展示太多数据，不像后台。

首页推荐表达：

- “在市场形成共识前，看见 AI 商业变化。”
- “把 AI 热点压缩成可讨论、可追踪、可复盘的商业判断。”
- “每天从融资、客户、收入、产品采用和产业落地中筛选关键变化。”

首页避免：

- “我们每天自动同步 Markdown。”
- “这是 AI 商业机会情报系统。”
- “老板 / 行业大咖 / 资源方 / 民间投资者”直接写成板块标题。
- 大量解释产品内部功能。

## Daily Brief 文案规范

Daily Brief 是“每日商业内参”，不是行动建议书。

结构建议：

1. 今日主线
2. 关键 Signals
3. 机会卡观察
4. 趋势观察
5. 反证 / 风险边界

推荐表达：

- “今天的核心变化集中在……”
- “它的价值不在单点新闻，而在后续是否能形成连续信号。”
- “本页呈现信号、机会卡和趋势观察之间的关系。”
- “不替代最终经营、投资或合作判断。”

避免：

- “下一步应该做……”
- “建议立即验证……”
- “可行动机会……”
- “必须进入第一批验证清单。”

## Signals 文案规范

Signal 标题必须是：

```text
事件 + 商业含义
```

推荐：

- “Amazon Quick 桌面端预览发布：企业 Agent 开始抢占员工桌面入口”
- “Haast 完成 A 轮融资：合规引擎正在变成 Agent 生产化前置件”

避免：

- “Amazon Quick”
- “Haast”
- “AI Agent”
- “融资新闻”

Signal 摘要应该回答：

- 发生了什么？
- 为什么这不是普通新闻？
- 它可能对应什么商业变化？
- 有什么证据和反证？

Signals 前台用词：

推荐：

- “信源状态”
- “单信源”
- “多信源”
- “来源与事实”
- “支撑信息”
- “指向机会”
- “关联趋势”
- “查看信号”

避免：

- “证据链”
- “强证据”
- “来源明确”
- “阅读证据”
- “机会确定”
- “本页只做筛选和判断入口”
- “下一步验证”

Signals 文案必须克制：只阐述信号和来源边界，不把单条新闻写成确定机会，也不替用户下最终判断。

## Opportunities 文案规范

机会卡名称必须是机会名称，不是公司名称。

推荐：

- “AI 招投标 Agent”
- “企业桌面端 Agent 执行层”
- “合规引擎 / Agent 治理”

避免：

- “AI工程仿真软件-JuliaHub”
- “Hightouch”
- “某公司机会”

机会卡说明要写：

- 适合谁
- 解决什么问题
- 为什么现在值得观察
- 有哪些证据
- 有哪些不确定性

## Trends 文案规范

趋势不是“热度词”，而是“连续证据的方向”。

推荐：

- “企业 Agent 正在从聊天入口走向桌面执行层。”
- “AI 治理从合规报告进入生产化前置件。”
- “端侧推理仍有融资热度，但商业落地证据需要继续观察。”

避免：

- “AI Agent 很火。”
- “趋势持续上升。”
- “未来可期。”

## CTA 文案规范

推荐：

- “申请查看完整信号”
- “申请进入完整情报层”
- “查看今日简报”
- “进入机会库”
- “查看信号”
- “查看来源”

避免：

- “立即抢占红利”
- “马上赚钱”
- “免费领取财富密码”
- “一键暴富”

## 禁用词与谨慎词

禁用：

- 财富密码
- 风口红利
- 颠覆未来
- 确定性机会
- 保证收益
- 必投
- All in
- 生态闭环
- 赋能
- 中台
- 一站式

谨慎使用：

- 行动建议
- 投资机会
- 战略机会
- 高确定性
- 必须
- 立即

## Copy Agent 工作流程

每次改文案前，必须先确认：

1. 页面目标。
2. 目标用户正在做什么判断。
3. 哪些表达是对外利益，哪些是内部流程，内部流程必须删除或转译。
4. 是否存在“替用户下最终判断”的风险。
5. 是否符合本 `COPY.md`。

每次输出文案后，必须检查：

- 是否有内部流程词。
- 是否有过度承诺。
- 是否有空泛概念。
- 是否言之有物、简洁克制、精炼有力。
- 是否存在删掉也不影响理解的废话。
- 是否有公司名误写成机会名。
- 是否有标题过长导致页面断行难看。
- 是否和 `DESIGN.md` 的页面层级匹配。
