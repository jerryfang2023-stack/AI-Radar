## 1. 文案定位

观澜AI的文案不是 AI 新闻站、工具导航站、热点内容号，也不是咨询公司白皮书。

它面向企业经营者、资源型合伙人、行业操盘手、投资观察者，以及真正关心 AI 如何影响商业的人。它要帮助用户：

- 看懂变化。
- 判断影响。
- 识别行业、岗位、流程中的机会。
- 找到企业 AI 起步点。

全站文风定位为：

```text
冷静、有判断、有温度的商业内参风。
```

读起来要像一个长期在一线观察产业变化、亲自做项目、懂企业现实处境的人，在帮用户过滤噪音、判断影响、找到起点。

## 2. 品牌声音

观澜AI同时具备四种气质：

| 气质 | 说明 | 禁区 |
|---|---|---|
| 商业记者的敏锐 | 能看出一条新闻背后的变量变化 | 只复述新闻标题 |
| 产业研究者的冷静 | 有证据、有反证、有边界 | 把热度写成结论 |
| 企业顾问的判断 | 能落到行业、岗位、流程、成本、风险 | 给空泛口号 |
| 创业者的体感 | 理解企业落地时的犹豫、预算和执行难度 | 装腔、说教、恐吓 |

最终效果：

- 有判断，但不装。
- 有锋芒，但不攻击。
- 有深度，但不晦涩。
- 有温度，但不煽情。
- 有共鸣，但不讨好。

## 3. 最重要的硬规则：避免 AI 味

宁可短一点、真一点，也不要完整但空泛。

### 3.1 不要机械分点

避免动不动写：

```text
首先 / 其次 / 最后 / 总之 / 值得注意的是 / 综上所述
```

如果需要分层，可以自然写，不要像模型生成的标准答案。

### 3.2 不要重复句式

以下句式可以偶尔用，但不能整页反复出现：

```text
不是……而是……
表面看……真正……
对企业来说……
这意味着……
核心在于……
```

重复句式会让文案看起来像自动生成。

### 3.3 不要空泛判断

没有具体行业、岗位、流程、场景支撑时，不写：

```text
AI 正在重塑各行各业
企业必须拥抱 AI
未来已来
这是一次产业级机会
AI 将带来效率革命
```

要把判断落到更具体的位置：销售线索、客服知识库、内容生产、采购应付、合规审计、研发测试、数据问答、门店运营、供应商协同。

### 3.4 不要堆概念

以下词不是绝对禁用，但公开前台必须高度克制：

```text
赋能
重构
闭环
生态
范式
底座
抓手
价值链路
数智化升级
中台
一站式
```

能用朴素语言说清楚，就不要堆概念。

### 3.5 不要写成冷冰冰的报告

观澜AI不是冰冷的咨询报告。文案可以有一点人的思考、迟疑、提醒和分寸感。

好的观澜句子不是“更完整”，而是让读者少一点噪音，多一个清楚的下一步想法。

## 4. 用户心理

用户不是来听热闹的。他们关心：

- 这件事和我有什么关系？
- 会不会影响我的行业？
- 会不会影响我的团队？
- 哪些岗位会先变化？
- 哪些流程值得先试？
- 我现在不动，会不会错过？
- 我现在动，又怕不值得。

所以不要只写“AI 发生了什么”，要写“这件事可能怎样影响企业经营”。

## 6. V2 栏目边界

V2 普通前台只保留四个一级栏目：

```text
今日要点 / 关键信号 / 机会解码 / 商业内参
```

| 前台栏目 | 用户任务 | 文案重点 | 不要写成 |
|---|---|---|---|
| 今日要点 | 今天先看什么，为什么 | 短、稳、有结论，像早上给老板看的商业提醒 | 新闻摘要、日报列表、行动建议书 |
| 关键信号 | 哪些变化值得继续跟踪 | 不只写发生了什么，还要写为什么进入观澜AI | 公司名列表、融资新闻、工具资讯 |
| 机会解码 | 哪些方向可继续观察或小范围试 | 适合谁、不适合谁、前置条件、风险 | 万物皆机会、公司案例库、执行方案 |
| 商业内参 | 跨周期组合判断 | 核心判断、关键事实、商业影响、机会与风险 | 情绪公众号、泛观点合集、公开热榜 |

降级或后台化：

- The Point：统一表达为 `观点校准` 或 `Builders 观点`，不是一级栏目。
- Trends：统一表达为 `趋势背景`、`热力输入`、`证据变化`，不是一级栏目。
- Scoring / Priority Engine：后台能力，不进入普通前台文案。
- Tags：搜索、筛选和关系网络能力，不作为一线栏目。

## 7. 核心表达原则

### 7.1 每句话有信息增量

每句话至少提供一种东西：

- 事实。
- 判断。
- 来源状态。
- 商业含义。
- 风险边界。
- 后续观察变量。

删掉不影响理解的句子，删掉。不能回答“所以呢”的句子，重写。

### 7.2 有判断，但不替用户下最终判断

推荐：

- “这个方向值得继续看。”
- “更适合先小范围试。”
- “证据还不够。”
- “风险主要在交付成本和数据可得性。”
- “这件事可能先影响销售、客服、内容、运营这类高频流程。”

避免：

- “必须马上行动。”
- “确定性机会。”
- “一定值得投资。”
- “不学就会被淘汰。”
- “保证收益。”

### 7.3 写商业含义，不写页面说明

公开前台不要解释“这个页面做什么”。不要写：

```text
本页用于
入口
字段
后台
同步
JSON
Markdown
脚本
```

把内部话转成用户能得到的判断：

| 内部说法 | 对外说法 |
|---|---|
| 本页用于展示关系网络 | 这几条内容指向同一个商业变化 |
| 同步脚本更新网站 | 今日内容已更新 |
| 字段自动沉淀 | 这条信息已进入后续观察 |
| 后台评分 | 证据状态 / 观察优先级 |

### 7.4 写来源与事实，不做证据表演

避免：

```text
证据链
强证据
来源明确
阅读证据
机会确定
```

推荐：

```text
来源与事实
来源状态
支撑信息
多源支撑
单信源
证据缺口
仍需观察
```

## 8. 标题系统

| 标题类型 | 规则 | 长度建议 |
|---|---|---|
| 首页 Hero | 品牌价值 + 判断对象，不写功能说明 | 16-28 中文字 |
| 一级栏目标题 | 一句话价值表达，不加解释型副标题 | 14-24 中文字 |
| 卡片标题 | 信息完整，不只写公司名 | 18-34 中文字 |
| Signal 标题 | 事件 + 商业含义 | 24-42 中文字 |
| Opportunity 标题 | 机会方向，不写公司名 | 8-18 中文字 |
| 商业内参标题 | 周期判断或主题判断 | 16-32 中文字 |
| 知识卡标题 | 对象 + 判断变量 | 12-28 中文字 |

字体适配：

- 页面大标题使用中文衬线，句子要短。
- 卡片标题优先一到两行可读。
- 标题过长时先压缩文案，不靠放大容器硬塞。
- 英文标签可用全大写，但正文不混入过多英文术语。

## 9. 摘要与导语

摘要回答四件事：

1. 发生了什么。
2. 它可能影响什么行业、岗位或流程。
3. 当前来源状态如何。
4. 还缺什么证据。

推荐：

```text
这条信号目前有多源支撑，但客户付费证据还需要继续观察。
它更像一次入口竞争的前奏，不是已经完成验证的机会。
本期关注的不是单家公司，而是一组信号共同指向的流程变化。
```

避免：

```text
这是一条强证据链。
机会已经确定。
建议马上进入验证。
本模块用于展示来源字段。
```

## 10. 四个一级栏目文案规范

### 10.1 今日要点

文风：短、稳、有结论，像一个早上给老板看的商业提醒。

必须回答：

- 今天最值得关注的变化是什么？
- 背后的商业信号是什么？
- 可能影响哪个行业、岗位或流程？
- 哪些地方还不能下结论？

推荐语感：

```text
今天更值得关注的，不是某个模型参数又提升了多少。

真正的变化是，AI 正在从单点工具，慢慢进入企业流程。

对企业来说，下一步要看的不是哪个工具更火，而是哪些重复、标准化、耗人力的流程，会先被 AI 接过去。
```

避免：

- 新闻列表。
- 今日新增数量汇总。
- “建议立即验证”。
- “行动清单”。
- Raw / Pool / Heat Candidate / M 级线索。

### 10.2 关键信号

文风：像商业雷达，不像新闻列表。

Signal 标题必须是：

```text
事件 + 商业含义
```

推荐：

```text
Haast 完成 A 轮融资：合规引擎正在变成 Agent 生产化前置件
Microsoft Agent 365 GA：企业开始为 Agent 控制层单独付费
```

避免：

```text
Haast
Microsoft
AI Agent
融资新闻
```

摘要必须说明：

- 发生了什么。
- 为什么不是普通新闻。
- 它可能改变哪个行业、岗位或流程。
- 来源和反证在哪里。

### 10.3 机会解码

文风：有边界，不把所有东西说成机会。

机会标题必须是方向、客户场景或能力，不是公司名。

推荐：

```text
AI 招投标 Agent
企业桌面端 Agent 执行层
合规引擎与 Agent 治理
采购应付流程 Agent
```

必须说明：

- 适合谁。
- 不适合谁。
- 前置条件是什么。
- 风险在哪里。
- 哪些证据还不够。

推荐语感：

```text
这个方向值得看，但不适合所有企业马上跟进。

它更适合那些已经有稳定线索来源、销售流程相对标准化、愿意沉淀数据的团队。

如果企业本身流程混乱，销售动作不可复盘，直接上 AI 系统，最后很可能只是多买了一套工具。
```

### 10.4 商业内参

文风：给决策者看的简报，不是公众号情绪文，也不是研究报告黑话。

应有层次：

- 核心判断。
- 关键事实。
- 商业影响。
- 行业 / 岗位 / 流程映射。
- 国内迁移可能性。
- 机会与风险。
- 观澜结论。

推荐：

```text
本期关注的不是单家公司，而是一组信号正在指向同一个商业变化。
热度已经升高，但客户付费证据还集中在早期采用阶段。
```

禁止：

- “本期告诉你马上做什么。”
- “确定性机会已经出现。”
- “趋势很热，所以机会成立。”
- 把单篇机会解码直接等同于商业内参。

## 11. 嵌入模块表达

| 模块 | 允许表达 | 禁止表达 |
|---|---|---|
| 观点校准 | “这条观点可帮助修正判断，但不替代事实来源。” | “大佬已经证明趋势成立。” |
| Builders 观点 | “这位 Builder 的关注变量正在变化。” | “The Point 一级栏目。” |
| 趋势背景 | “过去 30 天，证据主要集中在……” | “趋势很热，未来可期。” |
| 商业热力图 | “热力集中在销售线索、客服知识库和内容生产流程。” | “公开机会榜 / 公司榜 / 岗位替代榜。” |
| 反证与边界 | “客户付费、交付成本和数据可得性仍未确认。” | 空泛免责声明 |

## 12. CTA 与会员态

CTA 要克制，像产品动作，不像销售话术。

| 状态 | 推荐 CTA | 避免 |
|---|---|---|
| 访客 | 查看今日判断 / 阅读商业信号 / 了解商业内参 | 立即抢占红利 |
| 登录前 | 登录继续阅读 / 注册试读 | 不看就晚了 |
| 试读会员 | 阅读完整内参 / 查看来源摘要 | 最后机会 |
| 到期用户 | 查看会员方案 / 续订后继续阅读 | 马上逆袭 |
| 付费会员 | 阅读完整报告 / 查看来源附录 / 进入关联内容 | 开通后稳赢 |
| 企业内参 | 申请企业内参 / 预约闭门简报 | 立即获得定制答案 |

锁定态规则：

- 不整页遮挡。
- 公开部分保留主题、主判断、部分来源摘要和边界。
- 会员层展开完整热力、来源、证据缺口和后续观察变量。
- 不制造错失焦虑。

## 13. 每日监测产物规则

| 产物 | 角色 | 标题规则 | 摘要规则 | 禁止 |
|---|---|---|---|---|
| Front Signal | 深度信号报告 | 事件 + 商业含义 | 多源事实 + 商业变量 + 反证 | 普通新闻稿 |
| Structured Signal | 入库信号卡 | 事件 + 指向变量 | 快速说明事实、来源、关联对象 | 堆字段 |
| Opportunity Deep Dive | 深度机会报告 | 机会方向，不写公司名 | 机会判断 + 成立边界 + 观察变量 | 硬凑深挖 |
| Trend Update | 趋势背景更新 | 趋势方向 + 证据变化 | 7 / 30 / 90 天证据变化 | 热度即趋势 |
| Point / 观点校准 | 判断输入 | 人物 / 机构 + 观点变化 | 支持、修正或质疑什么判断 | 观点当事实 |
| Business Brief | 周期组合判断 | 主题 / 周期判断 | 信号链 + 机会组合 + 风险边界 | 泛泛观点合集 |

每日内容不得把 AI HOT、follow-builders、HN、X、Reddit 等 discovery 通道直接写成事实主证据。

## 14. 知识卡片表达

知识卡片不是后台字段展示，而是可复用的判断资产。

| 卡片类型 | 标题 | 摘要 | 标签语气 |
|---|---|---|---|
| 信号库 | 事件 + 商业含义 | 这条信号关联哪些机会、趋势和来源边界 | 正式主题词，不随手造词 |
| 机会库 | 机会方向 | 关联哪些信号、客户场景和风险变量 | 描述客户 / 场景 / 流程 |
| 观点库 | 人物 / 机构 + 观点变量 | 观点支持、修正或质疑什么判断 | 不把人名当趋势 |
| 案例库 | 公司 / 产品 + 代表证据 | 这个案例能支撑哪个机会或信号 | 公司是证据，不是机会 |
| 趋势背景 | 趋势方向 + 状态 | 证据如何在 7 / 30 / 90 天变化 | 不写成热词墙 |

知识卡摘要公式：

```text
它是什么 + 支撑哪个判断 + 证据边界 / 反证
```

## 15. 禁用语与替代表

| 类型 | 禁用 / 谨慎 | 推荐替代 |
|---|---|---|
| 内部流程词 | 本页用于、入口、同步、字段、后台、JSON、Markdown、脚本、Raw、Pool | 删除，或转译为用户可见价值 |
| 证据表演词 | 证据链、强证据、来源明确、阅读证据 | 来源与事实、来源状态、支撑信息、多源支撑、证据缺口 |
| 确定性词 | 确定性机会、一定、必然、唯一、保证、稳赚 | 值得继续观察、可能、证据正在增强、仍需观察 |
| 行动指令 | 必须、立即行动、马上入局、下一步验证、行动清单 | 后续观察、建议关注变量、可小范围试 |
| 营销词 | 财富密码、风口红利、错过再等十年、颠覆未来、弯道超车 | 商业变化、早期信号、流程变化、结构性变化 |
| 概念堆叠 | 全链路智能化矩阵、生态闭环、一站式中台、智能认知引擎 | 具体流程、客户采用、成本变化、交付边界 |
| 观点越界 | 大佬证明、观点坐实、趋势已成立 | 观点校准、观点分歧、判断输入 |

## 17. 改写流程

当文案像 AI 写的、像新闻摘要、像营销号时，按这个顺序改：

1. 保留事实，不乱加信息。
2. 删除夸张词、口号词、空泛判断。
3. 找到这条信息背后的商业含义。
4. 尽量落到行业、岗位、流程、组织、效率、成本、风险。
5. 加入一点真实判断，但不要装腔。
6. 让语言更自然，有节奏，有人的体感。
7. 避免所有段落都用同一种句式。
8. 每段都要有信息增量，不要重复换句话说。

## 18. Copy QA 检查表

后续页面、内容、会员态和知识库任务按下表验收：

| 检查项 | 通过标准 |
|---|---|
| V2 栏目名 | 前台只出现今日要点 / 关键信号 / 机会解码 / 商业内参 |
| 降级模块 | The Point、Trends 不作为一级栏目表达 |
| 去 AI 味 | 无机械分点、重复句式、标准答案感和空泛套话 |
| 内部词 | 无 Markdown、JSON、同步、字段、后台、脚本、Raw、Pool 等 |
| 标题 | Signal 为事件 + 商业含义；Opportunity 不写公司名 |
| 判断边界 | 不写确定性机会、保证收益、马上行动 |
| 来源表达 | 使用来源与事实、来源状态、支撑信息、证据缺口 |
| 观点边界 | Builder / VC / Founder 观点只作校准，不作事实主证据 |
| 会员态 | CTA 克制，锁定态不制造焦虑 |
| 今日要点 | 不像新闻列表，不输出行动指令 |
| 机会解码 | 不硬凑 Deep Dive，不写公司名机会 |
| 商业内参 | 像周期判断报告，不像泛泛观点合集 |
| 知识卡片 | 摘要说明关联判断和证据边界，不堆字段 |
| 字体适配 | 大标题短、卡片标题不拥挤、移动端可读 |
| 信息量 | 删除后不影响理解的句子已删除 |

## 19. Copy Agent 工作流程

每次改文案前，必须确认：

1. 页面或内容的用户任务。
2. 目标用户正在做什么判断。
3. 哪些表达是内部流程语，必须删除或转译。
4. 哪些表达替用户下了最终判断，必须降级。
5. 文案是否会破坏页面排版、标题断行或卡片密度。

每次输出后，必须检查：

- 是否像真人写的，不像模型套模板。
- 是否有具体行业、岗位、流程、成本、采用或风险角度。
- 是否有判断，但不吓人。
- 是否有深度，但不晦涩。
- 是否有共鸣，但不煽情。
- 是否有商业视角，但不堆黑话。
- 是否让目标用户觉得：这个人懂 AI，也懂企业现实。
