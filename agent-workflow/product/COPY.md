## 1. 文案定位

观澜AI的文案不是 AI 新闻站、工具导航站、热点内容号，也不是咨询公司白皮书。

它面向企业经营者、资源型合伙人、行业操盘手、投资观察者，以及真正关心 AI 如何影响商业的人。它要帮助用户：

- 看懂变化。
- 判断影响。
- 识别行业、岗位、流程中的机会。
- 找到企业 AI 起步点。

全站文风定位为：

```text
冷静、有判断、有温度的商业内参风。
```

读起来要像一个长期在一线观察产业变化、亲自做项目、懂企业现实处境的人，在帮用户过滤噪音、判断影响、找到起点。

## 2. 品牌声音

观澜AI同时具备四种气质：

| 气质 | 说明 | 禁区 |
|---|---|---|
| 商业记者的敏锐 | 能看出一条新闻背后的变量变化 | 只复述新闻标题 |
| 产业研究者的冷静 | 有证据、有反证、有边界 | 把热度写成结论 |
| 企业顾问的判断 | 能落到行业、岗位、流程、成本、风险 | 给空泛口号 |
| 创业者的体感 | 理解企业落地时的犹豫、预算和执行难度 | 装腔、说教、恐吓 |

最终效果：

- 有判断，但不装。
- 有锋芒，但不攻击。
- 有深度，但不晦涩。
- 有温度，但不煽情。
- 有共鸣，但不讨好。

## 3. 最重要的硬规则：避免 AI 味

宁可短一点、真一点，也不要完整但空泛。

### 3.1 不要机械分点

避免动不动写：

```text
首先 / 其次 / 最后 / 总之 / 值得注意的是 / 综上所述
```

如果需要分层，可以自然写，不要像模型生成的标准答案。

写作线程正文额外禁用：

```text
显而易见 / 不可否认 / 总而言之 / 由此可见
```

不要用过渡词假装文章有结构。结构要从事实、场景和判断里长出来。

### 3.2 不要重复句式

以下句式可以偶尔用，但不能整页反复出现：

```text
不是……而是……
表面看……真正……
对企业来说……
这意味着……
核心在于……
```

重复句式会让文案看起来像自动生成。

### 3.3 不要空泛判断

没有具体行业、岗位、流程、场景支撑时，不写：

```text
AI 正在重塑各行各业
企业必须拥抱 AI
未来已来
这是一次产业级机会
AI 将带来效率革命
```

要把判断落到更具体的位置：销售线索、客服知识库、内容生产、采购应付、合规审计、研发测试、数据问答、门店运营、供应商协同。

### 3.4 不要堆概念

以下词不是绝对禁用，但公开前台必须高度克制：

```text
赋能
重构
闭环
生态
范式
底座
抓手
价值链路
数智化升级
中台
一站式
```

能用朴素语言说清楚，就不要堆概念。

### 3.5 不要写成冷冰冰的报告

观澜AI不是冰冷的咨询报告。文案可以有一点人的思考、迟疑、提醒和分寸感。

好的观澜句子不是“更完整”，而是让读者少一点噪音，多一个清楚的下一步想法。

### 3.6 不要写抽象名词泡沫

三个 writer 的正文严禁使用带有“XX感 / XX性 / XX化”的抽象名词。

这条规则适用于文章作者自己的表达，不适用于原文标题、原文摘录、来源名称、数据表字段和内部治理文档。引用材料中出现这些词，可以保留；观澜自己的解释要改成具体动作。

常见问题：

```text
科技感 / 未来感 / 故事感
确定性 / 复杂性 / 重要性 / 可能性
智能化 / 数字化 / 商业化 / 场景化 / 规模化
```

改成具体动作：

```text
不要写“场景 AI 化正在加快”。
写“AI 正在进入客服、销售、采购这些具体流程”。

不要写“商业化路径更清晰”。
写“客户愿意为哪一段流程付钱，价格从哪里来”。

不要写“规模化落地困难”。
写“从 10 家客户做到 100 家客户时，交付团队会先被拖住”。
```

### 3.7 句子要有人的节奏

多用短句。长句可以有，但不能连着压人。

禁止连续写超过 3 个句式整齐的排比句。那会像机器在铺垫。

语气可以有一点主观判断。像在咖啡馆跟朋友聊一个生意变化。别端着，也别吓人。

## 4. 用户心理

用户不是来听热闹的。他们关心：

- 这件事和我有什么关系？
- 会不会影响我的行业？
- 会不会影响我的团队？
- 哪些岗位会先变化？
- 哪些流程值得先试？
- 我现在不动，会不会错过？
- 我现在动，又怕不值得。

所以不要只写“AI 发生了什么”，要写“这件事可能怎样影响企业经营”。

## 6. V2 栏目边界

V2 普通前台只保留四个一级栏目：

```text
今日观察 / 商业信号 / 趋势追踪 / 商业内参
```

| 前台栏目 | 用户任务 | 文案重点 | 不要写成 |
|---|---|---|---|
| 今日观察 | 今天市场怎么动，为什么是这些变化 | 当日市场综述、热点解释、精选变化卡选择理由 | 新闻摘要、日报列表、行动建议书 |
| 商业信号 | 哪些变化值得继续跟踪 | 不只写发生了什么，还要写为什么进入观澜AI | 公司名列表、融资新闻、工具资讯 |
| 趋势追踪 | 哪些变化正在连成更大的方向 | 深度长文、变化簇、趋势卡、机会判断和反证 | 卡片聚合页、公司案例库、执行方案 |
| 商业内参 | 跨周期组合判断 | 核心判断、关键事实、商业影响、机会与风险 | 情绪公众号、泛观点合集、公开热榜 |

降级或后台化：

- The Point：前台统一表达为 `前沿观点`，不是一级栏目。
- 机会判断：不再作为一级栏目，降级为趋势追踪和商业内参中的 `机会判断` 段落。
- Scoring / Priority Engine：后台能力，不进入普通前台文案。
- Tags：搜索、筛选和关系网络能力，不作为一线栏目。

## 7. 核心表达原则

### 7.1 每句话有信息增量

每句话至少提供一种东西：

- 事实。
- 判断。
- 来源状态。
- 商业含义。
- 风险边界。
- 后续观察变量。

删掉不影响理解的句子，删掉。不能回答“所以呢”的句子，重写。

### 7.2 有判断，但不替用户下最终判断

推荐：

- “这个方向值得继续看。”
- “更适合先小范围试。”
- “证据还不够。”
- “风险主要在交付成本和数据可得性。”
- “这件事可能先影响销售、客服、内容、运营这类高频流程。”

避免：

- “必须马上行动。”
- “确定性机会。”
- “一定值得投资。”
- “不学就会被淘汰。”
- “保证收益。”

### 7.3 写商业含义，不写页面说明

公开前台不要解释“这个页面做什么”。不要写：

```text
本页用于
入口
字段
后台
同步
JSON
Markdown
脚本
```

把内部话转成用户能得到的判断：

| 内部说法 | 对外说法 |
|---|---|
| 本页用于展示关系网络 | 这几条内容指向同一个商业变化 |
| 同步脚本更新网站 | 今日内容已更新 |
| 字段自动沉淀 | 这条信息已进入后续观察 |
| 后台评分 | 证据状态 / 观察优先级 |

### 7.4 写来源与事实，不做证据表演

避免：

```text
证据链
强证据
来源明确
阅读证据
机会确定
```

推荐：

```text
来源与事实
来源状态
支撑信息
多源支撑
单信源
证据缺口
仍需观察
```

## 8. 标题系统

| 标题类型 | 规则 | 长度建议 |
|---|---|---|
| 首页 Hero | 品牌价值 + 判断对象，不写功能说明 | 16-28 中文字 |
| 一级栏目标题 | 一句话价值表达，不加解释型副标题 | 14-24 中文字 |
| 卡片标题 | 信息完整，不只写公司名 | 18-34 中文字 |
| Signal 标题 | 公司 / 产品 / 机构 + 动作 + 场景，不用判断标签冒充事件 | 18-38 中文字 |
| Trend Report 标题 | 趋势方向或客户场景，不写公司名 | 8-18 中文字 |
| 商业内参标题 | 周期判断或主题判断 | 16-32 中文字 |
| 知识卡标题 | 对象 + 判断变量 | 12-28 中文字 |

字体适配：

- 页面大标题使用中文衬线，句子要短。
- 卡片标题优先一到两行可读。
- 标题过长时先压缩文案，不靠放大容器硬塞。
- 英文标签可用全大写，但正文不混入过多英文术语。

商业信号标题硬规则：

- 标题先讲事实，不先讲结论。优先写清楚“谁做了什么”，例如 `Claude 发布法律行业部署指南`、`PwC 扩大部署 Claude`、`OpenAI 发布 Codex Windows 沙箱指引`。
- 如果标题里没有公司、产品、机构、动作或明确场景，默认不合格。
- 禁止把判断压成标题，例如 `账单开始按用量走`、`围栏成了采购前提`、`门禁进入工具链`、`采用率开始被数据盯上`。
- 禁止半截标题。任何前台标题不得因为函数截断、CSS line-clamp 或人工压缩显示为 `...` / `…`。标题可以换行，但意思必须完整。
- 抽象判断可以放在摘要或商业含义里，但必须跟在事件之后，不能抢占标题位置。

首页商业信号硬规则：

- 首屏必须形成“左侧主判断，右侧信号简报”的主次关系。左侧是今日主信号，右侧是简报，不是两块平级内容。
- 左侧主信号必须先讲事实或事件，再给观澜判断。不能只放大标题和一句摘要，留下大面积空白。
- 左侧必须包含紧凑证据摘要：判断、案例、来源、证据状态、观察窗口。
- 右侧每条简报只写事件、短摘要、为什么值得看和标签；不写成三篇小文章。
- 数字统计要写成内参提示，例如 `今日信号 / 案例卡 / 前沿观点`，不要像后台 KPI。
- 禁止在前台显示 `判断：`、`证据链`、`字段`、`后台`、`待复核字段` 等内部词。
- 右侧小标题必须像人写的事件判断，不能写 `专业流程开始接 AI`、`会动手就要先管住`、`用量账单开始抬头` 这类机械标签。
- 如果一句话离开正文就看不懂，说明标题失败；重写为“公司 / 产品 / 机构 + 动作 + 场景”的完整表达。

## 9. 摘要与导语

摘要回答四件事：

1. 发生了什么。
2. 它可能影响什么行业、岗位或流程。
3. 当前来源状态如何。
4. 还缺什么证据。

推荐：

```text
这条信号目前有多源支撑，但客户付费证据还需要继续观察。
它更像一次入口竞争的前奏，不是已经完成验证的机会。
本期关注的不是单家公司，而是一组信号共同指向的流程变化。
```

避免：

```text
这是一条强证据链。
机会已经确定。
建议马上进入验证。
本模块用于展示来源字段。
```

## 10. 四个一级栏目文案规范

### 10.1 今日观察

文风：像一篇老板每天能读完的 AI 商业行情综述。它不是新闻摘要，而是当天市场风向、热点解释、变化选择理由和观澜判断。

必须回答：

- 今天最值得关注的变化是什么？
- 背后的商业信号是什么？
- 可能影响哪个行业、岗位或流程？
- 哪些地方还不能下结论？

今日观察主文写法：

- 用“商业观察者 / 主编判断”的连续表达，不写成页面说明、问答卡片或行动清单。
- 信息上必须覆盖：今天最值得关注的变化、为什么重要、判断依据、为什么选择这些变化卡、可能影响的行业 / 岗位 / 流程 / 企业决策。
- 不要写“结论先放在前面”“本页用于”“以下几点”“行动清单”这类提示语。
- 不要为了完整而把页面拉长；若短期 / 长期变量不是页面主诉求，收进正文一句话，不单独展开成 5、6、7 个模块。
- 语气要像懂企业现场的人在做判断：具体、克制、有边界，不讲空话。

推荐语感：

```text
今天更值得关注的，不是某个模型参数又提升了多少。

真正的变化是，AI 正在从单点工具，慢慢进入企业流程。

对企业来说，下一步要看的不是哪个工具更火，而是哪些重复、标准化、耗人力的流程，会先被 AI 接过去。
```

今日观察主文示例：

```text
企业会先为“可控”付费，再为“更强”付费。

今天值得看的变化，不是又多了几个能调用工具的 Agent，而是连接器和 Agent 平台正在把权限、审计、回滚和停用推到产品核心。Agent 一旦进入客服、销售、采购、财务或研发流程，就不再只是回答问题，而是在接触客户、数据和系统。企业真正要判断的，是它能否被授权、被限制、被复盘，并在出错时及时止损。
```

避免：

- 新闻列表。
- 今日新增数量汇总。
- “建议立即验证”。
- “行动清单”。
- Raw / Pool / Heat Candidate / M 级线索。

### 10.2 商业信号

文风：像商业雷达，不像新闻列表。

Signal 标题必须先讲事件：

```text
公司 / 产品 / 机构 + 动作 + 对象 / 场景
```

推荐：

```text
Claude 发布法律行业部署指南
PwC 扩大部署 Claude
OpenAI 发布 Codex Windows 沙箱指引
Ramp 数据显示 Anthropic 企业采用率超过 OpenAI
```

避免：

```text
Haast
Microsoft
AI Agent
融资新闻
账单开始按用量走
围栏成了采购前提
门禁进入工具链
采用率开始被数据盯上
```

摘要必须说明：

- 发生了什么。
- 为什么不是普通新闻。
- 它可能改变哪个行业、岗位或流程。
- 来源和反证在哪里。

### 10.3 趋势追踪

文风：深度长文 + 趋势卡并重。它不是卡片聚合页，也不是公司案例库。

趋势追踪必须把变化簇、趋势卡、案例、观点、技术路线、客户需求、竞品动作和反证写成一篇可读的研究报告。

趋势追踪下有两种文章：

- `趋势快报`：突然升温方向的临时深读，2000-3500 中文字，公开或登录可读。
- `深度报告`：完整 Trend Report，6000-10000 中文字，会员重点内容。

趋势快报也必须遵守观澜文案风格。它不是快讯，不是自媒体爆款，不用“重磅 / 颠覆 / 风口 / 必看 / 全面爆发”。它要像资深商业观察者在当天临时插入的一页判断：快，但有来源；短，但有反证；有画面，但不煽动。

趋势快报推荐开法：

```text
一个企业 IT 负责人今天真正关心的，不是某个模型参数又涨了，而是 Agent 开始接触内部系统之后，谁来负责权限、回滚和事故记录。
```

报告中的 `机会判断` 段落必须说明：

- 谁可能付钱。
- 为什么现在可能发生。
- 哪段流程先被改写。
- 预算、交付、数据、信任和组织阻力卡在哪里。
- 哪些证据还不够。

推荐语感：

```text
这个方向值得看，但不适合所有企业马上跟进。

它更适合那些已经有稳定线索来源、销售流程相对标准化、愿意沉淀数据的团队。

如果企业本身流程混乱，销售动作不可复盘，直接上 AI 系统，最后很可能只是多买了一套工具。
```

趋势快报写作骨架只给 writer 使用，不在前台露出：

1. 场景开头。
2. 今天为什么突然升温。
3. 它可能改变谁的流程。
4. 市面上谁在动。
5. 技术路线的商业含义。
6. 反证与观察变量。
7. 30 天后看什么。

### 10.4 商业内参

文风：给决策者看的简报，不是公众号情绪文，也不是研究报告黑话。

应有层次：

- 核心判断。
- 关键事实。
- 商业影响。
- 行业 / 岗位 / 流程映射。
- 国内迁移可能性。
- 机会与风险。
- 观澜结论。

推荐：

```text
本期关注的不是单家公司，而是一组信号正在指向同一个商业变化。
热度已经升高，但客户付费证据还集中在早期采用阶段。
```

禁止：

- “本期告诉你马上做什么。”
- “确定性机会已经出现。”
- “趋势很热，所以机会成立。”
- 把单篇趋势追踪直接等同于商业内参。

## 11. 嵌入模块表达

| 模块 | 允许表达 | 禁止表达 |
|---|---|---|
| 前沿观点 | “这条观点可帮助修正判断，但不替代事实来源。” | “大佬已经证明趋势成立。” |
| 前沿观点 | “这位 Builder 的关注变量正在变化。” | “The Point 一级栏目。” |
| 趋势背景 | “过去 30 天，证据主要集中在……” | “趋势很热，未来可期。” |
| 商业热力图 | “热力集中在销售线索、客服知识库和内容生产流程。” | “公开机会榜 / 公司榜 / 岗位替代榜。” |
| 反证与边界 | “客户付费、交付成本和数据可得性仍未确认。” | 空泛免责声明 |

## 12. CTA 与会员态

CTA 要克制，像产品动作，不像销售话术。

| 状态 | 推荐 CTA | 避免 |
|---|---|---|
| 访客 | 查看今日观察 / 阅读商业信号 / 了解商业内参 | 立即抢占红利 |
| 登录前 | 登录继续阅读 / 注册试读 | 不看就晚了 |
| 试读会员 | 阅读完整内参 / 查看来源摘要 | 最后机会 |
| 到期用户 | 查看会员方案 / 续订后继续阅读 | 马上逆袭 |
| 付费会员 | 阅读完整报告 / 查看来源附录 / 进入关联内容 | 开通后稳赢 |
| 企业内参 | 申请企业内参 / 预约闭门简报 | 立即获得定制答案 |

锁定态规则：

- 不整页遮挡。
- 公开部分保留主题、主判断、部分来源摘要和边界。
- 会员层展开完整热力、来源、证据缺口和后续观察变量。
- 不制造错失焦虑。

## 13. 每日监测产物规则

| 产物 | 角色 | 标题规则 | 摘要规则 | 禁止 |
|---|---|---|---|---|
| 精选变化卡 | 深度信号报告 | 事件 + 商业含义 | 多源事实 + 商业变量 + 反证 | 普通新闻稿 |
| 变化卡 | 入库信号卡 | 事件 + 指向变量 | 快速说明事实、来源、关联对象 | 堆字段 |
| Trend Report Flash | 趋势快报 | 趋势方向，不写公司名 | 多源信号 + 商业影响 + 反证 + 30 天观察变量 | 新闻快讯、热度稿、爆款标题 |
| Trend Report Full | 趋势深度报告 | 趋势方向，不写公司名 | 趋势判断 + 机会判断 + 反证 + 案例 / 竞品 / 技术路线 / 客户场景 | 硬凑深挖 |
| Trend Card | 趋势卡 | 趋势方向 + 证据变化 | 7 / 30 / 90 天证据变化 | 热度即趋势 |
| Point / 前沿观点 | 判断输入 | 人物 / 机构 + 观点变化 | 支持、修正或质疑什么判断 | 观点当事实 |
| Business Brief | 周期组合判断 | 主题 / 周期判断 | 信号链 + 机会组合 + 风险边界 | 泛泛观点合集 |

每日内容不得把 AI HOT、follow-builders、HN、X、Reddit 等 discovery 通道直接写成事实主证据。

## 14. 知识卡片表达

知识卡片不是后台字段展示，而是可复用的判断资产。

| 卡片类型 | 标题 | 摘要 | 标签语气 |
|---|---|---|---|
| 信号库 | 事件 + 商业含义 | 这条信号关联哪些机会、趋势和来源边界 | 正式主题词，不随手造词 |
| 趋势报告库 | 趋势方向 | 关联哪些信号、客户场景和风险变量 | 描述客户 / 场景 / 流程 |
| 观点库 | 人物 / 机构 + 观点变量 | 观点支持、修正或质疑什么判断 | 不把人名当趋势 |
| 案例库 | 公司 / 产品 + 代表证据 | 这个案例能支撑哪个机会或信号 | 公司是证据，不是机会 |
| 趋势背景 | 趋势方向 + 状态 | 证据如何在 7 / 30 / 90 天变化 | 不写成热词墙 |

知识卡摘要公式：

```text
它是什么 + 支撑哪个判断 + 证据边界 / 反证
```

## 15. 禁用语与替代表

| 类型 | 禁用 / 谨慎 | 推荐替代 |
|---|---|---|
| 内部流程词 | 本页用于、入口、同步、字段、后台、JSON、Markdown、脚本、Raw、Pool | 删除，或转译为用户可见价值 |
| 证据表演词 | 证据链、强证据、来源明确、阅读证据 | 来源与事实、来源状态、支撑信息、多源支撑、证据缺口 |
| 确定性词 | 确定性机会、一定、必然、唯一、保证、稳赚 | 值得继续观察、可能、证据正在增强、仍需观察 |
| 行动指令 | 必须、立即行动、马上入局、下一步验证、行动清单 | 后续观察、建议关注变量、可小范围试 |
| 营销词 | 财富密码、风口红利、错过再等十年、颠覆未来、弯道超车 | 商业变化、早期信号、流程变化、结构性变化 |
| 概念堆叠 | 全链路智能化矩阵、生态闭环、一站式中台、智能认知引擎 | 具体流程、客户采用、成本变化、交付边界 |
| 观点越界 | 大佬证明、观点坐实、趋势已成立 | 前沿观点、观点分歧、判断输入 |

## 17. 改写流程

当文案像 AI 写的、像新闻摘要、像营销号时，按这个顺序改：

1. 保留事实，不乱加信息。
2. 删除夸张词、口号词、空泛判断。
3. 找到这条信息背后的商业含义。
4. 尽量落到行业、岗位、流程、组织、效率、成本、风险。
5. 加入一点真实判断，但不要装腔。
6. 让语言更自然，有节奏，有人的体感。
7. 避免所有段落都用同一种句式。
8. 每段都要有信息增量，不要重复换句话说。

## 18. Copy QA 检查表

后续页面、内容、会员态和知识库任务按下表验收：

| 检查项 | 通过标准 |

## 19. Copy Agent 工作流程

每次改文案前，必须确认：

1. 页面或内容的用户任务。
2. 目标用户正在做什么判断。
3. 哪些表达是内部流程语，必须删除或转译。
4. 哪些表达替用户下了最终判断，必须降级。
5. 文案是否会破坏页面排版、标题断行或卡片密度。

页面 / 文案开发前必须输出 Copy-first 表：

| 页面/模块 | 用户任务 | 最终文案 | 字数/行数约束 | 证据边界 | 禁用语检查 | Dev 实现说明 |
|---|---|---|---|---|---|---|

每次输出后，必须检查：

- 是否像真人写的，不像模型套模板。
- 是否有具体行业、岗位、流程、成本、采用或风险角度。
- 是否有判断，但不吓人。
- 是否有深度，但不晦涩。
- 是否有共鸣，但不煽情。
- 是否有商业视角，但不堆黑话。
- 是否让目标用户觉得：这个人懂 AI，也懂企业现实。
