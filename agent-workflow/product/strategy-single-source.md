## 1. 一句话定位

观澜 AI｜WaveSight AI 是面向商业决策者的 AI 机会判断系统。

它每天从融资、客户采用、产品落地、监管、招投标、并购和行业变化中筛出商业信号，形成 Daily Brief，并把高价值方向沉淀为可验证的机会库和趋势判断。

## 2. 核心卖点

主卖点：

> 每天帮你看清少数真正值得验证的 AI 商业机会。

辅助表达：

> 比市场早一步，看清哪些 AI 机会值得验证。

核心承诺：

> 在市场形成共识前，看见值得验证的 AI 商业机会。

定位语：

> 观AI之澜，识商业之势。

## 3. 用户真正要完成的事

用户来到观澜 AI，不是为了看更多 AI 新闻，而是为了完成四个判断任务：

1. 今天该看什么。
2. 为什么值得看。
3. 哪些机会值得继续观察。
4. 这个方向是否持续成立。

对应产品结构：

| 用户任务 | 产品模块 |
|---|---|
| 今天该看什么 | Daily Brief |
| 为什么值得看 | Signals |
| 哪些机会值得继续观察 | 机会解码 |
| 这个方向是否持续成立 | Trends |

对外不要生硬罗列“老板、行业大咖、资源方、投资人”，而应抓住他们共同的购买动机：

- 减少信息噪音。
- 提高判断效率。
- 在共识形成前锁定值得验证的机会。
- 更清楚看见产品、客户、资金、场景之间的关系。
- 更稳地决定要见谁、投什么、连接什么资源、验证什么场景。

## 4. 第一阶段产品核心

第一阶段只聚焦四个前台产品模块。

### Daily Brief

Daily Brief 是每天打开的理由，也是第一付费交付物。

它不是今日新增内容列表，而是一份可独立付费阅读、可归档、未来可邮件发送的商业内参。

固定结构：

1. 今日判断
2. 关键 Signals
3. 机会观察
4. 趋势变化
5. 判断依据
6. 风险与反证

合格标准：

- 有观点。
- 有证据。
- 有优先级。
- 有边界感。
- 用户读完知道今天哪些方向更值得关注，哪些判断仍需继续观察。
- 不替客户下最终决策，不承诺结果，不输出确定性投资或经营建议。

### Signals

Signals 是可信基础，是观澜 AI 判断的证据来源。

Signal 不是新闻、公司动态或工具列表，而是：

> 一个已经出现商业证据、可能改变赛道判断或机会优先级的 AI 变化。

每条 Signal 必须同时具备：

1. 事件：融资、产品发布、客户采用、收入增长、并购、监管、渠道变化、招投标、重大合作。
2. 证据：客户、收入、融资、部署、复购、合作、政策、行业采用、明确产品更新。
3. 商业含义：说明某个方向正在升温、分化、成熟、受阻，或出现可验证机会。

没有商业含义的新闻，不进入 Signals。

### Opportunities

Opportunities 是最终机会库，是用户愿意持续订阅的长期资产。

机会卡不是公司档案，不是项目列表，也不是概念库。它必须回答：

- 机会是什么。
- 适合什么客户。
- 对应什么场景。
- 为什么现在值得验证。
- 证据等级如何。
- 趋势状态如何。
- 下一步怎么验证。

Priorities 不再作为独立前台栏目。Priority Engine 的评分、优先级、证据等级和验证动作进入每张 Opportunity Card。

### Trends

Trends 是第二层判断能力，用于支撑 Daily Brief 和 Opportunities。

Trends 不是趋势名称列表，而是判断某个 AI 方向是否正在形成商业势能的模型。

它必须回答：

- 这个方向处于哪个阶段。
- 它为什么升温、分化、降温或进入风险区。
- 哪些证据支持判断。
- 哪些反证会推翻判断。
- 它关联哪些 Signals 和 Opportunities。

趋势状态：

- `rising`：持续升温
- `splitting`：分化中
- `cooling`：降温
- `emerging`：新出现
- `mature`：成熟化
- `risk`：风险变量
- `invalidating`：反证增强

## 5. 前台栏目

V2 当前前台导航以 `v2-current-rule-overrides.md` 和 `column-architecture.md` 为准：

```text
今日要点 / 关键信号 / 机会解码 / 商业内参
```

栏目任务：

| 栏目 | 用户任务 | 核心内容 | 不应承担 |
|---|---|---|---|
| 今日要点 | 每天先看判断 | 今日判断、发生了什么、值得关注、趋势温度、机会观察、观点与思考、风险与边界、后续观察 | 新闻列表或行动指令 |
| 关键信号 | 看发生了什么变化 | Front Signal、Structured Signal、Builders 观点、来源和证据状态 | 只展示公司名 |
| 机会解码 | 判断哪些机会值得继续观察 | 深度机会报告、成熟度、证据等级、趋势状态、客户、场景、风险边界 | 用公司名当机会名 |
| 商业内参 | 阅读跨周期组合判断 | AIBrief Issue、专题、周/月复盘、趋势与机会地图、观点分歧、来源证据 | 泛泛观点合集 |

降级或后台化规则：

- `Scoring` 不作为前台栏目，后台化为 `Priority Engine`。
- `Priorities` 不作为独立前台栏目，能力合并进 `Opportunities`。
- `The Point` 不作为 V2 一级栏目，进入 `关键信号 > Builders 观点` 和各栏目观点校准模块。
- `Trends` 不作为 V2 一级栏目，作为趋势背景、热力输入和商业内参材料。
- `Tags` 不作为当前一级导航，作为搜索、筛选和机会网络能力。
- `Method` 不作为当前前台栏目，作为内部方法论资产。
- `Admin` 为后台入口，不进入普通用户导航。


命名规则：


规则：


## 7. 监测来源与关键词体系

监测来源分三层：

### 一级来源：一手证据

- 公司官网、产品更新、博客、开发者文档。
- 客户案例、合作公告、上线公告。
- 监管文件、政策文件、招投标公告。
- 招聘页、价格页、API 文档变化。
- GitHub、云市场、插件市场、App Store、Chrome Web Store。

### 二级来源：高质量商业媒体

- 海外：WSJ、FT、Bloomberg、The Information、TechCrunch、Axios、CNBC。
- 中文：36氪、晚点、甲子光年、钛媒体、财新、机器之心、量子位。
- 产业垂直媒体：医疗、工业、金融、营销、企业服务等行业媒体。

### 三级来源：市场与产业数据

- 融资数据库、并购公告、企业工商变更。
- 招投标平台、政府采购、央国企合作公告。
- G2、Product Hunt、云市场、应用商店评价。
- 搜索趋势、社媒热度、招聘岗位变化。

关键词分层：

- 赛道词
- 证据词
- 场景词
- 风险词
- 行动词

关键词不是静态清单，应每周输出质量报告：高产词、低效词、新增词、候选移除词。

## 8. Signal Score

Signal Score 使用 100 分制：

| 维度 | 权重 |
|---|---:|
| 商业证据强度 | 30% |
| 赛道相关性 | 20% |
| 新变化程度 | 15% |
| 市场速度 | 15% |
| 中国/目标客户适配度 | 10% |
| 可行动性 | 10% |

建议阈值：

- 80-100：核心 Signal，进入 Daily Brief。
- 65-79：有效 Signal，进入 Signals 库。
- 50-64：待观察，进入后台候选池。
- 50 以下：普通新闻，不进入系统。

## 9. Trends 判断模型

Trends 使用五个模型：

1. 证据阶梯：概念出现 → 产品发布 → 融资验证 → 客户采用 → 收入增长 → 复购扩张 → 行业标配。
2. 采用曲线：早期探索 / 先锋采用 / 主流扩散 / 成熟竞争 / 红海分化。
3. 机会温度：资金温度、客户温度、产品成熟度、竞争强度、政策/监管阻力、中国适配度。
4. 趋势分化：概念降温但真实业务场景升温、海外升温但中国仍验证、工具降温但基础设施增强等。
5. 反证清单：客户不续费、融资领先收入过多、缺少采购方、平台依赖、监管风险、只是大厂功能等。

## 10. 商业化路径

### 第一阶段：会员制情报访问

核心交付：

- Daily Brief
- Signals
- Opportunities
- Trends

用户路径：

```text
访客 -> 注册 -> 7 天试读 -> 月/季/年订阅
```

### 第二阶段：行业定制简报 / 闭门简报

面向高意向用户，围绕特定行业、赛道、资源类型输出定制判断。

### 第三阶段：机会连接与项目化服务

围绕机会方向提供资源连接、项目筛选、验证路径和合作撮合。

## 11. 账号、访问与会员原则

前台必须补齐：

- 登录
- 注册
- 忘记密码
- 会员状态
- 访问状态
- 权限分级
- 未授权页面提示

权限分级：

| 状态 | 可访问内容 | 页面提示 |
|---|---|---|
| 访客 | 首页、少量样例、注册入口 | 注册后试读 7 天 |
| 试读会员 | 关键信息、部分今日要点、部分机会解码 | 试读剩余天数 |
| 试读到期 | 公开内容和账号中心 | 购买月/季/年订阅 |
| 付费会员 | 完整今日要点、关键信号、机会解码、商业内参、搜索和关系网络 | 订阅到期时间 |
| 管理员 | 全部内容与后台功能 | 运营控制台 |

规则：

- 未登录用户看到注册 / 登录。
- 注册后进入 7 天试读。
- 试读到期后提示月 / 季 / 年订阅。
- 已登录用户不重复走普通申请访问。
- 企业版、私享内参、闭门简报保留申请入口。
- 初期可以不接真实支付，但必须先有商业入口和权限状态。

## 16. 文案风格

观澜 AI 的文案风格应是克制的商业判断，而不是热闹的营销表达。

总体气质：

- 像商业内参，不像新闻摘要。
- 像决策辅助，不像投资建议。
- 有观点，但不替客户做最终判断。
- 有倾向，但保留证据边界和不确定性。
- 有商业洞察，但不制造焦虑。
- 有高级感，但不堆砌空泛大词。

表达原则：

- 少用绝对词：避免“必然、一定、唯一、确定、马上、颠覆、爆发”等表达。
- 少用命令句：避免“你应该、必须、立刻去做、马上投资”等表达。
- 多用判断型表达：可以使用“值得关注、可能升温、需要验证、证据正在增强、仍需观察、短期不宜过度乐观”。
- 多用证据型表达：每个判断尽量对应客户、收入、融资、采用、政策、招投标、产品落地、并购或渠道变化。
- 少讲后台过程：不说“同步、拆解、沉淀、字段、脚本”，只说用户能获得的判断价值。
- 不夸大收益：不承诺赚钱、成功、投资回报或确定性商业结果。

推荐语气：

```text
这个方向正在出现更强的客户采用证据。
该机会值得进入观察池，但还需要验证付费意愿和交付成本。
目前更像早期信号，不宜直接等同于赛道拐点。
趋势正在升温，但反证仍然存在。
```

不推荐语气：

```text
这是下一个确定性机会。
马上行动，否则会错过窗口。
我们帮你找到稳赚方向。
这个项目一定会爆发。
```

## 17. 页面 UI 风格

观澜 AI 的页面 UI 应服务于高端商业判断，而不是展示技术感或信息堆砌。

视觉气质：

- 高端、克制、清晰、有留白。
- 像商业内参和高级咨询产品，而不是普通 SaaS 模板。
- 第一眼先看到判断价值，再看到数据和证据。
- 页面不追求“内容多”，而追求“判断路径清楚”。

设计原则：

- 首页只承载品牌价值、核心判断入口和少量代表性内容。
- 栏目页承载完整内容，不把所有内容塞进首页。
- 卡片标题优先使用“事件 + 商业含义”，避免只展示公司名。
- Scoring 不在前台作为独立视觉中心，评分结果应自然进入机会卡和排序逻辑。
- Tags 不做长列表堆砌，应作为筛选、搜索和关联网络出现。
- 普通用户页面不出现编辑、同步、恢复、访问状态等后台痕迹。
- 管理员能力集中在运营控制台，不污染前台体验。

页面节奏：

- 大标题少而有力，不要每个模块都用同等强度的黑体大字。
- 每屏只完成一个主要任务。
- 重要判断用短句呈现，证据放在下一级。
- 避免过密表格、过长标签墙和过多同级卡片。
- 移动端优先保证阅读顺序，不把桌面栅格硬压到小屏。

核心页面要求：

| 页面 | UI 重点 | 文案重点 |
|---|---|---|
| 首页 | 品牌感、判断价值、少量代表样例 | 让用户明白为什么值得登录或试读 |
| 今日要点 | 每日判断流、证据摘要、趋势温度、关键词表 | 给出观察重点和判断依据，不给行动指令 |
| 关键信号 | Front Signal 报告、Structured Signal 卡、Builders 观点 | 说明为什么这不是普通新闻 |
| 机会解码 | 机会方向、优先级、证据等级、趋势状态、风险边界 | 说明为什么值得继续观察或暂缓判断 |
| 商业内参 | 跨信号、趋势、机会和观点的组合判断 | 形成可复盘的商业判断报告 |
| Admin | 运营效率、同步状态、审核入口 | 面向内部管理，不面向客户展示 |

## 18. 内容判断标准

一个内容能否进入观澜 AI，需要回答：

1. 是否有明确事件？
2. 是否有商业证据？
3. 是否能改变机会判断？
4. 是否能关联机会或趋势？
5. 是否能形成后续观察问题或判断边界？

如果不能回答这些问题，就不应进入核心产品层。
