# Strategy Single Source｜观澜 AI 战略单一事实源

状态：current
更新时间：2026-05-24

## 定位

观澜 AI 是一个由 Agent 驱动的 AI 商业情报系统。

它持续监测全球 AI 商业变化，将外部信息转化为结构化情报资产，再生成面向企业老板、资源型合伙人、行业操盘手的商业信号、机会判断、AI 内参和 AI 商业热力图。

它不是内容站，也不做 AI 新闻搬运、工具导航或普通日报，而是一个会持续感知、判断、表达、发布、复盘和进化的 AI 商业决策系统。

V2.2.1 当前及后续的系统建设目标是 AI Native：Agent 之间要形成联动，通过结构化 handoff、质量门和自动化任务，把 Source / Raw / Pool / Card / Column / Site 串成可持续进化的生产链路。

## 核心链路

```text
Source -> Raw -> Pool -> Asset -> Column -> Site
```

- Source：AI HOT、follow-builders、HN、GDELT、官方来源、媒体、研究、开发者、关键词搜索。
- Raw：保留原始来源、full text、snapshot、source level、hash 和关键摘录。
- Pool：按商业信号价值分流。
- Asset：商业信号卡、前沿观点卡、变化候选、场景候选、趋势候选；正式变化和趋势只在材料成熟后升级。前沿观点卡必须执行四档评级：`feature` / `sidebar` / `archive` / `discard`，再决定是否进入前台。
- Column：今日观察、商业信号、趋势追踪、商业内参。
- Site：只展示通过证据、文案和同步门禁的内容。

## 当前栏目

- 今日观察：每天只讲最值得看的主线。
- 商业信号：展示通过门禁的产品 / 服务、融资、案例信号，承接成熟变化短专题和前沿观点入口。
- 趋势追踪：用趋势快报和深度报告追踪变化候选聚合。
- 商业内参：周期性融合判断和复盘。

## 当前非目标

- 不恢复已删除站点或已停止自动化。
- 不把 前沿观点 作为一级导航。
- 不把未评级、`archive` 或 `discard` 的观点卡同步到前台。
- 不把机会判断做成独立一级中心。
- 不把聚合源、社区热度或工具官网当作事实主证据。
- 不为页面或文章堆砌空泛趋势表达。

## 当前优先级

1. 稳定每日监测质量。
2. 稳定 Raw / Pool / Card 证据链。
3. 稳定商业信号和趋势追踪的前台表达。
4. 在规则稳定后恢复今日观察自动写作。
