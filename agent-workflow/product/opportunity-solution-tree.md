# Opportunity Solution Tree

## 北极星结果

让用户更快判断哪些 AI 机会值得关注、验证和连接。

## 机会点

### 机会点 1：信息太多，难以判断什么重要

用户问题：

- 每天 AI 新闻很多，不知道哪些值得看。
- 容易被融资、热词、公司宣传带着走。

解决方案：

- Daily 每天输出少数重点判断。
- 首页只展示高价值入口。
- Signals 用“事件 + 商业含义”做标题。

实验：

- 首页减少内容密度后，看用户是否更愿意进入 Daily 或申请访问。
- Daily 每天只保留 3-5 个重点判断，测试可读性。

### 机会点 2：知道热点，但不知道是否值得行动

用户问题：

- 看见热点，却不知道该投、该连、该观察还是暂缓。
- 缺少统一判断尺度。

解决方案：

- Priorities 代替 Scoring，展示机会优先级。
- 评分对象从公司改为机会方向或赛道。
- 每个优先级项关联证据和下一步动作。

实验：

- 将评分表改成优先级判断板。
- 测试用户是否能在 30 秒内知道前三个值得验证的方向。

### 机会点 3：机会与资源连接不清楚

用户问题：

- 不知道谁有需求、该连接谁、场景在哪里。
- 机会描述过于概念化。

解决方案：

- Opportunities 拆成客户、场景、商业模式、验证动作。
- Tags 串起客户、行业、场景、证据和行动。

实验：

- 给每张机会卡增加“下一步验证动作”。
- 抽样检查机会卡是否能转成一次可执行沟通。

### 机会点 4：内容长期积累后难以维护

用户问题：

- 每天新增内容后，重复、空字段、错位、权限痕迹容易累积。

解决方案：

- schema-check、relation-check、dedupe-check、content-quality-check。
- PM review 与 release checklist。
- 普通端和 Admin 分离。

实验：

- 每日自动生成质量报告。
- 每周 PM review 合并重复项并调整优先级。
