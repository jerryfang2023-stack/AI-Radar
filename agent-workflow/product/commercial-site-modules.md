# 商业化官网模块规划

## 官网定位

观澜 AI 官网不是普通展示页，而是会员制商业情报产品的入口。

它必须同时完成三件事：

1. 建立高端、可信、克制的品牌印象。
2. 让用户理解核心卖点：比市场早一步，看清哪些 AI 机会值得验证。
3. 引导用户注册/登录、订阅 Daily Brief 或进入会员体系。

## 前台模块

### 1. 账号与访问权限

必须补齐：

- 登录。
- 注册。
- 忘记密码。
- 会员状态。
- 访问状态。
- 权限分级：访客 / 试读用户 / 会员 / 管理员。
- 未授权页面提示。

关键规则：

- 未登录用户看到“注册 / 登录”。
- 注册后即拥有基础账号，不再二次申请访问。
- 普通注册会员可浏览关键信息 7 天。
- 7 天试读期结束后，保留登录状态，但付费内容提示购买订阅。
- 登录后根据会员状态展示可访问内容；如果权限不足，提示购买订阅或联系开通。
- “申请访问”只作为未登录访客的转化说法，本质动作是注册或预约开通，不应在已登录状态重复出现。

### 访问状态

| 状态 | 可访问内容 | 页面提示 |
|---|---|---|
| 访客 | 首页、少量样例、注册入口 | 注册后试读 7 天 |
| 试读会员 | 关键信息、部分 Daily Brief、部分机会卡 | 试读剩余天数 |
| 试读到期 | 仅保留公开内容和账号中心 | 购买月/季/年订阅 |
| 付费会员 | 完整 Daily Brief、Signals、Opportunities、Trends | 订阅到期时间 |
| 管理员 | 全部内容与后台功能 | 运营控制台 |

### 2. 会员订阅与商业化

初期可以不直接接支付，但必须先有商业入口：

- 会员方案页。
- 7 天试读说明。
- 月度订阅。
- 季度订阅。
- 年度订阅。
- 企业版 / 私享内参申请。
- 闭门简报或咨询预约。
- 续费 / 到期提醒。
- 发票或付款信息入口。

订阅规则：

- 月 / 季 / 年是标准个人会员周期。
- 年度会员应突出稳定跟踪、长期复盘和持续机会库访问。
- 企业版不与个人订阅混在一起，作为单独咨询或私享内参入口。

### 3. Daily Brief 产品模块

- 每日简报详情页。
- 历史 Brief 归档。
- 邮件订阅入口。
- 收藏 / 标记已读。
- PDF 或分享链接。
- 固定结构：今日结论、关键 Signals、优先验证机会、趋势变化、行动建议、风险提醒。

### 4. Opportunities 机会库

Priorities 合并进入 Opportunities 后，机会库必须成为核心产品资产。

每张机会卡应包含：

- 机会详情页。
- 优先级评分。
- 证据等级。
- 趋势状态。
- 相关 Signals。
- 相关 Trends。
- 下一步验证动作。
- 适合连接的资源类型。
- 收藏 / 跟进状态。
- 申请进一步了解。

### 5. Trends 趋势雷达

- 趋势地图。
- 趋势详情页。
- 升温 / 分化 / 降温 / 风险区。
- 7 天 / 30 天 / 90 天变化。
- 证据阶梯。
- 反证观察点。
- 相关机会入口。

### 6. 搜索与标签网络

- 全站搜索。
- 标签详情页。
- 赛道标签。
- 场景标签。
- 客户类型标签。
- 证据类型标签。
- 行动建议标签。
- 标签之间的关系图或主题簇。

### 7. 信任与转化

必须补齐：

- 样例报告。
- 关于观澜 AI。
- 注册 / 登录页。
- 联系我们。
- 隐私政策。
- 服务条款。
- 免责声明：非投资建议。
- 数据来源说明。

说明：

- Method 暂不做对外独立页面。
- 判断方法论作为内部资产保留，用于支撑样例报告、销售沟通、产品说明和未来可能开放的信任内容。

## 后台模块

Admin 应升级为运营控制台：

- 内容同步。
- 数据健康检查。
- Signal 去重确认。
- Opportunity 合并。
- 标签管理。
- 用户审批。
- 会员权限管理。
- Daily Brief 发布状态。
- 邮件发送状态。
- 错误日志 / 同步日志。

## 优先级

P0：

1. 登录 / 注册。
2. Daily Brief 详情与归档。
3. Opportunities 详情页升级。
4. 7 天试读与到期提示。
5. 会员状态与权限提示。
6. 管理员用户权限管理。

P1：

1. 月 / 季 / 年会员方案页。
2. 邮件订阅入口。
3. 趋势地图。
4. 标签网络。
5. 样例报告。

P2：

1. 支付与续费。
2. PDF 导出。
3. 收藏与跟进状态。
4. 企业版工作台。
5. 资源连接流程。
