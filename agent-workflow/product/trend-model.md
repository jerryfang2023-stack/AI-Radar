# Trends 趋势模型

## 产品定义

Trends 不是趋势名称列表，而是判断某个 AI 方向是否正在形成商业势能的模型。

它要回答：

1. 这个方向处于哪个阶段。
2. 它为什么升温、分化、降温或进入风险区。
3. 哪些证据支持这个判断。
4. 哪些反证会推翻这个判断。
5. 它与哪些机会卡和 Signals 相关。

## 趋势状态

建议使用以下状态：

| 状态 | 含义 |
|---|---|
| rising | 持续升温，证据增多 |
| splitting | 分化中，部分场景成立，部分概念降温 |
| cooling | 降温，新增证据减少或商业化受阻 |
| emerging | 新出现，信号早但可能值得观察 |
| mature | 成熟化，进入竞争和规模化阶段 |
| risk | 风险变量，监管、并购、地缘或安全因素影响判断 |
| invalidating | 反证增强，暂不成立或需要下调优先级 |

## 模型一：证据阶梯

用于判断一个趋势从概念到商业化的位置。

```text
概念出现
→ 产品发布
→ 融资验证
→ 客户采用
→ 收入增长
→ 复购扩张
→ 行业标配
```

趋势越靠后，确定性越强；越靠前，想象空间更大但风险更高。

## 模型二：采用曲线

用于判断市场采用阶段：

```text
早期探索 / 先锋采用 / 主流扩散 / 成熟竞争 / 红海分化
```

每个阶段对应不同机会：

- 早期探索：适合观察和小范围验证。
- 先锋采用：适合找标杆客户和合作伙伴。
- 主流扩散：适合规模化服务和渠道连接。
- 成熟竞争：适合并购、整合、垂直化。
- 红海分化：只保留有客户、数据或渠道壁垒的机会。

## 模型三：机会温度

每个趋势按 6 个维度评估：

| 维度 | 问题 |
|---|---|
| 资金温度 | 是否持续融资、并购或大厂投入 |
| 客户温度 | 是否有真实客户采用和复购 |
| 产品成熟度 | 是否从 demo 进入可交付产品 |
| 竞争强度 | 是否快速拥挤，是否已有头部 |
| 政策/监管阻力 | 是否受到合规、安全、数据限制 |
| 中国适配度 | 是否适合本土客户、场景、渠道和监管环境 |

## 模型四：趋势分化

很多 AI 方向不是简单上升或下降，而是分化。

典型分化：

- 概念降温，但真实业务场景升温。
- 海外升温，中国仍在验证。
- 工具热度下降，但基础设施机会增强。
- 创业机会变窄，但并购与整合机会增强。
- C 端声量高，但 B 端付费证据更强。

## 模型五：反证清单

每个重要趋势都要记录反证。

反证问题：

- 客户是否只试用不续费。
- 融资是否领先于收入证据太多。
- 是否缺少明确采购方。
- 是否依赖单一平台或政策窗口。
- 是否存在安全、隐私、监管、成本瓶颈。
- 是否只是大厂功能，而不是独立创业机会。

## 页面表现建议

Trends 页面应从列表升级为趋势地图：

- 高确定性趋势。
- 正在升温趋势。
- 分化中趋势。
- 风险变量。
- 暂不成立方向。

趋势详情页应展示：

- 趋势一句话判断。
- 当前状态。
- 证据阶梯位置。
- 机会温度六维图。
- 最近 7 / 30 / 90 天变化。
- 相关 Signals。
- 相关 Opportunities。
- 反证观察点。
