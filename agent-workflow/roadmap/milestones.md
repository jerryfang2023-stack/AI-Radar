# 里程碑

## M0：产品基线

目标：

- 明确项目定位、用户任务、栏目边界和 agent 职责。

验收：

- PM Agent 机制建立。
- 产品战略和栏目架构有文档。
- 对外文案边界明确。
- feature_list 能作为统一任务清单。

## M1：数据标准

目标：

- 让 Obsidian 内容源具备云端可解析、可关联、可维护的字段基础。

验收：

- Signals、Priority Engine、Trends、Opportunities 有稳定 frontmatter。
- slug、ID、title、tags、status 不冲突。
- 机会卡公司名清理完成。
- 标签体系字典成型。

## M2：决策闭环

目标：

- 让 Signal、Priority Engine、Trend、Opportunity、Tag 真正互相关联。

验收：

- Scoring 后台化为 Priority Engine，并合并进入 Opportunities。
- 每个高优先级 Opportunity 能追溯到 Signal、Trend 或评分依据。
- 每张 Opportunity 能追溯证据来源。
- Tags 成为搜索、筛选和机会网络能力。
- relation-check 可发现孤立项。

## M3：产品体验

目标：

- 普通用户看到的是高端、清晰、可信的商业判断产品；管理员看到的是可维护的工作台。

验收：

- 普通端无编辑、同步、恢复等后台痕迹。
- Admin 可视化编辑和审批路径清楚。
- 首页、Daily Brief、Signals、Opportunities、Trends 有稳定体验。
- 桌面和移动端不拥挤、不溢出。

## M4：云端运营

目标：

- 从本地 Obsidian + 静态站，升级为可长期云端运营的产品。

验收：

- 内容路径配置化。
- 自动更新、同步、质检、备份、回滚流程完整。
- 注册、登录、7 天试读、会员权限、企业版/私享内参申请和管理员权限可用。
- 发布前检查清单通过。
