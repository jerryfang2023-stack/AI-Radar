# WSD-20260504-24 登录 / 注册页面优化收口

## 1. 任务目标

本轮在当前调度窗口直接执行 `P0-9`。用户明确要求“在此窗口执行任务”，因此未再开启独立执行窗口；同一窗口内按 UI/UE、Copy、Dev、QA 四段顺序完成。

目标：

- 优化 `04-Site/login.html` 与 `04-Site/register.html`。
- 明确注册后的默认 7 天阅读有效期。
- 明确登录后的去向：权限有效时进入 Daily Brief，权限到期时进入账户页。
- 登录 / 注册页达到上线前普通前台页面质量，不像模板表单或后台组件堆叠。
- 不暴露 Admin、JSON、同步、编辑、恢复、字段、后台、申请访问、申请试读等普通前台禁用表达。

## 2. UI/UE 页面规范表

| 项 | 规范 |
|---|---|
| 页面类型 | 会员 / 账号页，普通前台页面 |
| 页面目标 | 用户 5 秒内知道该登录还是注册，并知道注册后默认获得 7 天阅读有效期 |
| 设计基准 | `frontend-design + Linear 表单克制结构 + FT/Economist 内参阅读气质`，不做营销落地页式夸张 hero |
| 布局基准 | 主容器 `960px`；桌面表单约 `440px`，说明区约 `360px`；移动端单列，表单优先 |
| 字体层级 | H1 `34-44px`；表单标题约 `25px`；正文 / 说明 `14-16px`；字段 label `12px` |
| 间距基准 | 标题区与表单区保持清楚间隔；字段高 `46px`；主按钮高 `48px` |
| 组件克制规则 | 保留单层表单卡片 + 单层深墨绿说明区；不做卡片套卡片、不堆徽章、不做大面积装饰 |
| 前台 / Admin 边界 | 普通页面不出现 Admin、后台能力、同步、JSON、编辑、恢复、字段等表达 |
| 桌面端验收点 | 登录 / 注册标题位置一致；表单起点一致；按钮宽度一致；无横向溢出；注册主按钮回到 900px 首屏内 |
| 移动端验收点 | 单列；字段宽度稳定；按钮可点击；无横向溢出；注册页因字段较多允许垂直滚动 |
| 禁止项 | 申请访问、申请试读、保证结果、立即抢占机会、必投机会、内部审批、后台、字段、同步、JSON |
