# 商业内参 V2 页面设计规范

## 1. 栏目定位

商业内参不是普通内容栏目，也不是公开热榜。它是 V2 的会员 / 增值产品入口，承接周度或月度高密度商业判断。

核心用户任务：

| 用户 | 正在判断什么 | 商业内参提供什么 |
|---|---|---|
| 创始人 / 业务负责人 | 哪些 AI 变化正在影响预算、流程和客户采用 | 周期判断、热力变化、证据摘要和观察边界 |
| 投资观察者 / 资源方 | 哪些方向从单点新闻变成连续商业信号 | 高热三元组、升温 / 降温 / 争议解释 |
| 产品 / 战略负责人 | 哪些变化值得进入内部讨论和持续跟踪 | 判断列表、关联信号、趋势背景和来源摘要 |
| 会员用户 | 为什么这一期值得完整阅读 | 封面、目录、热力图、证据展开和往期对比 |

产品结论：

- 商业内参继续作为 V2 一级导航。
- 商业热力图是商业内参核心模块和后台判断资产的会员层前台表达。
- 普通用户看到样张、目录、核心判断摘要和会员边界。
- 登录用户可看到更完整目录、部分试读和往期索引。
- 会员用户可阅读完整内参、热力图、证据展开、往期变化对比。
- 管理员生产过程不进入普通前台。

## 2. 页面阅读路径

### 2.1 栏目列表页

目标：让用户在 30 秒内判断“最新一期是否值得读、商业热力正在指向哪里、会员价值是什么”。

首屏结构：

1. 栏目标题：商业内参。
2. 一句话说明：本周 / 本月哪些 AI 变化正在形成更高密度的商业判断。
3. 最新一期封面样张：期号、周期、标题、会员标签、阅读时长。
4. 本期总判断：1 段，最多 90 字。
5. 会员 CTA：普通用户为“申请进入完整情报层”，登录用户为“查看本期目录”，会员为“阅读完整内参”。

模块顺序：

1. 最新一期封面与总判断。
2. 本期热力摘要，展示 3 个高热三元组，不做榜单排名。
3. 核心判断列表，3-5 条。
4. 证据来源摘要，按来源类型和增量事实说明，不写内部字段。
5. 往期内参，周 / 月归档。
6. 会员说明，说明可读范围和判断边界。

### 2.2 内参详情页

目标：像一份周期性商业判断文件，而不是文章流、新闻专题或后台面板。

首屏结构：

1. 报告封面区：Logo、Issue No.、周期、标题、会员标签、阅读时长。
2. 本期总判断：1 段衬线核心判断。
3. 目录锚点：总判断 / 商业热力图 / 高热三元组 / 升降温 / 来源摘要 / 观察边界。
4. 会员边界：非会员显示可读范围，不遮挡标题和判断。

正文模块顺序：

1. 本期总判断。
2. 商业热力图。
3. 高热三元组。
4. 升温 / 降温 / 争议。
5. 证据展开。
6. 观点校准。
7. 趋势背景。
8. 观察边界。
9. 往期关联。

### 2.3 热力图模块

角色：后台判断资产的前台摘要，帮助会员理解“为什么这组变化升温、降温或存在争议”。

呈现方式：

- 不做公开排行榜。
- 不按公司排名，不做岗位替代榜。
- 使用行业 x 岗位 x 流程三元组。
- 每个三元组展示变化方向、证据密度、商业强度和争议边界。
- 颜色只使用深澜蓝、雾灰、少量香槟金；风险或降温用低饱和红，不使用彩虹热力。
- 分数不作为绝对结论，优先用“升温 / 降温 / 争议 / 稳定”表达。

### 2.4 会员预览态与锁定态

普通用户：

- 可见最新一期标题、核心判断摘要、3 个热力摘要项、目录前 3 项。
- 不显示完整证据展开、完整热力矩阵、往期变化对比。
- CTA：申请进入完整情报层。

登录用户：

- 可见目录、部分试读、往期标题和本期来源类型摘要。
- CTA：申请阅读本期完整内参。

会员用户：

- 可见全文、热力图、证据展开、往期对比。
- CTA 转为保存、分享、查看往期。下载 / PDF 暂不作为本轮必做。

锁定态规则：

- 锁定层只覆盖受限模块，不整页遮挡。
- 锁定文案说明“完整判断和来源展开保留在会员层”，不写技术或后台原因。
- 预览内容必须仍有判断价值，不能只剩营销文案。

## 4. 桌面与移动布局

桌面端：

- 栏目页主容器 1200-1280px。
- 详情页采用 860px 正文 + 320px 侧栏，热力图模块可短暂突破到 1200-1440px。
- 章节之间保持 64-96px，报告内部段落保持 20-28px。
- 热力摘要使用非等宽三列或两列主次结构，避免模板化三卡并列。

移动端：

- 左右 20px，严格单列。
- 页面标题 30/42，内参标题 28/40。
- 封面样张保持纸面比例，但不强制满屏。
- 热力图改为纵向三元组卡，每项包含状态、变化原因和边界一句。
- CTA 固定在模块末尾，不做底部悬浮遮挡正文。

## 5. Copy 文案规范表

| 场景 | 建议文案 | 禁用 / 避免 |
|---|---|---|
| 栏目主标题 | 商业内参 | AI商业热力图、会员中心、商业情报入口 |
| 栏目说明 | 本周 / 本月哪些 AI 变化正在形成更高密度的商业判断。 | 本页用于展示会员报告和热力图字段 |
| 最新一期标题规则 | 方向 + 商业变化，不写公司名；如“企业 Agent 从工具入口走向流程治理” | OpenAI / 某公司商业内参、确定性机会 |
| 热力图模块标题 | 本期热力变化 | 商业热力排行榜、岗位替代榜 |
| 热力图说明 | 看哪些行业、岗位和流程正在升温、降温或进入争议。 | 强证据证明机会成立 |
| 会员预览 | 你可以先阅读本期判断摘要和目录，完整来源与边界保留在会员层。 | 立即解锁财富密码 |
| 锁定态 | 完整热力图、来源展开和往期对比仅向会员开放。 | 付费后获得确定机会 |
| 普通用户 CTA | 申请进入完整情报层 | 立即行动、马上赚钱 |
| 登录用户 CTA | 申请阅读本期完整内参 | 下一步验证 |
| 会员 CTA | 阅读完整内参 / 查看往期内参 | 进入后台 |
| 判断边界 | 不替代最终经营、投资或合作判断。 | 本系统给出投资建议 |

禁用语检查：不得出现 Admin、JSON、同步、字段、后台、证据链、强证据、来源明确、机会确定、下一步验证、落地方案、立即行动、保证收益、赋能、风口红利。

## 6. 数据字段 / 内容模块映射

| 前台模块 | 内容来源 | 前台表达 | 隐藏或后台化 |
|---|---|---|---|
| 最新一期 | AIBriefIssue | 期号、周期、标题、总判断、阅读时长 | 生成状态、脚本、发布过程 |
| 本期热力摘要 | HeatCard / HeatmapTriple | 3 个高热三元组、升降温或争议 | HeatScore 计算细节 |
| 商业热力图 | HeatEvidence / HeatCard | 影响坐标、变化方式、证据密度、速度、商业强度、边界 | Raw / Pool / Structured 字段 |
| 核心判断列表 | AIBriefIssue summary | 3-5 条判断 | 内部评分、编辑状态 |
| 证据来源摘要 | FrontSignal / OpportunityReport / TrendContext / PointCalibration | 来源类型、增量事实、观察边界 | C/M 级发现通道、未解析 URL |
| 高热三元组 | industry / role / workflow | 行业 x 岗位 x 流程 | 标签治理字典后台 |
| 升温 / 降温 / 争议 | TrendContext / CounterEvidence | 变化原因和反证 | 评分公式默认展开 |
| 观点校准 | PointCalibration | 支持、质疑或修正什么 | 观点热榜、人物墙 |
| 往期关联 | AIBriefIssue archive | 与上期相比的变化 | 数据不足时不强行对比 |

## 7. Design Director 质检

| 质检项 | 分值 | 评分 | 证据 |
|---|---:|---:|---|
| 商业内参感 | 15 | 14 | 封面、Issue No.、周期判断和纸面节奏明确 |
| 判断成品质感 | 15 | 14 | 先总判断，再热力、证据和边界，不像新闻列表 |
| 热力图克制专业 | 15 | 13 | 三元组 + 变化解释，禁止公开榜单和彩虹热力 |
| 信息层级清晰 | 15 | 14 | 列表页和详情页路径分离，会员态不抢主阅读 |
| 视觉克制与高级感 | 15 | 13 | 使用 `#FFFDF8`、深澜蓝、雾灰、少量香槟金 |
| 阅读效率 | 10 | 9 | 详情正文窄读宽，热力模块允许局部加宽 |
| 模块节奏 | 10 | 8 | 首屏、热力、证据、边界形成节奏 |
| 移动端可读性 | 10 | 9 | 全部单列，热力矩阵折为三元组卡 |
| 商业情报产品感 | 10 | 9 | 不做营销页、后台面板或模板页 |
| 总分 | 110 | 103 | 折算 93.6 / 100，通过会员核心页建议线 85 |

Squint Test：通过。第一眼应是“商业判断报告 + 会员内参样张”，不是科技 dashboard、新闻列表或营销落地页。

必须重做清单：

- 若实现时出现公开榜单、岗位替代榜、公司热度榜，必须重做热力模块。
- 若实现时把完整数据字段直接摊开，必须重做详情页阅读路径。
- 若锁定态整页遮挡标题和总判断，必须重做会员预览。

可延后优化：

- PDF 下载、保存、分享可后置，待会员系统和权限方案明确后再做。
- 月度完整版热力图可后置，先做周度 MVP。

## 8. QA 可开发性验收

| 检查项 | 结论 |
|---|---|
| 是否继承 failed 任务成果 | 否。本规范不继承 `V2-SITE-QUALITY-AUTO` 的任何成功结论或 mockup。 |
| 是否改变一级导航 | 否。继续采用今日要点 / 关键信号 / 机会解码 / 商业内参。 |
| 是否使用 frontend-design | 否。使用项目 VI 与 `design-taste-frontend` 的审美检查边界。 |
| 是否遵守 VI / token / 字体 | 是。使用 `--gl-bg-page`、`--gl-*` token、中文衬线标题、黑体正文、Mono 数字。 |
| 是否区分普通前台、会员预览和内部后台 | 是。生产过程、字段、脚本和计算细节后台化。 |
| 是否可交给 Dev 实现 | 是。已给出页面类型、模块顺序、字体、宽度、状态和禁用项。 |
