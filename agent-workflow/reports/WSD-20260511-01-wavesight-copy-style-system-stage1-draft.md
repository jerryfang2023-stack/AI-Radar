## 4. 保留 / 替换 / 删除 / 新增规则清单

### 4.1 建议保留

1. 文案不是普通营销页，也不是技术说明，而是面向商业决策者的商业内参表达。
2. 每句话必须提供信息、判断或边界。
3. 有观点，但不替客户下最终判断。
4. 对外讲利益，不讲内部流程。
5. Signal 标题必须是“事件 + 商业含义”。
6. Opportunity 标题不能写公司名，公司只能作为证据、案例或来源。
7. Builder / Point 类内容只作观点校准，不替代事实证据。
8. 商业内参必须区分事实、观点、观澜判断和证据缺口。
9. 公开前台不得出现 Markdown、JSON、同步、字段、后台、脚本、编辑、恢复等内部词。

### 4.2 建议替换

| 旧规则 / 旧表达 | 替换为 |
|---|---|
| Daily Brief | 今日要点；英文仅作为内部语义 |
| Signals | 关键信号；Signal 可用于内容类型，不作前台栏目名 |
| Opportunities | 机会解码；Opportunity 可用于内部对象 |
| Trends | 趋势背景 / 热力输入 / 证据积累 |
| The Point | 观点校准 / Builders 观点 |
| 申请查看完整信号 | 登录查看完整信号 / 试读查看完整信号 / 会员查看完整信号 |
| 申请进入完整情报层 | 了解商业内参 / 查看会员内参 / 申请企业内参 |
| 下一步验证 | 建议关注变量 / 后续观察变量 |
| 可行动机会 | 值得继续观察的机会方向 |
| 强证据 / 证据链 | 来源与事实 / 支撑信息 / 多源支撑 / 证据缺口 |

### 4.3 建议删除

1. 将 Trends 写成一级栏目或独立频道的语气。
2. “这个页面用于 / 本页只做 / 入口 / 模块用于”等页面说明式文案。
3. “马上行动 / 立即验证 / 必须进入清单 / 机会确定”等行动指令。
4. “财富密码 / 风口红利 / 颠覆未来 / 确定性机会 / 保证收益”等营销或投资承诺。
5. “Raw / Pool / Heat Candidate / M 级线索 / Structured 生产过程”等内部漏斗词的前台表达。
6. “中国迁移难点”作为机会解码固定栏目。

### 4.4 建议新增

1. 品牌声音四项：高级、克制、东方智识、商业内参、判断感。
2. V2 栏目表达总表。
3. 标题系统：页面标题、栏目标题、卡片标题、详情标题、内参标题、知识卡标题。
4. 摘要系统：一句话判断、来源状态、风险边界、观察口径。
5. CTA 系统：访客、登录、试读、到期、会员、企业内参。
6. 每日监测产物文案规则。
7. 知识库卡片表达规则。
8. 禁用语 / 替代表。
9. 20 条以上 before / after。

## 5. 新版观澜AI V2 文案风格系统草案

### 5.1 一句话定义

观澜AI 的文案是面向商业决策者的 AI 机会判断表达：从外部变化中筛出信号，说明它可能意味着什么，并保留来源、反证和观察边界。

### 5.2 品牌声音

| 维度 | 应该像 | 不应该像 |
|---|---|---|
| 高级 | 商业内参、研究报告、判断摘要 | 低价营销页、工具站、流量标题 |
| 克制 | 有倾向，但留出证据边界 | 鸡血、焦虑、强推 |
| 东方智识 | 稳、准、含蓄、有余地 | 国潮、书法化、玄学化 |
| 商业内参 | 先判断，再讲事实和边界 | 新闻聚合、公司通稿 |
| 判断感 | 明确指出为什么值得看 | 只堆信息，不给解释 |

### 5.3 语气边界

必须做到：

- 写信号，不写噱头。
- 写判断，不写定论。
- 写来源与事实，不写“证据链”表演。
- 写建议关注变量，不写行动指令。
- 写不确定性，不写免责声明套话。
- 写给商业决策者，不写给内部运营团队。

不得出现：

- 投资建议、经营指令、合作指令。
- 收益承诺、确定性承诺、窗口焦虑。
- 内部生产流程、后台字段、同步状态。
- 把观点来源写成事实主证据。

### 5.4 固定品牌写法

| 项 | 固定写法 |
|---|---|
| 中文品牌 | 观澜AI |
| 中英文并列 | 观澜AI｜WAVESIGHT AI |
| 英文品牌 | WAVESIGHT AI |
| 定位语 | 观AI之澜，识商业之势。 |
| 产品定位 | 面向商业决策者的 AI 机会判断系统 |
| 核心逻辑 | 从 AI 热点中筛出商业信号，从信号形成判断，从判断发现机会 |

避免写法：

- WaveSight AI
- WavesightAI
- wavesight ai
- AI 商业机会情报系统
- AI 新闻雷达
- AI 工具导航

## 6. 栏目表达规则草案

| 栏目 / 模块 | 用户任务 | 推荐表达 | 禁止表达 |
|---|---|---|---|
| 今日要点 | 今天先看什么，为什么 | “今天值得先看的变化集中在……” | 今日新闻列表、日报汇总、行动建议 |
| 关键信号 | 哪些变化值得进入判断资产 | “这条信号的价值不在事件本身，而在……” | 公司名列表、融资新闻、来源明确 |
| 机会解码 | 哪些方向值得继续观察 | “这个机会方向仍处在证据增强阶段……” | 确定性机会、马上验证、公司名机会 |
| 商业内参 | 跨周期组合判断 | “本期关注的是一组信号共同指向的变化……” | 泛泛观点合集、公开热榜、单篇机会卡 |
| 观点校准 | 用观点修正判断 | “该观点可作为判断校准，不替代事实来源。” | 大佬已证明、观点就是事实 |
| 趋势背景 | 说明证据积累方向 | “过去 30 天，证据主要集中在……” | 趋势很热、未来可期 |
| Builders 观点 | 观察建造者判断变化 | “这位 Builder 的关注变量从模型能力转向客户采用。” | The Point 一级栏目、马上跟进 |

### 6.1 今日要点

今日要点是每日商业判断，不是新闻流。

标题推荐：

- “今天，企业 Agent 的桌面入口竞争继续升温”
- “模型能力之外，客户采用开始成为更重要变量”
- “合规、桌面和销售自动化构成今天的三条主线”

摘要结构：

```text
一句今日判断 + 3-5 个值得关注变化 + 证据不足处 / 后续观察变量
```

避免：

- “今日新增 12 条线索”
- “建议立即验证以下机会”
- “本页用于展示每日同步结果”

### 6.2 关键信号

关键信号负责说明“为什么这不是普通新闻”。

Signal 标题公式：

```text
事件 + 商业含义
```

摘要结构：

```text
发生了什么 + 指向什么商业变量 + 来源状态 + 反证或缺口
```

推荐字段转译：

| 内部字段 | 前台表达 |
|---|---|
| source_count | 来源数量 / 多源支撑 |
| evidence_level | 来源状态 / 支撑强度 |
| related_opportunity | 指向机会 |
| related_trend | 关联趋势背景 |
| risk | 反证与边界 |

### 6.3 机会解码

机会解码是低频、少而精的深度机会分析报告，不是机会卡列表。

机会标题公式：

```text
客户 / 场景 / 能力方向
```

推荐：

- “企业桌面端 Agent 执行层”
- “AI 招投标 Agent”
- “合规引擎与 Agent 治理”

避免：

- “某公司机会”
- “AI 工程仿真软件-JuliaHub”
- “下一代确定性投资方向”

结论表达分级：

| 判断状态 | 推荐表达 |
|---|---|
| 早期观察 | 当前更像早期信号，适合继续观察。 |
| 证据增强 | 来源和客户采用证据正在增加，值得重点跟踪。 |
| 可小范围验证 | 可围绕明确客户场景做小范围验证。 |
| 风险升高 | 证据仍不足，短期不宜过度乐观。 |
| 暂缓判断 | 反证增加，暂缓写成正式机会。 |

### 6.4 商业内参

商业内参是会员 / 增值层的周期判断报告，不是普通文章栏目。

标题推荐：

- “企业 Agent 从聊天入口走向执行层”
- “AI 合规正在从报告工具变成生产前置件”
- “模型热度之外，客户采用证据正在分化”

摘要结构：

```text
本期总判断 + 关键证据簇 + 争议或反证 + 后续观察变量
```

锁定态原则：

- 不整页遮挡。
- 公开部分保留判断摘要和来源状态。
- 会员部分展开热力图、证据矩阵、完整来源和长期观察变量。
- CTA 克制，不制造错失焦虑。

## 7. 标题规范草案

| 标题类型 | 规则 | 长度建议 |
|---|---|---|
| 首页 Hero | 品牌价值 + 判断对象，不写功能说明 | 16-28 中文字 |
| 一级栏目标题 | 一句话价值表达，不加解释型副标题 | 14-24 中文字 |
| 卡片标题 | 信息完整，避免只写公司名 | 18-34 中文字 |
| Signal 标题 | 事件 + 商业含义 | 24-42 中文字 |
| Opportunity 标题 | 机会方向，不写公司名 | 8-18 中文字 |
| 商业内参标题 | 周期判断或主题判断 | 16-32 中文字 |
| 知识卡标题 | 对象 + 判断变量 | 12-28 中文字 |

字体适配提醒：

- 页面大标题使用中文衬线，句子要短，避免长串名词。
- 卡片标题使用中文黑体，优先一到两行可读。
- 英文标签使用全大写和正字距，但正文不混入过多英文术语。
- 标题过长时退回 Copy 压缩，不靠放大容器硬塞。

## 8. 摘要与导语规范草案

摘要必须回答四件事：

1. 发生了什么。
2. 它可能意味着什么。
3. 当前来源状态如何。
4. 仍需观察什么。

推荐句式：

- “这条信号目前由多源支撑，但客户付费证据仍需继续观察。”
- “它更像一次入口竞争的前奏，而不是已经完成验证的机会。”
- “本期内参关注的不是单家公司，而是一组信号共同指向的流程变化。”

避免句式：

- “这是一条强证据链。”
- “机会已经确定。”
- “建议马上进入验证。”
- “本模块用于展示来源字段。”

## 9. CTA 与会员态规范草案

| 用户状态 | 推荐 CTA | 避免 CTA |
|---|---|---|
| 访客 | 查看今日要点 / 了解商业内参 / 登录试读 | 立即抢占红利 |
| 未登录但可预览 | 登录继续阅读 / 注册试读 7 天 | 马上解锁财富密码 |
| 试读会员 | 继续阅读完整信号 / 查看完整内参 | 立即付费否则错过 |
| 试读到期 | 续订后继续阅读 / 查看会员方案 | 最后机会 |
| 付费会员 | 阅读完整报告 / 查看来源附录 / 进入关联内容 | 开通后稳赢 |
| 企业或私享内参 | 申请企业内参 / 预约闭门简报 | 立即获得定制答案 |

CTA 语气规则：

- 动词要平实：查看、阅读、继续、了解、申请。
- 不用命令式和焦虑式表达。
- 不把会员态写成“更聪明的人才看得到”。
- 锁定态说明只讲访问边界，不讲夸张收益。

## 10. 每日监测产物文案规则草案

| 产物 | 角色 | 标题规则 | 摘要规则 | 禁止 |
|---|---|---|---|---|
| Front Signal | 深度信号报告 | 事件 + 商业含义 | 多源事实 + 商业变量 + 反证 | 写成普通新闻 |
| Structured Signal | 入库信号卡 | 事件 + 指向变量 | 快速说明事实、来源、关联对象 | 堆字段 |
| Opportunity Deep Dive | 深度机会报告 | 机会方向，不写公司名 | 机会判断 + 成立边界 + 观察变量 | 硬凑深挖 |
| Trend Update | 趋势背景更新 | 趋势方向 + 证据变化 | 7 / 30 / 90 天证据变化 | 热度即趋势 |
| Point / 观点校准 | 判断输入 | 人物 / 机构 + 观点变化 | 观点如何支持、修正或质疑主判断 | 观点当事实 |
| Business Brief | 周期组合判断 | 主题 / 周期判断 | 信号链 + 机会组合 + 风险边界 | 泛泛观点合集 |

每日内容禁用：

- Raw、Pool、Heat Candidate、M 级线索。
- “今日暂无内容”式空泛占位。
- “建议立即验证”“行动清单”。
- C / M 级发现通道直接写成事实结论。

## 11. 知识卡片表达规则草案

知识卡片不是后台字段展示，而是可复用的判断资产。

| 卡片类型 | 标题 | 双链摘要 | 标签语气 |
|---|---|---|---|
| 信号库 | 事件 + 商业含义 | 这条信号关联哪些机会、趋势和来源边界 | 用正式主题词，不随手造词 |
| 机会库 | 机会方向 | 关联哪些信号、客户场景、风险变量 | 标签描述客户 / 场景 / 流程 |
| 观点库 | 人物 / 机构 + 观点变量 | 观点支持、修正或质疑什么判断 | 不把人名当趋势 |
| 案例库 | 公司 / 产品 + 代表证据 | 这个案例能支撑哪个机会或信号 | 公司是证据，不是机会 |
| 趋势背景 | 趋势方向 + 状态 | 证据如何在 7 / 30 / 90 天变化 | 不写成热词墙 |

知识卡摘要公式：

```text
它是什么 + 支撑哪个判断 + 证据边界 / 反证
```
