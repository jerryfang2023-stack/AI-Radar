# 观澜AI｜网站模块与设计规范系统评审

## 0. 结论摘要

本轮评审只做 PM / UI / Copy 层面的系统梳理，不修改 `04-Site/`，不进入 Dev，不生成首页海报图，不把 `P0-2B` failed 结果当成成功版本。

核心结论：

1. 当前产品主线成立：`首页 / Daily Brief / Signals / The Point / Opportunities / Trends` 可以继续作为前台一线结构。
2. 需要立刻优化的不是“再加模块”，而是全站高级感、模块主次、页面母版和前后台边界。
3. 首页当前仍保留旧方向：抽象雷达图、四张利益海报卡、泛营销式利益表达，和观澜AI“高端商业情报产品”的调性不匹配，应作为 P0-10 / P0-11 的首要问题处理。
4. `scoring.html` 与 `tags.html` 虽未在主导航出现，但仍是可直接访问的前台页面。它们不应作为普通用户独立产品页继续强化，应分别后台化 / 合并进 Opportunities 与搜索筛选能力。
5. `Signals` 已接近商业情报列表，但顶部指标、筛选区和雾化背景仍有后台仪表盘感。需要收敛为“信号检索台”，而不是数据面板。
6. `Opportunities` 应保留并强化，但右侧 `机会排行` 容易把 Priority Engine 误读成榜单；建议改成“优先观察”或“验证序列”，并弱化分数的视觉权重。
7. `The Point` 作为观点层有价值，但它不能变成观点归档或人物墙；首页、栏目页和详情页都应强调“共识、分歧、边界”，而不是热榜或日期列表。
8. `Admin` P0 工作台已 accepted，但邀请码、访问统计、申请审批仍是本地演示能力；上线前必须进入 P0-6 / P0-4 做前后台边界与生产边界复查。
9. `styles.css` 已超过 9000 行，并存在多轮 `final / override / lock / polish` 样式层。高级感难以继续靠追加样式获得，P0-10 必须先输出 DESIGN v2，再做 CSS 母版收敛。
10. 移动端本轮暂不作为高级感主目标；后续验收只保留“不溢出、不遮挡、不破坏阅读”的底线。

本报告分为两部分：

- 第一部分：当前优化版。目标是在不新增栏目、不大拆结构、不进入 Dev 的前提下，把现有页面优化到更符合观澜调性的高级商业情报产品。
- 第二部分：后续重构版。目标是为 P0-10 / P0-11 重新定义 DESIGN v2、桌面母版、视觉资产规则和上线前结构边界。

## 第一部分：当前优化版

当前优化版的原则是：保留现有产品结构，先修“看起来不高级、不像观澜、不像商业内参”的地方。它不要求重写全站，只要求后续执行窗口能马上按页面改动。

### 当前优化版立即执行总表

| 页面 / 模块 | 当前不合理或不好看之处 | 为什么不符合观澜调性 | 当前优化方向 | 可马上派发给谁 |
|---|---|---|---|---|
| 首页首屏 | H1 抽象，右侧是雷达流程图，四张利益海报卡偏营销 | 像 AI 能力说明页，不像商业情报内参 | H1 改为具体商业判断；删除雷达图与 poster 卡；右侧改成 Intelligence Desk 样张 | P0-10 / P0-11 |
| 首页第二屏 | `判断路径`、`看变化 / 看观点 / 看来源 / 看边界` 有内部教学感 | 高端产品应直接呈现判断，不先解释怎么读 | 改成“今日主判断 + 3 条支撑信号 + 机会 / 趋势摘要” | P0-10 |
| Signals | 顶部三张指标卡和 6 个筛选控件像后台查询台 | 信号页应像商业信号检索台，不是运营仪表盘 | 指标压缩为一行 summary；筛选合并为 command bar；列表标题和来源成为主视觉 | P0-5 / P0-6 后 Dev |
| Signals 源码 | `editorDialog` 含编辑、删除、保存 | 普通前台出现后台能力痕迹 | P0-6 先验收，后续移到 Admin-only 或删除前台 dialog | P0-6 |
| Daily Brief | `daily-primer` 像内部阅读路径说明 | Daily 应像每日商业内参，不像功能教程 | 首屏直接展示最新 Brief 主判断、日期、三条支撑，不显示 1/2/3 教程 | P1-4 或 P0-10 |
| Opportunities | 右侧 `机会排行` 和评分视觉像榜单 | 机会库不是排行榜，评分是后台判断辅助 | 改名为 `优先观察` / `验证序列`；分数降为 meta，机会定义成为主视觉 | P0-10 后 Dev |
| Trends | 趋势页容易成为趋势卡片墙 | 趋势应表达阶段、分化和反证，而不是热词集合 | 分组为升温 / 分化 / 降温 / 风险；每组先给主判断，再列证据 | P0-5 / P1-4 |
| The Point | 侧栏归档和主题热容易把页面拉回内容库 | The Point 的价值是共识、分歧和边界 | 主区固定展示今日共识 / 分歧 / 边界；往期和主题降级 | P0-5 |
| `scoring.html` | 独立 Priorities 页面仍可访问 | Scoring 是后台 Priority Engine，不是一线产品 | 普通前台隐藏或 Admin-only；前台结果并入 Opportunity | P0-6 / P0-4 |
| `tags.html` | 独立标签索引像内部资料库 | Tags 是筛选和关系能力，不是一线栏目 | 暂不强化独立页；先合并到栏目内筛选 | P0-6 / Data 后续 |
| Pricing / Account | 可能像通用 SaaS 套餐页 | 观澜是情报订阅，不应像工具订阅 | 用“阅读权限、试读有效期、完整情报层”组织页面 | P0-4 / P1-3 |
| 色彩 | 页面存在雾化背景、浅绿扩散和多轮样式叠加 | 会显得像模板修补，不像统一品牌系统 | 收敛为象牙白、纸面白、深墨绿、铜棕、细线 | P0-10 |
| 图片 / 视觉资产 | `hero-radar.svg`、`home-hero-radar.png` 抽象且含 Scoring | 表达内部流程，不表达真实判断价值 | 停用为首页主视觉；改为真实判断样张或纸面情报桌面 | P0-11 |

### 1.3 新增功能门禁记录与 WAVE 评分

本报告不建议新增任何前台一级栏目、后台模块或会员功能。以下仅是“已有模块内的强化视图 / 视觉承载方式 / 后台化处理建议”，不得绕过用户确认直接进入 Dev。

### 3.1 首页 Intelligence Desk 样张

| 门禁项 | 结论 |
|---|---|
| 功能名称 | 首页 Intelligence Desk 样张 |
| 功能类型 | 前台 / 视觉承载 / 现有首页强化 |
| 对应用户任务 | 判断平台是否值得注册；今天看什么 |
| 目标用户场景 | 访客打开首页，5 秒内判断观澜AI不是新闻站，而是商业机会判断系统 |
| 预期商业结果 | 提升注册意愿、品牌可信度和付费前信任 |
| 现有模块为什么不能承载 | 现有首页可以承载；问题是当前雷达图和海报卡表达不够具体 |
| 是否可改为已有模块内视图 | 是，必须作为首页内视觉样张，不新增页面 |

| 数据与运营依赖 | 使用当天真实 Signal / Opportunity / Trend 的脱敏短句；不嵌入后台字段 |

### 2.1 DESIGN v2 修改建议

### 5.1 Art Direction

观澜AI应采用：

- 产品气质：高端商业内参、研究桌面、判断样张、克制会员产品。
- 视觉关键词：克制、研究感、纸面秩序、深墨绿重点、真实情报样张。
- 非目标视觉：AI 紫蓝渐变、赛博大屏、机器人头像、抽象雷达流程图、模板 SaaS 卡片墙、投融资榜单页。
- 用户感受：可信、有判断、有边界、值得读，而不是炫技、营销、堆功能。

### 5.2 色彩系统

建议 DESIGN v2 固定以下桌面端主色板：

| 用途 | 建议色 | 用法 |
|---|---|---|
| 页面背景 | `#F6F1E8` | 全站底色，保留纸面温度 |
| 纸面 / 卡片 | `#FFFDF8` | 内容容器，不做强玻璃拟态 |
| 主文字 | `#17231F` | 标题和正文 |
| 次级文字 | `#5E6B66` | meta、说明 |
| 深墨绿 | `#13251F` | 首屏重点面板、Admin 侧栏、主要按钮 |
| 铜棕 | `#A45F36` | eyebrow、少量强调、日期 |
| 低饱和青绿 | `#0F6F62` | 链接、状态、趋势 |
| 边框 | `#D8D0C4` | 细线、表格分隔 |

执行规则：

- 首页和栏目页禁止大面积雾化渐变背景；如果需要层次，只用非常浅的纸张阴影或局部线性过渡。
- 深墨绿只用于一个重点面板或主按钮，不整页铺满。
- 铜棕只用于小面积识别，不做大面积背景。
- 不再新增紫蓝、霓虹蓝、彩色节点、强光束。

### 5.3 字体层级

| 层级 | 桌面建议 | 使用场景 |
|---|---:|---|
| 首页 H1 | 64-72px / 1.08 | 仅首页主标题 |
| 栏目页 H1 | 52-60px / 1.10 | Daily、Signals、The Point、Opportunities、Trends |
| 详情页 H1 | 38-46px / 1.18 | Signal / Trend / Opportunity / Point 详情 |
| Section H2 | 26-32px / 1.22 | 模块标题 |
| 卡片标题 | 18-22px / 1.35 | 列表与卡片 |
| 正文 | 15.5-17px / 1.7-1.85 | 详情正文 |
| Meta / Eyebrow | 11-13px / 1.2 | 日期、来源、状态 |

执行规则：

- 栏目页标题必须一行表达，不加解释型副标题。
- 卡片标题不得超过 24px。
- 详情正文用报告阅读节奏，避免字段式卡片堆叠。

### 5.4 页面母版

| 母版 | 结构 | 立即执行规则 |
|---|---|---|
| 首页母版 | 左侧价值判断 + 右侧 Intelligence Desk + 下沿露出今日内容 | 删除旧雷达和 poster 卡；首屏只保留一个主任务 |
| 栏目页母版 | 统一标题区 + command bar + 主列表 / 主卡组 | 指标、筛选、徽章都降级，不抢标题和列表 |
| 详情页母版 | 报告正文 + 右侧摘要 / 关联 | 左侧读判断，右侧看来源和关联，不做字段平铺 |
| 会员页母版 | 权限状态 + 表单 / 套餐动作 + 克制说明 | 不做营销落地页，不使用焦虑 CTA |
| Admin 母版 | 左侧窄导航 + 顶部状态条 + 高密度工作区 | 不与前台共用栏目页节奏 |

### 5.5 组件克制规则

- 卡片只用于独立内容项、工具面板、模态框；页面区块不要都做大卡片。
- 徽章最多用于状态和少量分类；同一行超过 3 个徽章要折叠或降级为 meta。
- 表格 / 列表优先用细分隔线，不用重阴影。
- 按钮只保留主动作和次动作；普通查看链接可用文本链接。
- 首页视觉资产不嵌入关键文字，关键文字由 HTML/CSS 承载。

### 5.6 首页右侧视觉资产规则

P0-11 应在 P0-10 accepted 后执行，视觉资产必须满足：

1. 内容是“商业判断样张”，不是抽象 AI 氛围。
2. 桌面端画面应包含：今日 Brief 摘要、2 条 Signal、1 条 Opportunity、1 个 Trend 状态，彼此有克制关系线。
3. 不出现 `Scoring`、`JSON`、`同步`、`后台`、`字段`、`编辑`。
4. 不出现机器人头像、霓虹科技背景、纯雷达网格、股市 K 线堆叠、通用 SaaS 插画。
5. 如果使用生成图，必须只做氛围和纸面结构，不嵌中文小字；真实文字由页面渲染。
6. 如果使用 HTML/CSS 样张，应优先用真实站内数据，截断为 1-2 行。
