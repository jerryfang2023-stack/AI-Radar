# 商业内参 V2 模块矩阵

## 1. PM 新增功能门禁

| 门禁项 | 结论 |
|---|---|
| 商业内参是否继续作为一级导航 | 是。它是 V2 会员 / 增值层入口，不改变四导航。 |
| 商业热力图是否为前台模块 | 是，但只作为商业内参核心模块和会员层完整表达；普通用户仅看摘要样张。 |
| 报告封面是否为前台模块 | 是。封面负责建立内参产品感、期号和周期判断。 |
| 会员预览态是否为前台模块 | 是。普通用户、登录用户、会员必须差异化。 |
| 锁定态是否为前台模块 | 是，但只锁受限模块，不整页遮挡。 |
| CTA 是否为前台模块 | 是。CTA 根据用户状态变化。 |
| 普通用户开放内容 | 样张、标题、总判断摘要、热力摘要、目录、往期标题。 |
| 会员 / 增值内容 | 全文、完整热力图、来源展开、往期变化对比、观察边界。 |
| 是否新增专题报告 | 是，作为 AIBriefIssue 周 / 月期刊；不新增独立一级栏目。 |
| 是否新增 Issue No. | 是。作为报告封面和归档识别。 |
| 下载 / 保存 / 分享 | 延期复核。会员 MVP 先不绑定支付或下载。 |
| 模块后台化 | HeatScore 计算、Raw / Pool / Structured、生产状态、质量检查、标签治理后台化。 |
| 是否可进入 Dev | 可以进入下一步 Dev 派发，但不得顺带修改会员定价、支付、自动化或 schema。 |
