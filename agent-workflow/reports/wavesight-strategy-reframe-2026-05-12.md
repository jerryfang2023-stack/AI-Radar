# WaveSight AI 战略重梳草案

日期：2026-05-12  
状态：draft / strategy-reframe  
用途：记录本轮战略、产品、内容资产、栏目和前沿观点系统的重梳共识。  

> 本文件是草案报告，不直接覆盖 `strategy-single-source.md`、`column-architecture.md`、`tag-taxonomy.md` 或 V2 当前生产规则。后续如需升级为正式规则，应另行派发治理更新任务。

## 1. 暂时保留与开放重审

本轮重梳只默认保留两项：

- 品牌名：观澜AI｜WaveSight AI。
- 大方向：AI × 商业机会 / 商业变化判断。

其他历史结论均视为当前假设，不直接视为正确答案。

## 2. 新战略主轴

观澜的主轴从“AI 机会判断系统”调整为：

> AI 商业变化解释与趋势推演系统。

更准确的工作目标是：

> 帮用户理解一个 AI 变化为什么发生、会影响什么、可能走向哪里，以及类似产品和场景正在怎样演化。

机会判断仍然重要，但它不再是每条内容的唯一结果。观澜优先解释已经可见的变化；只有当存在可追踪来源和足够材料时，才进一步给出趋势、机会或周期判断。

## 3. 第一性用户

第一性用户是：

> 有资源、有预算、有业务判断权或连接能力，正在判断 AI 能否变成具体生意、投资、合作、项目机会或内容影响力的人。

包括但不限于老板、投资人、产业资源方、创新负责人和类似角色。

不把普通 AI 爱好者、工具使用者、泛职场用户和开发者社区作为第一性用户。

## 4. 双层目标

观澜同时服务两个目标。

第一层：老板自己的判断资产系统。

- 短期：快速了解 AI 商业趋势。
- 中期：积累数据库、案例库、观点库，形成行业嗅觉和判断力。
- 长期：从持续观察中发现创业机会、合作机会、投资机会、资源连接机会和内容选题。

第二层：对外内容增长系统。

- 将内部判断资产二次创作为公众号、视频号、小红书、播客、朋友圈、社群分享等内容。
- 通过高质量内容获得流量、信任和高质量连接。

第一阶段采用“先内后外”：先建设个人 / 小团队使用的 AI 商业判断资产库，再从中筛选可传播内容做自媒体分发。

## 5. 三层系统

观澜不是单一网站，而是三层系统。

### 5.1 Obsidian / 本地知识库

判断资产主库。变化卡、案例卡、观点卡、趋势卡、来源记录、关系链和趋势脉络优先在本地知识库沉淀。

### 5.2 网站发布层

网站负责发布和呈现一部分整理后的内容，包括今日观察、商业信号、趋势追踪、商业内参和相关卡片详情。网站不是全部研究和判断工作的承载层。

### 5.3 自动化 / 数据处理层

负责搜索、抓取、初筛、Raw / Pool 管理、变化卡草稿生成、自动关联、相似卡片发现、前沿观点监测和二创候选输出。

## 6. 来源与数据源规则

所有判断必须外显来源；所有数据必须外显数据源；原文链接要让用户可以快速打开核对。

内容来源和数据源分开管理。

内容来源回答“这件事从哪里看到”。  
数据源回答“这个数字、规模或可量化判断从哪里来”。

前台来源可信度表达采用用户可读语言：

- 原始出处：来自公司、机构、监管、产品、财报、论文、招采等原始材料。
- 专业报道：来自可信媒体、研究机构、融资数据库、正式访谈或行业报告。
- 市场线索：来自社媒、社区、榜单、评论、开源活跃度、搜索热度等辅助观察信号。

数据源前台表达：

- 公开数据：财报、监管文件、官方统计、公开数据库。
- 机构估算：研究报告、行业数据库、市场规模测算。
- 行为信号：招聘、下载、评论、GitHub、云市场、搜索热度等代理指标。

后台可继续保留更细的 S/A/B/C/D 或权重体系。

## 7. 核心资产类型

正式长期资产四类：

1. 变化卡：记录一个 AI 商业变化。
2. 案例卡：记录一个公司、产品、项目、客户采用、商业模式或技术路线案例。
3. 观点卡：记录重要人物或机构观点。
4. 趋势卡：记录一组变化背后的阶段性趋势判断。

临时工作资产一类：

5. 变化簇：自动聚合相关变化卡，用来观察方向是否正在形成。

变化卡是主库母体。案例卡支撑变化，观点卡校准变化，趋势卡聚合变化。

## 8. 变化卡

变化卡写“现象”，案例卡写“对象”。

变化卡第一版必填 7 项：

1. 变化标题。
2. 明确变化。
3. 原始出处。
4. 数据来源。
5. 一句解释。
6. 技术路线 / 方法变化。
7. 同类产品 / 相邻案例。

数据来源不允许为空。没有可靠来源时，写：

> 数据来源：暂无公开信息

同类产品 / 相邻案例不允许为空。未发现时，写：

> 暂未监测到同类案例

条件满足时，应继续补充完整扩展项：

- 变化原因。
- 影响范围。
- 客户需求迹象。
- 场景 AI 化进程。
- 相关观点。
- 市场规模 / 付费数据。
- 可二创角度。
- 可升级为内参的理由。

内部后台字段：

- 可信度边界。
- 来源等级。
- 数据等级。
- 准入状态。
- 发布状态。
- 二创适配度。
- 内参升级建议。

## 9. 变化卡生命周期

后台状态：

- 新出现。
- 持续出现。
- 形成变化簇。
- 趋势候选。
- 并入趋势。
- 历史沉淀。

前台简化展示：

- 最近观察：对应新出现。
- 正在升温：对应持续出现 + 形成变化簇。
- 趋势追踪：对应趋势候选 + 并入趋势。

历史沉淀不作为显性前台状态展示，但可被搜索、引用和关联。

从“最近观察”升级到“正在升温”，不按天数自动升级，而按证据增强和关联密度升级。可满足任意两项：

- 7-14 天内出现 2 张以上相关变化卡。
- 出现新的同类产品 / 相邻案例。
- 出现高影响 Builder / VC / Research 观点呼应。
- 出现更强来源，如原始出处、客户案例、融资、招采、财报、产品文档。
- 被今日观察连续引用。
- 所属变化簇密度上升。

进入“趋势追踪”，以所在变化簇形成方向性判断为准。

## 10. 自动关联机制

新变化卡生成后，系统自动建立关联，不要求人工逐条确认。

关联类型包括：

- 同类产品。
- 相同技术路线。
- 同一场景。
- 相同客户需求。
- 同一趋势。
- 同一公司 / 人物 / 机构。
- 同一数据源或来源链。
- 同一内容选题。

审核机制应减少人的反复确认：

- 自动关联默认成立。
- 只将异常关联推给人看。
- 提供今日新增关联、低置信关联、高影响关联、被多次引用核心关联等批量审查视图。
- 系统定期检查误连、漏连、孤立卡片和重复误判模式。
- 前台只展示稳定、高置信或经批量审查保留的关系。

## 11. 趋势卡与变化簇

趋势卡只在多张变化卡形成簇之后再生成。

趋势卡最低门槛：

- 至少 3 张相关变化卡。
- 至少 2 个不同案例或对象。
- 至少 2 类来源。
- 至少 1 条技术路线或场景进程说明。
- 必须说明当前属于早期趋势、增强趋势、分化趋势或风险趋势。

变化簇是变化卡和趋势卡之间的临时工作台。它自动生成，可合并、拆分、沉降、升级，不作为正式长期知识卡。

变化簇状态建议：

- 新出现。
- 增强中。
- 分化中。
- 降温。
- 升级候选。
- 归档。

## 12. 前台主导航

第一版前台主导航：

> 今日观察 / 商业信号 / 趋势追踪 / 商业内参

四个栏目更新频率不同：

- 今日观察：每日更新。
- 商业信号：持续入库，前台精选展示。
- 趋势追踪：周更或不定期，宁缺毋滥。
- 商业内参：默认周刊，内容不足可合并半月刊 / 月刊，重大主题可出增刊。

四个栏目的一等对象：

- 今日观察：每日行情长文 + 精选变化卡。
- 商业信号：变化卡 + 案例卡。
- 趋势追踪：趋势深度报告 + 趋势卡。
- 商业内参：周期刊物文章。

## 13. 今日观察

今日观察是全站最高优先级栏目。

定位：

> 每个老板每天必读的一篇 AI 商业行情综述。

它不是卡片集合，也不是普通日报，而是基于 Raw 全量监测后的市场理解来撰写。

正确生产逻辑：

```text
Raw 广泛监测
→ Pool 候选筛选
→ 对当天市场形成整体判断
→ 撰写今日观察长文
→ 同时选出 5-8 张精选变化卡作为研究入口和证据样本
→ 变化卡进入商业信号资产库，并与文章互相关联
```

今日观察常规篇幅：

- 常规日：3000-5000 字。
- 低信号日：1500-2500 字。
- 高信号日：5000-6000 字。

今日观察上游 Raw 监测覆盖：

- 公司公告和产品发布。
- 融资、并购、资本市场动态。
- 可信媒体报道。
- Builders / VC / Researcher 观点。
- 论文、技术报告、开源项目。
- 客户案例、招采、合作、企业采用。
- 应用商店、云市场、GitHub、社区讨论等市场线索。
- 如有条件，加入付费数据库或市场规模数据。

每日生产漏斗范围：

- Raw 广泛监测：常规 80-150 条。
- 低信号日：50-80 条。
- Pool 候选：20-40 条。
- 变化卡候选：8-15 张。
- 今日精选变化卡：常规 5-8 张，低信号日 3-5 张，高信号日 8-12 张。
- 文章重点详写：3-5 张。

今日观察文章可以引用未入选变化卡的 Raw / Pool 信息，但要分清层级。只有经过来源确认的内容才能写成具体事实；未入选变化卡的内容可作为行情背景、热度观察或市场线索，不能包装成强判断。

今日观察页面结构：

- 完整长文为主阅读。
- 精选变化卡作为右侧、文中或文末研究入口。
- 每张变化卡可进入独立详情页。
- 卡片详情页反向显示被哪些今日观察文章引用。
- 提供日期筛选、日历视图、tags 筛选、主题筛选、相关历史文章和同标签下其他日期观察。

今日观察中的 Builders / 前沿观点：

- 后台每天固定检查 follow-builders 和其他观点源。
- 前台文章只在有高质量观点时自然写入。
- 没有高质量观点时不显示空模块，也不使用后台语言提示。

## 14. 商业信号

商业信号定位：

> 变化卡 + 案例卡的研究入口。

它不是另一套文章栏目，也不是普通新闻流。

商业信号应分层展示：

- 变化卡为主：回答什么正在变。
- 案例卡为支撑：回答谁在怎么做。

核心能力：

- 展示全部精选变化卡。
- 展示相关案例卡。
- 技术路线筛选。
- 同类产品 / 相邻案例。
- 来源与数据源。
- 关联今日观察文章。
- 关联趋势追踪和商业内参。

变化卡详情页展示相关案例卡；案例卡详情页展示它支撑哪些变化卡。

## 15. 趋势追踪

趋势追踪不是卡片聚合页。

定位：

> 深度趋势分析报告 + 趋势卡。

趋势深度报告和趋势卡都是一等对象。

趋势深度报告负责建立专业判断和认同感，应覆盖：

- 行业分析。
- 赛道分析。
- 公司分析。
- 竞品分析。
- 技术路线。
- 客户需求。
- 商业案例。
- 变化簇。
- 观点分歧。
- 风险与反证。
- 观澜判断。

趋势报告主交付是深度长文 / 深度研究报告，不是结构化填充页。

篇幅规则：

- 常规趋势报告：8000-15000 字。
- 重点赛道报告：15000-30000 字。
- 轻量追踪更新：3000-6000 字。
- 节奏：周更或不定期，宁缺毋滥。

趋势卡负责长期沉淀和持续追踪：

- 趋势状态。
- 关联变化卡。
- 关联案例卡。
- 关联观点卡。
- 更新记录。
- 来源与数据。
- 阶段变化。

趋势报告引用趋势卡和变化簇；趋势卡记录该趋势被哪些报告分析过。

## 16. 商业内参

商业内参定义为：

> 周期性 AI 商业观察刊物。

它不是把每日观察拼起来，而是在一个周期结束后，对这段时间的变化、案例、观点和趋势做二次融合、修正和升级。

发行节奏：

- 默认周刊。
- 低信号周期可合并为半月刊。
- 内容更少时可做月刊。
- 重大主题可出专题增刊。

商业内参是刊物文章，不是结构化模块填充页面。它可以有目录、章节、图表和引用卡片，但前台应读起来像完整刊物。

固定骨架：

1. 本期总判断。
2. 周期行情回看。
3. 关键变化复盘。
4. 重点案例观察。
5. 趋势修正。
6. 前沿观点。
7. 下周期观察。
8. 内容选题池。
9. 来源与数据附录。

内容选题池可内部可见，不一定前台完整展示。

## 17. 前沿观点

“判断风向”命名废弃，改为：

> 前沿观点。

前沿观点不进入主导航，但应该有聚合页面，方便查看和检索。

观点卡不作为事实主证据，但可以作为判断触发器和行业预期信号。高影响观点卡可以触发变化卡候选，但不能单独让变化卡正式成立。

观点来源三层：

1. 人物白名单：长期跟踪的 Builder、Founder、VC、Researcher、Operator、企业 AI 负责人。
2. 机构来源：a16z、Sequoia、YC、OpenAI、Anthropic、Google DeepMind、Meta、NVIDIA、Hugging Face、Databricks、Snowflake、Gartner、Forrester 等。
3. 临时高影响观点：非白名单人物因重大事件、行业身份或被广泛引用，临时进入观点卡。若后续连续出现，再纳入白名单候选。

采用混合模型：

- 观点卡是最小单元。
- 高影响人物聚合成人物档案页。

前沿观点前台三层展示：

1. 单张观点卡：原文可信度 + 观澜解读。
2. 人物档案页：时间线 + 观点变化。
3. 前沿观点聚合页：按 tags、主题、人物、关联资产检索和浏览。

### 17.1 观点卡

观点卡前台优先展示：

- 人物 / 机构。
- 当时 title / 身份。
- 原文摘录。
- 发表时间。
- 原文出处。
- 观澜解读。
- tags。
- 关联资产：变化卡、案例卡、趋势卡、今日观察、商业内参。

不再单独设置前台“观点摘要”。如原文为英文，可提供中文转述作为辅助，但不替代原文摘录。

后台保留结构化字段：

- 结构化观点主张。
- 观点状态：新观点 / 延续 / 转向 / 分歧 / 被验证 / 被反证。
- 观点倾向。
- 观点对象。
- 是否高影响观点。
- 是否触发变化卡候选。
- 关联质量。

观点状态放后台，不在前台直接展示。

### 17.2 人物档案页

人物档案页应包含：

- 人物基本信息：姓名、机构、title、角色类型。
- 关注主题。
- 观点时间线。
- 连续观点脉络解读。
- 代表观点。
- 原文出处。
- 观澜解读。
- 关联变化卡 / 趋势卡 / 今日观察 / 商业内参。
- tags。

连续观点脉络解读以周为默认总结单位，30 天 / 90 天作为延展回看。

前台不展示“新观点 / 转向 / 被验证”等机械标签，而通过自然语言解释观点变化。

### 17.3 前沿观点聚合页

定位：

> 观点资料库 + 高影响人物时间线。

聚合页支持：

- 按人物。
- 按机构。
- 按主题 / tags。
- 按时间。
- 按关联栏目。
- 查看高影响观点卡。
- 查看重点人物档案入口。
- 查看观点变化时间线。
- 查看本周观点分歧。
- 查看被多篇文章引用的观点。
- 查看关联变化卡和趋势卡。

## 18. Tags 沿用与升级

现有 `tag-taxonomy.md` 不重建，沿用其 9 类底层体系：

- `track`：赛道 / 技术方向。
- `function`：业务职能。
- `scenario`：应用场景。
- `customer`：客户类型。
- `evidence`：证据类型。
- `stage`：阶段 / 成熟度。
- `region`：地域 / 市场。
- `source`：来源类型。
- `point`：人物 / 观点主题。

升级方向：

- Tags 升级为变化卡、案例卡、观点卡、趋势卡都可使用的统一关系标签。
- 前台不展示机械 group 名称，不展示内部 tag_id。
- 前台表达为自然筛选，例如看方向、看场景、看行业 / 客户、看依据、看观点、看阶段。

变化卡建议标签规则：

- 至少 1 个 `track`。
- 至少 1 个 `scenario` 或 `function`。
- 至少 1 个 `evidence`。
- 至少 1 个 `source`。
- 如涉及观点，加 `point`。
- 如可判断阶段，加 `stage`。
