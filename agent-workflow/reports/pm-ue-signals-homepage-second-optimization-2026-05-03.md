# Signals 栏目首页二次优化方案

## 1. 前提变化

现在已经有独立 `signal.html` 详情页。  
这会改变 Signals 首页的职责：

- 首页不再承担完整阅读。
- 首页不再重复展示机会拆解、评分依据、长段证据说明。
- 首页应成为“快速筛选、按日复盘、发现高价值 Signal、进入详情”的雷达入口。
- 详情页承接“为什么这是 Signal、证据与来源、机会拆解、关联 Opportunity / Trend”的完整阅读。

一句话定位：

> Signals 首页负责判断“先看哪条”，Signal 详情页负责说明“为什么值得看”。

## 2. PM Agent 判断

### 2.1 首页核心任务

用户进入 Signals 首页后，10 秒内应完成 5 个判断：

1. 今天/本周有多少商业信号。
2. 哪些变化类型正在集中出现。
3. 哪些 Signal 证据更强。
4. 哪些 Signal 已经连接到 Opportunity / Trend。
5. 按日期复盘时，某一天的主要变化是什么。

### 2.2 首页不再承担的任务

以下内容应主要交给 Signal 详情页：

- 长段要闻正文。
- 完整机会拆解。
- 完整评分依据。
- 大量原始来源链接。
- 详细 Opportunity / Trend 列表。
- 复杂关系说明。

首页只保留“进入详情的理由”，不在卡片里讲完所有判断。

### 2.3 P0 二次优化目标

把当前“证据雷达页”进一步收敛成三层：

```text
顶部：日期与证据概览
中部：日期分组 Signal 队列
侧边：当前筛选下的结构摘要
```

重点不是增加信息，而是减少首页重复阅读负担。

## 3. UI / UE Agent 方案

### 3.1 页面结构建议

推荐结构：

1. 页面标题：保留现有一行标题。
2. 顶部雷达条：日期范围 + 今日信号 + 强证据 + 已关联判断。
3. 筛选工具条：搜索、日期、变化类型、证据强度、赛道、关联状态。
4. 主内容区：日期分组 Signal 卡片流。
5. 右侧洞察栏：当前筛选范围的结构摘要。

### 3.2 顶部雷达条优化

当前 3 个指标保留，但需要让它们更像“当前视图摘要”：

- 今日信号：固定显示最新日期数量。
- 强证据：随筛选变化。
- 已关联判断：随筛选变化。

可新增一个轻量日期标识：

- 当前视图：全部日期 / 今日 / 昨日 / 本周 / 更早 / 某个具体日期。

不建议新增第四个大指标，避免顶部过重。

### 3.3 筛选工具条优化

工具条应更紧凑，像研究产品的筛选栏，而不是表单。

建议：

- 搜索框保持较宽。
- 其他筛选项使用紧凑 select。
- 移动端改成两列或横向滚动。
- “新增 Signal”只在 Admin 编辑模式出现，普通用户完全不可见。

### 3.4 Signal 卡片二次收敛

有了详情页后，首页卡片应从“解释型卡片”改成“判断入口卡片”。

每张卡片保留：

- 变化类型。
- 证据强度。
- 标题。
- 一句话商业含义。
- 关联状态：机会 / 趋势。
- 原始来源状态。
- 进入详情入口。

卡片删除或弱化：

- 过多 Opportunity 名称。
- 过多 Trend 名称。
- 长段摘要。
- 卡片内重复来源链接。

推荐卡片结构：

```text
[变化类型] [证据强度]
标题：事件 + 商业含义
一句话商业含义
机会 2 / 趋势 1 / 来源已识别
查看详情
```

### 3.5 右侧洞察栏

当前右侧详情预览可以调整为“当前视图摘要”，不再直接复刻详情页。

右侧建议模块：

1. 当前筛选摘要
   - 当前视图内 Signal 数。
   - 强证据数量。
   - 已关联 Opportunity / Trend 数。

2. 变化类型分布
   - 展示当前视图前 5 个变化类型。
   - 点击后可筛选。

3. 证据强度分布
   - 强证据 / 中等证据 / 待补证据。

4. 选中 Signal 预览
   - 标题。
   - 一句话商业含义。
   - 证据强度。
   - 查看详情按钮。

这样右侧既有雷达感，也保留选中项反馈。

### 3.6 日期分组优化

日期组头部保留，但更强调“当日结构”：

- 日期。
- 当日 Signal 数。
- 主变化类型。
- 强证据数量。
- 已关联判断数量。

组内卡片按优先级排序：

1. 强证据。
2. 已关联 Opportunity / Trend。
3. 高 Signal Score 或评分项较高。
4. 同分时按原始顺序。

## 4. 推荐 P0 改动清单

### P0-1 首页卡片瘦身

把卡片从解释型改成入口型：

- 保留标题、变化类型、证据、商业含义、关联计数、来源状态。
- 增加“查看详情”链接。
- 不在卡片中展开 Opportunity / Trend 名称列表。

### P0-2 右侧从详情预览改成视图摘要 + 选中预览

右侧不再承接完整详情，而是：

- 当前筛选摘要。
- 变化类型分布。
- 证据强度分布。
- 选中 Signal 简短预览。
- 查看详情按钮。

### P0-3 日期组头增强

日期组头从纯说明变成当日小结：

- 当日 Signal 数。
- 主变化类型。
- 强证据数量。
- 已关联判断数量。

### P0-4 首页到详情页路径统一

所有首页卡片、Daily Brief 里的 Signal 链接、Trend 详情里的关联 Signal 链接，统一进入：

```text
signal.html?slug=...
```

Signals 首页只在需要筛选和复盘时使用。

## 5. P1 后续可做

- 日期视图与 Daily Brief 互相回链。
- 变化类型分布做成可点击的紧凑条形图。
- 当前筛选结果导出为“复盘清单”。
- Signal 合并提示，但不在普通前台展示。
- Tags 网络视图进入搜索/关系能力，不作为 Signals 首页主视觉。

## 6. 验收标准

- 首页第一屏能看到指标、筛选和至少一组日期 Signal。
- 用户不点详情也知道“哪条值得点进去”。
- 用户点击任一 Signal 后进入独立详情页，而不是在首页读完整内容。
- 首页卡片不拥挤，不重复详情页内容。
- 右侧不再像完整详情页，而像当前视图的结构摘要。
- 移动端保留日期分组，卡片不横向溢出。
- 普通前台不出现 Admin、JSON、同步、编辑、恢复等后台痕迹。

## 7. 自动化影响判断

本方案只调整前台页面结构和跳转路径，不改变 Markdown 命名、字段结构、同步脚本入口或自动化运行顺序。

当前判断：不影响 `ai-the-point`、`ai-2`、`ai-3`。

若后续为了右侧分布新增数据字段，应先复用现有 `eventTypes`、`sourceTier`、`relatedOpportunityIds`、`relatedTrendIds`，不要新增 Markdown 必填项。
