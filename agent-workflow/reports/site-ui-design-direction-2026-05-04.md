# 观澜AI｜网站 UI 方向与 DESIGN v2 草案

## 0. 结论

本轮按 `frontend-design` 与 UI / UE Design Director 规则执行，只做全站视觉方向和规范，不修改 `04-Site/`，不生成最终首页海报图，不改变栏目、数据模型、权限或自动化任务。

核心结论：

1. 观澜AI的视觉方向应从“AI 科技页 / 模板 SaaS / 后台面板”收敛为“商业情报桌面”。
2. 当前问题不是缺模块，而是缺统一 Art Direction、页面母版和可验收的字体 / 间距 / 色彩系统。
3. 首页旧雷达图、四张 poster 卡、泛营销利益表达和 P0-2B failed 结果都不应作为后续基础。
4. P0-11 应聚焦右侧 `Intelligence Desk` 判断样张，且只能在本任务 accepted 后执行。
5. 后续 Dev 不应继续靠追加 `final / override / lock / polish` 样式层推进高级感，应先按 DESIGN v2 拆 tokens、母版和组件边界。

## 1. Art Direction

| 项目 | 方向 |
|---|---|
| 产品气质 | 高端商业内参、研究桌面、判断样张、克制会员产品 |
| 视觉关键词 | 克制、研究感、纸面秩序、深墨绿重点、连续证据 |
| 目标感受 | 可信、有判断、有边界、值得读 |
| 记忆点 | AI 变化被压缩成商业判断的样张 |
| 非目标 | AI 紫蓝渐变、霓虹大屏、机器人头像、抽象雷达、模板 SaaS 卡片墙、投融资榜单 |

本轮使用的设计参考：

- `frontend-design`：明确采用有辨识度的设计方向，避免通用 AI 审美。
- Apple：留白与首屏主次。
- Linear：中高密度信息组织和克制边界。
- Bloomberg / FT / Economist：内参阅读、标题、正文和侧栏层级。
- taste-skill 适配规则：只吸收反过度卡片化、CSS Grid、移动端稳定和低动效原则。

## 2. 全站页面母版

| 母版 | 结构 | 关键规则 |
|---|---|---|
| 首页 | 左侧价值判断 + 右侧 Intelligence Desk + 下沿露出今日内容 | 删除旧雷达和 poster 卡；首屏只完成一个主任务 |
| 栏目页 | 统一标题区 + command bar + 主列表 / 主卡组 | 指标、筛选和徽章降级，列表成为主角 |
| 详情页 | 报告正文 + 右侧摘要 / 来源 / 关联 | 正文优先，侧栏辅助，不做字段平铺 |
| 会员页 | 权限状态 + 可读层级 + 注册 / 登录 / 订阅动作 | 像情报订阅产品，不像通用 SaaS 套餐页 |
| Admin | 左侧窄导航 + 顶部状态条 + 高密度工作区 | 与前台视觉分离，内部语言只在 Admin 出现 |
| 移动端 | 单列阅读 + 样张卡收缩 + command bar 折叠 | 本阶段底线为不溢出、不遮挡、不破坏阅读 |

## 3. 字体 / 间距 / 色彩系统

色彩采用象牙白纸面底、深石墨正文、深墨绿重点、铜棕小面积强调、低饱和青绿链接和状态。

字体层级：

| 层级 | 桌面建议 | 说明 |
|---|---:|---|
| 首页 H1 | `64-72px / 1.08` | 仅首页使用 |
| 栏目页 H1 | `52-60px / 1.10` | 一级栏目统一 |
| 详情页 H1 | `38-46px / 1.18` | 报告标题 |
| Section H2 | `26-32px / 1.22` | 模块标题 |
| 卡片标题 | `18-22px / 1.35` | 列表与卡片 |
| 正文 | `15.5-17px / 1.7-1.85` | 阅读正文 |
| Meta | `11-13px / 1.2` | 日期、来源、状态 |

页面节奏：

- 栏目标题区高度建议 `220-260px`。
- 首页主容器建议 `1480px` 左右，首屏必须露出第二屏内容。
- 详情正文主栏建议 `760-860px`，侧栏 `320-380px`。
- 卡片内边距建议 `18-24px`，列表行距比卡片堆叠更优先。

## 4. 首页右侧主视觉方向

P0-11 的主视觉应是 `Intelligence Desk`，不是海报轮播，也不是抽象图。

必须包含：

- 今日 Brief 摘要。
- 2 条 Signal。
- 1 条 Opportunity。
- 1 个 Trend 状态。
- 克制关系线，表达从事实变化到机会和趋势的连接。

禁止：

- `Scoring`、`JSON`、`同步`、`后台`、`字段`、`编辑`。
- 机器人头像、霓虹科技背景、纯雷达网格、股市 K 线堆叠。
- 生成图里嵌入关键中文小字。

推荐路线：

1. 先做 HTML/CSS Intelligence Desk 样张，使用真实站内数据截断。
2. 如果需要生成图，只做纸张、桌面和氛围，不承载关键文字。
3. 桌面端作为右侧主视觉，移动端收缩为一张判断样张卡。

## 5. Copy 文案边界检查

后续页面和视觉资产必须遵守：

- 首页 H1 推荐继续使用 `在市场形成共识前，看见 AI 商业变化`。
- CTA 推荐 `查看今日简报` / `申请完整情报层`。
- `机会排行` 改向 `优先观察` 或 `验证序列`。
- `Scoring` 与 `Tags` 不作为普通前台一线表达。
- 普通前台不得出现 `本页用于`、`入口`、`同步`、`字段`、`后台`、`证据链`、`强证据`、`来源明确`、`机会确定`、`下一步验证`。
- 不替用户下经营、投资或合作结论。

## 6. 审美阻塞项

以下问题出现时，UI / UE Design Director 应阻塞 Dev 或 QA acceptance：

- 首页继续沿用旧雷达图、四张 poster 卡或 P0-2B failed 结果。
- 页面看起来像后台组件堆叠、模板页或临时拼装。
- 栏目页标题位置、字号、行高、模块起点不一致。
- 详情页没有报告阅读路径，只是字段卡片平铺。
- 徽章、数字、按钮和摘要等权抢注意力。
- 文案破坏布局但未退回 Copy 压缩。
- 普通前台出现后台痕迹。
