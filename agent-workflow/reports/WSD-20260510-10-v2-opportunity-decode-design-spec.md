# WSD-20260510-10 机会解码 V2 设计规范

日期：2026-05-10  
状态：ready_for_review  
Owner：UI / UE Design Director  
适用范围：`机会解码` 列表页、详情页、机会卡与嵌入判断模块  
非目标：不修改站点代码、不修改内容、不修改 VI 资产、不改变一级导航

## 1. 栏目定位

机会解码不是旧机会库，也不是每日机会卡列表。它承担的用户任务是：当一组 AI 变化已经经过多源搜索、论证和反证后，帮助商业决策者判断“这个方向为什么值得继续读、证据在哪里累积、边界是什么、后续应观察哪些变量”。

前台表达应保持低频、少而精。证据不足时，页面允许出现“观察中方向”，但不得把候选包装成结论，也不得为了更新频率硬凑 Deep Dive。

## 2. 判断链

机会解码详情页的阅读链路：

```text
一句话机会判断
-> 变化背景
-> 问题与对象
-> 流程变化
-> 价值来源
-> 趋势背景
-> 观点校准
-> 风险 / 反证
-> 成立边界
-> 建议关注变量
-> 关联内容
```

页面上的模块命名需面向用户，不暴露内部字段名。`Deep Dive` 在前台可称为“深度报告”；`Opportunity` 称为“机会方向”或“机会报告”；`Trend Context` 称为“趋势背景”；`Point Calibration` 称为“观点校准”；`HeatEvidence` 前台只转译为“内参引用 / 证据积累 / 热力摘要”，不得直接显示字段名。

## 3. 列表页信息架构

首屏采用同一套一级栏目标题节奏：背景 `--gl-bg-page`，无突兀色块。左侧是栏目标题与一句判断，右侧是最新报告摘要纸面，不做满屏 hero。

模块顺序：

1. 栏目标题区：`机会解码`、一句话说明、低频更新边界。
2. 最新报告：展示 1 篇主报告，包含一句话判断、证据状态、适合关注对象、主要反证。
3. 观察中方向：展示 2-4 个候选方向，每个只说明缺口，不导向确定结论。
4. 历史报告：按日期或主题分组，优先使用列表 + 分隔线，避免三列等宽卡片墙。
5. 内参关联：说明哪些机会方向进入过商业内参，不做公开热力榜。

桌面布局建议：主容器 `1200-1280px`；首屏使用 `7:5` 非等分网格，主标题宽度不超过 `680px`，最新报告摘要宽度 `420-480px`。模块间距 `72-96px`。

移动布局建议：单列，标题先于报告摘要，最新报告后直接出现“观察中方向”。候选方向卡片使用紧凑列表，不横向滚动。

## 4. 详情页信息架构

详情页必须像商业分析报告，而不是字段展开页。

推荐结构：

1. 报告标题区：标题、发布日期、阅读时长、证据状态、来源概况。
2. 一句话机会判断：用衬线字体作为导语，不写确定性机会。
3. 正文报告主栏：按判断链连续阅读，主栏 `760-860px`。
4. 侧栏：摘要、关键变量、关联信号 / 趋势 / 内参引用，宽度 `320-380px`，不抢主阅读。
5. 反证与成立边界：必须在正文后半段明确出现，不能只放侧栏。
6. 关联阅读：关键信号、今日要点、商业内参。

桌面建议：`main + aside` 双栏，主栏优先；侧栏使用细分隔线和浅纸面，不使用后台式信息表。  
移动建议：侧栏内容折到正文后，标题、导语和反证模块必须完整可读，不出现遮挡或横向溢出。

## 5. 机会卡规范

机会卡只承担“是否值得进入报告”的预读任务，不承担完整解释。

必须包含：

- 机会方向标题：方向 / 客户场景 / 能力，不写公司名。
- 一句话机会判断。
- 证据状态：用“证据正在增强 / 观察中 / 仍缺客户采用”等自然语言。
- 适合关注对象：行业、岗位、部门或组织形态。
- 主要反证：一句话说明缺口。
- 入口：`阅读报告` 或 `查看观察`。

禁止：

- `机会确定`、`强证据`、`下一步验证`、`立即行动`。
- 公司名作为标题。
- 三列等宽营销卡片。
- 把 HeatEvidence、Raw、Pool、Structured、JSON、同步状态暴露到前台。

## 6. 证据与趋势模块

证据呈现方式：不叫“证据链”，使用“来源与事实”“支撑信息”“证据状态”。列表最多 3-5 条摘要，每条说明来源类型和增量事实，长链接放在详情或来源展开中。

趋势背景：展示证据积累、时间变化、影响坐标扩散、反证变化和内参引用。数据不足时只写文字，不强行画图。

观点校准：说明观点支持、质疑或修正什么；Builder / VC / 市场观点不能替代事实来源。

风险 / 反证：以“还不能下结论的地方”呈现，列出技术、商业化、监管、集成、客户习惯、交付成本等边界。

行动地图：前台不提供行动方案，改为“建议关注变量”。最多 3-5 个变量，例如客户上线规模、公开定价锚点、效果指标、责任条款、模板复用率。

## 12. Design Director 评分

基准：`DESIGN.md`、VI USAGE、brand tokens、typography-guidelines，辅以 `design-taste-frontend` 的反模板化检查；如有冲突，以项目 VI 为准。

| 质检项 | 分数 | 证据与扣分 |
|---|---:|---|
| 商业判断感 | 18/20 | 报告化链路明确；需 Dev 避免回退为卡片列表 |
| 信息层级清晰度 | 18/20 | 列表页主报告、观察中、历史报告分工清楚 |
| 视觉克制与高级感 | 17/20 | 使用暖白、墨海蓝、细线和纸面；需控制徽章数量 |
| 阅读效率 | 17/20 | 详情页主栏 + 侧栏；需限制侧栏信息密度 |
| 模块节奏 | 17/20 | 低频报告节奏成立；历史索引需避免过长 |
| 移动端可读性 | 17/20 | 单列方案明确；需实现时检查标题断行 |
| 商业内参产品感 | 18/20 | 不像新闻流或后台面板，边界清楚 |

综合：122/140，折算 87/100。达到一级栏目页 / 详情页通过线 `>=80`。

Squint Test：通过。远看应是“报告标题 + 判断摘要 + 少量报告项”，不是新闻站、后台面板或模板卡片墙。
