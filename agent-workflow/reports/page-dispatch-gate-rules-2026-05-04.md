# 页面与文案类任务派发闸门规则

## 1. 背景

近期首页和 Admin 开发过程中反复出现以下问题：

- 页面粗糙、简陋。
- 栏目间距不一。
- 标题位置不一。
- 字体大小随意。
- 排版丑陋。
- Admin 模块首屏节奏不一致。
- 公开文案不规范、内部话术、说服感过强、空泛、标题断行难看。

复盘后确认，根因不是 UI/UE Agent 没有规范，而是规范没有成为派发、实现和验收的硬闸门。

过去的派发单常见问题：

- 只要求“读取 `DESIGN.md`”，没有要求 UI/UE 先输出可验收的页面规范表。
- Dev 可以直接实现，但不需要逐条说明是否按 UI/UE 规范执行。
- QA 经常只检查 JS 语法、无横向溢出、页面能打开，没有按坐标、字号、间距、首屏节奏独立拦截。
- UI/UE、Dev、QA 可能在同一执行窗口连续完成，导致自验自收。
- 调度中枢验收 closeout 时缺少页面规范表，也可能标记 `accepted`。
- Copy 规范只写在 `COPY.md` 和 Copy Agent 岗位文件里，没有成为 Dev 必须落地、QA 必须验收的硬条件。
- Dev 可能为了完成页面自行补写文案，导致内部话术、空泛营销、过度承诺或容器不适配。

## 2. 新硬规则

凡任务涉及以下范围，一律视为“页面类任务”：

- 首页。
- Daily Brief、Signals、The Point、Opportunities、Trends 等栏目页。
- 详情页。
- 会员页。
- Admin。
- 移动端。
- 任意 `04-Site/*.html`。
- 任意 `04-Site/css/styles.css` 页面体验改动。

页面类任务必须遵守：

```text
PM 范围确认 -> UI/UE 页面规范表 + Copy 文案规范表 -> Dev 按表实现 -> QA 独立截图/测量/文案验收 -> Workflow 收口
```

没有 UI/UE 页面规范表的页面实现任务，不得标记为完成。
涉及可见文案但没有 Copy 文案规范表的页面实现任务，也不得标记为完成。

## 3. UI/UE 页面规范表

调度窗口派发页面类任务时，必须要求 UI/UE Agent 先输出页面规范表。

规范表至少包含：

- 页面类型。
- 页面目标。
- 设计基准。
- 布局基准。
- 字体层级。
- 间距基准。
- 组件克制规则。
- 前台 / Admin 边界。
- 桌面端验收点。
- 移动端验收点。
- 禁止项。

重点必须写清：

- 标题位置。
- 标题字号。
- 标题行高。
- 主容器宽度。
- 模块起点。
- 模块间距。
- 卡片/边框/阴影/徽章如何克制。
- 是否允许深色面板、背景块或特殊标题区。

## 4. Dev 按表实现

页面类任务 closeout 中，Dev 必须逐条回应 UI/UE 页面规范表和 Copy 文案规范表：

- 哪些条目已实现。
- 哪些条目未实现。
- 未实现原因。
- 是否新增临时样式。
- 是否自行新增、改写或扩写 Copy 未提供的关键文案。
- 是否影响其他栏目、Admin 模块或移动端布局。

如果无法按规范表实现，必须标记 `blocked` 或退回 UI/UE / PM，不得自行另写一套视觉。
如果文案缺失、过长、无法适配容器或语义不清，必须退回 Copy / PM，不得自行补成内部话术或空泛营销语。

## 5. QA 独立验收

页面类任务 closeout 中，QA 必须独立按 UI/UE 页面规范表验收。

必须包含：

- 桌面端截图路径。
- 移动端截图路径。
- 对比页面或对比模块。
- 标题位置、字号、行高、模块起点是否一致。
- 首屏主次是否清楚。
- 字体层级是否统一。
- 间距是否一致。
- 是否无横向溢出。
- 普通前台是否无后台痕迹。
- 禁用语检查。
- 判断边界检查。
- 标题长度与容器适配。

以下问题一律阻塞：

- 栏目之间标题位置、字号、行高或首屏节奏明显不一致。
- Admin 模块之间顶部工具栏、模块框起点、模块标题起点或标题区高度不一致。
- 页面像后台组件堆叠、模板页或临时拼装页。
- 字体大小随意，缺少统一层级。
- 桌面端或移动端出现横向溢出、遮挡、重叠。
- 普通前台出现 Admin、JSON、同步、编辑、恢复、字段、后台等痕迹。
- 公开前台出现内部流程语、空泛营销词、过度承诺或替用户下最终判断。
- Signal / Opportunity / Daily Brief / Trends / The Point 违反 `COPY.md` 专项规则。
- 文案过长导致标题断行难看、按钮拥挤、卡片压迫或移动端遮挡。

## 5A. Copy 文案规范表

凡页面类任务涉及可见文案，或任务本身是文案任务，调度窗口必须要求 Copy Agent 输出 Copy 文案规范表。

规范表至少包含：

- 页面/模块。
- 用户任务。
- 当前文案问题。
- 替换后文案。
- 禁用语检查。
- 判断边界。
- 标题长度与容器适配。
- QA 文案验收点。

重点必须写清：

- 哪些表达是内部流程语，必须删除或转译。
- 哪些表达可能替用户下最终判断，必须降级为观察、仍需验证、单信源、多信源或反证。
- 哪些标题、CTA 或卡片文案可能导致断行难看、按钮拥挤或卡片压迫。
- 哪些栏目专项规则必须执行，例如 Signal 标题不能退回公司名、Opportunity 标题不能写公司名、Daily Brief 不输出行动指令。

## 6. 调度中枢验收

调度中枢收到页面类 closeout 后，必须先检查：

1. 是否包含 UI/UE 页面规范表。
2. 如涉及可见文案，是否包含 Copy 文案规范表。
3. Dev 是否逐条对应实现。
4. QA 是否有桌面端和移动端截图。
5. QA 是否按页面规范表做坐标/字号/间距/首屏节奏验收。
6. QA 是否按 Copy 文案规范表做禁用语、判断边界、标题长度和容器适配验收。
7. 是否说明未满足项和阻塞项。

缺少任一项时：

- 不得代补事实。
- 不得标记 `accepted`。
- 必须退回执行窗口补齐。

## 8. 对当前项目的影响

本规则不修改网站代码，不改变内容源，不影响自动化任务。

它只改变后续调度窗口派发和验收页面类任务的标准。

自动化影响：

- `ai-the-point`：不影响。
- `ai-2`：不影响。
- `ai-3`：不影响。

## 9. 建议主调度窗口后续动作

建议主调度窗口新增或更新以下任务：

1. 首页 P0 重新派发：必须先产出 UI/UE 页面规范表，再进入 Dev。
2. Admin P0 返修验收：用新规则复查现有 Admin 模块间距、标题、字体层级和移动端。
3. 全站栏目标题矩阵验收：Daily / Signals / The Point / Opportunities / Trends 对齐标题位置、字号、行高和首屏节奏。
4. 全站前台 Copy 闸门验收：检查首页、栏目页、详情页、会员页是否存在内部话术、空泛、过度承诺、标题过长和容器不适配。
5. 更新所有未执行的页面 / 文案类派发单：补入 `7A 页面类任务硬性要求` 和 `7B 文案类任务硬性要求`。
