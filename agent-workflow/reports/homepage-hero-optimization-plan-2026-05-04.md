# 首页首屏与海报图优化规划

## 1. 规划结论

首页首屏不应继续以抽象雷达流程图作为主要记忆点。它需要在 5 秒内让用户明白：观澜AI不是 AI 新闻站，而是把 AI 热点压缩成商业判断的情报系统。

推荐首屏方向：

> 品牌第一眼清楚出现，主标题直接讲商业判断价值，右侧海报呈现“真实信号被压缩成判断”的情报桌面。

当前首页已有品牌、CTA、雷达图和 Decision Brief，但首屏存在三个问题：

- 主标题“从 AI 浪潮里，看见商业胜负手”有气势，但不够具体，用户不一定知道产品提供什么。
- 首屏缺少一段副标题解释“信号、趋势、机会、判断依据”之间的价值关系。
- 右侧 `hero-radar.svg` 更像内部流程示意，缺少真实商业情报产品的可信场景。

战略上应把首页从“视觉口号 + 内容入口”升级为“商业判断样张 + 今日信号入口”。用户第一眼看到的不是抽象能力，而是：哪些 AI 变化值得看、为什么值得看、它如何连到机会和趋势。

## 2. 首页首屏战略判断

### 2.1 首页承担的唯一主任务

首页首屏只回答一个问题：

> 观澜AI能否帮我更早看见值得继续验证的 AI 商业变化？

首屏不承担：

- 展示全部 Signals、Opportunities、Trends。
- 解释内部方法、字段、同步和数据处理。
- 证明“机会已经确定”。
- 替用户给出投资、经营或合作结论。

### 2.2 目标用户的 5 秒理解

用户扫过首屏后，应形成三层理解：

1. 这是一个面向商业判断的 AI 情报产品。
2. 它从融资、客户采用、产品落地和产业变化里筛信号。
3. 它帮助我更快决定今天看什么、哪些方向值得继续观察。

### 2.3 首页价值表达边界

可以讲：

- 市场形成共识前的商业变化。
- 从热点中筛出信号。
- 把碎片信息压缩成可讨论、可追踪、可复盘的商业判断。
- 观察证据、趋势状态和机会方向。

不讲：

- 确定性机会、稳赚、必投、马上行动。
- 后台、同步、字段、脚本、JSON、Markdown。
- “本页用于”“入口”“证据链”“强证据”等内部或说服式表达。

## 3. 首屏文案方向

### 3.1 推荐主方案

品牌锚点：

```text
观澜AI｜WaveSight AI
```

主标题：

```text
在市场形成共识前，看见 AI 商业变化
```

副标题：

```text
每天从融资、客户采用、产品落地和产业变化中筛出关键信号，形成可追踪的机会观察与趋势判断。
```

主 CTA：

```text
查看今日简报
```

次 CTA：

```text
申请进入完整情报层
```

### 3.2 备选标题

备选 A：

```text
从 AI 热点里，筛出值得继续验证的商业信号
```

适合更偏产品说明，清楚但冲击力略弱。

备选 B：

```text
少追概念，多看谁正在变成生意
```

适合更锋利的营销首页，但需要副标题补足产品边界。

备选 C：

```text
把 AI 变化压缩成可复盘的商业判断
```

适合强调内参与复盘能力，但首眼商业机会感略弱。

### 3.3 首屏辅助信息

首屏建议保留 3 个短标签，用来让用户立即知道判断依据来自哪里：

```text
融资与收入
客户采用
产品落地
```

不建议继续使用过多抽象利益点卡片，如“看见窗口 / 形成判断 / 连接资源 / 穿透概念”四张大海报。它们可以被收敛为第二屏的产品逻辑，不应抢首屏。

## 4. 海报图方向

### 4.1 推荐方向：情报桌面 + 真实判断样张

优先级：P0

画面内容：

- 一张高级商业内参桌面或屏幕界面。
- 左侧是当天 2-3 条关键 Signals 的短判断。
- 中间是一条从 Signal 到 Opportunity / Trend 的连线关系。
- 右侧是一段 “Today’s Brief” 风格的主判断摘要。
- 视觉像高端研究产品的工作台，不像后台面板。

优势：

- 最能解释观澜AI的商业判断价值。
- 能用真实站内内容或脱敏样例承载可信度。
- 后续可直接拆给 Dev 用现有数据渲染，不一定依赖一次性静态图片。

注意：

- 不能出现 Admin、JSON、同步、编辑、恢复等后台痕迹。
- 不要做密密麻麻的大屏数据看板。
- 桌面端可做右侧大视觉，移动端应转为首屏下方的“判断样张”。

### 4.2 备选方向：生成图海报

优先级：P1

画面内容：

- 深墨绿与象牙白基调的商业情报桌面。
- 桌面上有纸质简报、行情式标记、少量来源卡片和关系线。
- 不出现具体真实公司 logo，不承诺具体机会。

优势：

- 品牌感更强。
- 可快速形成首屏海报冲击力。

风险：

- 如果太抽象，会回到“好看但不说明产品”的问题。
- 如果过于科技感或金融大屏，会偏离克制内参气质。

### 4.3 备选方向：数据场景可视化

优先级：P1

画面内容：

- 用真实数据结构呈现 Signals、Trends、Opportunities 的关系网络。
- 采用克制线条、低饱和色和少量动态高亮。
- 每个节点只显示短标题或类型，不展示大量文字。

优势：

- 产品能力表达清楚。
- 可作为后续交互式 hero 的基础。

风险：

- 容易像后台关系图。
- 如果节点过多，首屏会变成技术演示，用户反而看不懂。

### 4.4 备选方向：真实资产拼贴

优先级：P2

画面内容：

- 使用站内真实 Daily Brief、Signal 详情、Opportunity 卡的局部截图进行拼贴。
- 形成 “从来源到判断” 的产品样张。

优势：

- 可信，开发成本低。
- 适合作为上线前临时版本。

风险：

- 如果当前页面成品感不足，会放大粗糙感。
- 截图内容更新成本较高。

### 4.5 不推荐方向

- 纯雷达抽象 SVG：容易像内部流程图，不够商业。
- 大面积深色科技大屏：容易像后台或数据监控系统。
- AI 生成的城市、海浪、宇宙、光束等氛围图：有气氛但无判断价值。
- 过度营销人物图：会削弱内参与判断产品的专业感。

## 5. 首页首屏信息架构

### 5.1 桌面端结构

```text
顶部导航
  品牌 / 首页 / Daily Brief / Signals / The Point / Opportunities / Trends / 登录或注册

首屏主体
  左侧 55%
    品牌名或短 eyebrow
    H1：在市场形成共识前，看见 AI 商业变化
    副标题：解释信号、机会观察、趋势判断
    三个依据标签：融资与收入 / 客户采用 / 产品落地
    CTA：查看今日简报 / 申请进入完整情报层

  右侧 45%
    情报桌面海报
    今日主 Signal
    一条机会观察
    一个趋势状态
    来源状态或日期，不展示后台指标
```

首屏下沿应露出第二屏的 “今日判断入口”，让用户知道可以继续下滑进入真实内容。

### 5.2 移动端结构

```text
顶部导航
  品牌 / 菜单

首屏
  H1
  副标题
  CTA
  三个依据标签
  情报样张缩略卡

第二屏
  今日判断入口
```

移动端海报不应强行保留桌面大图比例，应转为一张可读的样张卡，优先显示主判断和 1-2 条支撑信息。

### 5.3 首屏下方模块取舍

建议把当前 “核心客户利益点” 四卡收敛为一条方法感横向叙事：

```text
看变化 -> 看观点 -> 看机会 -> 看趋势是否延续
```

但对外文案应避免“判断路径”这类内部说明，可改成：

```text
从变化到机会
```

每一步用一句客户可理解的短句：

- 发生了什么变化。
- 谁正在采用或付费。
- 它指向什么机会。
- 这个方向是否持续升温。

## 6. 第一屏验收标准

Strategy 验收：

- 5 秒内能判断观澜AI不是新闻站，而是 AI 商业机会判断系统。
- 不新增商业化承诺，不夸大机会确定性。
- Scoring、Tags、Admin 不进入普通首屏。

PM 验收：

- 首屏只完成“说明价值 + 进入今日判断”一个主任务。
- 首页内容不超过精选样例，其余进入栏目页。
- CTA 明确但不制造焦虑。

UI / UE 验收：

- 使用 `frontend-design + Apple 留白 + Bloomberg / FT 内参式阅读 + Linear 信息密度`。
- 首屏主次清楚：H1、导语、CTA、海报互不抢权。
- 海报图不是后台面板，不是抽象流程图，不是模板化 SaaS hero。
- 桌面端和移动端均无文字遮挡、横向溢出、按钮挤压。

Copy 验收：

- 文案言之有物，简洁克制。
- 不出现内部流程语。
- 不使用“确定机会、强证据、立即行动、保证收益”等表达。
- 标题长度适合首屏，不造成难看的中文断行。

## 8. 自动化影响判断

本规划任务只输出首页方向、信息架构、海报候选和后续任务拆分。

未改动：

- Markdown 命名、目录或 frontmatter。
- Signal / Priority / Trend / Opportunity / Point 字段。
- `sync-data.mjs`、`check-relations.mjs`、`check-point-quality.mjs`、`unified-site-sync.mjs`。
- The Point 来源、素材笔记、全文/译文规则。
- 每日商业雷达机会拆解、评分表、趋势或机会卡生成规则。
- 自动化时间线、入站顺序、备份、回滚或发布闸门。

因此默认不影响：

- `ai-the-point`
- `ai-2`
- `ai-3`

## 9. 调度中枢建议

建议调度中枢将本任务标记为 `review` 后，按以下顺序继续派发：

1. Copy Agent：首屏最终文案。
2. UI / UE Agent：首屏结构与海报规格。
3. Visual / Asset：情报桌面图或真实样张资产。
4. Dev Agent：首页首屏实现。
5. QA Agent：浏览器截图与多身份验收。

如果调度中枢希望更快落地，可先走 Copy + UI 两个规划任务，再合并成一个 Dev 实现任务。
