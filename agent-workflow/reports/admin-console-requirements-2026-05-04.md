# Admin 管理后台独立升级需求报告

日期：2026-05-04  
owner：`pm`  
对应任务：`WSD-20260504-08-admin-console-requirements`  
状态：drafted for dispatch

## 1. 本轮结论

Admin 后台应从“本地控制台原型”升级为独立管理工作台。它不再只是同步按钮、JSON 编辑器和用户列表的组合，而是承接每日运营、内容管理、用户权限、订阅订单、质量检查和发布准备的后台系统。

本任务只输出开发需求和后续派发建议，不修改 `04-Site/admin.html`，不接入真实云端权限、数据库或支付。

## 2. 当前 Admin 页面观察

`04-Site/admin.html` 当前已有：

- 一键同步。
- 本机访问权限开通。
- 可视化编辑入口。
- 栏目与卡片 JSON 编辑。
- 用户与阅读权限。
- 订阅记录。
- 同步说明弹窗。

主要缺口：

- 仍使用普通前台顶部导航，后台信息架构不独立。
- 首屏不能直接判断“今天能否发布”。
- 内容管理默认依赖 JSON，不利于长期运营。
- 用户、订单、质量检查和发布准备没有形成完整后台工作流。
- 高风险操作需要更清楚的提示和二次确认。

## 3. Admin 后台模块清单

| 模块 | 目标 | P0 能力 | P1 能力 |
|---|---|---|---|
| 今日工作台 | 判断今天能否发布 | 同步状态、质量摘要、待处理、发布 checklist | 发布记录、备份与回滚摘要 |
| 内容管理 | 管理核心判断资产 | 栏目列表、搜索筛选、结构化摘要、高级 JSON、复制、删除、下架 | 多页编辑、操作日志、字段映射 |
| 用户与权限 | 管理阅读有效期 | 用户列表、状态筛选、延期、到期、默认有效天数 | 用户表映射、管理员身份预留 |
| 订阅与订单 | 管理套餐和支付占位 | 套餐显示、订单状态、手动确认支付 | 真实支付前的数据结构预留 |
| 质量检查 | 集中展示发布风险 | 关系检查、标签检查、The Point / Signal / Opportunity 检查入口 | 自动质量报告聚合 |
| 发布准备 | 降低上线失误 | 同步、质量、备份、前台边界 checklist | release 记录与回滚路径 |
| 系统设置 | 管理本地配置 | 默认有效期、本地服务、缓存清理、备份提示 | 云端配置和权限策略预留 |

## 4. 页面地图

P0 保持单页后台：

```text
admin.html
├─ Admin 专属导航
├─ 今日工作台
├─ 内容管理
├─ 用户与权限
├─ 订阅与订单
├─ 质量检查
└─ 系统设置
```

P1 可拆为多页后台：

```text
admin.html                 今日工作台
admin-content.html         内容管理
admin-members.html         用户与权限
admin-orders.html          订阅与订单
admin-quality.html         质量检查
admin-settings.html        系统设置
```

## 5. 页面设计方向

- Admin 是后台工具，不是普通前台栏目页。
- 后台应使用独立导航，不沿用前台导航节奏。
- 首屏优先状态和动作：同步、质量、内容、用户、订单。
- 内容管理以表格、列表、筛选、详情编辑为主，JSON 作为高级展开能力。
- 用户与订单以表格优先，便于扫读和批量管理。
- 高风险操作必须有明确提示或二次确认。
- 桌面端优先完成主要管理操作，移动端至少能查看状态和处理轻量任务。

## 6. P0 开发批次

目标：让现有 Admin 形成可用工作台，不改变底层权限和自动化代码。

开发项：

1. 重构 Admin 信息架构为后台专属导航。
2. 新增今日工作台，显示同步、质量、内容、用户、订单关键状态。
3. 内容管理从纯 JSON 默认编辑升级为列表 + 结构化摘要 + 高级 JSON。
4. 用户与权限区增加状态筛选、有效期显示和延期入口。
5. 订阅与订单区显示支付状态和手动确认入口。
6. 增加质量检查区和发布前 checklist。
7. 高风险操作增加二次确认。
8. 保证普通前台无后台入口。

## 7. P1 开发批次

目标：为云端正式后台做准备，但不在本批次接入真实数据库、支付或复杂权限。

开发项：

1. 拆分多页后台或建立可拆分路由结构。
2. 增加操作日志模型。
3. 增加备份、恢复、回滚提示和发布记录。
4. 为真实用户表、订单表、内容表预留字段映射。
5. 预留多管理员权限分级。
6. 与上线准备任务对齐部署、备份、回滚和权限方案。
