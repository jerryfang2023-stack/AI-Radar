# Admin Console UI/UE + PM Review

日期：2026-05-04  
参与 Agent：UI / UE Agent、PM Agent  
对象：`04-Site/admin.html` 当前 P0 工作台  
状态：review for next iteration  

## 1. 总体判断

当前 Admin 已从“本地控制台原型”进入“单页后台工作台”阶段，普通前台和 Admin 的边界比原来清楚，内容管理也已从单一栏目下拉升级为 `页面 / 栏目 / 主题` 三层分类。

但站在管理员角度，它目前仍是“可用后台”，不是“高效后台”。主要原因是：首页统计还不够运营化，各模块只是被分开显示，尚未形成足够清楚的任务流、状态流、批量处理能力和错误处理路径。

综合评分：`6.8 / 10`

- UI/UE 页面与标题规范：`7.0 / 10`
- 管理员操作效率：`6.5 / 10`
- 功能模块完整度：`6.8 / 10`
- 发布前可控性：`6.2 / 10`

## 2. UI / UE Agent 复核

### 2.1 已符合的规范

- Admin 已使用后台专属左侧导航，不再沿用普通前台导航。
- 首页与模块操作区已经分离，不再把所有内容混成一页长滚动。
- 内容管理已增加 `页面 / 栏目 / 主题` 分类，方向正确。
- 高风险操作已有确认提示，降低误操作风险。
- 目前没有把 Admin 入口暴露到普通前台。

### 2.2 仍需注意的页面和标题规范

1. 顶部标题仍固定为 `管理工作台`，进入 `内容管理 / 用户与权限 / 订阅与订单 / 质量检查 / 系统设置` 后，顶部主标题没有跟随模块变化。
   - 建议：顶部 H1 应随当前模块变化，例如 `项目统计`、`内容管理`、`用户与权限`。
   - `管理工作台` 可降级为左侧品牌或顶部小字。

2. 首页 `项目统计总览` 的视觉权重偏大，像一个 hero，而不是管理后台的统计首页。
   - 建议：Admin 首页应更像状态仪表盘，不需要大面积主视觉。
   - 统计首页优先展示数量、状态、最近更新时间、异常和下一步动作。

3. 模块内标题层级需要统一。
   - 当前顶部 H1、模块 H2、卡片标题之间的权重仍略混乱。
   - 建议每个模块固定结构：模块标题、模块状态摘要、筛选/动作区、主表格或主列表、右侧详情。

4. 内容管理右侧“结构化预览 + 高级 JSON”方向正确，但还不是管理员的主编辑界面。
   - 建议逐步把常用字段做成结构化表单，JSON 只作为高级折叠能力。

### 2.3 UI/UE 下一轮优先级

P0：
- 动态模块标题和面包屑。
- 首页从大 hero 改成项目统计仪表盘。
- 内容管理改成“左侧分类 / 中间表格 / 右侧详情编辑”的稳定后台布局。

P1：
- 增加空状态、错误状态、保存成功 / 失败状态。
- 移动端只保留查看和轻量处理，复杂编辑建议桌面端完成。

## 3. 管理员操作体验评分

### 3.1 是否便于管理

当前答案：`基本便于查看，不够便于高频管理。`

如果管理员每天只做：
- 看一下项目数据；
- 同步一次；
- 找一条内容微调；
- 延长用户权限；

当前版本可以胜任。

如果管理员每天要做：
- 批量复核今日 Signals；
- 快速处理软提醒；
- 判断能否发布；
- 查找某个主题下所有关联内容；
- 批量管理用户与订单；
- 追踪谁改了什么；

当前版本还不够顺手。

### 3.2 主要摩擦点

- 首页没有真正显示“今天哪些地方需要处理”。
- 内容管理有分类，但缺少日期、状态、标签、质量状态、关联状态等筛选。
- 条目列表还不是表格，批量扫读效率不足。
- 用户与订单没有形成管理表格，缺少搜索、排序、批量操作。
- 质量检查只是入口说明，还没有真正显示最近检查结果、硬错误、软提醒和报告链接。
- 发布前 checklist 不能记录处理状态，也不能关联检查报告。

## 4. PM Agent 功能模块改进建议

### 4.1 项目统计首页

目标：打开 Admin 后 5 秒内知道今天能不能继续处理或发布。

建议模块：
- 今日内容统计：Daily、Signals、Priority Engine、The Point、Trends、Opportunities 数量和最近更新时间。
- 待处理事项：缺字段、缺关联、软提醒、未知标签、到期用户、异常订单。
- 自动化状态：`ai-the-point`、`ai-2`、`ai-3` 最近运行状态。
- 发布状态：可发布 / 待复核 / 阻塞。
- 快捷动作：同步、查看质量检查、进入内容管理、查看前台。

### 4.2 内容管理

建议拆成更明确的管理对象：

- 页面：首页、Daily Brief、Signals、The Point、Opportunities、Trends。
- 栏目：Signals、Priority Engine、The Point、Trends、Opportunities、Daily Brief。
- 主题：趋势主题、The Point 主题、标签主题。
- 来源：The Point 来源、Signal 来源、外部媒体来源。
- 关系：Signal -> Trend、Signal -> Opportunity、Point -> Signal。

每个列表建议增加：
- 日期筛选。
- 状态筛选。
- 标签 / 赛道筛选。
- 质量状态筛选。
- 关联状态筛选。
- 最近更新时间。
- 批量选择和批量标记 review / published / archived。

### 4.3 用户与权限

建议从卡片改成表格优先：
- 用户、联系方式、公司、角色、阅读状态、到期时间、套餐、最近访问、操作。
- 支持搜索、状态筛选、即将到期筛选。
- 增加权限调整记录。
- 高风险操作继续二次确认。

### 4.4 订阅与订单

建议补齐：
- 订单状态筛选：待支付、已支付、失败、手动确认、已取消。
- 异常订单提醒：已支付但未延权、用户不存在、重复订单。
- 套餐管理：月 / 季 / 年是否展示、价格占位、说明文案。

### 4.5 质量检查

当前只是入口，下一步应产品化为质量控制台：
- 显示最近一次检查时间。
- 显示硬错误数量、软提醒数量。
- 展示最近报告链接。
- 一键运行关系检查 / 标签检查 / The Point 检查。
- 把问题按模块归类，允许跳转到对应内容管理条目。

### 4.6 系统设置

建议保留低频能力：
- 本地管理服务状态。
- 清缓存。
- 备份与恢复说明。
- 自动化任务说明。
- 后续云端配置占位。

系统设置不应承载高频编辑入口，高频入口应回到内容管理或质量检查。
