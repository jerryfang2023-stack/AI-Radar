## 1. 战略判断

The Point 首页当前不应被定义为“观点列表页”“人物观点墙”或“海外原文合集”。它在观澜AI中的战略价值是：

> 以一线建造者、研究者、产品负责人的关键观点为核心内容，在明确出处的基础上，用观澜注解帮助用户看见 AI 共识、分歧与边界。

用户来到 The Point，不只是想知道“谁说了什么”，而是想完成三个判断：

1. 哪些一线观点正在改变 AI 方向的理解。
2. 这些观点与当天 Signal、长期 Trend、可验证 Opportunity 有什么关系。
3. 当前观点是正在形成共识、出现分歧，还是只适合作为早期观察。

因此 The Point 首页改版的核心命题是：

> 从“观点展示”升级为“基于一线观点识别 AI 共识、分歧与边界”的判断入口。

### 1.1 与其他栏目边界

| 栏目 | 负责回答 | The Point 不应抢的任务 |
|---|---|---|
| Daily Brief | 今天先看什么判断 | 不替代每日主线简报 |
| Signals | 发生了什么商业变化 | 不把观点包装成事实新闻 |
| Opportunities | 哪些机会值得验证 | 不直接给机会排序或行动指令 |
| Trends | 哪些方向正在升温、分化或降温 | 不变成趋势列表 |
| The Point | 一线观点如何解释变化、补充证据、提示分歧 | 不做观点热榜、人物榜、原文搬运 |

### 1.2 价值表达重心

新版 The Point 首页必须突出四种价值：

- **标明一线出处**：人物、机构、来源类型和原文链接必须清楚，不能让用户觉得观点是无来源转述。
- **解释变化**：同一条商业信号，一线建造者怎么看，能帮助用户理解它为何不是普通新闻。
- **识别共识**：多位高质量来源是否在谈同一方向，是否形成连续观点。
- **保留分歧**：观点不一致时，要呈现分歧本身，而不是强行给出单一结论。
- **连接机会**：观点需要回到 Signal / Trend / Opportunity，而不是停留在精彩语录。

### 1.3 非目标

新版 The Point 首页不做：

- AI 名人榜。
- 社媒热度榜。
- 观点瀑布流。
- 长文素材库首页。
- 外部全文售卖页。
- 对外解释抓取、同步、字段或后台流程。
- 投资、经营或合作建议。
- The Point 数据模型、自动化规则或导航位置调整。

## 2. 页面目标

The Point 首页改版后，应让目标用户在首屏内理解：

- 这里追踪的是一线 AI 建造者观点，不是泛泛评论。
- builders、机构和原文出处是可信度核心，观澜注解是第二层增值。
- 用户可以沿着“人物观点 -> 观点含义 -> 关联 Signal / Trend / Opportunity”继续阅读。

页面成功标准：

- 5 秒内能看懂 The Point 的独特价值。
- 首屏不显得像普通资讯页或人物墙。
- 每条观点的“商业含义”和“判断边界”比人物头像、分数、来源徽章更突出。
- 页面明确引导用户进入观点详情、每日观点集合或关联判断资产。

## 3. 推荐页面结构

### 3.1 首屏：一线观点正在改变哪些判断

首屏不宜继续使用平铺卡片。建议结构：

- 栏目标题固定为：`从一线观点中，看见 AI 共识、分歧与边界。`
- 不加小字导语，不解释栏目功能。
- 主观点：当天或最近一条最高价值 Point，强调“谁说了什么”“出处在哪里”与“观澜怎么看”。
- 右侧或下方三条辅助线索：
  - 正在形成的共识。
  - 仍有分歧的方向。
  - 可连接的机会或趋势。

Copy Agent 后续不再重写 H1，只负责模块标题、卡片字段和 CTA 的精炼。

### 3.2 判断分组：共识 / 分歧 / 早期信号

首页不应只按时间倒序列观点。建议按判断状态组织：

| 分组 | 用户价值 | 展示内容 |
|---|---|---|
| 正在形成共识 | 看哪些方向被多位高质量来源反复提及 | topic、代表观点、相关 Trend / Opportunity |
| 出现分歧 | 看风险、反证和路径差异 | 支持 / 谨慎观点、分歧点 |
| 早期信号 | 看还未形成结论但值得观察的观点 | 单条高质量观点、需要继续观察的证据 |

第一版可以由现有 Point 字段和人工策展实现，不要求新增数据模型。

### 3.3 今日一线观点：页面核心，少而精

今日观点区是 The Point 首页核心，不应被弱化成侧栏或补充模块。建议展示 6-10 条，不追求铺满。

每条 Point 最小展示：

- 来源人物 / 机构。
- 来源类型：X / Blog / YouTube / Podcast / 官方博客等。
- 原文出处链接：哪怕国内用户可能无法访问，也必须标明出处。
- 原始观点摘要：用中文转述观点核心，不把摘要冒充为原文全文。
- 观澜解读：为什么这条观点值得看。
- stance 或判断状态。
- 关联 Signal / Trend / Opportunity 的轻入口。
- “查看观点”入口。

弱化或避免：

- 大面积人物头像墙。
- 分数作为主视觉。
- 原文大段展示或全文搬运。
- 过多装饰性徽章、标签、时间戳。

### 3.4 关联判断资产：让观点回到系统

页面需要一个明确区域展示 The Point 与观澜AI判断系统的关系，但不写内部流程说明。

推荐表达方式：

- “这些观点正在连接到哪些变化”
- “相关信号”
- “相关趋势”
- “相关机会”

不推荐：

- “数据关系网络”
- “系统自动关联”
- “字段解析”
- “证据链已沉淀”

### 3.5 出处与素材入口：必须标明，但不作为售卖主路径

builders 与原文出处仍然是 The Point 的可信度核心。首页必须清楚标明来源和外链，但不能把外部全文包装成观澜自有付费内容。

推荐入口：

- “查看出处”
- “查看来源”
- “阅读观点”

不在首页大段展示全文、译文、speaker/timecode 或授权说明；外部原文链接作为来源参考，不作为唯一阅读路径。

## 4. UI / UE 执行要求

UI / UE Agent 后续执行前必须使用 `frontend-design`，并明确参考：

- Bloomberg / FT / Economist 内参式阅读：用于标题、导语、主观点、侧栏摘要的层级。
- Linear：用于信息密度、列表行、状态标签和干净边界。

页面阅读路径：

1. 先看到 The Point 的价值判断。
2. 再看到一条高价值主观点。
3. 再按“共识 / 分歧 / 早期信号”扫描。
4. 最后进入观点详情或关联 Signal / Trend / Opportunity。

视觉约束：

- 一级栏目标题必须与 Daily Brief / Signals / Opportunities / Trends 保持同一套位置、字号、行高和首屏节奏。
- 首屏背景默认与页面背景一致，不新增突兀色块。
- 卡片半径、边框、阴影保持克制。
- 不做卡片套卡片。
- 不做人物头像墙或社媒动态流。
- 移动端优先保证主观点、分组和列表的阅读顺序。

## 5. Copy 执行要求

Copy Agent 后续应重写以下文案：

- 模块标题。
- 首屏主观点区字段。
- 主观点区标题与摘要。
- 共识 / 分歧 / 早期信号分组名。
- Point 卡片字段名。
- CTA。
- 空状态。

文案原则：

- 写给商业决策者，不写给内部团队。
- 强调观点对判断的帮助，不说“观点层”“抓取”“字段”“同步”。
- builders、机构和出处必须清楚，不把一线来源藏起来。
- 不把一线观点写成事实结论。
- 不替用户下最终经营、投资或合作判断。
- 避免“确定机会”“强证据”“下一步验证”“入口”等内部或说服式表达。

推荐 CTA：

- “查看观点”
- “查看来源”
- “查看相关信号”
- “查看相关趋势”
- “进入今日观点”

## 7. QA 验收标准

QA / Acceptance Agent 后续至少检查：

- The Point 首页没有后台话术、同步话术、字段话术。
- 页面不显示 JSON、Markdown、抓取、编辑、恢复等后台痕迹。
- 首屏能明确表达“一线观点如何帮助商业判断”。
- 页面没有退化为人物墙、观点热榜或原文搬运页。
- 标题位置、字号、行高和首屏节奏与其他一级栏目一致。
- 桌面端和移动端无横向溢出。
- 点进观点详情、每日集合、素材阅读、相关 Signal / Trend / Opportunity 的路径可用。
- 未登录、试读、到期、管理员状态下普通前台不泄漏后台能力。

浏览器检查应由 UI / Dev / QA 落地任务运行，本规划任务不运行浏览器检查。

## 9. 自动化影响判断

本任务只做页面方向与产品需求规划：

- 不修改 `05-Point/` 内容源。
- 不修改 The Point frontmatter、字段规则或数据模型。
- 不修改同步脚本。
- 不修改自动化时间线。
- 不修改 The Point 在导航中的位置。

因此默认不影响：

- `ai-the-point`
- `ai-2`
- `ai-3`

后续 Dev 任务如只改普通页面样式和文案，也应继续保持“不影响自动化任务”。若未来要改字段、文件命名、同步口径或自动化顺序，必须另走 Plan-first 并提示用户“可能影响自动化任务”。
