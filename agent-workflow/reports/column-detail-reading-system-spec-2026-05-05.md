## 2. UI/UE 规范表一：一级栏目页

适用页面：`daily.html`、`signals.html`、`the-point.html`、`opportunities.html`、`trends.html`。

| 规范项 | 标准 |
|---|---|
| 页面目标 | 用户 5 秒内知道这是哪类判断资产，并能进入主列表阅读。 |
| 设计基准 | 沿用 DESIGN v2 的“商业情报桌面”：象牙白纸面、深石墨文字、深墨绿重点、铜棕小面积强调。 |
| 布局基准 | 顶部导航后进入统一栏目标题区；主容器建议 `1480-1680px`；标题区、command bar、主列表的左边界保持一致。 |
| 标题区 | 高度 `220-260px`；背景与页面底色一致；不加独立色块、渐变、装饰图或解释型副标题。 |
| H1 | `52-60px / 1.10`；一句话价值表达；桌面端优先一行；不得追加小字说明。 |
| Command bar | 标题区后第一组控件；高度克制；筛选、排序、搜索合并为一行或两行以内；不能像后台表单。 |
| 主列表 / 主卡组 | 列表标题和摘要是第一视觉；卡片内边距 `18-24px`；标题 `18-22px / 1.35`；meta `11-13px`。 |
| 侧栏 | 只做摘要、当前筛选、相关主题或轻量导航；宽度不超过主内容的 35%；不得抢主列表。 |
| 空状态 | 只说明当前没有可读结果，并提供一个回到全部内容的动作；不写内部原因。 |
| 加载状态 | 使用安静骨架或文本状态；不出现同步、脚本、JSON 等字样。 |
| 移动端 | 单列；H1 收敛到 `34-42px / 1.12`；command bar 可折叠；不横向滚动。 |
| 禁止项 | 后台指标大卡、解释型副标题、卡片套卡片、大面积雾化渐变、排行榜视觉、字段表感。 |

### 2.1 栏目差异规则

| 栏目 | 页面任务 | 必须强化 | 必须弱化 |
|---|---|---|---|
| Daily Brief | 进入每日商业内参 | 最新 Brief 主判断、日期、关键支撑 | 日期归档感、阅读教程、行动指令 |
| Signals | 查看商业信号 | 事件、来源、商业含义、关联机会 / 趋势 | 后台仪表盘、指标卡过强、编辑痕迹 |
| The Point | 看一线观点的共识、分歧与边界 | 主观点、人物 / 机构、来源、观澜解读边界 | 人物墙、观点热榜、日期归档 |
| Opportunities | 看机会库和优先观察序列 | 机会定义、适用客户、证据状态、相关信号 | 排行榜、公司库、分数最大化 |
| Trends | 看趋势阶段和反证 | 阶段、连续证据、分化、风险 | 热词卡片墙、泛热度表述 |

## 3. UI/UE 规范表二：详情页阅读母版

适用页面：`daily-detail.html`、`signal.html`、`opportunity.html`、`trend.html`、`point.html`。

| 规范项 | 标准 |
|---|---|
| 页面目标 | 用户能像读研究笔记一样读完一个判断，并理解来源、边界和关联内容。 |
| 设计基准 | DESIGN v2 的详情页母版：报告正文 + 右侧摘要 / 来源 / 关联。 |
| 总布局 | 桌面端两栏；主栏 `760-860px`；侧栏 `320-380px`；两栏间距 `48-64px`。 |
| 顶部 meta | 类型、日期、来源状态、栏目归属放在 H1 上方；`11-13px / 1.2`；铜棕或次级文字。 |
| H1 | `38-46px / 1.18`；表达判断主题，不只写日期或公司名。 |
| 主判断 | H1 下方用一段 `18-20px / 1.55` 的导语承接；不重复 H1。 |
| 正文 | `15.5-17px / 1.75-1.85`；段距 `14-18px`；小标题 `22-26px / 1.25`。 |
| 侧栏 | 摘要、来源、关联 Signal / Opportunity / Trend / Point；不复制正文，不放长段落。 |
| 关联内容 | 底部或侧栏轻量列表；不做大卡片堆叠；不使用“证据链”等说服式词。 |
| 引用 | 同一引用只出现一次；顶部作为主引用时，正文不再重复截断文本。 |
| CTA | 只保留读者自然动作：查看信号、查看来源、进入机会库、继续阅读。 |
| 移动端 | 单列；侧栏移动到正文后；H1 `30-36px / 1.18`；正文行宽保持舒适。 |
| 禁止项 | 字段卡片平铺、正文过宽、侧栏重复正文、后台词、投资 / 经营指令。 |

### 3.1 详情页差异规则

| 详情页 | 主阅读任务 | 主栏必须呈现 | 侧栏只承接 |
|---|---|---|---|
| Daily Detail | 读今日主线 | 今日主判断、关键 Signals、机会观察、趋势观察、反证边界 | 日期、相关 Brief、轻量索引 |
| Signal Detail | 理解一个商业信号 | 发生了什么、来源与事实、商业含义、机会拆解 | 信源状态、相关机会 / 趋势 |
| Opportunity Detail | 理解一个机会方向 | 机会定义、目标客户、替代流程、证据状态、不确定性 | 相关 Signals、Trends、代表案例 |
| Trend Detail | 理解趋势是否持续 | 趋势阶段、证据阶梯、分化 / 降温、反证 | 相关 Signals、Opportunities |
| Point Detail | 理解一个观点的价值和边界 | 人物 / 机构观点、原始观点转述、观澜解读、边界 | 来源、相关主题、相关判断 |

## 4. UI/UE 规范表三：长文阅读母版

适用页面：`point-source.html`，以及后续全文文档、全文译文、来源说明页。

| 规范项 | 标准 |
|---|---|
| 页面目标 | 用户能安静阅读来源材料，清楚区分原文、译文、来源与授权状态。 |
| 设计基准 | 商业内参长文阅读：低噪音、窄正文、来源信息清楚但不压过正文。 |
| 主栏宽度 | `720-820px`；长文正文居中或与页面报告主栏对齐。 |
| 辅助栏 | 桌面端可有 `280-340px` 的素材信息栏；移动端移至正文后。 |
| 标题 | `36-44px / 1.18`；标题下只保留来源、日期、素材类型、授权状态。 |
| 正文 | `16-17px / 1.8-1.9`；段距 `16-20px`；列表和小标题有明确层级。 |
| 原文 / 译文 | 使用标签或分段切换；不能把摘要伪装成译文；译文说明要克制。 |
| 长引用 | 左侧细线或浅底，字号不小于正文；不使用重色块。 |
| 来源与版权 | 放在页尾或侧栏底部；弱化呈现，但必须可读。 |
| 目录 | 长文超过 3000 中文字时可使用轻量目录；不做浮动大面板。 |
| 移动端 | 单列；正文不小于 `16px`；不横向溢出；表格或长链接需要换行。 |
| 禁止项 | 把来源说明放成主视觉、正文过宽、授权状态像警告牌、摘要冒充全文。 |

## 5. Copy 规范表

### 5.1 通用语气

| 项目 | 标准 |
|---|---|
| 基本语气 | 有判断、克制、具体、保留边界。 |
| 句子长度 | H1 尽量 18 中文字以内；卡片标题尽量 24 中文字以内；CTA 2-8 字。 |
| 判断边界 | 使用“值得继续观察”“当前更像早期信号”“仍需反证”“证据正在增强”。 |
| 禁止语气 | 不替用户下最终经营、投资或合作判断；不制造焦虑；不夸大确定性。 |
| 容器适配 | 文案过长时先压缩文案，不靠放大容器或缩小字号硬塞。 |

### 5.2 栏目页标题规范

| 栏目 | 标题方向 | 可用标题 |
|---|---|---|
| Daily Brief | 每日主判断 | `今天值得跟踪的 AI 商业变化` |
| Signals | 商业信号 | `从 AI 热点中筛出商业信号` |
| The Point | 一线观点 | `从一线观点中，看见 AI 共识、分歧与边界` |
| Opportunities | 机会库 | `把连续信号沉淀为可观察机会` |
| Trends | 趋势判断 | `看 AI 商业变化是否正在形成趋势` |

说明：栏目页不加解释型副标题。若 Dev 落地时发现标题在桌面端无法一行显示，应先回到 Copy 压缩。

### 5.3 详情页 H1 / meta / CTA 规范

| 元素 | 标准 | 推荐表达 | 禁止表达 |
|---|---|---|---|
| Eyebrow | 类型 + 日期 / 来源 | `Signal · 2026-05-05`、`Opportunity · 优先观察` | `详情页入口`、`自动同步` |
| H1 | 判断主题 | `企业 Agent 正在靠近员工桌面入口` | 只有公司名、只有日期 |
| Meta | 来源与状态 | `单信源支撑`、`多信源观察`、`持续观察` | `强证据`、`来源明确` |
| CTA | 自然阅读动作 | `查看信号`、`查看来源`、`进入机会库`、`继续阅读` | `立即验证`、`马上行动` |

### 5.4 长文页文案规范

| 元素 | 标准 |
|---|---|
| 原文标签 | 使用 `原文` 或 `素材原文`。 |
| 译文标签 | 使用 `全文译文`；仅在逐段翻译完整时使用。 |
| 摘要标签 | 使用 `内容摘要`，不得写成译文。 |
| 来源说明 | 使用 `来源与授权`；说明材料来源和使用边界。 |
| 状态说明 | 使用 `已整理`、`待补全文`、`仅摘要可读` 等事实状态。 |

### 5.5 禁用语检查

公开前台不得出现：

- Admin
- JSON
- Markdown
- 同步
- 脚本
- 字段
- 后台
- 编辑
- 恢复
- 本页用于
- 入口
- 证据链
- 强证据
- 来源明确
- 机会确定
- 下一步验证
- 行动清单
- 必投
- 立即行动
- 保证收益

允许在 Admin 或内部报告中出现的词，不得出现在普通前台。
