# 2026-05-03 PM Agent｜Signals 首页改版方案

## 栏目定位

Signals 不是新闻列表、公司动态库或工具目录，而是观澜AI的证据层。

用户进入 Signals 的核心任务是：

1. 看清发生了什么商业变化。
2. 判断它为什么不是普通新闻。
3. 理解它指向哪个机会方向或趋势变化。
4. 查看证据强度、来源和原始链接。
5. 顺路进入相关 Opportunity / Trend 做下一步判断。

## 当前问题

当前 Signals 首页主要是左侧列表 + 右侧详情，更接近内部工作台。它能承载浏览和搜索，但对客户的价值表达不足：

- 首页第一屏没有回答“今天有哪些值得看的商业信号”。
- 列表项信息层级偏平，客户需要点进去才知道证据强度和商业含义。
- 搜索和赛道筛选有用，但缺少按事件类型、证据强度、趋势状态筛选。
- 缺少日期维度，历史 Signals 堆积在一起，用户难以按天复盘“某一天发生了哪些关键变化”。
- 详情页有机会拆解，但首页没有把 Signal 与 Opportunity / Trend 的关系显性化。
- 容易被误解为 AI 新闻列表，而不是机会判断系统的证据入口。

## 客户利益点

Signals 首页应让客户获得四类收益：

- 降噪：只看进入商业信号层的变化，而不是全量新闻。
- 提速：快速判断哪些变化值得进入 Daily Brief、Opportunity 或 Trend。
- 溯源：每条 Signal 有证据强度和原始来源，方便复核。
- 连接：从单条变化直接看到相关机会、趋势和后续验证方向。

## 改版目标

将 Signals 首页从“列表型页面”升级为“证据雷达页”：

```text
先看总体变化 -> 再筛选高价值 Signal -> 再进入详情验证证据与机会关系
```

## 推荐页面结构

### 1. 顶部判断区

展示 3 个总览指标：

- 今日新增 Signals
- 高证据 Signals
- 已关联机会 / 趋势的 Signals

同时增加一句栏目标识：

> Signals 记录已经出现商业证据、可能改变机会优先级或趋势判断的 AI 变化。

### 2. 变化类型雷达

使用标准 `eventTypes` 聚合：

- 融资
- 客户采用
- 收入验证
- 产品发布
- 监管/政策
- 采购/招标
- 并购整合
- 平台数据

功能：

- 每个类型显示数量。
- 点击后筛选列表。
- 不作为独立栏目，只作为 Signals 内部筛选。

### 3. 高价值 Signal 队列

优先展示：

- 高 Signal Score
- S/A 来源
- 已关联 Opportunity
- 已进入 Trend 的 Signals

每张卡片展示：

- 标题：事件 + 商业含义
- 变化类型
- 证据强度
- 关联机会 / 趋势
- 一句商业含义
- 原始来源入口

### 4. 信号列表区

替代现有左侧窄列表，改为按日期分组的主列表或双栏卡片。日期应成为默认组织方式，而不是藏在元信息里。

默认分组：

- 今日
- 昨日
- 本周
- 更早

每个日期组显示：

- 日期
- 当日 Signal 数量
- 当日主要变化类型
- 当日最高证据等级
- 当日关联机会 / 趋势数量

日期组内再展示 Signal 卡片。

筛选项：

- 日期 / 时间范围
- 搜索
- 赛道
- 变化类型
- 证据强度
- 是否关联 Opportunity
- 是否关联 Trend

排序项：

- 最新
- 按日期分组
- 证据强
- 机会相关
- 趋势相关

日期视图建议：

- 默认视图：按日期倒序分组。
- 筛选后：仍保留日期组，空日期组不显示。
- 点击日期：可进入当天 Daily Brief，形成 Signals 与 Daily Brief 的回链。

### 5. 右侧或下方详情预览

详情预览保留，但弱化内部工作台感。

预览模块顺序：

1. 要闻
2. 为什么这是 Signal
3. 证据与来源
4. 指向的 Opportunity
5. 关联 Trend
6. 机会拆解入口

## 不建议做

- 不做新闻瀑布流。
- 不做公司榜单。
- 不把 Score 做成普通用户主视觉排行榜。
- 不把 Tags 变成一线导航。
- 不在普通用户界面展示编辑、新增、JSON、同步等后台痕迹。

## P0 范围

P0 只做前台价值和筛选体验，不做复杂图谱。

必须实现：

- 顶部 3 个指标。
- 日期分组视图，默认按日期倒序展示。
- 日期 / 时间范围筛选。
- 变化类型筛选。
- Signal 卡片化列表。
- 卡片显示证据强度、变化类型、关联 Opportunity / Trend。
- 详情页保留原始来源跳转。

P1 再做：

- 趋势热度分布。
- 证据时间线。
- Signal 合并/重复提示。
- 标签网络视图。

## 验收标准

- 用户不点详情，也能知道哪些 Signal 最值得看。
- 用户能按日期复盘某一天出现的关键 Signals，并能从日期回到对应 Daily Brief。
- 用户能按变化类型、赛道和证据强度筛选。
- 每条卡片能说明“事件、证据、商业含义、关联机会/趋势”。
- 页面不被误解为新闻站、工具站或公司动态库。
- Admin 编辑入口只在管理员视觉编辑模式出现。
