# Sprint Plan

## Sprint 1：产品定义与判断系统

目标：

先把观澜 AI 的核心产品定义清楚，避免继续在栏目和页面层面打转。

建议任务：

1. 固化核心卖点：比市场早一步，看清哪些 AI 机会值得验证。
2. 建立 Signals 定义、来源体系、关键词自优化机制和评分算法。
3. 将 Daily Brief 定义为每日付费级独立报告。
4. 完成 Priorities 合并进 Opportunities 的产品方案。
5. 建立 Trends 趋势判断模型。

验收：

- 用户能在 5 秒内理解观澜 AI 为什么值得付费。
- 能区分新闻、Signal、Opportunity、Trend。
- 每个核心栏目都有明确用户任务和验收标准。

## Sprint 2：产品闭环与数据关系

目标：

让 Daily Brief、Signals、Opportunities、Trends 形成真实决策闭环。

建议任务：

1. 实现 schema-check。
2. 实现 relation-check。
3. 设计 keyword-quality-report。
4. 设计 brief-quality-check。
5. Opportunity detail 展示评分、证据等级、趋势状态、相关 Signals 和验证动作。

验收：

- 孤立项和重复项可被检查出来。
- 用户可以从 Daily Brief 进入机会，再追溯到证据与趋势。

## Sprint 3：会员制官网体验

目标：

让官网从展示页升级为会员制商业情报产品入口。

建议任务：

1. 登录 / 注册。
2. Daily Brief 详情与归档。
3. Opportunities 详情页升级。
4. 会员状态与权限提示。
5. 管理员用户权限管理。

验收：

- 普通用户不会看到管理入口。
- 首页、Daily Brief、Signals、Opportunities、Trends 可对外演示。

## Sprint 4：云端准备

目标：

完成云端部署前的最后结构准备。

建议任务：

1. deploy-check 初版。
2. 内容路径配置化复查。
3. 备份与回滚方案。
4. 注册登录与会员权限后端方案。
5. 发布检查清单。

验收：

- 可进入云端预发布。
