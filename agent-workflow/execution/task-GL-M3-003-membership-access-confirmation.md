# GL-M3-003 会员访问与订阅模块任务确认

更新时间：2026-05-02  
当前阶段：需求确认，待用户批准进入设计与开发  
负责长期 agent：PM Agent / Commercial Site Agent / UI-UE Agent / Copy Agent / Dev Agent / QA Agent

## 1. 用户确认后的需求调整

本任务从“申请访问 / 申请试读”模式调整为“注册登录 + 试读有效期 + 订阅续期”模式。

核心调整：

1. 不保留申请试读入口。
2. 去掉普通内容的申请访问流程。
3. 普通用户只有未登录、已登录两种基础状态。
4. 用户注册后自动获得试读体验。
5. 默认试读时长为 7 天。
6. 管理员可按周 / 月 / 季 / 年调整用户阅读权限。
7. 登录用户可通过订阅获得更多阅读时间权限。
8. 新增注册页、登录页、账户页、订阅套餐页、购买和支付占位页。
9. 首页增加注册 / 登录入口。
10. 删除旧首屏和尾部的申请访问入口或按钮。

## 2. 页面交付物

### 首页

- 右上角显示注册 / 登录。
- 已登录后显示账户入口或会员状态。
- 删除旧申请访问 CTA。
- CTA 改为注册开始试读、登录、查看订阅方案。

### 注册页

建议路径：`register.html`

功能：

- 用户创建账号。
- 注册后自动获得默认 7 天试读。
- 注册成功后进入账户页或 Daily Brief。

### 登录页

建议路径：`login.html`

功能：

- 已注册用户登录。
- 未注册用户跳转注册。
- 登录成功后根据权限跳转。

### 账户页

建议路径：`account.html`

功能：

- 展示当前登录状态。
- 展示试读或订阅到期时间。
- 到期前提示续订。
- 到期后引导订阅。

### 订阅套餐页

建议路径：`pricing.html`

套餐：

- 月度会员
- 季度会员
- 年度会员

### 购买与支付页

建议路径：`checkout.html`

第一阶段只做流程占位：

- 订单摘要。
- 支付方式占位。
- 支付状态。
- 支付成功后延长阅读权限。

## 3. Admin 交付物

Admin 需要增加：

- 用户列表。
- 用户权限状态。
- 用户试读 / 订阅到期时间。
- 按周 / 月 / 季 / 年调整权限。
- 套餐显示管理。
- 支付状态手动标记。

Admin 继续保留：

- 内容同步。
- Daily Brief 发布。
- Signals / Opportunities / Trends 编辑和合并。
- 质量检查和系统日志。

## 4. 权限验收状态

需要至少验收四种身份：

| 身份 | 预期 |
|---|---|
| 未登录 | 可看首页、注册、登录、订阅说明；受限内容提示登录/注册 |
| 已登录有效 | 可访问完整内容，看到权限有效期 |
| 已登录到期 | 受限内容提示续订，进入订阅套餐页 |
| 管理员 | 进入 Admin 后台管理内容、用户、订阅和支付状态 |

## 5. 不做事项

- 不接真实支付网关。
- 不做企业 CRM。
- 不保留普通内容申请访问表单。
- 不做申请试读入口。
- 不把 Admin 能力暴露在普通前台。

## 6. 下一步分工

待用户确认后：

1. UI-UE Agent 输出注册、登录、订阅、支付、账户页设计规范。
2. Copy Agent 输出权限、订阅、到期、非投资建议文案。
3. Dev Agent 实现本地账号状态、权限判断、页面入口和 Admin 管理。
4. QA Agent 按四种身份跑验收。
