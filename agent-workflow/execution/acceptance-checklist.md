# 阶段验收清单

## PM 验收

- 功能解决了明确的用户判断任务。
- 没有把后台能力误做成前台栏目。
- 栏目之间没有重复承担同一职责。
- 新功能有 PRD、范围、非目标和验收标准。
- 重大变更已写入栏目决策记录。

## 内容验收

- Signal 标题是事件和商业含义，不只是公司名。
- Opportunity 标题不含公司名。
- Priority Engine 的评分对象是机会方向、赛道或场景，并进入 Opportunity 展示。
- Trend 有方向和原因，不只是名称。
- Tags 作为搜索和关系网络能力，能连接多个栏目。
- 对外页面没有内部流程语言。

## 技术验收

- 同步脚本运行成功。
- 网站数据 JSON 可解析。
- 详情页独立 URL 可访问。
- 普通页面无管理入口。
- Admin 页面保留同步、编辑、审批能力。
- 重要改动有备份或可回滚。

## 视觉验收

- 首页少而有序。
- 字体层级不压迫。
- 移动端不拥挤、不溢出。
- 卡片不嵌套过多。
- 标签不过量占据首屏。
- 风格符合高端、克制、商业判断感。

## 云端验收

- 内容路径全部配置化。
- 构建流程可重复。
- 权限边界清楚。
- 注册、登录、7 天试读、会员权限和企业版/私享内参申请边界清楚。
- 发布前健康检查通过。
- 回滚方案可执行。
