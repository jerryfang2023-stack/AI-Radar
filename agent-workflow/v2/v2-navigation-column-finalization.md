# V2 导航与栏目收敛最终决策

## 0. 决策结论

V2 普通前台最终导航确认采用：

```text
今日要点 / 关键信号 / 机会解码 / 商业内参
```

对应英文与 URL 建议：

| 中文导航 | 英文语义 | URL 建议 | 内部承接 |
|---|---|---|---|
| 今日要点 | Daily Brief | `/daily` | AIBriefDaily / FrontSignal |
| 关键信号 | Key Signals | `/signals` | FrontSignal / StructuredSignal |
| 机会解码 | Opportunity Briefs | `/opportunities` 或 `/opportunity-briefs` | OpportunityReport + TrendContext |
| 商业内参 | Business Brief | `/brief` 或 `/business-brief` | AIBriefIssue + HeatCard |

`The Point` 不进入 V2 普通一级导航，统一降级为 `观点校准`。  
`Trends` 不进入 V2 普通一级导航，统一降级为 `趋势背景` 和 `商业热力输入`。  
`商业热力图` 不作为公开一级栏目，作为商业内参核心模块和后台判断资产。

## 1. PM 新增功能门禁复核

| 门禁项 | 结论 |
|---|---|
| 用户任务是否清楚 | 清楚。用户要在更少入口中完成“今天看什么、哪些信号重要、哪些方向值得深读、周期判断在哪里”。 |
| 是否新增复杂栏目 | 否。V2 是收敛栏目，不是扩张栏目。 |
| 是否能由既有模块承接 | 能。Daily / Signals / Opportunities / Brief 承接前台；Point / Trend / HeatEvidence 后台或嵌入。 |
| 是否强化付费理由 | 是。`商业内参` 作为会员 / 增值层，承接周度或月度判断。 |
| 是否破坏 V2-only 口径 | 否。本决策只定义 V2 前台与模块边界。 |
| 是否可进入 Dev 前规范 | 可以。交给 V2-8AUTO 输出页面母版、schema、Copy 与 QA 规范。 |

## 5. 命名与文案边界

- `今日要点`：回答“今天哪些变化值得看，为什么”。
- `关键信号`：回答“哪些外部变化足够重要，值得进入判断资产”。
- `机会解码`：回答“哪些方向经过多源搜索、论证和反证，值得写成深度机会分析”。
- `商业内参`：回答“本周 / 本月 AI 正在影响哪些行业、岗位和流程”。
- `观点校准`：只说明观点如何支持、质疑或修正判断，不做观点热榜。
- `趋势背景`：只说明证据是否持续积累，不做僵化趋势页。

禁用表达：

- 不写 `The Point 频道`、`Trends 频道` 作为 V2 普通前台入口。
- 不写 `机会确定`、`强证据`、`下一步验证`、`落地方案`。
- 不把公司名写成机会解码标题。
- 不把商业热力图写成新闻榜、公司榜、岗位替代榜或公开排行榜。
