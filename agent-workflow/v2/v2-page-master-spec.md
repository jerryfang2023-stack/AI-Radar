# V2 页面母版与设计验收规范

## 0. 设计基线

V2 页面统一采用 `方案 A：极简澜线型`：澜线、地平线、留白、文字秩序和商业内参文件感。页面应像高密度商业判断产品，不像资讯门户、模板 SaaS、后台数据屏或 AI 氛围图。

核心前台导航：

```text
今日要点 / 关键信号 / 机会解码 / 商业内参
```

## 1. 全站结构

| 区域 | 桌面规范 | 移动规范 |
|---|---|---|
| 顶部导航 | 72px，高透云白背景，Logo 左，四导航中，登录 / 会员右 | 64px，Logo 左，菜单右 |
| 页面宽度 | 主内容 `1200-1280px`，报告阅读 `760-860px` | 左右 20px，单列 |
| 模块间距 | `64-96px` | `48-72px` |
| 视觉元素 | 澜线、地平线、细分割线 | 减少装饰，保留标题与阅读节奏 |
| 卡片圆角 | 6-12px，避免卡片套卡片 | 稳定尺寸，文字不挤压 |

禁止：雷达、眼睛、机器人、蓝紫渐变、大屏面板、公开排行榜、字段表、后台语言。

## 2. Home 母版

用户问题：观澜AI是什么，今天为什么值得继续读。

结构：

1. Hero：`观 AI 之澜，识商业之势`，一句价值说明，两个动作。
2. 今日 3 条关键信号摘要。
3. 今日要点预览。
4. 最新 1-2 篇机会解码。
5. 商业内参样张与会员入口。

设计要求：

- 首屏必须露出下一段内容，不能做满屏孤岛。
- 右侧若使用判断样张，必须像纸面内参，不像后台仪表盘。
- CTA 建议：`查看今日要点`、`阅读机会解码`、`了解商业内参`。

## 3. 今日要点栏目页 / 详情页

栏目页结构：

- 日期与今日一句判断。
- 3-5 条要点。
- 今日 3 条关键信号。
- 观点校准摘录。
- 风险与边界。
- 历史日期索引。

详情页结构：

- 标题区：日期、阅读时长、当日判断标签。
- 核心判断：1 段总判断 + 3-5 条要点。
- 今日关键信号。
- 观点校准。
- 机会观察。
- 反证与证据缺口。

验收重点：不能像新闻摘要合集；观点要克制，边界要具体。

## 4. 关键信号栏目页 / 详情页

栏目页结构：

- 今日精选 3 条。
- 历史检索：日期、行业、岗位、流程、关联机会。
- 卡片展示：事件、商业含义、来源数量、影响对象、反证摘要。

详情页结构：

- 标题必须是“事件 + 商业含义”。
- 一句话判断。
- 事实背景。
- 二次来源摘要。
- 机会解析 6 维。
- 商业解释。
- 观点校准。
- 反证与缺口。
- 关联内容。

验收重点：列表页不堆完整报告；详情页不把字段直接摊成卡片。

## 5. 机会解码栏目页 / 详情页

栏目页结构：

- 说明低频、少而精。
- 最新报告。
- 观察中方向。
- 历史报告。
- 内参关联。

详情页结构：

- 报告标题、发布日期、证据状态。
- 一句话机会判断。
- 变化背景。
- 问题与对象。
- 流程变化。
- 价值来源。
- 趋势背景。
- 观点校准。
- 反证与限制。
- 成立边界。
- 建议关注变量。

验收重点：不输出落地方案，不写确定性机会，不用公司名做标题。

## 6. 商业内参栏目页 / 详情页

栏目页结构：

- 最新一期：期号、标题、核心判断、会员状态。
- 本期热力摘要。
- 核心判断列表。
- 证据来源摘要。
- 往期内参。
- 会员提示。

详情页结构：

- 封面区：期号、周期、标题、会员标签、阅读时长。
- 本期总判断。
- 商业热力图。
- 高热三元组。
- 升温 / 降温 / 争议。
- 证据展开。
- 观察边界。
- 往期关联。

验收重点：像周期性商业决策文件，不像普通文章栏目或后台图表。

## 7. 嵌入模块母版

| 模块 | 出现位置 | 展示方式 | 禁止 |
|---|---|---|---|
| 观点校准 | 今日要点 / 信号 / 机会 / 内参 | 来源、立场、支持或质疑什么 | 做成独立频道或热榜 |
| 趋势背景 | 机会解码 / 内参 | 证据积累、时间变化、反证 | 做成趋势词墙 |
| 热力摘要 | Home / 内参 | 三元组、升温 / 降温 / 争议原因 | 做成公开榜单 |
| 反证与边界 | 所有详情页 | 风险、反例、证据缺口 | 空泛免责声明 |

## 8. Design Director 评分表模板

| 质检项 | 通过线 | 证据要求 |
|---|---:|---|
| Style Purity | 16/20 | 是否遵守澜线 VI，是否有异类视觉 |
| Proportion & Rhythm | 16/20 | 首屏、标题、模块节奏是否成立 |
| Color Sophistication | 16/20 | 云白、墨海蓝、雾灰、香槟金比例是否克制 |
| Craftsmanship | 16/20 | 圆角、线宽、hover、状态、对齐是否统一 |
| Emotional Resonance | 16/20 | 第一秒是否像商业判断产品 |
| Squint Test | 通过 | 眯眼后主次仍清楚 |
| 总分 | 首页/母版 >=85，栏目/详情/会员 >=80 | 低于通过线必须重做 |

## 9. Dev 截图验收要求

页面实现后必须输出：

- 桌面截图：1440px 宽。
- 移动截图：390px 宽。
- 横向溢出检查。
- 首屏重心说明。
- 普通用户、登录用户、会员、管理员四状态截图或说明。
- Dev 偏差清单：与本母版不一致处必须解释。
