# V2.0 产品架构与 PRD

## 0. 修订结论

上一版 PRD 质量不达标，问题在于仍沿用 V1 的频道框架，没有充分吸收 V2 算法升级后的产品定位变化。本版按用户反馈重写。

V2.0 不应把所有内容形态都做成前台频道。新的前台结构应更克制：

```text
今日要点
关键信号
机会解码
商业内参
```

处理结论：

- `The Point`：不作为独立频道。它可采集内容少、碎片化，降级为 今日要点 / 关键信号 / 机会解码 / 商业内参中的“观点校准”栏目模块，同时在 Admin 保留素材与来源管理。
- `Trends`：不作为短期独立频道。趋势需要数据积累，短期容易僵化，合并进 `机会解码` 作为“趋势背景 / 势能观察 / 证据积累”模块。
- `Opportunities`：前台改名为 `机会解码`，从浅层机会卡升级为低频、少而精的机会分析报告。不是每天更新，只有当多源搜索、论证和反证足够支撑时才发布。
- `商业热力图`：作为商业内参核心模块和后台判断资产，不做公开榜单。
- `商业内参`：进入会员 / 增值产品层，承接周期性判断。

必须删除的旧口径：

- 不再采用旧版机会地图表述。
- 不提供客户落地方案。
- 不替用户下经营、投资、合作或执行决定。
- 只提供机会判断、证据解释、风险边界、观察变量和建议关注方向。

## 6. 首页重构方案

### 6.1 首页角色

V2 首页承担组合入口：

```text
品牌价值入口
+ 今日要点入口
+ 机会解码入口
+ 商业内参预览
```

首页不承担：

- The Point 独立频道入口。
- Trends 独立频道入口。
- 热力图公开榜单入口。
- 后台流程展示。

### 6.2 首屏结构

建议首屏：

```text
H1：观 AI 之澜，识商业之势
副标题：每天从 AI 变化中筛出关键信号，判断影响，沉淀值得关注的机会方向。
左侧：今日 3 条关键信号
右侧：本周机会观察 / 内参预览
CTA：查看今日要点 / 阅读机会解码
会员 CTA：了解商业内参
```

V1 的 `Intelligence Desk` 只保留“判断样张”思想，不保留原视觉。V2 必须按 `方案 A：极简澜线型` 转译，不做后台面板、雷达、大屏或复杂关系图。

### 6.3 首页模块

| 模块 | 展示 | 用户状态 |
|---|---|---|
| 今日 3 条深度 Signal | 展示 | 普通看标题 + 一句话判断；登录看摘要；会员看来源展开入口 |
| 今日要点摘要 | 展示 | 来自 Daily Brief，不重复全文 |
| 机会解码预览 | 展示 | 只展示最新 1-2 篇深度报告摘要，不承诺每日更新 |
| 观点校准摘录 | 轻量展示 | 作为 Daily / Signal 的辅助引用，不做频道入口 |
| 热力变化摘要 | 会员引导 | 普通只看极简样张，会员看内参热力图 |
| 商业内参预览 | 展示 | 普通看样张和价值；登录看目录；会员看最新一期 |

## 9. 商业热力度与解析维度

商业热力图不回答“这个机会怎么做”，而回答：

```text
AI 正在让哪些行业、岗位、流程发生结构性变化；
这种变化为什么升温、是否有争议、边界在哪里。
```

### 9.1 商业热力度 6 维

| 维度 | 回答什么 | 主要来源 | 用于哪些页面 |
|---|---|---|---|
| 影响坐标 | 哪个行业 / 岗位 / 流程被影响 | Signal tags / Trend Context / HeatEvidence | 商业内参、商业热力图、机会解码 |
| 变化方式 | AI 是替代、增强、压缩、重组、平台化还是外包化流程 | Structured Signal / Deep Dive | 关键信号、机会解码、热力图 |
| 证据密度 | 有多少信号、来源、案例、观点、融资、客户采用支撑 | Front Signal / Secondary Sources / Point Calibration | 商业内参、热力图、机会解码 |
| 变化速度 | 近期证据增长是否明显，还是旧趋势延续 | Trend Context / HeatEvidence period | 商业内参、热力图 |
| 商业强度 | 是否影响预算、收入、成本、效率、交付、风险或管理复杂度 | Business impact tags / Deep Dive | 关键信号、机会解码、内参 |
| 争议与边界 | 是否存在反证、失败反馈、监管限制、集成难度或客户习惯阻力 | Counter Evidence / Point Calibration / Evidence Gap | 所有详情页 |

商业热力度建议公式：

```text
商业热力度 =
影响坐标清晰度 15%
+ 变化方式明确度 15%
+ 证据密度 20%
+ 变化速度 15%
+ 商业强度 20%
- 争议与边界风险调整 15%
```

说明：

- `争议与边界` 不只是扣分项。争议高但证据密度和商业强度高时，可进入“争议升温”而不是降级删除。
- 商业热力度不是曝光度、热搜度或公司热度。
- Point 只校准争议和边界，不能单独推高热力。

### 9.2 机会解析 6 维

旧版维度中过于泛化的“时点原因”和过于宏大的“中国市场迁移判断”容易重复或空泛。本版改为更具体的成立条件。

| 新维度 | 替代旧维度 | 内容口径 |
|---|---|---|
| 解决什么具体问题 | 保留并收窄 | 具体流程里的摩擦、成本、风险或效率损耗 |
| 谁最先感受到这个问题 | 替代“目标客户是谁” | 行业、客户类型、岗位、部门或组织形态 |
| 改变了哪段流程 | 替代“替代或优化什么流程” | 被替代、增强、压缩、重组或重新分配的流程 |
| 价值从哪里来 | 替代“商业模式” | 收入、成本、效率、体验、合规、交付能力或管理复杂度 |
| 触发信号是什么 | 替代泛化的时点原因 | 客户采用、融资、产品成熟、价格下降、模型能力变化、监管变化、竞争动作、渠道变化等具体信号 |
| 成立边界是什么 | 替代宏大的市场迁移判断 | 数据、预算、监管、集成、客户习惯、交付成本、市场差异等边界 |

### 9.3 关键信号解析维度

关键信号详情页使用“机会解析 6 维”的轻量版本，并额外标记：

- 来源等级与二次来源数量。
- 影响坐标：行业 / 岗位 / 流程。
- 变化方式。
- 触发信号。
- 反证与证据缺口。
- 是否形成机会解码候选。

### 9.4 机会解码解析维度

机会解码报告使用完整机会解析 6 维，并叠加商业热力度：

- 机会解析 6 维。
- 商业热力度 6 维。
- 趋势背景：证据是否持续积累。
- 观点校准：一线观点支持、质疑或修正什么。
- 观察变量：后续值得继续看的变量。

机会解码不输出：

- 落地方案。
- 执行步骤。
- 投资、采购、合作或经营决策。

### 9.5 趋势背景解析维度

Trends 不独立成频道后，作为 `趋势背景` 使用以下维度：

| 维度 | 口径 |
|---|---|
| 证据积累 | 相关高质量 Signal 是否增加 |
| 时间变化 | 本周 / 本月相比上一周期是否升温、降温或稳定 |
| 影响坐标扩散 | 是否从单一行业 / 岗位 / 流程扩散到更多三元组 |
| 反证变化 | 反证是在减弱、增强，还是进入争议 |
| 内参引用 | 是否足够进入某期商业内参 |

### 9.6 商业内参解析维度

商业内参以商业热力度为主轴：

- 本期核心判断。
- 高热影响坐标。
- 高热变化方式。
- 证据密度与变化速度。
- 商业强度。
- 争议与边界。
- 关联关键信号、机会解码、观点校准和趋势背景。

## 12. Signals V2 详情要求

每日新增 3 条深度分析 Signal 后，Signals 列表页应变为“今日精选 3 条 + 历史检索”的结构。

Signal 详情页必须呈现：

- 一句话判断。
- 事实背景。
- 至少 3 个 S/A/B 二次搜索来源。
- 机会解析 6 维：具体问题、首要感受者、流程变化、价值来源、触发信号、成立边界。
- 商业解释：影响收入、成本、效率、客户体验、管理难度或合规风险中的哪一类。
- 观点校准：一线观点、质疑、失败经验或非共识，不替代事实证据。
- 反证与证据缺口。
- 关联机会分析报告 / 内参热力点。

## 13. Opportunities V2 模块要求

`Opportunities` 的前台名称确定为：

```text
机会解码
```

它不是机会卡列表，也不是行动计划库。它是低频、少而精的报告型内容。

发布条件：

- 至少出现一组值得关注的商业变化。
- 已完成多方面搜索和论证。
- 有足够来源说明客户、流程、商业模式、市场背景和反证。
- 能说明触发信号和关注理由，但不能写成用户应该怎么做。
- 证据不足时不更新，不用低质量内容填充页面。

报告结构：

1. 一句话机会判断。
2. 发生了什么变化。
3. 解决什么具体问题。
4. 目标客户和受影响对象。
5. 替代或优化什么流程。
6. 价值来源。
7. 触发信号。
8. 趋势背景和证据积累。
9. 反证、限制与不确定性。
10. 成立边界。
11. 建议关注变量。

禁止：

- 不给落地方案。
- 不写“下一步落地动作”。
- 不替客户决定是否进入、投资、合作或采购。
- 不用公司名做标题。
- 不为更新频率牺牲判断质量。

## 14. Trends V2 模块要求

`Trends` 不作为独立前台频道。它在 V2 中承担三种角色：

1. `Opportunities` 报告里的趋势背景和证据积累。
2. 商业内参里的热力变化来源。
3. Admin 中的长期趋势资产库。

Trend 资产需要积累：

- 相关高质量 Signal 数量。
- 时间跨度。
- 来源类型多样性。
- 观点校准与反证。
- 是否形成行业 / 岗位 / 流程三元组。
- 是否被某期商业内参引用。

积累不足时，只入库，不生成前台页面。
