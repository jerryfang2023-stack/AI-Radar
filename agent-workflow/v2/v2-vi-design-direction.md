# 观澜 AI｜WAVESIGHT AI V2.0 VI / Design Direction

## 0. 结论

V2.0 正式采用用户提供的 `方案 A：极简澜线型` 作为观澜 AI 的品牌视觉方向。

核心品牌表达：

```text
观 AI 之澜，识商业之势
```

核心价值线：

```text
看懂变化，判断影响，找到行动点
```

V2.0 的视觉不再强调雷达、眼睛、信号点、箭头、复杂山峰或蓝紫科技感，而是围绕 `澜线 / 地平线 / 留白 / 文字秩序` 建立识别系统。品牌气质应是：

```text
东方留白、商业智识、克制高级、面向未来。
```

后续 V2 页面、Logo、报告、内参、热力图、会员页和社交物料均应以本文件为默认设计口径。任何 V2 页面 Dev 必须先吸收本文件，不得直接沿用 V1 的雷达、poster、后台面板或抽象 AI 视觉。

### 0.1 品牌资产锁定规则

用户已确认：以后官网、页面、可能新增的栏目、组件、符号、元素和动效，都必须按本规范执行。

以下品牌资产不得被后续执行窗口随意更改、重画或替换：

- Logo 主方向：`方案 A：极简澜线型`。
- Logo 核心结构：`一条远方地平线 + 三道流动澜线 + 观澜 AI 字标 + WAVESIGHT AI 英文字标`。
- Logo 中英文呈现：中文主视觉为 `观澜 AI`，英文统一为 `WAVESIGHT AI`。
- 核心图形语言：澜线、地平线、细分割线。
- 商业内参符号系统：Signal、Point、Trend、Opportunity、Quote、Divider、Timeline Node、Data Snapshot、Rating Chip、Source Note。

## 2. 品牌总方向

观澜 AI 不做炫技型 AI 品牌，也不做资讯流量型媒体。它应呈现为一个更克制、更可信、更有判断力的 AI 商业洞察与内参品牌。

品牌母题：

| 母题 | 含义 | 使用方式 |
|---|---|---|
| 澜线 | 变化、流动、趋势 | Logo、底纹、卡片角标、报告页脚、趋势图辅助 |
| 地平线 | 视野、远见、判断边界 | Logo 顶部弧线、标题细线、章节分割、会员标识 |
| 留白 | 克制、专业、高级 | 首页首屏、报告封面、内参详情、模块间距 |
| 文字 | 智识、可信、决策感 | 中文衬线标题、黑体正文、等宽数字 |

目标用户感受：

```text
平静，但有判断力；克制，但有方向感。
```

非目标视觉：

```text
赛博霓虹、机器人手臂、蓝紫渐变、复杂雷达扫描、大量信号点阵、明显眼睛图形、金融机构式厚重金色、国潮山水插画、文旅品牌感、茶品牌感。
```

## 3. Logo 系统

### 3.1 Logo 核心概念

Logo 由两个核心元素组成：

```text
一条远方地平线
三道流动澜线
```

含义：

- 地平线：视野、远见、大局判断。
- 三道澜线：变化、流动、商业趋势。
- 横向舒展：方向、势能、未来感。

Logo 不加入信号点、雷达圈、眼睛、箭头、太阳、复杂山峰或栏目符号。

### 3.2 标准版本

| 版本 | 结构 | 适用场景 |
|---|---|---|
| 横版 Logo | `[澜线图形] 观澜 AI / WAVESIGHT AI` | 官网导航、报告页眉、名片、邮件签名、后台系统 |
| 竖版 Logo | `澜线图形 / 观澜 AI / WAVESIGHT AI` | 内参封面、品牌海报、PPT 首页、报告封面 |
| 图标版 | 只保留地平线和澜线 | favicon、App icon、头像、角标、小尺寸按钮 |
| 字标版 | `观澜 AI / WAVESIGHT AI` | 正式文件、法律文本、极简导航、品牌落款 |

横版 Logo 是网站默认主版本。竖版 Logo 适合高端和仪式感场景。图标版小尺寸时可以简化为一条地平线加两到三道澜线，不强行塞入文字。

### 3.3 Logo 颜色

正色版本：

| 元素 | 颜色 |
|---|---|
| 中文字标 | 墨海蓝 `#071827` |
| 英文字标 | 墨海蓝 `#071827` |
| 主澜线 | 深澜蓝 `#0D355C` |
| 辅助澜线 | 雾蓝灰 `#6F7F8F` |
| 地平线 | 香槟金 `#C8A766` |
| 背景 | 云白 `#F7F4EF` 或纸白 `#FFFFFF` |

深色背景版本：

| 元素 | 颜色 |
|---|---|
| 字标 | 云白 `#F7F4EF` |
| 主澜线 | 云白或浅雾灰 |
| 辅助澜线 | 蓝灰 |
| 地平线 | 香槟金 `#C8A766` |
| 背景 | 墨海蓝 `#071827` 或深海蓝 |

单色版本仅用于低成本印刷、合同页脚、极简页面或无法呈现品牌色的场景，可使用墨海蓝、黑色或白色反白。

### 3.4 Logo 比例、安全区与最小尺寸

以澜线图形高度为 `H`：

| 项目 | 建议 |
|---|---|
| 图形宽度 | `2.8H-3.2H` |
| 金色弧线宽度 | `2.9H-3.3H` |
| 中文字标高度 | `0.8H-1H` |
| 英文字标高度 | `0.18H-0.24H` |
| 图形与文字间距 | `0.45H-0.6H` |

安全区：

```text
Logo 四周至少保留中文字标高度的 1/2，或英文大写字母高度的 3 倍。
```

网页最小尺寸：

| 类型 | 最小尺寸 |
|---|---:|
| 横版 Logo | 宽度 120px |
| 竖版 Logo | 宽度 96px |
| 图标版 | 24px x 24px |
| favicon | 32px / 48px / 64px |

### 3.5 Logo 禁止用法

禁止：

- 拉伸或压缩 Logo。
- 改变澜线数量。
- 去掉金色地平线后仍称为主 Logo。
- 添加圆点、雷达圈、眼睛、箭头、太阳、山峰或复杂渐变。
- 添加发光特效、复杂阴影或彩虹渐变。
- 将 Logo 放在杂乱图片中央。
- 随意替换中文字标字体。
- 将英文放得过大。

主 Logo 必须稳定，不因 Signal、Point、Opportunity、Trend 或 Brief 栏目不同而改变图形结构。

| 雾蓝灰 | `#6F7F8F` | 辅助文字、说明、图表辅助线 |

```text
云白 / 纸白：70%
墨海蓝 / 墨黑：18%
雾灰 / 远山灰：8%
香槟金：2%-4%
```

香槟金只做点睛，不做主色。它可以出现在细线、小标签、高亮点、会员标识、章节编号和按钮 hover 中，不得大面积铺开。

| 用途 | 颜色 | 规则 |
|---|---:|---|
| Signal | `#2F6F9F` 或深澜蓝 | 用于轻微波纹、变化标识、信息变化 |
| Point | `#1A2633` 或墨海蓝 | 用于深度判断、关键观点 |
| Opportunity | `#B88A42` 或香槟金 | 用于机会切口、会员和商业价值点 |
| Trend | `#627386` 或雾蓝灰 | 用于长期曲线、时间轴、趋势辅助 |
| Positive | `#2F8C57` | 低饱和正向变化 |
| Risk | `#B85C5C` | 低饱和风险变化 |

禁止使用彩虹色图表、大红大绿涨跌、霓虹蓝、紫蓝渐变或高饱和点阵。

## 5. 字体系统

### 5.1 品牌字标

中文 Logo 字标应使用定制化现代宋体方向，不直接使用普通系统宋体。气质应是：

```text
现代宋体、高对比、稳重、有东方智识感。
```

英文字标统一为：

```text
WAVESIGHT AI
```

英文使用全大写，字距拉开，建议 `letter-spacing: 0.12em-0.18em`。正式品牌英文不写作 `WaveSight AI`、`WavesightAI` 或 `WAVE SIGHT AI`。

## 6. 图形语言与图标系统

### 6.1 辅助图形

观澜 AI 的辅助图形只使用三类：

```text
澜线、地平线、细分割线。
```

澜线规则：

- 线条不超过 5 条。
- 透明度建议 `4%-18%`，按场景控制。
- 不抢正文，不做复杂波纹，不做动态炫光。
- 可用于首页 Hero 背景、内参封面、报告页脚、卡片角落、会员页背景。

地平线规则：

- 线条高度 `1px-2px`。
- 长度 `32px-96px`。
- 颜色为香槟金，透明度 `80%-100%`。
- 可用于标题下划线、章节分割、重点判断卡顶部、按钮 hover。

分割线规则：

- 默认 `1px` 实线。
- 使用雾灰或远山灰。
- 必要时中心加入一小段香槟金细线。
- 不使用粗线、彩色分割条或复杂边框。

### 6.2 图标

图标统一采用：

```text
线性图标、1.5px stroke、圆角端点、24px 网格、不填充、不渐变。
```

栏目图标方向：

| 栏目 | 图标方向 | 气质 |
|---|---|---|
| Signal | 轻微波纹 / 小型脉冲线 | 信息变化 |
| Point | 聚焦点 / 单点圆环 | 深度判断 |
| Opportunity | 切口 / 开口圆 / 小箭角 | 行动机会 |
| Trend | 延展曲线 / 长线走势 | 长期趋势 |
| Brief | 文件页 / 细线封面 | 内参文件 |

禁止使用机器人、芯片、火箭、闪电、大箭头、眼睛、复杂数据网络或强 3D 图标。

## 7. 图片与插画

推荐图片方向：

```text
远山、湖面、海面、雾气、地平线、抽象流线、低饱和商业场景、深色纸张质感。
```

图片处理：

- 降低饱和度。
- 提高灰度和蓝灰调。
- 避免高对比、鲜艳人物和商业图库感。
- 可叠加墨海蓝 `8%-16%` 或云白 `10%-20%` 品牌滤镜。

禁止图片方向：

```text
机器人脸、AI 大脑、芯片特写、赛博城市、蓝紫霓虹、握手商务照、夸张 3D 数据球、股票 K 线屏、密集代码背景。
```

## 8. 网站页面系统

### 8.1 总体气质

网站应像：

```text
在线商业内参、AI 趋势判断系统、企业决策参考平台。
```

不要像：

```text
资讯门户、博客站、AI 工具导航、普通 SaaS 官网、流量媒体站。
```

页面核心原则：

```text
先判断，再解释；先结构，再内容；先克制，再强调。
```

桌面导航：


当前栏目状态：

- 文字加深。

移动导航：

- 英文可隐藏。
- 右侧为搜索或菜单按钮。

### 8.4 首页母版

推荐结构：

```text
顶部导航
Hero：观 AI 之澜，识商业之势
今日精选 Signal
深度 Point
机会 Opportunity
长期 Trend
内参 / 会员入口
页脚
```

Hero 方向：

- 云白背景。
- 极淡远山 / 水面 / 澜线。
- 大号中文标题。
- 小号英文副标题。
- 一个主按钮和一个次按钮。

推荐主标题：

```text
在这里看懂变化，
判断影响，找到行动点
```

推荐英文：

```text
See change. Assess impact. Find the right action.
```

Hero 不放过多产品截图，不做卡片堆叠。第一印象应是判断力，而不是功能展示。

### 8.5 栏目页母版

统一结构：

```text
页面标题
一句克制说明
筛选 / 日期 / 搜索
主内容列表或卡组
必要时出现侧栏摘要
```

栏目方向：

| 栏目 | 标题建议 | 视觉重点 |
|---|---|---|

| Point | 深度 Point | 更大标题、更强摘要、研究文章感 |
| Opportunity | 机会 Opportunity | 可验证、可行动、低销售感 |
| Trend | 长期 Trend | 时间轴、档案感、长期方向 |
| Brief | 商业内参 | 文件感、会员感、判断密度 |

栏目页可以有说明文字，但文案必须服务判断，不写内部页面说明。

### 8.6 详情页母版

桌面端建议三栏：

```text
左侧目录：180px
正文内容：720px-780px
右侧决策栏：260px-300px
```

移动端改为单栏：

```text
Logo + 菜单
标题区
核心判断卡
内容模块
行动提示
相关内容
会员 CTA
```

移动端不保留左右栏；目录可变为横向吸顶锚点。

## 9. 内参与热力图视觉方向

### 9.1 内参定位

AI 商业内参是 V2 可能成立的增值产品层，视觉上应更像周期性决策文件，而不是普通栏目页或资讯文章。

推荐首屏：

```text
观澜 AI 商业内参
GUANLAN / WAVESIGHT AI BUSINESS BRIEF

Vol.012 ｜ 2026.05.06 ｜ Member Brief

AI Agent 正在从效率工具进入企业流程重构阶段

一句话判断：
真正值得关注的，不是员工会不会用 AI，
而是哪些岗位、流程和客户触点会被 AI 重新分配。
```

### 9.2 内参详情页模块

统一模块顺序建议：

```text
Overview
Signal
Point
Trend
Opportunity
Action Note
```

右侧决策栏内容：

- 信号强度。
- 商业价值。
- 落地难度。
- 适合对象。
- 一句话总结。
- 相关标签。
- 阅读时长。
- 下载 PDF。

右侧栏不放广告，不堆太多相关推荐。

### 9.3 热力图方向

AI 商业热力图不得做成普通榜单或泛仪表盘。它应更像一张“商业变化的判断地图”。

推荐图表形式：

| 场景 | 推荐形式 | 禁止 |
---|---|---|
| 行业热度 | 矩阵图 / 卡片矩阵 | 彩虹热力表、排名榜 |
| 岗位影响 | 四象限 / 层级矩阵 | 强销售化效率榜 |
| 流程变化 | 流程线 / 地平线式阶段图 | 后台流程图堆叠 |
| 趋势延续 | 时间轴 / 长线曲线 | 金融 K 线和交易屏 |
| 反证与争议 | 双栏判断框 | 红绿对打式评分牌 |

热力图颜色：

- 主线：深澜蓝。
- 辅助线：雾蓝灰。
- 重点点位：香槟金。
- 风险项：低饱和红。
- 正向项：低饱和绿。
- 背景网格：浅灰。

## 11. 商业内参符号与元素

本节吸收用户补充的商业内参符号与元素规范。它适用于 `Brief / AI 商业内参`、专题报告、热力图详情、会员 PDF、PPT 和需要强化“文件感”的页面。

### 11.1 符号系统

| 符号 | 英文 | 含义 | 使用方式 |
|---|---|---|---|
| 信号 | Signal | 识别重要信号，提示变化或关注点 | 小型波纹或脉冲线，不使用雷达圈 |
| 关键判断 | Point | 聚焦核心结论，帮助快速定位重点 | 圆点 / 圆环 / 聚焦线，配合判断块 |
| 趋势 | Trend | 表示发展方向与趋势变化 | 克制上行折线或长线，不金融化 |
| 机会 | Opportunity | 识别潜在机会，强调探索与可能性 | 开口线框或切口符号，不做箭头冲刺 |
| 引述 | Quote | 引用观点或原文，增强权威与可信度 | 双引号符号 + 左侧金线 |
| 分隔线 | Divider | 划分信息结构，提升阅读清晰度 | 细线，多线时保持低对比 |
| 时间节点 | Timeline Node | 标记关键时间点，构建时间脉络 | 细线 + 小圆点 |
| 数据快照 | Data Snapshot | 呈现关键数据指标，辅助判断 | 小柱图 / 折线，不做仪表盘 |
| 评级标签 | Rating Chip | 表达评价结果，便于横向比较 | 胶囊形态，金色只点睛 |
| 来源说明 | Source Note | 标注信息来源，保证信息透明 | 页脚或卡片底部的小号说明 |

符号原则：

- 使用克制的线性符号和结构化元素。
- 保持一致的视觉语言与间距规范。
- 采用低饱和色彩与高可读文字。
- 聚焦关键信息，精简层级。
- 以数据与判断为核心驱动内容表达。

禁止：

- 使用过多或风格混杂的图标。
- 使用高饱和色彩和强烈装饰效果。
- 堆砌装饰性元素或复杂背景。
- 层级混乱、重点不清晰。
- 过度科技感或炫技式视觉表达。

### 11.2 信息元素

| 元素 | 结构 | 用途 |
|---|---|---|
| 期号编号 | `No. 042` | 内参期号、报告编号、归档索引 |
| 日期 | `2026.05.20` | 发布日、数据截止日、观察窗口 |
| 会员内参标签 | `会员内参 / Member Brief` | 强化身份与内容层级 |
| 阅读时长 | `8 分钟` | 预估阅读成本，帮助选择阅读深度 |
| 核心判断框 | 图标 + 标题 + 一句话判断 | 正文前置结论 |
| 引述框 | 引号 + 引文 + 作者 | 增强观点可信度 |
| 数据卡 | 指标 + 数字 + 变化 + 小趋势线 | 快速呈现商业证据 |
| 趋势时间线 | 年份 / 阶段 / 节点 | 呈现长期演化 |
| 机会矩阵 | 影响力 / 确定性 | 结构化比较机会优先级 |
| 数据来源脚注 | 来源 + 截止日期 | 保持信息透明 |

### 11.3 内参版式原则

| 版式 | 结构 | 使用场景 |
|---|---|---|
| 封面页 | Logo、标题、期号、日期、淡澜线或远山图 | PDF / PPT / 年度报告 |
| 正文详情页 | 左侧阅读主栏，右侧结构化补充信息 | 内参详情、深度报告 |
| 侧边决策栏 | 关键判断、机会、数据、来源 | 桌面端详情页 |
| 三栏摘要页 | 三个核心结论或机会卡 | 周报 / 月报摘要 |
| 报告章节页 | 大标题、淡图、细分割线 | PPT / PDF 章节过渡 |

内参页面不能像后台数据表，也不能像普通文章列表。每一页应先给判断，再给证据和解释。

## 12. 动效与交互规范

本节吸收用户补充的商业内参与网站动效规范。动效应该像水面微动：轻、慢、稳，不喧宾夺主。

### 12.1 动效原则

| 原则 | 含义 |
|---|---|
| 克制内敛 | 轻量微动，不抢内容，降低视觉负担 |
| 目的明确 | 每一次动效都有清晰的信息意义 |
| 从容沉稳 | 节奏不急不躁，传递专业与可信赖 |
| 清晰可读 | 动效服务信息呈现，不影响阅读与理解 |
| 高端质感 | 细节精致、过渡自然，提升品牌气质 |

### 12.2 基础动效参数

| Token | 建议 |
|---|---|
| 快速反馈 | `120ms` |
| 常规过渡 | `180ms` |
| 内容显现 | `240ms` |
| 结构切换 | `320ms` |
| Easing | `ease-out` 或 `cubic-bezier(0.22, 1, 0.36, 1)` |
| Delay | `0-120ms`，只用于分段入场 |
| Hover Lift | `+2px` |
| Opacity Fade | `0 -> 100%` 或 `100% -> 0` |
| Soft Blur | `0 -> 8px`，只用于背景或大面积过渡 |

### 12.3 常用动效

| 动效 | 用法 | 推荐参数 | 禁止 |
|---|---|---|---|
| 导航下划线滑动 | 指示当前导航和切换位置 | underline slide `160ms / ease-out` | 使用粗重色块或大面积背景 |
| 按钮悬浮反馈 | 可点击元素反馈 | hover `+2px / 180ms / ease-out` | 放大、弹跳、旋转 |
| 卡片悬浮升起 | 提升可点击性与层级感 | lift `+2px / 180ms / shadow 8%` | 过度放大或剧烈阴影 |
| 滚动进入显现 | 内容随阅读进入视野 | fade-in + slight up / `240ms` | 大面积同时播放或延迟过长 |
| 澜线缓动 | 品牌背景或普通装饰 | subtle drift / `12s-16s` 循环 | 剧烈波动或发光 |
| 图表线条绘制 | 展示趋势数据出现 | draw-in `600ms-900ms` | 让线条反复播放 |
| 数字递增 | 数据指标展示 | count-up `800ms` | 快速跳动或无限循环 |
| 标签页切换 | 状态或分类切换 | underline slide `160ms` + fade `200ms` | 内容闪烁或强缩放 |
| 手风琴展开 | FAQ / 目录 / 摘要展开 | expand / collapse `240ms` | 过快导致内容跳动 |
| 弹窗淡入淡出 | 订阅、确认、分享 | fade-in/out `240ms` | 弹跳、旋转、遮挡阅读 |

### 12.4 内参阅读体验动效

| 场景 | 动效 |
|---|---|
| 阅读进度 | 顶部细线随阅读推进 |
| 锚点高亮 | 目录当前项轻微加深或左线变金 |
| 右侧导读栏 | 吸顶时保持稳定，不抖动 |
| 引用块显现 | fade-in + slight up / `240ms` |
| 数据卡聚焦 | hover lift + subtle shadow |
| 来源说明 | hover 展开或 tooltip，但不遮挡正文 |

### 12.5 禁用动效

禁止：

- 霓虹闪烁或高饱和光效。
- 弹跳效果。
- 夸张视差和多层深度位移。
- 噪点循环动画。
- 强制自动播放视频或持续动效。
- 让关键内容在动画结束前不可读。

## 13. 文案语气边界

品牌主张：

```text
观 AI 之澜，识商业之势
```

核心价值句：

```text
在这里看懂变化，判断影响，找到行动点
```

英文辅助：

```text
See change. Assess impact. Find the right action.
```

文案应像：

```text
商业判断、内参摘要、战略备忘录、投研观点。
```

不要像：

```text
标题党、营销软文、科技公关稿、热点追踪号。
```

推荐用词：

```text
正在发生、值得关注、短期影响、长期趋势、可验证机会、需要警惕、更适合观察、可以小规模试点。
```

避免用词：

```text
颠覆、革命、赋能、重磅、爆发、遥遥领先、史诗级、一文看懂、彻底改变、超级风口、万亿赛道、一夜爆火。
```

注意：`行动点`、`行动建议` 可作为 V2 品牌价值线和内参输出维度出现，但不能写成替用户下最终经营、投资或合作判断。推荐使用 `可验证行动点`、`小规模试点`、`观察切口` 等克制表达。
