# V2.0 Transition Charter

日期：2026-05-07  
状态：draft-for-v2-planning  
owner：`strategy` / `pm`

## 1. V2.0 目标

把观澜AI从“AI 商业信号网站 + 工作流”升级为“AI 机会判断系统”。

V2.0 是正式产品升级项目，不是测试项目。它会实际更新 VI、内容体系、算法、Agent 体系、内容源与监测规则。测试页、test-only 管线和隔离 worktree 只用于降低迁移风险，不代表 V2.0 只在测试环境中完成。

V2.0 的核心不是更多页面，而是更高质量的判断链路：

```text
更多来源 -> 更强去重 -> 更准 Signal -> 更稳 Judgment -> 更少但更深 Opportunity -> 可积累 Trend
```

## 2. 目标用户

- 需要判断 AI 对业务影响的创始人、投资人、产品负责人和业务负责人。
- 需要从海外 AI 变化中识别可迁移商业机会的人。
- 需要高信噪比 AI 判断，而不是新闻流、工具导航或观点堆叠的人。

## 3. V2.0 核心差异

| 方向 | V1.0 | V2.0 |
|---|---|---|
| Agent 体系 | 长期岗位 + 调度中枢 | 新判断流水线 + 长期岗位治理 |
| 算法 | Signal / Priority / Trend 基础解析 | 多源采集、二次搜索、反证、迁移判断、机会深挖 |
| 内容源 | 日常雷达与手工补充 | 来源分层、可信度、X/LinkedIn/Product Hunt/YC/VC/Builder |
| VI | 商业情报桌面方向 | 可长期识别的品牌视觉系统 |
| 网站 | 栏目呈现 | 判断产品与内容资产入口 |
| 自动化 | 生产日更 + test-only v2 管线 | 以正式替换为目标，先分阶段验证，再迁入生产链路 |

## 4. V2.0 非目标

V2.0 不做：

- 泛 AI 新闻站。
- AI 工具导航。
- 公司排行榜或项目黄页。
- 开发者社区。
- 通用聊天助手。
- 未经 PM 门禁的新增前台栏目。
- 未经过验收的生产自动化直接替换。

## 5. 关键原则

- 少即是多：前台展示宁少但强，不追求数量。
- 判断优先：每条内容必须服务商业判断。
- 可追溯：来源、分析、反证、迁移判断必须可追踪。
- 可积累：Signal、Opportunity、Trend 不能只是当天内容，应进入长期资产。
- 分层上线：V2 是正式升级，但必须先经过规划、实现、QA、用户验收和迁移闸门，再替代 V1 生产页面或自动化链路。

## 6. 正式升级口径

V2.0 的目标是实际落地，不是停留在规划或测试环境。

正式升级范围包括：

- 新 Agent 体系。
- 新算法与 HeatEvidence / AI商业热力图数据模型。
- 新 VI 与页面母版。
- 更多内容源与监测规则。
- AI商业内参与增值产品结构。
- 必要的生产页面、内容入库、同步脚本和自动化链路迁移。

执行方式：

```text
规划确认 -> 分支 / worktree 隔离开发 -> 证据化 QA -> 用户验收 -> 合并 / 部署 / 生产替换
```

限制：

- 在用户验收前，不直接覆盖 V1.0 稳定版本。
- 任何影响生产自动化、内容源字段、同步脚本或前台导航的任务，都必须单独派发并通过对应硬闸门。
