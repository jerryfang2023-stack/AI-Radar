# V2 Copy / Editorial System

## 0. 文案总原则

V2 文案要像商业内参摘要：有判断、有证据边界、简洁克制。公开前台只说明信号、事实、来源和观察边界，不以说服别人为目标，不替用户下经营、投资、合作或采购判断。

固定表达：

```text
观澜AI｜WaveSight AI
观 AI 之澜，识商业之势
```

## 1. 栏目命名与一句话价值

| 栏目 | 一句话价值表达 | 禁止 |
|---|---|---|
| 今日要点 | 今天值得看的 AI 商业变化，以及它们为什么值得继续观察。 | 新闻流、日报列表、自动摘要 |
| 关键信号 | 从外部变化中筛出足以进入判断资产的商业信号。 | 公司 PR、低价值新闻、强证据 |
| 机会解码 | 对证据足够的方向做低频、克制的机会分析。 | 落地方案、确定机会、公司名标题 |
| 商业内参 | 把一段时间内的信号、观点和趋势压缩成商业判断。 | 榜单、公开热力排名、投资建议 |
| 观点校准 | 一线观点如何支持、质疑或修正当前判断。 | 独立观点频道、观点热榜 |
| 趋势背景 | 相关证据是否持续积累，变化正在升温、降温还是进入争议。 | 趋势词墙、泛图表 |

## 2. Home 文案框架

H1 推荐：

```text
观 AI 之澜，识商业之势
```

副标题推荐：

```text
每天从 AI 变化中筛出关键信号，判断影响，沉淀值得继续观察的机会方向。
```

CTA：

- `查看今日要点`
- `阅读机会解码`
- `了解商业内参`

禁用：`立即行动`、`抢占红利`、`财富密码`、`确定机会`。

## 3. Signal 标题 / 摘要 / 6 维表达

Signal 标题必须是：

```text
事件 + 商业含义
```

摘要必须回答：

- 发生了什么。
- 为什么不是普通新闻。
- 它可能影响什么行业、岗位或流程。
- 当前证据和反证在哪里。

机会解析 6 维用语：

| 维度 | 推荐标题 |
|---|---|
| 解决什么具体问题 | 具体问题 |
| 谁最先感受到这个问题 | 首要感受者 |
| 改变了哪段流程 | 流程变化 |
| 价值从哪里来 | 价值来源 |
| 触发信号是什么 | 触发信号 |
| 成立边界是什么 | 成立边界 |

禁用表达：`强证据`、`机会确定`、`下一步验证`、`落地动作`。

## 4. 机会解码报告模板

标题规则：

- 标题写方向，不写公司名。
- 公司只能作为代表事件、案例、来源或反证出现。

报告结构：

1. 一句话机会判断。
2. 发生了什么变化。
3. 解决什么具体问题。
4. 谁最先感受到这个问题。
5. 改变了哪段流程。
6. 价值从哪里来。
7. 触发信号是什么。
8. 趋势背景与证据积累。
9. 观点校准。
10. 反证、限制与不确定性。
11. 成立边界。
12. 建议关注变量。

推荐语气：

- `这个方向值得继续观察。`
- `证据正在积累，但仍需要看客户采用是否持续。`
- `它更像早期变化，而不是已经完成验证的机会。`

禁止：

- `建议立即进入`
- `确定成立`
- `必须布局`
- `落地方案如下`
- `保证收益`

## 5. 商业内参表达边界

公开用户可见：

- 最新一期标题。
- 期号和周期。
- 1-2 条核心判断摘要。
- 热力图样张或高层摘要。
- 会员边界说明。

登录用户可见：

- 目录。
- 部分试读。
- 历史期入口。

会员可见：

- 全文。
- 热力图。
- 三元组证据展开。
- 反证与观察变量。

会员引导文案推荐：

- `查看完整内参，继续阅读本期热力变化和证据来源。`
- `完整内容包含高热三元组、证据摘要和反证边界。`

避免：

- `升级后获得确定机会`
- `马上掌握风口`
- `完整热力榜单`

## 6. 禁用语表

内部语言禁用：

- 后台
- 字段
- JSON
- 同步
- 自动化
- 入库
- 解析脚本
- validator
- content-paths
- 编辑 / 恢复

确定性承诺禁用：

- 确定机会
- 强证据
- 必须
- 立即行动
- 必投
- 保证收益
- All in
- 下一步落地

营销空话禁用：

- 赋能
- 生态闭环
- 一站式
- 颠覆未来
- 风口红利
- 财富密码
- 爆发式增长

## 7. QA 文案检查表

| 检查项 | 通过标准 |
|---|---|
| 是否言之有物 | 每句话提供事实、判断或边界 |
| 是否克制 | 不煽动、不推销、不替用户决策 |
| 是否无内部话术 | 普通前台不出现后台、字段、同步等词 |
| 标题是否可显示 | 标题不因过长破坏布局 |
| Opportunity 标题 | 不以公司名命名 |
| Signal 标题 | 是事件 + 商业含义 |
| 会员引导 | 说明内容边界，不承诺商业结果 |

## 8. Dev 交接

页面实现时，关键文案由本文件或后续 Copy 表提供。Dev 不得自行补写涉及商业承诺、会员权益、机会判断、反证边界的关键句。
