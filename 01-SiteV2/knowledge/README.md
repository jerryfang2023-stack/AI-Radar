# WaveSight Knowledge｜观澜 AI 判断资产主库

本目录是观澜 AI 的长期判断资产库，服务 Obsidian / 本地知识库使用。

它不是每日 Raw 生产漏斗，也不是网站发布目录。每日监测、候选池、变化簇、自动生成过程文件和前台文章写入 `01-SiteV2/content/`；本目录只保留可复用、可追溯、可复核、可关联的长期知识卡。

## 当前结构

| 目录 | 用途 | 保留标准 |
|---|---|---|
| `00-MOC/` | 关系索引、主题索引、周期索引 | 只保留当前有用索引，不保留过期日索引 |
| `01-Signal-Cards/` | 长期商业信号资产 | product_service / funding / case；必须有事实证据和前台文案门禁 |
| `02-Opinion-Cards/` | 前沿观点资产 | 必须有原文 / 摘录、出处、事实边界和四档评级 |
| `03-Asset-Candidates/` | 后台判断候选 | 变化、场景、趋势候选；未成熟前不直接前台展示 |
| `04-Publication-Index/` | 发布索引 | 今日观察、趋势报告、商业内参等前台内容索引 |
| `10-Templates/` | 模板 | Obsidian 卡片模板 |

## 不进入 knowledge 的内容

- 每日 Raw 全量。
- Pool 候选。
- 自动生成的变化簇候选过程文件。
- 没有关联资产的社交观点。
- 只有标题、摘要或“暂无公开信息”的卡片。
- 重复日期版本。
- 调试、过程、临时、低质量或无效文件。

这些内容应留在 `content/` 的生产过程层；不具备长期价值时直接删除。

## 卡片保留原则

### 商业信号卡

商业信号卡记录可被证据支撑的显性事实。前期只保留三种主类型：

- `product_service`：新产品、新服务或新能力。
- `funding`：融资与资本押注方向。
- `case`：真实客户、行业场景、岗位流程或业务工作流里的 AI 使用。

同一事件只保留一张稳定卡。后续更新写入同一张卡，不按日期重复生成。合作、采购、定价、风险等只作为变量或观察理由，不作为前期主类型。

### 前沿观点卡

观点卡记录“谁在何时何处说了什么”。它不是二手中文摘要，短观点优先保留完整原文，长观点保留关键原文摘录，并必须保留原文出处链接。

观点卡不作为公司事实主证据，只作为前沿判断资产、判断触发器和行业预期信号。观点中的公司动作、客户采用、融资、收入或市场规模必须另有事实来源支撑。

观点卡必须执行四档评级：

| 档位 | 展示位置 | 保留规则 |
|---|---|---|
| `feature` | 今日观察主推观点 | 高价值人物 / 机构，观点完整，直接支撑当日主线 |
| `sidebar` | 商业信号页观点模块 / 侧栏 | 有判断价值，能作为商业信号参照 |
| `archive` | 知识库归档 | 有保留价值，但信息量、上下文或关联度不足 |
| `discard` | 隐藏或后续清理 | 玩笑、祝贺、纯转发、纯营销、无上下文、无判断价值 |

只有 `feature` 和 `sidebar` 可以进入前台；`archive` 与 `discard` 不得因存在 `frontend` 字段而被同步。

### 后台判断候选

后台候选包括：

- `change`：变化候选。单一材料不能直接生成正式变化判断，必须积累多条商业信号、前后对比、商业变量和风险边界。
- `scene`：场景候选。必须写清行业 / 部门、岗位 / 使用者、具体流程 / 任务，以及 AI 替代、增强或重组了哪一个步骤。
- `trend`：趋势候选。必须由多个变化和场景、多类来源共同支撑，并写清信息缺口或后续观察变量；时间窗口可按内容需要使用，但不作为硬门槛。

后台候选不是普通前台卡片。变化成熟后优先通过 `content/04-business-signals/change-topics/` 的变化短专题表达；趋势成熟后进入趋势追踪、热力图或商业内参。

## 命名规则

```text
YYYY-MM-DD--type--slug.md
```

示例：

```text
2026-05-20--change--agent-permission-control.md
2026-05-20--case--collibra-ai-command-center.md
2026-05-20--frontier-opinion--aaron-levie-agent-control.md
2026-05-20--trend--enterprise-agent-governance.md
```

## 清理规则

- 空目录直接删除。
- 过期索引直接删除。
- 同名或同事件跨日重复卡只保留质量最高的一张。
- 无关联、无身份、无上下文、无原文的观点卡降为 `discard` 或直接删除。
- 变化簇候选留在 `content/`，不进入 knowledge。
- 不设置长期归档目录；无效内容不搬家，直接清理。

## 与 content / site 的关系

```text
content/ 生产候选和文章
knowledge/ 长期判断资产
site/ 前台展示
```

`knowledge/` 只沉淀判断资产，不承担发布页面和每日生产过程。
