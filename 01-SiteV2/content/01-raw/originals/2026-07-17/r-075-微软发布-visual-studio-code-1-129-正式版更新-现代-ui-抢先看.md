现代 UI 抢先看：微软发布 Visual Studio Code 1.129 正式版更新，AI Agent 能力加强
2026/7/16 13:42:15
来源： IT之家
作者： 问舟
责编： 问舟
评论：
感谢IT之家网友 带带大师弟 的线索投递！
IT之家 7 月 16 日消息，微软今日发布了 Visual Studio Code 1.129 版本。新版本将采用分阶段方式向稳定版用户推送，用户也可以通过“检查更新”或前往官网下载安装最新版本。
相比以往节奏较快的周度更新，本次版本更新除了继续完善 AI Agent 能力外，还首次提供了现代化 UI 界面预览，并对 Agents 窗口、聊天交互和编辑体验进行了多项调整，例如 Agents 窗口中的新编辑器面板、支持用“!”前缀直接运行终端命令。
此次最受关注的变化之一是 现代化用户界面 。该功能目前仍处于实验预览阶段，用户可在设置中启用 workbench.experimental.modernUI （或搜索“Modern UI”）进行体验。
启用后，VS Code 工作台整体视觉风格将发生变化，包括左右侧边栏及底部面板改为带有圆角和留白的悬浮卡片样式，同时刷新整体界面设计语言，与新版 Agents 窗口保持一致。微软表示，该功能在 Insiders 预览版本中默认开启，未来还将根据反馈继续完善。
AI 智能体相关功能依然是微软的重点。VS Code 1.129 引入了全新的 Agent Host 架构，将 Copilot、Claude、Codex 等智能体运行在独立进程中。
借助这一架构，同一个 Agent 会话可以同时在多个 VS Code 窗口中访问和展示，并为后续更多 AI 功能提供统一基础。微软介绍，Copilot Agent 已基于 Copilot SDK 构建，使其行为与 GitHub Copilot CLI、独立 Copilot 应用等产品保持一致。
与此同时， Agents 窗口 新增了一项实验性编辑器面板。此前，开发者需要分别查看聊天内容和 Agent 生成的文件、差异内容；新版则将编辑器与详情区域整合到同一个停靠面板，共享标签页，开发者可以直接在聊天旁查看文件、Diff 以及修改结果，并支持标签管理、内联与并排 Diff 切换、恢复上一次会话状态等功能，减少在多个面板之间切换的操作。
聊天功能方面，VS Code 1.129 新增了一个快捷交互方式：用户只需在聊天输入内容前添加 “!” 前缀，即可将该内容直接作为终端命令执行，无需手动切换到终端窗口。这项能力适用于基于 Agent Host 的编辑器会话和 Agents 窗口，可进一步简化开发过程中执行命令的流程。
针对使用多个 AI 会话的开发者，微软还加入了一系列 会话管理工具 。运行于 Agent Host 的 Copilot、Claude、Codex 等智能体现在可以列出已有会话、读取其他会话上下文、创建新的会话或聊天，并在用户确认后向其他会话发送消息，以便拆分子任务、协同完成开发工作。同时，Agents 窗口的新建会话流程也进行了优化，包括记住上一次的默认配置、新增 Git Worktree 独立工作目录选项等。
除 AI 功能外，本次更新还改进了编辑器体验。例如，当文件支持多个编辑器时，用户可以直接通过工具栏中的 Reopen Editor With 菜单切换不同编辑器，而无需再通过命令面板完成操作。此外，企业版 GitHub 用户现在也能够在 Agent Host 中直接登录并使用 GitHub Enterprise 提供的 Copilot 服务，进一步完善企业开发场景支持。
