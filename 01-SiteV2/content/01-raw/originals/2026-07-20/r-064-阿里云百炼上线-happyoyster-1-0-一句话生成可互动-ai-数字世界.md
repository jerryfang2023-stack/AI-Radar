# 阿里云百炼上线 HappyOyster 1.0，一句话生成可互动 AI 数字世界

## clean_text

IT之家 7 月 19 日消息，今日，阿里云百炼正式上线可实时构建和交互的开放式世界模型产品 HappyOyster 1.0 （快乐生蚝 1.0），更多消息现已公布。
据官方介绍，该模型深度学习物理世界状态转移规律，能主动推演从动作到反馈的因果链，并保持人物和环境的长程一致性。HappyOyster 1.0 具备世界探索（Adventure）和实时导演（Directing）两大模式， 用户输入一句话或一张图即可生成可互动、可探索的 AI 数字世界 。目前已提供 Android / iOS / Web 三端 SDK，在阿里云百炼开启灰测。
文生视频模型的学习目标是文本到视频的单向映射，而 HappyOyster 1.0 的学习目标是从“当前状态 / 动作”到“下一状态 / 动作”的转移规律， 因此支持用户在任意节点介入并改变数字世界走向 。相较于大语言模型日趋成熟的架构与范式，世界模型仍属于新兴赛道，而 HappyOyster 1.0 已在探索该赛道的前沿，它从海量自然视频中学习现实里人物、事物和环境的演变，自主推演从动作到反馈的因果链，从而能自然迁移到没见过的场景，根据用户指令推理出新的因果，并维持数字世界的长程稳定和音视频的长时协同。
世界探索模式：支持 1 分钟连续实时交互
用户输入文字或上传图片即可生成可探索的数字世界，通过前冲、下蹲、跳跃、攻击等操控按键丰富人物运动轨迹，支持载具驾驶与武器战斗等物理交互。世界探索模式支持长达 1 分钟的连续实时位移与镜头控制，生成内容可导出原片。
实时导演模式：支持暂停、回溯与分支叙事
实时导演模式下，用户可在任意节点暂停，回跳至剧情关键节点，输入新提示词改变剧情走向。支持生成 3 分钟以上的 480p 或 720p 实时画面，生成内容可导出原片。
阿里云方面称，HappyOyster 1.0 正在探索下一代多模态的互动内容形态。 在交互式游戏领域 ，开发者通过上传图片和输入提示词，可快速生成开放世界原型、角色交互和战斗场景。 在互动短剧和虚拟陪伴场景中 ，用户通过自然语言指令构建人物和剧情，并可在任意节点改写走向。在文旅领域，可为游客提供沉浸式探索体验。
目前，开发者服务端通过 Open API 管理世界，客户端 SDK 封装了 RTC 连接、视频渲染和实时交互，支持 Adventure 与 Directing 两种模式，体验结束后可导出视频。产品支持企业用户和开发者通过 SDK 快速构建可玩世界、互动剧、AI 陪伴等沉浸式应用。
