# 微软 GitHub Copilot 接入首个开源模型 Kimi K2.7 Code

## clean_text

IT之家 7 月 3 日消息，月之暗面 Kimi 今日宣布，GitHub Copilot 正式接入其开源模型 Kimi K2.7 Code，这是 GitHub Copilot 首次接入开源模型 。
IT之家获悉，Kimi K2.7 Code 由 GitHub 托管于微软 Azure 平台，采用 按量计费 模式，正逐步向 Copilot Pro、Pro+ 以及 Max 订阅方案开放。
GitHub 官方表示，此次推送将分批逐步进行，会持续监测该模型的质量与性能。在接下来的几周内，将把支持范围扩大至 Copilot Business、Enterprise 版本及其他应用平台。
据IT之家此前报道，6 月 12 日， 月之暗面 Kimi 发布并开源 Kimi K2.7 Code 编程模型 。
月之暗面表示，内外部基准评估显示，Kimi K2.7 Code 相比 K2.6 模型显著提升了长上下文编程场景的指令遵循能力、长程编程任务的性能表现，并且大幅改善了在长程任务中的过度思考倾向， 平均 token 消耗减少 30% 。

## full_text

IT之家 7 月 3 日消息，月之暗面 Kimi 今日宣布，GitHub Copilot 正式接入其开源模型 Kimi K2.7 Code，这是 GitHub Copilot 首次接入开源模型 。
IT之家获悉，Kimi K2.7 Code 由 GitHub 托管于微软 Azure 平台，采用 按量计费 模式，正逐步向 Copilot Pro、Pro+ 以及 Max 订阅方案开放。
GitHub 官方表示，此次推送将分批逐步进行，会持续监测该模型的质量与性能。在接下来的几周内，将把支持范围扩大至 Copilot Business、Enterprise 版本及其他应用平台。
据IT之家此前报道，6 月 12 日， 月之暗面 Kimi 发布并开源 Kimi K2.7 Code 编程模型 。
月之暗面表示，内外部基准评估显示，Kimi K2.7 Code 相比 K2.6 模型显著提升了长上下文编程场景的指令遵循能力、长程编程任务的性能表现，并且大幅改善了在长程任务中的过度思考倾向， 平均 token 消耗减少 30% 。
