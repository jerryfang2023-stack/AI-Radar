Orthogonal - Agentic Payments for APIs
APIs weren’t built for agents, we fixed that.
Christian Pickett
Orthogonal
6 months ago
Embed
https://orthogonal.com
TL;DR: APIs weren’t built for agents, we fixed that. Developers and agents get instant access to hundreds of APIs through our MCP or SDK. No key management, no billing headaches, pay as you go. API providers list once and become instantly discoverable.
Ask: We’d love an intro to teams building agents to understand how they’re handling paid APIs today. Contact us at founders@orthogonal.com
https://www.youtube.com/watch?v=CixNTItWDjI
The Problem:
AI agents are about to become the biggest consumers of APIs on the internet. But they struggle to access tools.
MCP helped standardize tool calling, but created fragmentation. Some services have MCPs, others don't. Some map to single APIs, others wrap entire workflows. Agents end up with 30+ tools, bloating context.
Worse, any MCP requiring identity or payment means managing accounts, API keys, billing, and auth across dozens of services. And most APIs don't even have MCPs.
You could have your agent find APIs, sign up, handle auth, and buy credits, but the failure rate is high. The entire API economy was built for humans.
Meanwhile, API providers are leaving money on the table. Billions of agents could be paying for their APIs right now, but they have no way to find or transact with them.
These APIs are locked behind API keys and human payment methods, which agents do not have access to.
The Solution:
Orthogonal is where agents go to find and pay for APIs.
For AI Agents and Developers:
Add our MCP to your agent for instant access to hundreds of paid APIs
Or use our SDK as a single, unified gateway to those APIs
You purchase Orthogonal credits, and we handle paying the APIs
Natural language search: "I need an API to get Garry Tan’s latest tweets"
No context bloat, no API keys to manage, no billing relationships to maintain
We can handle rate limits, retries, and dynamic routing
For API Providers:
List your API once, and it's instantly discoverable by every AI agent and developer using our platform
Get paid per request. No subscriptions, no invoices, no payment disputes
Set your own prices, including dynamic pricing
We also enable x402 on your APIs
We're starting with Search, Scraping, Datasets, and AI Model APIs .
About Us:
We're Christian and Bera. We met at McGill and have been close friends ever since. Christian worked on payments at Coinbase and billing at Vercel. Bera worked on reCAPTCHA and Maps APIs at Google. Between the two of us, we've lived in payments and API infrastructure, and this problem sits right at the intersection.
The Ask:
Intros to teams building AI agents that need easy access to paid services
Intros to teams managing APIs across many platforms who'd love to unify them
If you are a developer or have agents, sign up to get $10 of credits towards API usage : https://www.orthogonal.com
We appreciate our early believers: Precip (YC W24), Riveter (YC F24), Andi (YC W22), Shofo (YC W26), Fiber AI (YC S23), Sixtyfour (YC X25), Olostep, ScrapeGraph, Linkup
Find us:
