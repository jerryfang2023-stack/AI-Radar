# GitHub Copilot切换按量计费后6月使用量激增，创史上最佳月度业绩

## clean_text

IT之家 6 月 25 日消息，得益于 AI 编程需求持续攀升，GitHub 刚迎来了史上业绩最佳的一个月。
微软旗下这款开发者平台的首席技术官弗拉基米尔 · 费多罗夫在当地时间周三的员工会议上透露，平台调整旗下 AI 编程工具 Copilot 的计费模式后，6 月用户使用量出现大幅激增。
费多罗夫表示：“6 月无疑是我们成立以来表现最好的一个月。”由于公司本财季即将收官，他并未披露具体数据。
眼下 GitHub 正与多款增速迅猛的竞品同台竞争，包括 Cursor、OpenAI 的 Codex 以及 Anthropic 的 Claude Code，这类工具均可实现代码编写、编辑与漏洞修复。
微软方面拒绝就此置评。费多罗夫还称，单从此次使用量激增的情况来看，他个人认为 GitHub 无需大幅涨价，但并未透露公司任何确定的调价方案。
6 月 1 日，GitHub 调整了 Copilot AI 编程工具的收费规则：由原先按固定次数、单人统一包月定价，改为按用户实际使用量计费。此次调整，让 GitHub 的收费模式与一众同类竞品看齐，均采用按量付费模式。
微软于 2018 年收购了 GitHub，该平台是工程师存储、管理代码以及开展项目协作的主流平台，早年凭借超高人气占据行业先发优势。
但近期 GitHub 面临来自 Cursor 与 Anthropic 旗下 Claude Code 的激烈竞争。Business Insider 获取的一段内部会议录音显示，去年年末一场内部会议上，微软一名高管提及，GitHub 必须全面革新产品，才能与上述工具抗衡。
IT之家注意到，受使用量暴涨影响，GitHub 在 2026 年已发生数十起大规模服务中断事故。如今微软选择向其头号云服务商竞争对手亚马逊求助，用以解决平台算力承载不足的问题。
