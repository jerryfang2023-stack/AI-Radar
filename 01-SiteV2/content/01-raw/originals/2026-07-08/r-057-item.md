Microsoft Marketplace AI app and agent publishing checklist
Feedback
Summarize this article for me
AI apps and agents quality
Expectations
Clearly define the agent's purpose, target users, and business value.
Document explicit boundaries (what the agent will NOT do).
Publish transparent data handling and privacy practices.
Set baseline performance expectations (latency, accuracy, uptime).
Ensure UI and experience meet accessibility requirements (WCAG).
Quality controls
Build evaluation sets covering normal, edge, ambiguous, and risky scenarios.
Define success metrics (accuracy, grounding, deflection, etc.).
Enable continuous regression testing across builds and prompt changes.
Maintain output stability across versions.
Add human-in-the-loop for high-risk actions.
Implement safe fallbacks and failure responses.
Security
Enforce tenant isolation across all data paths.
Use Entra ID + managed identities, following least privilege access.
Secure all webhooks/APIs with auth, validation, and idempotency.
Encrypt all data at rest and in transit; avoid logging sensitive content.
Manage secrets exclusively via Azure Key Vault.
Define safe logging and retention policies.
Enforce prompt and response level safety filters to prevent jailbreaks and unsafe outputs.
Apply inference time policy checks (safety, compliance, DLP) on every response.
Validate supply chain integrity for models, MCP tools, connectors, and OSS components.
Log tool calls and agent actions securely with sensitive content redacted.
Limit model access to sensitive data using RBAC/ABAC and automatic redaction.
Enforce strict RAG context boundaries to prevent cross tenant data exposure.
Apply per tenant isolation for inference, embeddings, and agent context.
Compliance
Provide Responsible AI documentation (transparency, limitations, safety).
Align to SOC/ISO/etc., as applicable.
Define retention/deletion/consent workflows.
Validate accessibility and localization.
Prepare an evidence package for reviewers.
Development requirements
Core build requirements
Choose the correct offer type (SaaS, Azure App, Container, VM).
For SaaS: fully implement Fulfillment API v2 (Resolve, Activate, Update, Suspend, Cancel).
Build a landing page that processes the SaaS token correctly.
Implement tenant linking workflow (OAuth/consent as needed).
Implement Marketplace metering (if using consumption billing).
Design clear plan/SKU structure and entitlements.
Build a production ready multi tenant architecture.
Set up Dev > Stage > Prod environments with isolated credentials.
Engineering best practices
Version and modularize prompts/models (with rollback).
Implement caching (semantic, retrieval, standard HTTP).
Add API resilience (retry, timeout, throttling, circuit breaker).
Add guardrails: grounding, errant/incorrect information checks, safe outputs.
Optimize cost: token limits, batching, efficient vector usage.
Add observability: logs, traces, dashboards, alerts.
Automate CI validation for functionality, AI safety, and security.
Use safe deployment strategies (blue/green, rings, feature flags).
Samples and templates to use
SaaS Accelerator (reference fulfillment implementation).
Landing page sample (token processing).
Metering API sample.
Partner Center configuration template.
SKU <-> meter mapping table.
Prompt lifecycle governance (Draft > Review > Approved > Deployed).
Responsible AI document templates (transparency, limitations, safety FAQ).
Architecture diagram template.
Incident response / on call runbook.
Release notes & versioning log.
Publishing steps in Partner Center
Setup and content
Complete Publisher profile (legal, tax, payout, support contacts).
Add offer description, benefits, keywords, categories, industries.
Provide screenshots and documentation (Terms of Use, Privacy Policy).
Configure landing page URL and redirect URIs (for SaaS).
Configure fulfillment webhooks (+ authentication token).
Define plans/SKUs, trials, hidden SKUs, CSP reseller settings.
Define meter dimensions and pricing (if used).
Testing
Test end to end purchase > landing page > activation.
Test plan changes, suspend/resume, cancellation.
Test metering events appear in Partner Center.
Test idempotency and edge cases (duplicate events, instant cancel).
Validate accessibility across UI and listing materials.
Certification readiness
Perform full pre submission checklist (fields, URLs, assets).
Provide reviewer instructions or short demo video.
Provide security/architecture documentation.
Validate webhook performance (fast 200s).
Validate token landing flow with invalid and valid tokens.
After Go-Live
Track major updates for potential recertification.
Maintain transparency note, security docs, RAI docs.
Monitor usage, errors, subscription lifecycle events.
Prepare for Co-sell Readiness and Marketplace Rewards programs .
Stay current on Marketplace policy & RAI updates.
Next steps
Build AI apps and agents
Publish AI apps and agents to Microsoft Marketplace
