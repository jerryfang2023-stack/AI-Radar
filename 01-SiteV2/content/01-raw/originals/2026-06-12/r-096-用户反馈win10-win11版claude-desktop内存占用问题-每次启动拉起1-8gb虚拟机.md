IT之家 6 月 11 日消息，科技媒体 NeoWin 今天（6 月 11 日）发布博文，报道称 Anthropic 旗下适用于 Windows 10 、 Windows 11 的 Claude Desktop 存在内存占用问题。
IT之家援引博文介绍，用户在 Windows 平台使用桌面版 Claude 程序，一旦使用过 Claude Cowork 或者智能体模式，该程序就会启动一个 1.8GB 的 Hyper-V 虚拟机。
用户后续即便只是开启聊天模式， 每次启动 Claude Code 后都会创建 1.8GB 的 Hyper-V 虚拟机进程。 根据 GitHub 页面，用户早在今年 2 月就提交反馈，至今未获修复，部分用户已选择卸载。
该问题不影响 CLI 及其它平台，在 Windows 任务管理器中表现为 Vmmem 进程，CPU 占用 0%，内存却占用 1.8GB。不少用户认为该进程应仅在显式调用 Agent 模式后启动，并在使用后清理会话文件。

## 原始摘要 / 采集文本

Anthropic旗下Windows版Claude Desktop存在内存占用问题。用户只要使用过Claude Cowork或智能体模式，后续每次启动Claude Code都会创建一个1.8GB的Hyper-V虚拟机进程（任务管理器中表现为Vmmem），即使仅使用聊天模式也如此。该进程CPU占用0%，内存占用1.8GB。用户自今年2月在GitHub提交反馈至今未修复，部分用户已卸载。CLI及其他平台不受影响。
