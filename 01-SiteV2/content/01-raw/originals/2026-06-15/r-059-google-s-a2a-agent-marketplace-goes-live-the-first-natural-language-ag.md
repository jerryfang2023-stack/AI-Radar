# Google's A2A Agent Marketplace Goes Live: The First Natural Language Agent Discovery Platform | AgentMarketCap

## clean_text

Google Cloud just made it possible to shop for AI agents the same way you ask a question — in plain English. The new AI Agent Marketplace, integrated directly into Google Cloud Marketplace, lets enterprise teams discover, evaluate, and deploy third-party agents built on the Agent2Agent (A2A) protocol using Gemini-powered natural language search. It is the first major cloud marketplace where the discovery layer itself is agentic.
For an industry that has spent years debating interoperability standards, this is a concrete product moment: a transactional marketplace where ISVs list A2A-compatible agents, enterprises browse with conversational queries, and Google handles billing, security validation, and protocol compliance. The question is no longer whether agent-to-agent communication will standardize — it is whether platform-mediated discovery becomes the default procurement channel before standalone agent vendors can build direct sales motions.
From Protocol to Platform: A2A's First Year
The Agent2Agent protocol launched in April 2025 as Google's answer to a fragmented agent ecosystem. One year later, the numbers tell a story of rapid adoption: over 150 organizations have joined the A2A project, the protocol has landed natively in all three major cloud platforms, and the Linux Foundation now governs it through the Agentic AI Foundation (AAIF) alongside Anthropic's Model Context Protocol (MCP).
Version 0.3 of the A2A specification, released in early 2026, introduced gRPC support, signed security cards, and extended Python SDK capabilities — features specifically designed to make the protocol enterprise-production-ready. Industry analysts project a $2.3 billion market for A2A-related technologies by the end of 2026, with approximately 35% of AI-focused enterprises actively exploring integration.
The critical shift is that A2A has moved from a Google-led proposal to genuine multi-cloud infrastructure. Microsoft embedded A2A support in Azure AI Foundry and Copilot Studio. AWS added compatibility through Amazon Bedrock AgentCore Runtime. This cross-platform commitment changes the calculus for enterprise teams evaluating whether to invest in A2A-native architectures — vendor lock-in risk drops substantially when all three hyperscalers speak the same protocol.
How the Marketplace Actually Works
The AI Agent Marketplace sits inside Google Cloud Marketplace, which already processes billions in annual cloud software transactions. The agent-specific layer adds three capabilities that didn't exist in traditional marketplace listings.
Natural language discovery. Enterprise teams can describe what they need — "an agent that monitors Kubernetes clusters and auto-remediates common failures" — and Gemini-powered search returns validated agents matching that description. This replaces the keyword-and-category browsing model that dominates existing cloud marketplaces.
Agent Card validation. Every listed agent exposes an A2A Agent Card: a standardized self-description that includes capabilities, supported interaction patterns, authentication requirements, and skill tags. Google validates these cards against the A2A specification before listing, creating a minimum quality bar that raw protocol adoption alone cannot guarantee.
Transactional integration. Partners sell agents directly through the marketplace, with Google handling billing, committed-spend drawdown, and procurement compliance. For enterprises already spending through Google Cloud Marketplace, adding agent subscriptions requires no new procurement workflows — a significant friction reduction for teams that historically needed months of vendor evaluation for new AI tooling.
Who Is Listing and What They Are Selling
The early marketplace reflects a clear pattern: enterprise software incumbents are leading, not agent-native startups.
Adobe is listing agents that make its distributed AI capabilities interoperable with Google Cloud's ecosystem, enabling multi-system process automation across creative and marketing workflows. ServiceNow, a founding A2A partner, contributes agents through its AI Agent Fabric — a multi-agent communication layer designed for IT service management and enterprise workflow orchestration.
EPAM launched seven specialized agents covering complex industry challenges across multiple verticals. Elastic developed an agent targeting Site Reliability Engineers and SecOps teams, helping them interpret log messages, optimize code, and generate reports.
The vendor distribution skews heavily toward IT operations, customer service, and enterprise workflow automation. Notably absent are coding agents, research agents, and the kind of general-purpose autonomous systems that dominate consumer AI discourse. The marketplace is solving for enterprise procurement teams, not individual developers — and the listing categories reflect that priority.

Enterprise Services EPAM Industry-specific workflow automation

The A2A + MCP Stack: Complementary, Not Competing
One of the most common misconceptions in enterprise AI architecture is that A2A and MCP are competing standards. They solve different problems at different layers of the stack.
MCP standardizes how agents access tools, APIs, and data sources — it is the universal adapter layer. With 97 million monthly SDK downloads, MCP has become the de facto standard for agent-to-tool communication. A2A standardizes how agents communicate with each other — it is the coordination layer for multi-agent workflows where autonomous systems need to delegate tasks, share context, and negotiate outcomes.
The emerging consensus architecture, endorsed by the Linux Foundation's AAIF, is a three-layer stack: MCP for tool access, A2A for agent coordination, and WebMCP for web-native interactions. By February 2026, over 100 enterprises had signed on as supporters of this unified approach.
For the Google Cloud Agent Marketplace specifically, this means listed agents can leverage MCP for their internal tool integrations while exposing A2A interfaces for inter-agent collaboration. An enterprise team deploying a ServiceNow ITSM agent alongside an Elastic observability agent can orchestrate them through A2A without either agent needing to understand the other's internal MCP tool connections.
What Platform-Mediated Discovery Changes
The strategic question is whether cloud marketplace distribution becomes the primary channel for enterprise agent procurement — and what that means for agent vendors building direct sales.
Three dynamics favor platform mediation:
Committed-spend drawdown. Enterprises with existing Google Cloud commitments can purchase agents against those commitments. This eliminates net-new budget approval, which is historically the single largest friction point in enterprise software sales. An agent vendor listed on the marketplace gets access to pre-committed budgets that would otherwise require 3-6 months of procurement process through direct sales.
Security and compliance validation. Google validates A2A compliance, authentication mechanisms (OAuth 2.0, API Keys, mTLS), and basic agent behavior before listing. For enterprise security teams, marketplace agents come pre-vetted against a baseline that standalone vendors must demonstrate independently — often through lengthy security reviews.
Discovery network effects. As more agents list, the natural language discovery layer becomes more valuable. Enterprise teams searching for capabilities find a richer selection, which attracts more vendors, which improves discovery further. This flywheel is difficult for standalone agent directories to replicate without the transaction layer and committed-spend incentive.
Three dynamics push against it:
Margin compression. Cloud marketplace fees reduce agent vendor margins. For high-value enterprise agents with strong direct demand, the marketplace take rate may not justify the distribution benefit.
Differentiation limits. Marketplace listings compress agent capabilities into standardized Agent Cards and category tags. Agents with nuanced differentiators — custom training, domain-specific fine-tuning, unique architectural approaches — may find that marketplace presentation flattens their value proposition.
Platform dependency. Building distribution through a single cloud marketplace creates concentration risk. Agents that succeed primarily through Google Cloud Marketplace face the same platform dependency challenges as mobile apps that rely entirely on App Store discovery.
The 40% Threshold: Enterprise Agent Adoption by End of 2026
Industry analysts project that by the end of 2026, 40% of enterprise applications will include task-specific AI agents. The Google Cloud Agent Marketplace is positioned to capture a meaningful share of that deployment by reducing three barriers simultaneously: discovery (natural language search), procurement (committed-spend billing), and interoperability (A2A protocol validation).
The 65-75% year-over-year growth rate in A2A adoption suggests the protocol itself is not the bottleneck. What has been missing is a transactional layer that converts protocol compatibility into deployable products. The marketplace provides exactly that.
For enterprise teams evaluating agent procurement strategy, the practical recommendation is straightforward: if you are already on Google Cloud, the Agent Marketplace is now the lowest-friction path to deploying third-party agents. If you are multi-cloud, watch for equivalent marketplace offerings from AWS and Azure — both have A2A support, and both have existing marketplace infrastructure that could host similar agent directories.
The broader signal is structural. Agent distribution is following the same pattern as SaaS distribution a decade ago: from direct sales to marketplace aggregation to platform-mediated discovery. The vendors that build for this channel early — with clean Agent Cards, standardized A2A interfaces, and marketplace-optimized go-to-market — will have a distribution advantage that compounds as enterprise teams default to marketplace-first procurement.
