# Anthropic 工程负责人：Claude Code 让程序员更孤独

## clean_text

IT之家 6 月 22 日消息，编程向来是一项容易让人感到孤独的工作，AI 智能体的普及反而加剧了这种孤立感。据《商业内幕》22 日（今天）报道，负责 Claude Code 和 Claude Cowork 团队的 Anthropic 工程负责人 Fiona Fung 表示，团队发现，工程师越依赖 AI 智能体，彼此之间的交流就越少。
Fung 坦言：“我们注意到，时间一长，这种工作方式可能会让人感到孤独，因为大家都把太多时间花在 和自己的智能体一起工作 上。”
据IT之家了解，为了缓解这个问题，团队开始 重新创造面对面交流的机会 。团队组织编程午餐、黑客松和共同开发时段，让工程师坐在一起工作，并互相学习不同的 AI 工作流程。“每个人都在用 Claude Cowork，但使用方式差别很大。进行结对编程时，我们能从彼此身上学到很多东西。”
Claude Code 已迅速成为软件开发领域最常用的产品之一。针对 20 多名创业公司创始人和风险投资人进行的调查发现，Claude Code 已成为创业公司使用最多的 AI 编程工具。部分创始人表示，遇到复杂工程任务时，自己通常会优先使用 Claude Code。
工程师也开始把更多时间用于 安排智能体工作、检查生成结果和协调多个并行任务 。与此同时，能够通过自然语言指令开发软件“氛围编程”正迅速走红。即使没有技术背景，创业者也能在不组建传统工程团队的情况下制作定制工具，“单人创业者”因此越来越常见。
然而，单枪匹马完成所有工作同样容易陷入孤独，许多创业者仍然认为协作不可或缺。Fung 表示，即使团队成员使用 AI 的方式不同，Anthropic 仍在努力为工程师 创造更多并肩工作的机会 。“每次观察别人怎么工作，我自己也会有所收获。”
