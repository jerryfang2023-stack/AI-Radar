Request: Copilot Coding Agent for Enterprise Cloud with Data Residency
Context:
We're a UK-based logistics company evaluating GitHub Enterprise Cloud with EU data residency for compliance requirements. We're excited about the new Copilot coding agent feature but discovered it's only available on GitHub.com, not on the isolated GHE.com infrastructure used for data residency deployments.
Current Limitation:
✅ Copilot chat, completions, and code review work on data residency
❌ Copilot coding agent is unavailable on data residency deployments
This appears to be an architectural limitation due to the separation between GitHub.com and GHE.com
Business Impact:
For enterprises requiring data sovereignty (GDPR, financial services, government), this creates a significant feature gap. The coding agent represents a major productivity advancement that we can't access while maintaining compliance.
Questions:
Are there any plans to bring Copilot coding agent to Enterprise Cloud with data residency?
If so, is there any timeline or roadmap visibility?
Are there technical alternatives or workarounds being considered?
Is this limitation permanent due to architectural constraints?
We understand the complexity of maintaining feature parity across different infrastructure deployments, but this feature gap affects our adoption decision significantly.
Our Setup:
GitHub Enterprise Cloud with EU data residency
C# backend + Angular frontend
Strong interest in AI-powered development workflows
Would appreciate any guidance from the GitHub team or insights from other Enterprise Cloud users facing similar limitations.

Aug 19, 2025
Thanks @trentsteel84 , really thoughtful question and you’ve outlined the limitation clearly. Based on current documentation and similar enterprise discussions, here’s the situation:
The Copilot coding agent is indeed only available on GitHub.com today. Data residency deployments (GHE.com infrastructure) currently support chat, completions, and code review, but not the full agent functionality.
The reason is largely architectural: the coding agent relies on GitHub.com infrastructure for orchestration, integrations, and secure execution. That separation makes feature parity slower to achieve in environments with strict residency constraints.
There has been no public roadmap update yet announcing when or if the coding agent will be supported in Enterprise Cloud with data residency. GitHub typically rolls out high-compliance features in stages, so it may be a matter of time, but there is no confirmed ETA.
Alternatives today:
Continue using chat, completions, and code review, which are supported under EU residency.
For agent-like automation, some enterprises pair Copilot with custom GitHub Actions or bots running within their residency-compliant cloud. This can mimic certain agent workflows (like triggering refactors or responding to issues) while staying compliant.
Keep an eye on GitHub’s Enterprise roadmap and reach out to your GitHub account rep or support channel to flag your use case. Feature prioritization for residency often depends on customer demand.
Is the limitation permanent? Likely not permanent, but it is non-trivial. Residency services usually lag behind global GitHub.com releases, and complex features like the agent may take longer (or arrive in a more limited form).
Summary: The Copilot coding agent is not yet supported in Enterprise Cloud with data residency, and there’s no official timeline. The best step now is to log this with GitHub support or your account manager so your business need is factored into prioritization. In the meantime, partial alternatives like Actions or internal bots can help cover some automation gaps.
