# 微信支付 AI 接入工具箱 2.0 发布，Token 消耗降低 50%

## clean_text

IT之家 6 月 15 日消息，今年 4 月， 微信支付官宣发布一整套面向 AI 的支付接入 Skill ，涵盖 Skill 技能包、AI 友好文档、AI 友好 API 三大工具。据微信公开课消息，微信支付 AI 接入工具箱迎来 2.0 版本，围绕“安全、高效、原生流畅”的方向，重点更新了五大内容。
据介绍，微信支付 AI 接入工具箱 2.0 现在 全面支持中文、英语、日语、韩语、法语、俄语、葡萄牙语、西班牙语、阿拉伯语这 9 种语言 。全球开发者基本可以用自己的母语对接微信支付。
工具箱 1.0 的 Skill 仅覆盖“支付”和“券”两大方向，2.0 版本升级为全线覆盖 —— 囊括微信支付所有产品的知识 ，同时还提供两大 AI 专家能力：
技术专家： 支持从支付场景选型、接入流程解读，到动态排障的全链路问答，帮助开发者通过对话完成【报错 → 定位 → 修复】的完整闭环，全程无需脱离开发环境。
金融级研发专家： 内置代码质量审查机制，从金融级安全标准和最佳实践角度评估开发者的代码，精准识别容易被忽略的质量审计盲区。
1.0 版本的静态排障手册只能对照查阅已知问题，2.0 新增了 CLI 动态排障能力。开发者在编码环境中，直接通过自然语言调用 CLI 查询支付单、退款单的实时状态，结果即时返回。不用切出编码环境，不用登商户后台，让调试节奏不断，问题链路更短。
IT之家注意到， 微信支付 AI 接入工具箱 2.0 将文档中所有时序图、状态图全部转成了 Mermaid 格式 ，让无多模态能力的模型可直接理解图表语义，有多模态能力的模型也无需再调用图像识别，配合已有的 Markdown 格式文档和站点索引文件， 整体 Token 消耗可比原始 HTML 文档直接降低 50% 。
另外，工具箱升级 2.0 后，Skill 启动时会自动比对本地知识库与官网知识库的版本差异。一旦官网有更新， 就会自动触发本地内容同步 。开发者无需主动检查版本，也无需手动更新，可确保每次使用的都是最新的知识和能力。
