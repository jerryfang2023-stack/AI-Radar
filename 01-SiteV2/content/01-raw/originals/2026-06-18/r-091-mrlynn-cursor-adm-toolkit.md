Why ADM Toolkit?
Enterprise AI deployments require more than licenses. They need account intelligence — revenue context, ROI models, health scores, competitive positioning, and deployment plans — all in one place, inside the AI assistant you already use.
ADM Toolkit gives AI Deployment Managers everything they need to run enterprise rollouts: rules, agents, commands, hooks, and 25 MCP tools that work together.
Install in One Command
/add-plugin https://github.com/mrlynn/cursor-adm-toolkit
This installs rules, agents, commands, hooks, and the MCP server. Open your project in Cursor and type /adm-help to see all commands.
Other options
Method

CLI
npm install -g @admtoolkit/cli then adm init and adm setup
Clone
git clone https://github.com/mrlynn/cursor-adm-toolkit.git then npm install

Typical Workflow
adm account new → Create account interactively
adm research <name> → Research account (revenue, tech stack, AI strategy)
adm roi <name> → Model ROI (conservative/base/aggressive)
adm health <name> → Compute health score (0-100)
adm qbr <name> → Generate quarterly business review
adm new-deck <name> → Scaffold a presentation deck
Learn Cursor → Customer Content
/adm-learn lab mcp → Guided lab (7 modules)
/adm-learn capture → Journal insight to .adm/learn/
/adm-content enablement Adobe → Enablement pack from your learnings
Web: Learn · Content Studio · CLI: adm learn , adm content build
Customer hackathons
adm hackathon new --template customer-virtual-2day --account <name>
adm hackathon status "<event>" # readiness %, next actions
adm hackathon recap "<event>" # RECAP + account HACKATHON handoff

Open .adm/ as an Obsidian vault (name: ADM ). Insights are atomic notes in learn/insights/ with YAML frontmatter for Dataview.
adm obsidian setup # scaffold MOC + insights folder
adm obsidian open learn/Learn \ MOC.md
See Obsidian integration .
