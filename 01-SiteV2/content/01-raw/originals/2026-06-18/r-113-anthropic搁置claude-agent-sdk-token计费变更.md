IT之家 6 月 17 日消息，科技媒体 Ars Technica 昨日（6 月 16 日）发布博文，报道称 Anthropic 为缓解开发者的不满情绪， 宣布搁置 Claude Agent SDK 基于 Token 的计费变更计划。
IT之家注：Claude Agent SDK 是 Anthropic 推出的开源开发框架，支持开发者在 Python 或 TypeScript 应用程序中，直接调用驱动 Claude Code 的底层能力，让 AI 自主使用工具和完成复杂任务。
Anthropic 于 5 月 13 日发布公告，宣布调整 Claude Agent SDK（包括通过第三方应用及程序化命令“claude -p”调用）计费方式，从标准订阅调整为按照 API 调用的 Token 用量计费。
Anthropic 表示，自 6 月 15 日起，此类外部 SDK 调用将按照 Anthropic 现行 API 费率计费，订阅用户每月仅可获得一笔与订阅费等额的使用额度抵扣。
开发者 Matthew Diakonov 在分析中指出，如果以 Opus 作为主编码助手，“第一周就会超过盈亏平衡点”。代码编辑器 Zed 背后的团队更直接警告用户，这是一次“重大成本增加”。
在 Anthropic 于 6 月 15 日官更新其支持页面，表示“暂停所述变更”，并强调“目前没有任何变化”，正在更新计划以更好支持用户。
