# 腾讯云发布 Agent Bucket，为亿级智能体提供原生独立云空间

## clean_text

IT之家 7 月 9 日消息，今天，腾讯云 Agent Bucket（智能体桶）正式发布。这个“桶”可以为亿级 Agent 提供独立云空间，统一存放用户上传的资料，以及 Agent 生成的报告、图片、代码等文件和工作产物。
IT之家附官方详细介绍如下：
开发者仅需兼容标准 S3 接口，就能用熟悉的接入方式，快速为海量 Agent 构建原生独立空间，无需自研多租户隔离、权限管理等复杂能力。
给 Agent 找一个“能存”的方案不难，但想要适配海量并发场景，在“隔离、体验、性能”上达到最优解，必须重新设计底层架构。
（三种存储共同撑起了过去二十年的数字世界，但不原生适配 Agent 需求） 腾讯云依托对象存储“成本实惠、啥都能装、想装多少装多少”的优势，专门为 Agent 打造了这款原生存储底座。
目前，Agent Bucket 已搭载在 WorkBuddy、QClaw、ClawPro 等腾讯 Agent 应用中，经受“免费安装龙虾”全民热潮的高并发考验，同时服务华硕、东风日产等客户，在 AI PC 和车载云空间等场景中，为用户提供安全隔离、可长期管理的云端文件空间。
亿级空间原生隔离
企业和开发者通常用 1 个 Bucket 承接 1 个业务或应用，再通过文件路径、命名前缀等方式管理海量数据。但在 Agent 场景下，这两种传统方式都不够用。
❌为每个 Agent 单独建 Bucket，数量上限难以支撑海量实例；
❌让所有 Agent 共用 1 个 Bucket，又会让隔离过度依赖业务代码，也难以对单个 Agent 独立设置配额、限速和生命周期策略。
Agent Bucket 想了个办法，引入了“Space”。
Space 是 Bucket 下的独立空间单元，每个 Agent 或用户都可以拥有一个 Space，并具备独立的访问凭证、容量配额和读写限速。
隔离从“代码逻辑”变成“系统能力”，资源管理也可以精细到单个 Agent。
高并发也不怕，Agent Bucket 底层架构自动实现负载均衡，彻底消除单点热点瓶颈，不会因为某个热门 Agent 被集中访问就把整个桶拖垮。
“网盘级”的文件管理，开箱即用
解决隔离之后，第二个问题是体验。Agent 在 1 次任务中可能生成代码、图片、报告等多种文件。用户需要跨会话继续使用，团队也可能需要共享这些成果。
如果底层只有对象存储接口，开发者还需要自行实现目录管理、搜索、分享、回收站等能力，成本较高。
Agent Bucket 将部分通用能力前置到存储层👇
例如秒传去重，通过 SHA-256 生成文件指纹，避免重复存储；
支持 POSIX 文件系统语义，让云端操作更接近本地文件系统；
支持多模态检索，可按内容查找图片、视频或文本文件。
此外，还内置外链分享、回收站和内容审核能力，提升文件管理的完整性和安全性。
开发者无需再单独构建一套文件管理系统，仅用存储就能实现丰富的文件能力。
低成本也有高性能
与传统应用不同，Agent 会在执行过程中频繁读写数据，并在多轮任务中反复调用历史文件。大量小文件和高并发访问，会持续增加系统压力。
对象存储适合海量数据，但高频访问直接命中远端存储，容易成为瓶颈。
Agent Bucket 引入了 GooseFS 数据加速能力，通过分布式缓存，将高频访问数据放在更靠近计算的位置，减少远端访问，提高响应速度。
同时，系统采用冷热分层策略：常用数据进入缓存层，不常访问的数据保留在低成本对象存储中，在性能与成本之间取得平衡。
这个桶，Agent、车主、PC 用户都能用！
依托 Agent Bucket 这套原生存储能力，腾讯 Agent 产品打造了多样的文件管理体验👇
-WorkBuddy 用户：外链分享、在线预览、编辑、云端协作一应俱全。
-部署 ClawPro 的企业，“公域”和“私域”存储都搞定，既能存组织的技能库，也可以存每个 Agent 的产物。
不止 Agent，车主、PC 用户也能用
-AI PC 场景，华硕基于它构建个人空间盘，为每台设备提供 TB 级云空间，研发周期从年缩短到月。
-智能汽车场景，为东风日产 N7、NX8 等全系新能源车主提供独立云空间，用于存储和检索哨兵视频。
