HeyMate Skill (Tech & Product Design Team)
Open Source AI Tech & Product Design Team Skill
HeyMate is an open-source AI skill framework for founders, CTOs, product managers, engineers, QA teams, designers, consultants, and AI coding agents.
It turns a generic AI coding agent into a context-aware technical partner that can handle product discovery, design governance, architecture review, implementation planning, QA, DevOps review, handoff, documentation, and customer-safe support behavior.
This repository is the HeyMate Skill Package , not the full HeyMate runtime/platform. It is designed to be used with Claude Code, Cursor, Codex, Windsurf, Roo Code, OpenHands, and other AI coding agents.
Created By
Duong Dang
Founder & CEO, Dotbase Technology & Solution
Email: duong@dotbase.vn
Telegram: @duongdang
What HeyMate Does
HeyMate helps teams build better software through:
AI CTO and multi-role technical governance
Product discovery, PRD, user stories, acceptance criteria, and MVP scope
Enterprise product design workflow from idea to CTO/Codex handoff
UX, information architecture, user flows, wireframes, design system, visual design, and design QA
UI consistency and design system drift prevention
Architecture, backend, frontend, QA, security, DevOps, data, AI, CRM, ERP, POS, invoice, and e-contract perspectives
Knowledge commands for docs, APIs, specs, UI, rules, stack, and context
Handover, training, user guide, and documentation modes
Permission-safe customer support behavior
Command Format
All HeyMate commands use the mate-* format:
mate-{position}
mate-{action}
Examples:
mate-cto
mate-prod
mate-design
mate-qa
mate-devops
mate-audit
mate-plan
mate-review
mate-handoff
Every command response should start with an identity banner:
This is Mate CTO.
This is Mate Design.
This is Mate Review.

Bootstrap command:
mate-install
mate-install checks for project instruction files such as CLAUDE.md , AGENTS.md , agent.md , agents/agent.md , and .cursor/rules/hey-mate.mdc . It creates a missing file or appends a marked HeyMate Project Handle block to an existing file.
After that block exists, HeyMate is active by default for the project. Normal prompts and tasks are routed through Mate first, even when the user does not type mate-* .
Guarded commands:
mate-ship
mate-deploy
mate-rollback
mate-learn
mate-compact
Guarded commands require explicit confirmation and a concrete target before action.
Mate Design
mate-design integrates the former HeyMate Product Designer Enterprise workflow into the main HeyMate skill.
The full enterprise source package is preserved as reference material:
references/heymate-product-designer-enterprise/
The main skill contains the compact operating layer. The reference folder keeps the detailed workflows, rules, templates, schemas, checks, Figma MCP conventions, examples, and CTO handoff contracts so the enterprise detail is not lost.
Pipeline:
Idea Intake
-> Discovery
-> Product Spec / PRD
-> UX Research
-> Information Architecture
-> User Flow
-> Wireframe
-> Design System
-> Visual Design
-> Figma MCP Build Plan
-> Design QA
-> CTO / Codex Handoff
Modes:
mate-design autonomous
mate-design guided
mate-design figma
mate-design review
mate-design handoff
mate-design owns product experience and design readiness. mate-cto owns architecture, implementation strategy, engineering risk, infrastructure, testing, and deployment safety.
Quick Start
Claude Code
Copy the skill into your project root:
cp skills/hey-mate.md CLAUDE.md
Then start Claude Code and say:
Hey Mate
Use commands:
mate-install
mate-audit
mate-plan
mate-design autonomous: CRM dashboard for a sales team
mate-review-ui
mate-handoff

Codex / Other Agents
cp skills/hey-mate.md AGENTS.md
Then run the installer prompt from:
templates/MATE_INSTALL_PROMPT.md
