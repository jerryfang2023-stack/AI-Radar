# 06-asset-candidates｜后台资产候选

本目录保存变化、场景、趋势的轻量候选。候选不直接前台展示，也不得写成确定结论。

```text
change/ -> 变化候选
scene/  -> 场景候选与案例补全研究
trend/  -> 趋势候选 / 变化簇候选
```

升级规则：

- 变化候选需要多条商业信号、至少一个案例 / 场景、前后对比、商业变量和风险边界。
- 场景候选需要真实行业 / 部门、岗位 / 使用者、流程 / 任务和 AI 改变步骤。
- 趋势候选需要多个变化、多个场景 / 案例和多类来源，并写清信息缺口或后续观察变量。

未成熟时只允许写 `watchlist_only`、`needs_backfill` 或 `threshold_pending`。
