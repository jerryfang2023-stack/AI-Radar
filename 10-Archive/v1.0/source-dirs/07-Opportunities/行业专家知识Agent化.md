## 机会摘要

- **一句话机会定义：** 将稀缺行业专家（设备巡检、工程报价、合规审查等）的判断经验和决策规则编码为专用 AI Agent，替代人工完成重复性专业判断。
- **为什么是现在：** Cloneable 以"关键基础设施现代化"为场景，通过捕获专家知识并部署为专用 AI Agents，完成 460 万美元种子轮融资。IFS Loops 推出 Agent Studio，强调"预制行业模板 + 可调工作流"降低数字员工部署门槛。这说明资本和产品都在验证"专家经验产品化"比泛知识库更容易形成行业壁垒。中国有大量重资产行业（电力、建筑、制造、采矿）面临专家退休、知识断层的压力。
- **这是谁的问题：** 设备运维公司、工程公司、建筑企业、能源/电力/制造行业的运营部门——岗位经验强、人员流动率高、培训成本高的领域。
- **不做的代价：** 行业专家知识是 Agent 领域少数能形成可持续壁垒的方向——数据飞轮和行业知识积累本身就是护城河。如果现在不做，等头部行业软件厂商（如西门子、施耐德、金蝶）内置此类能力后，独立创业者的窗口期会大幅缩短。

## 标签

- 行业：能源/电力、建筑/工程、制造、采矿、农业
- 业务职能：运营 / 维护 / 质检 / 报价 / 合规
- 场景：设备巡检判断、工程报价审核、合规审查、施工资料审核、质量检验
- 企业规模：中型以上重资产企业
- 成熟度：2
- 中国适配度：4
- 决策紧迫度：3

## 问题与解决方案

- **原始痛点：** 关键岗位（设备巡检、工程报价、合规审查等）依赖有多年经验的专家，专家退休或离职后知识断层严重；培养新人周期长（6-12 个月），且判断标准不一致。
- **当前替代方案：** 老带新（效率低、规模受限）、SOP 文档（落地难、无法覆盖边缘情况）、培训课程（不能代替判断）。实质上没有真正的替代方案，大多数企业只能忍受效率损失。
- **AI解决方案类型：** 专家知识捕获 + 规则引擎 + LLM 推理的混合 Agent，覆盖"输入场景 → 按专家逻辑判断 → 输出建议/决策"的闭环。
- **需要什么数据/系统前提：** 2-3 位行业专家深度访谈，提取判断规则；20-50 个真实案例用于调优 Agent 输出质量；对接企业工单/巡检系统（或支持导入导出）。
- **典型实施路径：** 选定岗位 → 访谈专家 → 整理决策树 → 构建 Agent 原型 → 用历史案例测试 → 专家反馈修正 → 试点上线 → 持续积累案例优化。

## ROI假设

- **可能节省的成本：** 减少对高级专家的依赖，初级员工 + Agent 可承担原来高级专家 60-80% 的日常判断工作；人员培训周期从 6-12 个月缩短到 1-2 个月。
- **可能提升的收入：** 以项目制 + 年维护费定价，单客户 ARR 10-30 万；以行业为单位可复制。
- **可能缩短的时间：** 单次设备巡检/工程报价的判断时间从 30 分钟-2 小时缩短到 5-10 分钟。
- **回本周期假设：** 项目制切入 3-6 个月回本；产品化后 6-12 个月。
- **适合先做试点还是直接规模化：** 强项目制切入。选定一个行业（如电力巡检或工程报价），做透 3-5 家客户，沉淀行业知识库后再考虑跨行业复制。

## 代表性案例

- **案例公司：** Cloneable、IFS Loops
- **发生了什么：** Cloneable 获 460 万美元种子轮融资，方向是把专家知识编码为可扩展的 AI Agents，服务能源、公用事业、建筑、铁路等重资产行业。IFS Loops 推出 Agent Studio 和 Digital Workers，预置行业上下文、成熟工作流和企业级信任控制。
- **关键数据：** Cloneable 460 万美元种子轮；IFS Loops 强调"预制岗位 + 可调工作流 + 权限治理"。
- **成功前提：** 需要行业专家愿意配合知识提取（需要行业关系）；Agent 输出质量需经专家验证达到"可作为初稿"水平（>70%）；需要持续用真实案例迭代知识库。
- **失败风险：** 专家不配合或知识提取不全导致 Agent 质量不达标；客户行业太分散导致难以复制；一个行业一个模型导致规模不经济；专家本身认为 Agent 是威胁而不愿配合。

## 典型供给方

- **海外代表：** Cloneable、IFS Loops
- **国内代表：** 暂无直接对标；部分大型国企在自建设备巡检知识库；电力/能源行业有 AI 应用创业公司但多聚焦预测维护而非专家知识复制。
- **自建/采购/合作建议：** 建议自建，但需要找到愿意配合的行业专家作为联合创始人或深度顾问。先用服务换行业知识，再沉淀为产品。不建议采购通用 Agent 平台来改造，行业深度才是壁垒。
- **供应商筛选标准：** 如果涉及模型层，推荐用国产模型（GLM-5.1、DeepSeek-V4）做私有化部署，满足重资产企业的数据安全要求。

## 风险与反证

- **最大风险：** 行业专家知识提取是"人工密集"操作，每个行业都需要重新访谈、整理、调优，导致产品难以规模化。如果找不到"跨行业通用的专家知识框架"，本质上是一个咨询生意而非产品生意。
- **哪类企业不适合：** 没有行业资源的纯技术团队；追求纯产品化和快速规模化的团队。
- **反对观点：** "专家知识的价值被高估了，很多所谓的专家判断其实就是 Pattern Matching，通用模型很快就能覆盖"——但对于设备巡检、工程报价这类高风险场景，企业不可能接受"通用模型说大概可以"，场景特殊性决定了垂直 Agent 的不可替代性。
- **需要进一步验证的问题：** 在一个具体行业（如电力巡检），Agent 输出质量能否达到专家认可的"可作为初稿"的比例超过 70%？这决定了产品有没有实际替代价值。

## 行动建议

- **建议优先级：** 中
- **适合谁发起：** 有重资产行业资源和人脉的创业者；有行业专家合伙人或顾问的团队。
- **下一步最小验证动作：** 选择一个具体岗位（如电力设备巡检），访谈 2 位行业专家，把判断规则整理成决策流程，用 10 个真实案例测试 Agent 输出质量，验证"可作为初稿"比例是否达到 70%。
- **建议预算范围：** 行业调研 + 专家访谈 + MVP 原型 5-10 万。
