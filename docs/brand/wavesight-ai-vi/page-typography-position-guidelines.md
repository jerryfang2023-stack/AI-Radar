# 观澜AI 页面位置字体规范

状态：V2.2 current  
owner：Experience & Editorial  
适用范围：`01-SiteV2/site/` 公开前台页面、首页模块、栏目页、详情页、导航栏、页脚和后续页面任务。  

本文件补足 `typography-guidelines.md` 没有细化到页面位置的问题。后续新页面、新模块和页面重构，必须同时遵守：

1. `typography-guidelines.md`
2. `brand-tokens.css`
3. 本文件

页面类任务必须在生成前执行 Typography-first：先输出 `Typography 页面位置表`，再进入 Build & Release。`$guanlan-typography-qc` 必须用于 Build 前审查该表，不能只在页面完成后做事后检查。

如果三者冲突：

```text
page-typography-position-guidelines.md 的页面位置规则 > typography-guidelines.md 的通用层级 > 单个页面 CSS
```

## 1. 为什么新增本规范

当前问题不是“没有字体规范”，而是规范停留在通用层级：

- 有 Hero / 页面标题 / 模块标题 / 卡片标题，但没有规定它们分别出现在首页、栏目页、详情页、导航栏的哪个位置。
- 页面实现中出现 `clamp(54px, 5vw, 88px)`、`clamp(42px, 4vw, 68px)` 等海报级字号，容易让普通栏目页和模块也变成 Hero。
- 字重出现 650、700、760、780 混用，卡片、标签和按钮主次容易混乱。
- 有些页面把卡片标题、模块标题、详情页标题都做得接近，导致阅读路径不清。
- 位置规范缺失，导致同样是栏目页标题、详情页标题或导航文字，不同页面上下位置和大小不一致。

本文件目标是把“字体大小、粗细、行高、位置和使用场景”固定下来。

## 2. 总原则

### 2.1 一页只有一个最大标题

- 首页只有 Hero H1 可以使用 `56px / 72px`。
- 栏目页只使用 `44px / 58px`。
- 详情页只使用 `40px / 56px`。
- 模块标题不得超过 `36px / 48px`。
- 卡片标题不得超过 `24px / 34px`，绝大多数使用 `20px / 30px`。

### 2.2 字号不随视口任意放大

禁止在普通标题、栏目标题、卡片标题上使用：

```css
font-size: 5vw;
font-size: clamp(54px, 5vw, 88px);
font-size: clamp(42px, 4vw, 68px);
```

只允许首页 Hero H1 使用受控 token。其他位置必须使用固定 token 或清晰的桌面 / 移动断点。

### 2.3 字重控制

公开前台常用字重只允许：

| 用途 | 字重 |
|---|---:|
| 正文 | 400 |
| 正文重点 / 导航 | 500 |
| 卡片标题 / 模块标题 | 600 |
| Hero / 页面标题 | 600 |
| 英文标签 / 小标签 | 600 |

避免使用 `760`、`780` 这类过重字重。数字或标签需要强调时，优先用颜色、位置、留白和字母间距解决，不靠极粗字重。

### 2.4 字体角色固定

| 位置 | 字体 |
|---|---|
| 首页 Hero H1 / 页面标题 / 详情页 H1 / 核心判断句 | `--gl-font-serif-cn` |
| 导航 / 正文 / 摘要 / 按钮 / 卡片正文 | `--gl-font-sans-cn` |
| 英文品牌 / 英文标签 / 日期标签 | `--gl-font-en` 或 `--gl-font-mono` |
| 日期 / 编号 / 指标 / 报告号 | `--gl-font-mono` |

大段正文不得使用衬线字体。英文品牌和标签不得使用中文字体模拟。

## 3. 全站导航栏

| 位置 | 桌面规范 | 字体 | 字重 | 位置 |
|---|---:|---|---:|---|
| 导航栏高度 | 72px | - | - | 与页面背景融合，无边框、无阴影 |
| Logo | 宽度 160-180px | SVG | - | 左侧，垂直居中 |
| 一级导航文字 | 14px / 20px | `--gl-font-sans-cn` | 500 | 中间或右侧导航组，垂直居中 |
| 当前导航 | 14px / 20px | `--gl-font-sans-cn` | 600 | 不加大字号，只用颜色或下划线/状态区分 |
| 右侧按钮 | 14px / 20px | `--gl-font-sans-cn` | 600 | 右侧，按钮高度 38-42px |
| 英文小标签 | 11px / 16px | `--gl-font-en` | 600 | 只用于必要的英文品牌辅助 |

禁止：

- 导航文字超过 15px。
- 当前导航靠加粗到 700+ 或放大字号强调。
- 导航栏增加边框、阴影、色块底。
- 不同页面导航位置、字号、间距不一致。

## 4. 首页

### 4.1 首页 Hero

| 位置 | 桌面规范 | 字体 | 字重 | 位置 |
|---|---:|---|---:|---|
| Hero 顶部间距 | 导航下 96-120px | - | - | 首屏必须露出下一段内容 |
| Eyebrow / 英文标签 | 11px / 16px | `--gl-font-en` | 600 | H1 上方 18-24px，字距 `0.12em` |
| Hero H1 | 56px / 72px | `--gl-font-serif-cn` | 600 | 最大宽度 760-840px |
| Hero 副标题 | 18px / 32px | `--gl-font-sans-cn` | 400 | H1 下方 24-32px，最大宽度 680-760px |
| Hero CTA | 14px / 20px | `--gl-font-sans-cn` | 600 | 副标题下方 32-40px |

禁止：

- Hero H1 超过 56px / 72px。
- H1 使用负字距。
- 副标题超过 20px 或行高过紧。
- CTA 文案使用 16px 以上大按钮文字。

### 4.2 首页模块

| 位置 | 桌面规范 | 字体 | 字重 | 位置 |
|---|---:|---|---:|---|
| 模块上间距 | 80-96px | - | - | 模块之间保持稳定节奏 |
| 模块编号 / Eyebrow | 12px / 18px | `--gl-font-mono` 或 `--gl-font-en` | 600 | 模块标题上方 12-16px |
| 首页模块标题 | 36px / 48px | `--gl-font-serif-cn` | 600 | 普通模块最大标题，不得用 Hero 尺寸 |
| 模块说明 | 15px / 26px | `--gl-font-sans-cn` | 400 | 标题下方 16-20px，最大宽度 640px |
| 主卡标题 | 28px / 38px | `--gl-font-serif-cn` | 600 | 只用于首页每屏的主信号 / 主内参卡 |
| 普通卡标题 | 20px / 30px | `--gl-font-sans-cn` | 600 | 卡片内部 |
| 卡片摘要 | 14px / 24px | `--gl-font-sans-cn` | 400 | 不超过 3-4 行 |
| 标签 / 日期 | 12px / 18px | `--gl-font-en` / `--gl-font-mono` | 600 | 颜色弱于标题 |

首页商业信号模块特别规则：

- 模块标题最多 `36px / 48px`。
- 今日主信号标题可用 `28px / 38px`，不能用详情页 H1。
- 右侧列表卡片标题使用 `18px / 28px` 或 `20px / 30px`，不能与主信号同级。
- 摘要先讲事件，再讲一句判断，不用字段名当标题。

## 5. 栏目页

适用：今日观察、商业信号、趋势追踪、商业内参。

| 位置 | 桌面规范 | 字体 | 字重 | 位置 |
|---|---:|---|---:|---|
| 页面顶部起点 | 导航下 88-112px | - | - | 四个一级栏目保持同一节奏 |
| 栏目 Eyebrow | 11px / 16px | `--gl-font-en` | 600 | 字距 `0.12em`，标题上方 14-18px |
| 栏目 H1 | 44px / 58px | `--gl-font-serif-cn` | 600 | 最大宽度 760px |
| 栏目说明 | 16px / 28px | `--gl-font-sans-cn` | 400 | 最大宽度 720px |
| 筛选栏文字 | 13px / 20px | `--gl-font-sans-cn` | 500 | 不抢 H1 |
| 筛选按钮 | 13px / 20px | `--gl-font-sans-cn` | 600 | 高度 34-38px |
| 列表分组标题 | 24px / 34px | `--gl-font-serif-cn` 或 `--gl-font-sans-cn` | 600 | 分组顶部 |
| 主列表卡标题 | 22px / 32px | `--gl-font-serif-cn` | 600 | 只用于栏目主卡 |
| 普通列表卡标题 | 18px / 28px | `--gl-font-sans-cn` | 600 | 默认列表 |
| 列表摘要 | 14px / 24px | `--gl-font-sans-cn` | 400 | 不超过 4 行 |
| 列表元信息 | 12px / 18px | `--gl-font-mono` / `--gl-font-en` | 500 | 日期、编号、来源等级 |

禁止：

- 栏目 H1 使用首页 Hero 字号。
- 筛选栏文字大于正文。
- 列表卡标题超过 24px。
- 栏目说明写成大段正文。
- 不同栏目页 H1 起点和字号不同。

## 6. 详情页

适用：今日观察详情、商业信号详情、趋势报告详情、商业内参详情。

| 位置 | 桌面规范 | 字体 | 字重 | 位置 |
|---|---:|---|---:|---|
| 详情页顶部起点 | 导航下 88-112px | - | - | 文章标题区稳定 |
| 详情页标签 / 栏目 | 12px / 18px | `--gl-font-en` / `--gl-font-mono` | 600 | H1 上方 14-20px |
| 详情页 H1 | 40px / 56px | `--gl-font-serif-cn` | 600 | 最大宽度 860px |
| 导语 / Deck | 18px / 32px | `--gl-font-sans-cn` | 400 | H1 下方 20-28px |
| 元信息 | 12px / 18px | `--gl-font-mono` | 500 | 日期、编号、来源 |
| 正文 | 16px / 30px | `--gl-font-sans-cn` | 400 | 阅读宽度 760-860px |
| 正文重点 | 17px / 30px | `--gl-font-sans-cn` | 500 | 少量使用 |
| 正文 H2 | 26px / 38px | `--gl-font-serif-cn` 或 `--gl-font-sans-cn` | 600 | 段落大节 |
| 正文 H3 | 20px / 30px | `--gl-font-sans-cn` | 600 | 小节 |
| 引用 / 判断块 | 20px / 34px | `--gl-font-serif-cn` | 500 | 不超过 4-6 行 |
| 来源链接 | 14px / 22px | `--gl-font-sans-cn` | 600 | 同一组链接水平或列表对齐 |

详情页禁止：

- H1 超过 40px / 56px。
- 正文用 18px 以上大字号铺开。
- 正文行高低于 1.7。
- 每个模块都使用大标题，导致主次不清。
- 把字段名当模块标题，例如“businessMeaning”“sourcePath”。
- 侧栏标题与正文 H2 同级。

## 7. 详情页侧栏 / 关联模块

| 位置 | 桌面规范 | 字体 | 字重 |
|---|---:|---|---:|
| 侧栏分组标签 | 11px / 16px | `--gl-font-en` | 600 |
| 侧栏标题 | 16px / 24px | `--gl-font-sans-cn` | 600 |
| 侧栏摘要 | 13px / 22px | `--gl-font-sans-cn` | 400 |
| 侧栏元信息 | 12px / 18px | `--gl-font-mono` | 500 |
| 关联卡标题 | 17px / 26px | `--gl-font-sans-cn` | 600 |

侧栏永远不能比正文主线更大。关联模块是辅助阅读，不是第二个页面主标题。

## 8. 页脚

| 位置 | 桌面规范 | 字体 | 字重 |
|---|---:|---|---:|
| 页脚品牌名 | 14px / 22px | `--gl-font-sans-cn` | 600 |
| 页脚说明 | 12px / 20px | `--gl-font-sans-cn` | 400 |
| 备案号 / 法务 | 12px / 20px | `--gl-font-sans-cn` | 400 |
| 页脚链接 | 12px / 20px | `--gl-font-sans-cn` | 500 |

页脚不使用大标题，不使用 Hero 风格，不加重品牌英文。

## 9. 移动端基础口径

当前 V2.2 开发桌面端优先，移动端不作为页面任务硬阻塞。但所有新增页面必须满足基础不崩坏。

| 位置 | 移动规范 |
|---|---:|
| 导航高度 | 64px |
| 首页 Hero H1 | 34px / 46px |
| 栏目 H1 | 30px / 42px |
| 详情页 H1 | 28px / 40px |
| 模块标题 | 22px / 32px |
| 卡片标题 | 18px / 28px |
| 正文 | 15px / 26px |
| 摘要 | 13px / 22px |
| 标签 / 日期 | 12px / 18px |

移动端禁止横向溢出，标题可以换行，但不能被截断成省略号。

## 10. 位置与容器

| 页面位置 | 桌面容器 |
|---|---:|
| 导航内容 | 1200-1280px |
| 首页主内容 | 1200-1280px |
| 首页 Hero 文案最大宽度 | 760-840px |
| 栏目页主容器 | 1200-1280px |
| 栏目页标题最大宽度 | 760px |
| 详情页正文 | 760-860px |
| 详情页带侧栏总宽 | 1120-1280px |
| 详情页侧栏 | 320-380px |
| 数据密集宽表 | 1440-1520px，仅限热力图、宽表、证据矩阵 |

普通页面不得默认使用 1520px 以上宽度。宽度越大，字号越不能随之放大。

## 11. CSS 执行规则

新增页面或模块 CSS 必须优先使用 token：

```css
font-family: var(--gl-font-serif-cn);
font-size: var(--gl-type-page-title-size);
line-height: var(--gl-type-page-title-line);
font-weight: var(--gl-weight-title);
```

允许新增局部类，但不得新增随意字号。新增字号必须先进入 `brand-tokens.css` 或本文件。

禁止：

```css
font-size: 37px;
font-size: 46px;
font-size: 5vw;
font-weight: 760;
line-height: 1.02;
letter-spacing: -0.03em;
```

例外：

- 首页 Hero H1 可使用专用 token。
- 大数字可按数字字体规范使用轻微负字距。
- Logo SVG 内部字体不受页面 CSS 控制。

## 12. 页面任务验收清单

页面类任务 closeout 必须回答：

1. 是否在 Build 前输出 Typography 页面位置表？
2. `$guanlan-typography-qc` Build 前审查是否通过？
3. Build & Release 是否 100% 按 Typography 页面位置表实现？
4. 是否新增表外字体规则；如有，selector 是什么、为什么新增、是否经过 Experience & Editorial 复核？
5. `$guanlan-typography-qc` Build 后复核是否通过？
6. 导航字号是否为 `14px / 20px`，位置是否统一？
7. 首页是否只有 Hero H1 使用 `56px / 72px`？
8. 栏目页 H1 是否为 `44px / 58px`？
9. 详情页 H1 是否为 `40px / 56px`？
10. 模块标题是否不超过 `36px / 48px`？
11. 卡片标题是否不超过 `24px / 34px`？
12. 正文是否使用 `16px / 30px` 左右的黑体，行高不低于 1.7？
13. 是否存在 `vw` 或过大的 `clamp()` 字号？
14. 是否存在 `760 / 780` 等过重字重？
15. 是否存在页面标题、模块标题、卡片标题同级的问题？
16. 桌面截图中，标题、正文、卡片和导航的主次是否一眼可辨？

未回答以上检查，不得标记页面排版 accepted。
