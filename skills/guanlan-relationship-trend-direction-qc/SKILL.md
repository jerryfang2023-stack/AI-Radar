# 观澜关系与趋势方向卡 QC

## 使用场景

当需要生成或检查“关系与趋势方向”“趋势候选”“标签关系转趋势方向”“数据观察台方向卡”时使用。

## 目标

方向卡要呈现商业洞察：说清多条材料之间的内在共同点、关系方向和商业含义。

不要把“数量统计”误当作洞察。数量是证据范围，洞察是这些材料为什么属于同一个方向。

## 标题规则

标题必须回答：

- 这些材料的内在共同点是什么？
- 它们共同指向哪条商业链路、企业流程或能力转移？
- 为什么这些看似分散的材料应该放在一起看？

推荐结构：

```text
AI 产品发布的共同指向，是抢占企业采购、工程工具链和行业流程入口
Agent 的企业落点正在从聊天入口转向销售、客服和工作流
AI Coding 的竞争点正在转向工程工具链和运行环境
客服和语音 Agent 的共性，是先在客户接触点证明效率
```

日期、数量、来源和 Card 数量应该放在摘要、元信息或详情页，不要默认塞进标题。

## 禁止标题

禁止使用不能回答“什么、多少、为什么”的标题：

```text
产品发布密度高，但真实采用仍需证据过滤
2026.06.04 产品类 Card 11 张，集中在 AI 采购、Claude Code 和行业 Agent
11 张材料集中在 AI 采购、Claude Code 和行业 Agent
趋势正在形成
值得继续观察
商业落点正在出现
需要辨别
案例验证
关键约束
```

这些词可以作为内部状态，不应作为前台主标题。

## 摘要规则

摘要必须说明可见事实：

- 命中的 Card 范围。
- 支撑材料来自哪些主体或来源。
- 这些材料共同落在哪些场景、流程或对象上。
- 标题里的洞察是由哪些事实支撑的。

摘要不要写：

```text
说明趋势正在形成
证明价值正在显现
但仍需证据过滤
客户是否买单
下一步需要验证
```

## 详情页规则

详情页首屏只展示：

1. 标题。
2. 日期、数量范围。
3. 可见事实。
4. 关系链。
5. 支撑材料。

不得在首屏展示：

- Raw / Pool / Card 内部链路。
- 资产路径。
- JSON / 脚本 / 后台字段。
- 大量 tag 列表。
- 过程性判断或质量门描述。

## 字体规则

遵循观澜详情页字体规范：

- 详情标题：`40px / 56px / 600`，中文衬线。
- 事实导语：`18px / 32px / 400`，中文黑体。
- 正文：`16px / 30px / 400`。
- 元信息和标签：`12px / 18px / 500-600`。
- 支撑材料标题：`14px / 24px / 600`，不得抢主标题。

## QC 清单

检查时逐条回答：

- 标题是否说出了内在共同点，而不是只列对象？
- 标题是否像商业洞察，而不是统计描述？
- 数量是否放在摘要或详情中作为证据范围？
- 摘要是否解释为什么这些材料可以归为同一方向？
- 摘要是否给事实，而不是替用户下判断？
- 详情页首屏是否主次分明？
- 是否出现 Raw / Pool / 字段 / 资产路径等非前台信息？
- 字体是否符合观澜详情页规范？
