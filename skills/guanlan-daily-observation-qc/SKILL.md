# Guanlan Daily Observation QC｜今日观察可读性质检 Skill

## 1. QC 立场

QC 不是资料审计，也不是事实门禁。

QC 要像一个严厉商业主编判断读者会不会读下去：

- 标题有没有钩子；
- 开头有没有把人拽进去；
- 正文是否顺、紧、有推进；
- 句子是否像人写的，而不是模型压缩稿；
- 判断是否有记忆点；

事实错误、来源缺失、明显越界可以备注，但不作为本 Skill 的主评分轴。证据和正确性应交给事实门、monitor QC 或专项 source verification。

核心标准：

> 一篇好今日观察，首先要让读者愿意读、读得懂、读完记得住。

## 2. 栏目定位

今日观察不是新闻摘要、工具清单、多事件综述、Card 扩写或厂商案例精读。

它是：

> 每天从 AI 商业素材里挑一个最有冲突和叙事价值的对象，写成一篇有刺、有场面、有节奏、有可带走判断的商业观察。

QC 重点看成稿是否把这个定位转化成可读文本，而不是检查它是否填满内部 checklist。

## 3. 阅读阻断项

出现以下任一项，必须返修：

- 标题读不顺、断句别扭，或像内部报告标题；
- 开头 150 字内没有冲突、场面、反差或判断；
- 正文连续 3 段都在解释概念，没有新动作、新场面或新判断；
- 同一核心判断反复出现 3 次以上；
- 结尾只说“值得关注 / 仍需观察 / 未来可能”，没有落锤；
- 小标题为了“金句感”强行押句式，导致隐喻不贴合、主语别扭或语气过重；
- 前台出现明显内部生产语言，例如 Raw、Pool、候选、证据边界、放行、补证等。
- 网站展示字段像内部质检说明、素材摘要或旁观式转述，例如“需要克制使用”“说明一个方向”“可编排流程”“可管理工序”“公开案例里的主角是”“案例页给出的结果很亮眼”“这家公司在案例中宣称”，没有落到读者能自查的经营问题。

## 4. 评分维度

满分 100。

| 维度 | 分值 | 检查重点 |
|---|---:|---|
| 标题与入口 | 20 | 标题是否顺口、有钩子、不僵硬 |
| 开头吸引力 | 20 | 前 150 字是否有冲突、反差、场面或锋利判断 |
| 叙事推进 | 20 | 每段是否往前走，是否有动作、场面、角色变化或判断递进 |
| 语言质感 | 15 | 是否像商业记者写作，避免报告腔、翻译腔、模型腔和硬断句 |
| 信息密度 | 15 | 是否少废话、少重复、少正确空话，核心判断是否只说一次 |
| 结尾 | 10 | 结尾是否短、有力、有落点 |

## 5. 扣分规则

- 标题像“主题 + 冒号 + 长判断”硬拼：扣 10-20 分。
- 开头以“某某案例显示 / 随着 AI 发展 / 今天可以看到”起手：扣 10-20 分。
- 正文像在解释栏目为什么写这个题：扣 10-20 分。
- 多段反复说同一判断，只换词不增量：扣 15-30 分。
- 句子过长、断句奇怪、逗号堆叠，读起来喘不过气：扣 10-25 分。
- 频繁使用“真正值得看的是 / 本质上 / 关键在于 / 说明了一个趋势”：扣 10-20 分。
- 小标题没有承接段落推进，只是在追求对仗或标语感：扣 10-20 分。
- 结尾啰嗦，重复正文，或落在“继续观察”：扣 10-20 分。

## 6. 逐项检查

### 6.1 标题

好标题：

- 一口气能读完；
- 有具体对象或冲突；
- 不靠堆概念显得高级；
- 不把两个标题硬塞进一个标题。

坏标题：

- 冒号前后各讲一层判断；
- 长到必须换两三行才读完；
- 看起来正确，但没有点开欲望；
- 换一个公司名也成立。

### 6.2 开头

开头必须尽快进入：

- 一个反差；
- 一个场面；
- 一个角色变化；
- 一个能刺痛旧说法的判断。

开头不要先交代背景、来源、栏目选择理由或抽象定义。

### 6.3 正文节奏

逐段检查：

- 这一段有没有新增信息；
- 段与段之间是否有推进；
- 是否出现“同一句判断换三种说法”；
- 是否有足够短句让读者喘气；
- 是否能删掉 20% 而不损失意思。

### 6.4 语言质感

重点抓：

- 报告腔；
- 翻译腔；
- 模型腔；
- 内部术语；
- 莫名其妙的断句；
- 长串并列名词；
- 抽象词压过具体动作。

合格文字应该像编辑写给读者，不像系统写给系统。

### 6.5 网站展示字段

如果稿件包含首页摘要、栏目页标题、栏目页主判断、栏目页正文一、栏目页正文二、侧记和右侧卡片，必须单独检查这些字段。正文通过不代表展示字段通过。

逐项检查：

- 首页摘要是否有点开欲望，而不是复述正文主线。
- 右侧卡片是否是读者能秒懂的小判断，而不是内部概念标签。
- 栏目页主判断是否直接说明读者该看什么，不堆“流程 / 工序 / 能力 / 方向”等抽象词。
- 栏目页正文一是否直接进入具体处境、角色和动作，不能用“公开案例里的主角是 / 案例讲的是 / 这是一家……”这类旁观式开头。
- 栏目页正文二如果处理案例数字，必须把边界写成读者语言，再落到一个自查问题；不得写“案例页给出的结果很亮眼 / 需要克制使用 / 说明一个方向 / 更适合说明”等内部校对句。
- 各展示字段不能重复同一句判断，尤其不能让首页摘要、主判断、正文一、正文二都围绕同一个抽象词打转。

## 7. 二次质检与精校要求

当用户要求“严厉老主编二审”“二次打分”“检查重复啰嗦”“精校”时，QC 必须进入二审模式，不只给分，还要判断文章是否真的值得老板读。

二审额外评分维度：

| 维度 | 检查重点 |
|---|---|
| 价值性 | 是否让读者多一个商业判断，而不是只知道一个融资、发布或案例。 |
| 传播力 | 标题和核心句是否能被读者转述，是否有“我想转给别人”的钩子。 |
| 老板吸引力 | 是否落到成本、重复劳动、客户资产、组织能力、预算判断、风险责任或增长机会。 |
| 重复啰嗦 | 是否同一判断用不同词说了三遍，是否概念词压过具体动作。 |
| 句面精度 | 是否存在过长句、绕口句、校对腔、报告腔、抽象名词堆叠。 |

二审扣分规则：

- 只讲行业概念，不讲老板处境：扣 10-20 分。
- 只说“值得关注 / 说明趋势 / 组织能力”等抽象词，没有具体经营痛点：扣 10-20 分。
- 同一词或同一判断反复出现，例如“个人效率 / 组织记忆 / 私人聊天框 / 上下文”连续承担同一功能：扣 10-25 分。
- 标题顺但不扎心，老板读完不知道和自己有什么关系：扣 8-18 分。
- 开头没有进入成本、重复、客户、交付、预算、责任或角色变化：扣 10-20 分。

精校必须执行：

1. 先找出 3-5 个最啰嗦或重复的位置。
2. 删除过程性句子，例如“真正值得看的是”“本质上”“关键在于”“这个说法有营销味，但问题是真问题”等可替代校对腔。
3. 把抽象概念换成经营语言，例如“公司资产”“少重复一次”“客户研究能不能复用”“团队能不能接力”“新人能不能进入状态”。
4. 拆长句，让每段只推进一个判断。
5. 同步检查网站展示字段，避免正文改好了，首页摘要和栏目页仍保留旧口径。
6. 精校后必须再扫一遍前台禁用词和空话词。
7. 对栏目页正文一、正文二做前台化改写：正文一直接进入人和动作，正文二直接进入数字边界和读者问题，不输出内部质检口吻或旁观式转述。

```markdown

- 总分：
- 结论：

## 分项分数
| 维度 | 分数 | 主编判断 |
|---|---:|---|

## 最大问题
-

## 必改方向
-

## 精校动作
-
```

精校后的文章如果只做局部改动，应更新 `quality_score`、`qc_round` 和 QC 报告路径；如果没有重跑事实审计，不得把二审分数写成事实质量分。

```markdown

- 总分：

- 一句话主编判断：

按影响排序列 3-5 条。

|---|---|---|

## 5. 可直接删掉的内容
-

-
```

## 9. 最终通过标准

完整通过只看 6 条：

- 标题有钩子；
- 开头抓人；
- 正文有推进；
- 语言顺、不僵、不像报告；
- 重复被压掉；
- 结尾短、有力、有落点。
