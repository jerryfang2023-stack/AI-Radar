## 机会摘要

- **一句话机会定义：** 面向非技术创始人和小团队创业者，提供 AI Coding 驱动的一人公司工具链——用 AI 编码工具快速搭建原型、内部工具、数据看板和自动化流程，让单人能用 1-3 天验证一个产品想法。
- **为什么是现在：** 4-29 雷达中两个强信号：Google 披露 75% 新代码由 AI 生成（从 2024 年 10 月的 25% 增长到 2026 年 4 月的 75%），OpenAI Codex 编程 Agent 接入 AWS Bedrock 成为云平台标准服务。5-01 雷达进一步确认该趋势：Factory 以 15 亿美元估值融资 1.5 亿美元，客户包括 Morgan Stanley、EY、Palo Alto Networks 等企业级客户，说明 AI 编程已从个人开发工具演进为"企业级研发效能基础设施"。AI Coding 不仅支撑一人公司验证 MVP，也正在改变大型企业研发组织的交付方式——这意味着 AI Coding 的市场天花板比"仅限一人创业"大得多。
- **这是谁的问题：** 有产品想法但不会写代码的创始人；受限于开发成本无法快速验证 MVP 的创业者；需要频繁搭建内部工具的企业产品经理；想从 0 到 1 试水 AI 产品的独立开发者。
- **不做的代价：** AI Coding 工具正在快速成熟，窗口期不是"要不要用"而是"谁先用它构建起自己的工具链和流程"。后入者需要花更多时间学习和适应，而先行者已经在用几周时间跑完别人几个月才能完成的验证周期。

## 问题与解决方案

- **原始痛点：** 非技术创始人验证产品想法需要找技术合伙人或外包开发（成本高、沟通成本大、周期 1-3 个月）；现有 AI 编码工具（Cursor、Claude Code、Copilot）虽然强，但需要一定的技术判断力和工程经验才能高效使用；缺少面向非技术用户的"AI Coding 方法论 + 工具链 + 模板"体系。
- **当前替代方案：** 找技术合伙人（难找、难对产品 vision 对齐）；外包开发（贵、慢、迭代困难）；自学编程（周期 6-12 个月才能达到可用水平）；低代码/无代码平台（有天花板，复杂逻辑难实现）。没有专门服务于"单人创业"的 AI 开发方法论和工具组合。
- **AI解决方案类型：** AI Coding 工具链 + 一人公司 MVP 方法论（教程/模板/Workshop/实战陪跑）
- **需要什么数据/系统前提：** 无需数据前提。需要创业者有清晰的产品假设和 MVP 范围界定能力。AI Coding 工具需要接入对应 API（Claude Code / Cursor / Copilot 等）。
- **典型实施路径：** 定义 MVP 最小功能集 → 用 AI Coding 工具搭建首个可运行原型（1-3 天）→ 部署和测试 → 获取前 10 个用户反馈 → 迭代优化。关键在于"先让 AI 写 80% 的代码，人工聚焦 20% 的关键判断"。

## ROI假设

- **可能节省的成本：** 传统外包开发一个 MVP 需要 5-20 万 + 2-4 个月。AI Coding 工具链可以将成本降到 5000-20000 元 + 1-2 周。
- **可能提升的收入：** 对创业者来说，提升的是"试错速度"——同样预算和时间可以测试 5-10 个产品想法，而不是只测试 1 个。验证失败的成本从几万块降到几千块。
- **可能缩短的时间：** MVP 从 idea 到可运行原型从 2-4 个月缩短到 1-2 周。
- **回本周期假设：** 如果你提供"AI Coding 一人公司工作坊/模板"，定价 1999-4999 元/人，招募 20-50 人即可覆盖内容开发成本。如果是做自己的产品，回本周期取决于产品本身的商业模式。
- **适合先做试点还是直接规模化：** 适合先做自己的产品（dogfooding），跑通"AI Coding 做 MVP → 获取用户 → 迭代"的全流程。在实战中沉淀方法论和工具链后，再考虑做成面向他人的教程/模板/Workshop。

## 代表性案例

### 案例 1：Google / OpenAI Codex / Cursor / Claude Code / GitHub Copilot / Bolt.new
- **发生了什么：** Google CEO Sundar Pichai 披露截至 2026 年 4 月，75% 的新代码由 AI 生成并由工程师审查。该比例从 2024 年 10 月的 25% 增长到 2025 年秋季的 50%，再到目前的 75%，增长曲线陡峭。OpenAI Codex 编程 Agent 也通过 Bedrock 进入 AWS 生态。
- **关键数据：** 75% 的代码由 AI 生成（Google 内部数据。代表全球最大软件组织的工程效率变化。Bolt.new 等 AI 编程平台已让非技术用户能在 10 分钟内生成可运行的网页应用原型。
- **成功前提：** 创业者需要能清晰定义"什么让 AI 做，什么自己判断"——AI Coding 不是"说一句话就生成完整产品"，而是"AI 写 80% 代码 + 人决策 20% 关键逻辑"。需要对产品或行业有深刻理解（AI 写不了产品判断）。
- **失败风险：** AI 生成的代码质量不稳定，特别是复杂业务逻辑和多人协作场景；纯 AI 生成的产品缺乏差异化（AI 写出的代码风格类似，容易同质化）；过度依赖 AI 导致缺乏对核心系统的理解（后续难以 debug 和优化）；一人公司意味着创始人需要同时做产品、技术、运营、销售——AI Coding 只解决了"技术"这一块。

### 案例 2：Factory
- **发生了什么：** Factory 完成 1.5 亿美元融资，估值达 15 亿美元，由 Khosla Ventures 领投，Sequoia、Insight Partners、Blackstone 参投。其 AI 编程 Agent 面向企业工程团队，支持在 Claude、DeepSeek 等多种基础模型间切换。客户包括 Morgan Stanley、EY、Palo Alto Networks 等大型企业。
- **关键数据：** 15 亿美元估值、1.5 亿美元融资；客户来自金融、咨询、安全等高付费行业。
- **成功前提：** 企业级 AI 编程需要解决代码安全、权限管理、合规审计和模型切换等问题，非个人开发工具的简单升级。

## 典型供给方

- **海外代表：** Cursor、Claude Code、GitHub Copilot、Bolt.new、Windsurf、Replit Agent、Factory（企业级AI编程Agent，15亿美元估值）
- **国内代表：** 通义灵码（阿里）、CodeGeeX（智谱）、GLM-5.1 Code Agent（智谱）、Fittencode（字节）
- **自建/采购/合作建议：** 一人公司应直接使用最成熟的海外 AI Coding 工具（Cursor + Claude Code 组合），通过 API 或命令行使用。国内替代方案可选通义灵码或 CodeGeeX。
- **供应商筛选标准：** 选择对中文理解好、代码生成质量高的工具。优先级：Cursor > Claude Code > Bolt.new（偏前端）> Copilot。国内首选通义灵码或 CodeGeeX。

## 风险与反证

- **最大风险：** AI Coding 工具的同质化——用同一套工具生成的产品如果不加人的判断和行业理解，容易做出"看起来一样"的产品。一人公司真正的壁垒不是"会用 AI 写代码"，而是"用 AI 快速验证 + 用行业判断做差异化"。
- **哪类企业不适合：** 需要强安全合规的产品（金融、医疗等受监管行业）；需要复杂分布式架构的产品；多人协作的团队项目（AI 代码的协作和维护成本较高）。
- **反对观点：** "AI 写代码质量不够，还得靠人"——但 75% 这个数字来自 Google 的生产环境，说明 AI 代码达到生产标准是完全可行的。问题不在于"能不能用"而在于"怎么用"（哪些让 AI 写、哪些自己控制）。
- **需要进一步验证的问题：** 一位没有技术背景的创业者，用 AI Coding 工具链从 0 搭建一个 SaaS MVP 需要多久（从学习工具到上线可用版本）？第一个付费用户出现在第几天？

## 行动建议

- **建议优先级：** 高
- **适合谁发起：** 任何正在或计划做"单人 AI 创业"的人。这个方向与你的 AI 机会雷达系统、内容体系和创业方向高度一致。
- **下一步最小验证动作：** 用 AI Coding 工具在 3 天内搭建一个最小可用的工具（如 Obsidian/Dataview 前端、机会卡生成器等），上线获取前 5 个用户，记录从 idea 到上线的完整时间和遇到的关键瓶颈。同时用 Notion/飞书记录"一人公司 AI Coding 工作流"。
- **建议预算范围：** AI Coding 工具订阅费（Cursor $20/月、Claude API 按量付费），总启动成本 < 2000 元/月。
